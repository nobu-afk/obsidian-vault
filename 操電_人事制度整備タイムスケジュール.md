# 操電 人事制度整備タイムスケジュール

**会議資料** | 2026年1-6月 初期人事制度設計計画

---

## 📋 会議の目的

操電の人事制度整備について、2026年1-6月の初期設計スケジュールと今後の展開方針を共有し、方向性を確認する。

---

## 1. 現状分析

### ベンチャー企業の成長フェーズと人事制度

ベンチャー企業の組織成長は、社員数の増加に伴って4つのフェーズに分けられます。各フェーズで求められる人事制度の内容が異なります。

| フェーズ | 社員数 | 特徴 | 人事制度のポイント |
|---------|--------|------|------------------|
| **フェーズ1** | 10人未満 | 経営者の目が隅々まで届く時期 | 事業優先、制度は最小限<br>（評価基準や昇給ルールの簡単なメモ書き程度） |
| **フェーズ2** | 10〜30人 | 「20人の壁」が見え始める時期 | 評価の「見える化」開始<br>シンプルな評価基準（成果×行動の2軸）<br>1on1などのコミュニケーション重視 |
| **フェーズ3** | 30〜50人 | 「50人の壁」、マネージャー層が育ち始める | **本格的な制度設計のタイミング**<br>等級制度、評価制度、報酬制度の整備 |
| **フェーズ4** | 50〜100人 | 組織が安定、人事部門が本格的に機能 | 戦略的な人事へ<br>HRテック導入、キャリアパスの複線化 |

### 操電の現状

| 項目 | 内容 |
|------|------|
| **創業日** | 2022年6月30日（約3.5年経過） |
| **現在の社員数** | **28名** |
| **来期目標** | **50名体制** |
| **現在のフェーズ** | **フェーズ2（10-30人）の後半** |
| **移行予定フェーズ** | **フェーズ3（30-50人）への移行期** |
| **制度整備時期** | **2026年1-6月**（6ヶ月間） |

**なぜ今、制度整備が必要か**
- 現在28名でフェーズ2の後半に位置
- 来期50名体制を目指すため、フェーズ3への移行準備が必要
- フェーズ3は「本格的な制度設計のタイミング」とされており、このタイミングでの整備が最適

### 主な課題

1. **「20人の壁」を超えた組織運営の課題**
   - 経営者の主観だけでは評価が困難
   - 公平・透明な評価基準の整備が急務

2. **組織の階層化に伴う課題**
   - マネージャー層が育ち始めている
   - 昇進・昇給の基準が不明確
   - 社員の納得感やエンゲージメントに影響

3. **50名体制への準備**
   - 採用競争力の強化（明確なキャリアパス提示）
   - 組織運営の透明性・ガバナンスの確立

## 2. 設計の基本方針

### 4つの基本原則

| 原則 | 内容 | 理由 |
|------|------|------|
| **①シンプルさを最優先** | 運用負荷を最小化 | ベンチャー企業のリソース制約を考慮 |
| **②透明性と納得感** | 社員が理解できる内容 | 制度への信頼とエンゲージメント向上 |
| **③アジャイルな運用** | 完璧を目指さず、運用しながら改善 | 変化の激しい環境への適応 |
| **④戦略との整合性** | 事業戦略・バリューと連動 | 組織の方向性と個人目標のアラインメント |

## 3. 2026年1-6月：初期人事制度設計スケジュール

### 月次スケジュール概要

| 月 | フェーズ | 主要タスク | 成果物 | 重要ポイント |
|---|---------|-----------|--------|------------|
| **1月** | **全体構想** | • 現行課題整理（ヒアリング・アンケート）<br>• 制度の方向性すり合わせ<br>• 全体構想の策定 | • 課題整理書<br>• 全体構想書<br>• バリュー定義書 | 経営陣との方向性合意が重要 |
| **2月** | **等級制度** | • キャリアパス設計<br>• コース体系（マネジメント/専門職）<br>• 等級体系・定義（5-6段階）<br>• 昇格ルール<br>• 格付け移行シミュレーション | • 等級制度案<br>• キャリアパス図<br>• 移行シミュレーション結果 | 現職員28名の等級仮振り実施 |
| **3月** | **報酬制度** | • 報酬体系設計<br>• 等級別報酬水準（給与テーブル）<br>• 昇給ルール<br>• 賞与制度（原資算定・配分）<br>• 個人別シミュレーション | • 給与テーブル案<br>• 賞与規程案<br>• シミュレーション結果 | 激変緩和措置の必要性を判断 |
| **4月** | **評価制度** | • 評価体系設計（成果×行動の2軸）<br>• 評価シート・基準作成<br>• 評価決定プロセス<br>• 評価スケジュール<br>• 報酬への反映ルール | • 評価制度案<br>• 評価シート<br>• 評価基準書 | MBO/OKR + バリュー評価のハイブリッド |
| **5月** | **移行措置** | • 全体整合性確認<br>• 激変緩和措置検討<br>• 社員説明資料作成<br>• 社員説明会実施<br>• 就業規則・規程類作成 | • 社員説明資料<br>• 各種規程類一式 | 社員の納得感醸成が最重要 |
| **6月** | **制度導入** | • 制度正式導入<br>• 評価者研修実施<br>• 初回評価サイクル準備<br>• 運用フォロー体制構築 | • 導入完了報告<br>• 運用マニュアル | 評価者研修は必須 |

### 整備する主要要素（優先順位）

#### 【最優先】3つの基盤制度

1. **等級制度**
   - 5-6段階の等級体系
   - 等級定義（役割・責任・能力）
   - 昇格ルール

2. **評価制度**
   - 成果軸（What）と行動軸（How）の2軸評価
   - MBO/OKR + バリュー評価のハイブリッド
   - 評価頻度：半期ごと（年2回）

3. **報酬制度**
   - 等級別給与テーブル
   - 昇給ルール（評価結果連動）
   - 賞与制度（全社業績×個人評価）

#### 【高優先】2つの補完制度

4. **キャリアパス**
   - マネジメントコースと専門職コースの基本構造
   - 各等級への成長ステップ

5. **職務記述書（JD）**
   - 主要ポジションのJD整備
   - 全社員分は後回しでも可

#### 【必須】評価者研修

- マネージャー向け評価者研修（5-6月実施）
- 評価制度の理解、フィードバックスキル向上

## 4. 2026年7月以降：ブラッシュアップの方向性

### 段階的な制度拡張計画

| 時期 | 背景・タイミング | 主な実施内容 | 目的 |
|------|----------------|------------|------|
| **2026年<br>7-12月** | 制度導入直後<br>（初回評価サイクル後） | • 初回評価サイクルの振り返り<br>• 制度の微調整<br>• 1on1の質向上 | 運用の安定化 |
| **2027年<br>1-6月** | 50名体制を超える<br>（組織の階層化が進む） | • 360度評価の部分的導入（マネージャー層）<br>• キャリアパスの複線化<br>• HRテック導入検討 | 制度の拡張・効率化 |
| **2027年<br>7-12月** | 組織が安定<br>（外部競争力の確保） | • 外部報酬水準リサーチ<br>• 給与テーブル見直し<br>• 賞与制度の精緻化 | 報酬競争力の強化 |
| **2028年<br>1-6月** | 70-100名体制<br>（フェーズ4への移行） | • HRテック本格導入<br>• タレントマネジメント導入<br>• 戦略的人事への移行 | 戦略的人事の確立 |

### ブラッシュアップの基本方針

- **6ヶ月ごとの見直し**：運用状況を確認し、必要に応じて改善
- **成長フェーズに応じた拡張**：社員数や組織の成熟度に合わせて制度を進化
- **現場の声を反映**：定期的なヒアリングやサーベイで課題を把握

## 5. 成功のための重要なポイント

### 設計段階で注意すべきこと

| ポイント | 具体的なアクション |
|---------|------------------|
| **完璧を目指さない** | シンプルに始め、運用しながら改善 |
| **現場の声を聞く** | 社員ヒアリング（10-15名）、全社員アンケート |
| **経営陣のコミットメント** | トップが制度の意義を語り、現場と対話 |
| **バリューとの整合** | 会社の価値観を評価基準に組み込む |

### 運用段階で注意すべきこと

| ポイント | 具体的なアクション |
|---------|------------------|
| **形骸化の防止** | 定期的な見直し、「評価のための評価」にしない |
| **評価者の負荷軽減** | シンプルな評価項目、ツールの活用 |
| **透明性の維持** | 評価基準やプロセスをオープンに |
| **学習サイクルの確立** | 運用→振り返り→改善のサイクルを回し続ける |

### 主なリスクと対策

| リスク | 対策 |
|--------|------|
| **制度変更による混乱** | 段階的な導入、十分な説明、社員説明会の実施 |
| **評価の不公平感** | 評価者研修の徹底、評価の一貫性確保 |
| **運用負荷の増大** | ツール導入、プロセス簡素化、シンプルな設計 |
| **制度と文化のミスマッチ** | 定期的な文化との整合性確認、バリューとの連動 |

## 7. まとめ

### 重要な原則

1. **シンプルに始める**：最初は最小限の制度でスタート
2. **成長に合わせて拡張**：社員数や組織の成熟度に応じて制度を進化
3. **現場と対話する**：社員の声を反映し、納得感を高める
4. **継続的に改善**：一度作って終わりではなく、常に最適化を続ける

### 期待される効果

- **公平・透明な評価基準**の確立により、社員の納得感とエンゲージメント向上
- **明確なキャリアパス**の提示により、採用競争力の強化
- **組織運営の透明性・ガバナンス**の確立により、投資家・パートナーからの信頼獲得
- **50名体制への準備**が整い、組織拡大に対応可能な基盤の構築
