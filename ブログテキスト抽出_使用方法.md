# ブログテキスト抽出ツール 使用方法

## 概要

修士論文研究目的で、ブログサイトからテキストデータを抽出するためのツールです。
既に許可を得られているサイトからの学術研究用テキスト抽出に使用してください。

## セットアップ

### 1. 必要なライブラリのインストール

```bash
pip install -r requirements.txt
```

または個別インストール：

```bash
pip install requests beautifulsoup4 lxml
```

### 2. スクリプトの実行権限設定（macOS/Linux）

```bash
chmod +x blog_scraper.py
```

## 使用方法

### 基本的な使用方法

```bash
python blog_scraper.py --url https://terminalhill.jugem.jp/ --output 抽出記事.txt
```

### オプション一覧

| オプション | 必須 | 説明 | デフォルト値 | 例 |
|------------|------|------|--------------|-----|
| `--url` | ✓ | 抽出対象のブログURL | - | `https://terminalhill.jugem.jp/` |
| `--output` | | 出力ファイル名 | `blog_articles.txt` | `terminalhill_articles.txt` |
| `--max-articles` | | 最大抽出記事数 | 制限なし | `50` |
| `--delay` | | リクエスト間の待機時間（秒） | `1` | `2` |

### 使用例

#### 全記事を抽出
```bash
python blog_scraper.py --url https://terminalhill.jugem.jp/ --output 全記事.txt
```

#### 最初の50記事のみ抽出
```bash
python blog_scraper.py --url https://terminalhill.jugem.jp/ --output サンプル記事.txt --max-articles 50
```

#### サーバー負荷を考慮してゆっくり抽出（2秒間隔）
```bash
python blog_scraper.py --url https://terminalhill.jugem.jp/ --output 記事データ.txt --delay 2
```

## 出力ファイル形式

抽出されたテキストファイルは以下の形式で保存されます：

```

--------------------------------------------------------------------------------
記事の本文内容がここに続きます...

================================================================================

記事 2/150
URL: https://terminalhill.jugem.jp/?eid=124
...
```

## 注意事項

### 著作権とマナー

1. **許可の確認**: 事前にサイト運営者から研究目的での利用許可を得てください
2. **適切な引用**: 研究成果では適切な出典表示を行ってください
3. **サーバー負荷への配慮**: `--delay`オプションで適切な間隔を設定してください

### 技術的制約

1. **JavaScript**: JavaScriptで動的に生成されるコンテンツは取得できません
2. **認証**: ログインが必要なページは取得できません
3. **サイト構造**: サイトの構造によっては記事の検出精度が変わります

## トラブルシューティング

### よくある問題と解決方法

#### 1. 記事が検出されない
```
発見された記事数: 0
```

**解決方法**: サイトの構造が一般的でない場合、スクリプトを調整する必要があります。

#### 2. 接続エラー
```
エラー: https://example.com - 404 Client Error
```

**解決方法**: 
- URLが正しいか確認
- サイトがアクセス可能か確認
- `--delay`を長めに設定

#### 3. 文字化け
- 日本語サイトの場合、エンコーディングが自動検出されます
- 問題がある場合は出力ファイルのエンコーディングを確認

## 研究での活用

### データ分析での注意点

1. **メタデータの活用**: URL、タイトル、日付情報も分析に活用できます
2. **品質チェック**: 抽出されたテキストの品質を確認してください
3. **前処理**: 分析前に不要な文字列の除去などの前処理を行ってください

### 分析用データ形式への変換

抽出したテキストをCSVやJSONに変換したい場合は、別途変換スクリプトを作成することを推奨します。
