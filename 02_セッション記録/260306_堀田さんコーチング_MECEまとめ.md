## エグゼクティブサマリー

1. **サービス設計を「ウニ丼モデル」に転換せよ** — 自分の高付加価値（ウニ）だけを売るのではなく、低コスト人材の作業（ご飯）をまぶして提供することで、顧客満足度と利益率を同時に上げる
2. **経営者マインドでスケール構造を設計せよ** — 自分の稼働を最小化しながら売上を最大化する「デロイト・パートナーモデル」を初期から組み込み、個人スケールから組織スケールへ移行する
3. **リード獲得の仕組みを最優先で自分の手で作れ** — 案件が無限に入る状態を先に作ることが最重要。Meta広告×診断ツールでテストを回し、仕組みが確立するまでは自分でやり切る

## 1. サービス設計：「ウニ丼モデル」への転換

### 現状の課題
- 現行のGravityシリーズは月35万×6ヶ月（総額210万）の伴走型プログラム
- 月2回×1.5時間のセッションで「その場で作る」設計だが、顧客から見ると「1時間半で何が進むのか」という疑問が生じうる
- 自分の高付加価値だけを提供する「ウニ単体販売」状態になっている

### 堀田さんの提案：ウニ丼モデル
- **ウニ（高付加価値）**: 自分の45分の戦略ディスカッション・判断
- **ご飯（低コスト作業）**: 時給2,000〜5,000円の人材による作業（パワポ作成、プロンプトベースの資料作成、リサーチ等）
- ウニだけ売ると「職人しか買わない」が、ウニ丼にすると「3,500〜4,500円でも満足度が上がる」
- 顧客から見ると「こんなにアウトプットが出るのに35万？安くない？」となる状態を作る

### 具体的な設計変更

| 項目 | 変更前 | 変更後 |
|------|--------|--------|
| セッション | 月2回×1.5h、自分が作業 | 月2回×45min、ディスカッション＋判断に特化 |
| 作業 | セッション内で完結 | 「貫禄おばちゃん」（時給安めだが信頼感ある人材）がプロンプトベースで作成 |
| Slack | いつでも問い合わせ可 | 裏方人材が一次対応→石井が後からまとめてレビュー・突っ込み |
| 顧客体感 | 限られた時間で進める | 超ハイパフォーマンスなアウトプットが高速・正確に出続ける |

### 価格戦略：単価を上げるな、時給を上げろ
- 単価を上げると顧客が離れる（目標達成したら切られる）
- 単価を下げるとLTV（継続期間）が伸びる
- **自分の時給が下がらない方式で単価を下げていく**のが正解
- 月35万のままでも、自分の実稼働が月2〜3時間に圧縮されれば時給は大幅上昇

### 継続モデルの設計
- 6ヶ月のGravity終了後 → 「Gravityエグザイル」として継続コース提案
- 月1回×45min＋作業宿題で月20万程度
- 「実は基礎の3〜4段階しかやってない。本当は7〜10段階ある」形式で自然に継続へ
- 顧客が「まだあるんだ」と感じて離脱しにくい設計

## 2. スケール構造：デロイト・パートナーモデルの適用

### 堀田さんの自社事例
- 月額80万円のAIコンサル：自分は月4時間稼働、裏ではバイブコーダー（時給5,000円）が週30時間稼働
- 顧客から見ると「ベンチャーCPOに相当する価値」が出ている
- 原価16万で80万を受注 → 自分の時給は実質17万円
- エンジニアリングパワーで「翌日にはプロトタイプが出てくる」体験を提供

### デロイト・パートナークラスのKPI
- マネージングパートナーのKPI：ミニマム年間6億の案件創出
- 自分でもコンサルをやるし、部下にもやらせるセールスマン
- パートナーのミーティング同席だけで付加価値が成立
- シニアマネージャー以下は「そこそこの人」でも回る

### 経営者マインドの核心
- **「自分がやりたくない仕事は、他人からすればよだれが出るほどやりたい仕事」**
- 思考と行動のうち、多くの人は「思考がつらい」→ 行動だけやればいい仕事は喜ばれる
- 人を雇うことで「くだらない作業を売ることが楽になる」のが組織スケール
- 仕事ができる人ほど「全部自分でやれる」から経営の発想に至らない

### スケールの方程式
```
自分の稼働 = 戦略判断 + ミーティング同席（月数時間）
パートナー/人材の稼働 = 作業実行 + 一次対応（プロンプトベース）
顧客体感 = 超ハイパフォーマンス × 高速アウトプット × 低価格感
利益 = 高単価受注 − 低コスト人材原価 = 高い時給換算
```

### Gravityスキャン（フロント商品）への示唆
- フロント商品の段階で「こんなにやってくれるんだ」という体験を先に作る
- 原価30万かかってもいいから、最初にクオリティをぶっ込む
- 後ろのGravity本契約（月35〜50万）で回収するモデル

### 法人化のタイミング
- **急ぐ必要なし**
- 個人事業主のまま、利益を出さずにAI等への投資で黒字を打ち消す
- ブレイクイーブンちょい超えくらいを維持して投資に回す方がインフレ下では得
- 法人化するのは「取引先が法人を求める場合」のみ
- 今のターゲット（10〜40人規模）なら不要

## 3. リード獲得：仕組みを最優先で構築する

### 堀田さんの根本メッセージ
- **「案件を喜んでいる時間はない。リード獲得の仕組みができれば景色が変わる」**
- 案件が無限に入ってくる状態を先に作ることで、個別案件への執着がなくなる
- 地方案件など目先の機会に飛びつくより、仕組み構築が先

### 現状の取り組み
- Meta広告を5パターン作成、設定完了済み
- 無料診断ツール（12問）→ 組織課題の類型を診断 → 詳細結果取得のためにリード情報取得
- 月10〜30万円程度の広告予算でテスト開始予定

### 堀田さんのフィードバック

| ポイント | 内容 |
|----------|------|
| クリエイティブ | 「個人ポスト風」にした方が刺さる可能性あり（例：「この診断やったら○○型って言われました」） |
| フォーマット | 動画の方が効果的かもしれない。面白さが重要（例：事業に集中できない経営者のストーリー） |
| 媒体 | Instagram＋Facebook。ストーリーズからリンクで診断に誘導 |
| テスト方針 | いろんな切り口で出してみて、数値を見て収束させる |

### 仕組みづくりの鉄則
- **仕組みが確立するまでは自分でやるしかない**
- 人に頼むと「そんなにちゃんとした仕事はしない」
- 仕組みができた後に人に任せる

### 商品設計の見せ方
- LPの見せ方を改善する必要あり
- 「何をやるか」が曖昧だと高く見えてしまう
- 「Month 1でこれをやります、Month 2でこれをやります」と具体的なアウトプットを明示
- 顧客が「意外と安い」と感じるポイントを作る（「もっと高いと思ってました」の空気感）

## 売上ロードマップ（修正案）

| 年度 | 売上目標 | 構成 |
|------|---------|------|
| 2026年（1年目） | 2,500万 | Gravityシリーズ直接デリバリー中心 |
| 2027年（2年目） | 5,000〜8,000万 | パートナー経由売上拡大＋講座事業 |
| 2028年以降 | 1億 | 自分の稼働最小化、パートナー網＋プロダクト＋エクイティ |

### 講座事業（事業中心人事力養成講座）の価格感
- 当初設計：80万×少数 → 非現実的（HR人材は50万以上の自己投資をしない層が多い）
- 修正案：**30万×100人＝3,000万**
- 堀田さんのAI講座（28万）で100人超の実績あり、同水準は到達可能
- 100人集めるには広告仕組み（リード獲得）が必須
