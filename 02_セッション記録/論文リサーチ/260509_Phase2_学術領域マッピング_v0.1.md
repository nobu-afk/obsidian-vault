## 0. 検索クエリの設計原則

### 0.1 クエリ作成のルール（捏造防止 + 効率化）

1. **英語クエリを主軸**（学術論文は英語ベース・Elicit / Semantic Scholar も英語前提）
2. **3-5 語の自然文クエリ**（Elicit は質問形式 + キーワードのハイブリッドが最強）
3. **各コア主張に 3-5 クエリ**（多すぎず・観点が偏らない）
4. **「賛成側」+「反論側」両方のクエリを混ぜる**（確証バイアス回避・Phase 3 で反論論文 5-10 本含めるため）
5. **被引用数 > 50 を目安に質スクリーニング**（あまりに新しい論文は引用ベースで実在確認しづらい）

### 0.2 ツール別の使い分け

| ツール | 強み | 用途 |
|---|---|---|
| **Elicit**（https://elicit.com）| 質問形式の検索 + 論文要約 + 一括比較 | 主リサーチ・候補抽出 |
| **Semantic Scholar API** | 引用関係マップ + DOI / arXiv ID 確認 | 実在確認・引用ネットワーク辿り |
| **Google Scholar** | 被引用数 + 邦訳論文補完 | 補助検索・著者業績確認 |
| **Perplexity** | 補助 Q&A | キーワード探索のみ（**論文引用は捏造リスク高で禁止**）|
