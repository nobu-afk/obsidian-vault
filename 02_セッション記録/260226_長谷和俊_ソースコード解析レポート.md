## Phase 2：ドーパミン発生源の特定

上記8つの衝動行動を並べると、一本の構造線が浮かび上がる。

### 共通パターン：「壊れた構造を見つけた瞬間、再設計せずにいられない」

```
[検知] 不当・非効率な構造を発見する
    ↓ ── ここでドーパミンが最初に発火する
[接続] バラバラだった要素が「一本の線」に繋がる瞬間
    ↓ ── ここでドーパミンが最大化する
[再構築] 新しい仕組みとして社会に実装する
    ↓ ── ここで「本物が届いた」という報酬が返る
```

**具体的に分解すると：**

- **ドーパミン・トリガー（着火点）**：「この構造は間違っている」と気づいた瞬間
  - 村のヒエラルキー、FAXで情報共有する病院、広告費を垂れ流すEC構造、足で稼ぐ営業モデル
  - 彼はこれを"違和感"ではなく"怒り"に近い感情として体験している

- **ドーパミン・ピーク（最大興奮点）**：「点と点が線になる」瞬間
  - 彼自身の言葉：*「私の頭の中では点と点が線になるように、全てのスキルとキャリアがこの原料に繋がっていく未来が見えていました」*
  - 散在する要素が一つの新しい構造として結晶化する瞬間──これが彼の最大の快楽である

- **ドーパミン・リワード（報酬）**：本物の価値が、正しいルートで届いた確認
  - 1200人の応募殺到、1年で1000病院導入、3000病院ネットワーク──数字は「構造が正しく機能した証拠」として報酬になる

## Phase 6：演算可能な命令文への変換

この関数が最もパフォーマンスを発揮する入力条件：

```
IF   既存構造に明確な非効率/不正がある
AND  本物の価値（エビデンス、実績）が存在するが届いていない
AND  複数の異なる領域の知見を接続できる余地がある
THEN hase_kazutoshi() は最大出力を返す
```

逆に、この関数がフリーズする条件：

```
IF   構造に問題がない（壊すものがない）
OR   本物の価値が存在しない（再構築する意味がない）
OR   短期利益のために構造の正しさを妥協するよう求められた
THEN hase_kazutoshi() は停止する
```

**彼は「壊れた構造を検知し、本物の価値が正しく届くように再設計する機能」である。**
