# 翻訳タイム シミュレーション（5パターン×典型翻訳）

> **用途：** WSの翻訳タイム（1人5分）で石井が即座に翻訳するための事前準備
> **構造：** パターン別に「参加者が描くループ」→「社長依存の要素」→「経営者が言うこと」→「石井の翻訳」→「最初の一手」
> **ソース：** 50社の実務経験＋モニター実績＋YouTube発言

---

## 翻訳の型

石井の翻訳は常にこの構造で行う：

```
① 経営者が言うこと（表面）
  ↓
② 石井の翻訳（構造）
  ↓
③ だから最初にやるべきこと（一手）
```

**翻訳のコツ：**
- 経営者の言葉を否定しない。「それは違います」ではなく「それは正しい。ただ、裏にある構造は…」
- 「力」の名前を具体的に言い換える。「営業力」→「案件見極め基準の言語化力」
- 最初の一手は「明日からできること」レベルまで具体化する

## パターン1：単循環型

### 典型的なループ

```
[営業] → [提案・見積] → [受注] → [納品・サービス提供] → [顧客満足] → [紹介・リピート]
   ↑                                                                    │
   └────────────────────────────────────────────────────────────────────┘
```

### 社長依存しやすい要素：営業＋提案

### 翻訳パターン集

**翻訳1-A：「営業力が足りない」**

| ステップ | 内容 |
|---|---|
| 経営者が言う | 「営業力が足りない。自分以外に売れる人がいない」 |
| 石井の翻訳 | 「営業力ではなく、**案件の見極め基準が社長の頭の中にしかない**。どの案件を追うべきか、どの顧客に時間を使うべきかの判断基準が言語化されていないから、社長以外が営業できない」 |
| 最初の一手 | 「直近受注した5件と失注した5件を並べて、何が違ったかを書き出す。そこに見極め基準が隠れている」 |

**翻訳1-B：「採っても辞める」**

| ステップ | 内容 |
|---|---|
| 経営者が言う | 「採用しても半年で辞める。うちには合わない人ばかり来る」 |
| 石井の翻訳 | 「採用基準が"社長の過去の自分"になっている。事業の勝ち筋ループで**今足りない要素の力**から逆算した採用基準がないから、"良さそうな人"を採って"思ってたのと違う"が起きる」 |
| 最初の一手 | 「ループの中で一番詰まっている要素を1つ決めて、その要素を回すのに必要なスキルを3つ書く。それが採用基準」 |

**翻訳1-C：「任せたいのに任せられない」**

| ステップ | 内容 |
|---|---|
| 経営者が言う | 「権限移譲したいが、結局自分が判断している」 |
| 石井の翻訳 | 「権限を渡す"気持ち"はある。でも**判断基準が言語化されていない**から、渡しても"社長ならどうしますか"が返ってくる。渡しているのは権限ではなく作業。判断基準ごと渡さないと権限移譲にならない」 |
| 最初の一手 | 「今週、部下から"確認"が来た案件を3つメモする。それぞれ自分が何を基準に判断したかを書く。それが渡すべき判断基準」 |

**翻訳1-D：「紹介が来ない」**

| ステップ | 内容 |
|---|---|
| 経営者が言う | 「既存客からの紹介が少ない。口コミが広がらない」 |
| 石井の翻訳 | 「紹介を"偶然"に頼っている。**紹介が起きる仕組み（タイミング・依頼方法・インセンティブ）が設計されていない**。満足→紹介の間にもう1ステップ必要」 |
| 最初の一手 | 「納品完了後のフォロー連絡で"もし同じ課題を抱えている方がいたらご紹介ください"の一言を定型化する」 |

## パターン2：ストック型（SaaS/D2C/サブスク）

### 典型的なループ

```
[マーケ・認知] → [リード獲得] → [商談・成約] → [オンボーディング] → [継続利用] → [アップセル/紹介]
       ↑                                                                          │
       └── [収益 → マーケ再投資] ←─────────────────────────────────────────────────┘
```

### 社長依存しやすい要素：商談（FS）＋解約防止（CS）

### 翻訳パターン集

**翻訳2-A：「解約が多い」**

| ステップ | 内容 |
|---|---|
| 経営者が言う | 「チャーンが高い。入っても3ヶ月で辞める」 |
| 石井の翻訳 | 「解約の問題ではなく、**最初の30日で"価値を体感する体験"が設計されていない**。顧客は"いつか便利になるだろう"では待てない。初月に"あ、これ手放せない"の瞬間を作れているか」 |
| 最初の一手 | 「直近の解約者5名に"いつ辞めようと思ったか"を聞く。大半が最初の2週間で決まっている」 |

**翻訳2-B：「MRRが伸びない」**

| ステップ | 内容 |
|---|---|
| 経営者が言う | 「新規は取れているが、MRRの伸びが鈍い」 |
| 石井の翻訳 | 「CSが"解約防止"しかやっていない。**既存顧客の"次の課題"を発見してアップセルに繋げる導線がない**。CSの役割が守りだけで、攻めがない」 |
| 最初の一手 | 「CSチームのKPIに"既存顧客からのアップセル提案数"を追加する。月1件でいい」 |

**翻訳2-C：「商談が社長じゃないと決まらない」**

| ステップ | 内容 |
|---|---|
| 経営者が言う | 「最終的に自分が出ないと大型案件が決まらない」 |
| 石井の翻訳 | 「営業プロセスの**どのフェーズで社長が必要かが定義されていない**。全商談に出るのではなく、"この条件の案件は社長が出る"の基準を作れば、8割の商談は任せられる」 |
| 最初の一手 | 「直近の成約10件を"社長が出た/出なかった"で分ける。社長なしで決まった案件の共通点が、任せられる案件の基準」 |

## パターン3：M&A型

### 典型的なループ

```
[CORE：既存事業深化] → [実績・キャッシュ創出] → [資金調達] → [M&A候補発掘] → [買収・PMI]
         ↑                                                                    │
         └─── [BASE：基盤統合（人材・ノウハウ・バックオフィス）] ←──────────────┘
```

### 社長依存しやすい要素：M&A候補発掘＋PMI判断

### 翻訳パターン集

**翻訳3-A：「PMIがうまくいかない」**

| ステップ | 内容 |
|---|---|
| 経営者が言う | 「買収した会社がなかなか馴染まない。文化が合わない」 |
| 石井の翻訳 | 「"文化が合わない"のではなく、**COREの勝ち筋ループを買収先に翻訳する共通言語（BASE）がない**。評価基準もオンボーディングもバラバラだから、"同じ会社"になれない」 |
| 最初の一手 | 「CORE事業と買収先で"共通にすべきもの"と"各社固有でいいもの"を分ける。評価基準と行動指針は共通。業務プロセスは各社固有」 |

**翻訳3-B：「事業拡大と組織のスピードが合わない」**

| ステップ | 内容 |
|---|---|
| 経営者が言う | 「新しい事業をどんどん取りに行きたいが、組織が追いつかない」 |
| 石井の翻訳 | 「事業拡大を先行させて、BASEを後から作ろうとしている。**建物の4階から建てている**。BASEの構築が買収スピードに追いつかないと、買うたびに組織が薄まる」 |
| 最初の一手 | 「次の買収の前に、"買収先を受け入れる体制（BASE）のチェックリスト"を作る。人事・経理・法務の統合手順が3つずつあればいい」 |

## パターン4：PF型

### 典型的なループ

```
[認知事業（赤字OK）] → [会員獲得] → [PF内回遊] → [収益事業で課金] → [収益でクラスタ拡大]
                                                                         │
                                                              [新領域への進出] ←┘
```

### 社長依存しやすい要素：新クラスタ拡張の意思決定＋事業間導線

### 翻訳パターン集

**翻訳4-A：「面は取れたが伸びない」**

| ステップ | 内容 |
|---|---|
| 経営者が言う | 「事業はたくさんある。でも全体の売上が思ったほど伸びない」 |
| 石井の翻訳 | 「各事業は個別に回っているが、**事業間の回遊導線が設計されていない**。認知事業で獲得した会員が収益事業に自然に流れる仕組みがないから、点で終わっている」 |
| 最初の一手 | 「"どの事業の顧客が、どの事業に移動しているか"のデータを取る。回遊が起きている導線と起きていない導線が見える」 |

**翻訳4-B：「赤字事業を切るべきか迷う」**

| ステップ | 内容 |
|---|---|
| 経営者が言う | 「この事業単体では赤字。でも切ると会員が減りそう」 |
| 石井の翻訳 | 「各事業の役割が**"認知（赤字OK）"と"収益（黒字必須）"で定義されていない**。役割が曖昧だから、切るべきか残すべきかの判断基準がない」 |
| 最初の一手 | 「全事業を"認知事業"と"収益事業"に分類する。認知事業は"何人の会員を連れてきているか"で評価。収益事業は"黒字かどうか"で評価。評価軸が違う」 |

## パターン5：エキスパート型

### 典型的なループ

```
[専門知識の発信] → [信頼構築] → [案件獲得] → [デリバリー] → [実績・知見の蓄積]
                                                                    │
                                                              [発信に還元] ←┘
```

### 社長依存しやすい要素：全部（特にデリバリー）

### 翻訳パターン集

**翻訳5-A：「自分の時間が足りない」**

| ステップ | 内容 |
|---|---|
| 経営者が言う | 「案件は来るが、自分がデリバリーしないと品質が保てない」 |
| 石井の翻訳 | 「デリバリーの中で**"自分にしかできない判断"と"誰でもできる作業"が切り分けられていない**。全部自分でやっているから時間が足りないが、品質を決めているのは"判断"の部分だけ。リサーチ・資料作成・データ整理・定型連絡などの"作業"は品質に影響しない。ほとんどの場合、自分にしかできない判断は全体の2-3割程度」 |
| 最初の一手 | 「直近の案件で自分がやったことを全部書き出す。"判断"と"作業"に分ける。作業は明日からパートナーやAIに渡せる」 |

**翻訳5-B：「スケールしない」**

| ステップ | 内容 |
|---|---|
| 経営者が言う | 「1対1でしかサービスが届けられない。スケールしない」 |
| 石井の翻訳 | 「**知識が"頭の中"にしかない**。講座化・テンプレート化・プロセス化すれば、1対多でも1対1と同じ価値を届けられる。スケールしないのではなく、スケールする形に変換していないだけ」 |
| 最初の一手 | 「毎回のセッションで繰り返し話していることを3つ書き出す。それがコンテンツ化の第一候補」 |

**翻訳5-C：「発信が続かない」**

| ステップ | 内容 |
|---|---|
| 経営者が言う | 「Note書かなきゃ、SNSやらなきゃ、でも時間がない」 |
| 石井の翻訳 | 「発信を"新しく作る"と思っているから辛い。**デリバリーで話したことを発信に還元する仕組みがない**。セッション→録音→文字起こし→記事化のパイプラインがあれば、発信は"作る"のではなく"流れる"」 |
| 最初の一手 | 「次のクライアントセッションを録音して、その内容を1本のNote記事にする。新しく書くのではなく、話したことを変換する」 |

## 翻訳の精度を上げるための質問リスト

ループを見ただけでは翻訳の精度が上がらない場合、以下の深掘り質問を使う（1人5分の中で1-2問）：

| 質問 | 何がわかるか |
|---|---|
| 「このループの中で、一番"楽しい"と感じる要素はどれですか？」 | want toの所在。ここは仕組み化しにくい（本人がやりたい） |
| 「このループが止まったら、最初にどこが止まりますか？」 | 最大のボトルネック。本人の直感が最も正確 |
| 「この1年で、ループの中で一番変わった要素はどれですか？」 | 変化への対応力。変わっていない要素が構造的に固着している |
| 「"この人が辞めたらこの要素が止まる"という人はいますか？」 | エース依存の特定 |

## WSの翻訳タイム進行スクリプト

```
石井：「Aさん、画面共有お願いします」

（ループを見る・5秒沈黙）

石井：「なるほど。[パターンX]ですね。
      ループの中で社長にチェックが入っているのは[Y]と[Z]。

      Aさん、[Y]について"[経営者が言いそうな課題]"と感じていませんか？」

Aさん：「はい、まさにそうです」

石井：「ですよね。でも私の見立ては少し違います。
      [Y]の問題は[表面的な力]ではなく、
      **[翻訳した構造的な力]**が足りていないことです。

      なぜなら、[1-2文で構造を説明]。

      最初の一手は[具体的なアクション]です。
      明日からできます」

（次の参加者へ）
```

**1人5分の内訳：**
- ループを見る＋質問1つ：1分
- 翻訳を伝える：2分
- 参加者の反応＋補足：2分
