### 0-2. 各タスクの詳細説明とチェックポイント

#### **フェーズ1：全体構想（1月）**

**1-1 現行課題整理**
- **実施内容**:
  - 社員ヒアリング（10-15名、各部署から選定）
  - 全社員アンケート調査（匿名可）
  - 現行の評価・報酬に関する課題の抽出
  - 事業戦略・バリューの再確認
- **チェックポイント**:
  - ヒアリング対象者の選定（多様性を確保）
  - アンケートの質問設計（課題を引き出す質問）
  - 課題の優先順位付け
- **成果物**: 課題整理書、ヒアリング結果、アンケート結果、バリュー定義書

**1-2 制度の方向性すり合わせ**
- **実施内容**:
  - 経営陣との対話（制度の目的・方向性の確認）
  - 人事制度整備プロジェクトチームの組成
  - 外部支援の検討（必要に応じて）
  - 予算・リソースの確認
- **チェックポイント**:
  - 経営陣のコミットメント確認
  - プロジェクト体制の明確化
  - スケジュール・予算の合意
- **成果物**: 方向性合意書、プロジェクト体制図、予算計画

**1-3 全体構想の策定**
- **実施内容**:
  - 人事制度の全体像の設計
  - 各制度要素（等級・評価・報酬）の関係性の整理
  - 導入スケジュールの確定
  - 成功指標（KPI）の設定
- **チェックポイント**:
  - 制度要素間の整合性
  - スケジュールの実現可能性
  - 成功指標の明確化
- **成果物**: 全体構想書、制度設計図、スケジュール表、KPI設定書

#### **フェーズ2：等級制度（2月）**

**2-1 キャリアパス**
- **実施内容**:
  - 社員の成長ステップの設計
  - マネジメントコースと専門職コースの基本構造
  - キャリアパスの可視化（図表作成）
- **チェックポイント**:
  - 各コースの明確な区別
  - 成長ステップの妥当性
  - 社員が理解しやすい表現
- **成果物**: キャリアパス図、キャリアパス説明書

**2-2 コース体系**
- **実施内容**:
  - 各コースの定義と特徴
  - コース間の移行ルール
  - 将来的な複線化の方向性
- **チェックポイント**:
  - コースの定義が明確か
  - 移行ルールが公平か
  - 将来の拡張性
- **成果物**: コース体系図、コース定義書

**2-3 等級体系・等級定義**
- **実施内容**:
  - 等級の階層数と名称の決定（推奨：4-5段階）
  - 各等級の定義（役割・責任・能力・成果）
  - 等級と職種の対応関係
- **チェックポイント**:
  - 等級数が適切か（多すぎず、少なすぎず）
  - 等級定義が具体的か
  - 職種間の公平性
- **成果物**: 等級制度案、等級定義書

**2-4 昇格・降格ルール**
- **実施内容**:
  - 昇格要件の明確化（在籍期間、成果要件、能力要件）
  - 昇格プロセスの設計（自己申告、マネージャー推薦、面接）
  - 降格ルールの設定（例外的なケース）
- **チェックポイント**:
  - 昇格要件が明確で公平か
  - プロセスが透明か
  - 降格ルールが適切か
- **成果物**: 昇格規程案、昇格プロセスフロー

**2-5 格付け移行シミュレーション**
- **実施内容**:
  - 現職員の等級仮振り
  - 移行時の影響分析（給与変動、評価への影響）
  - 激変緩和措置の必要性の判断
- **チェックポイント**:
  - 仮振りの妥当性
  - 影響範囲の把握
  - 激変緩和の必要性
- **成果物**: 移行シミュレーション結果、格付け一覧表、影響分析レポート

## 1. 2026年1-6月：初期人事制度設計の詳細アウトライン

### 1-1. 等級制度（グレード）設計の詳細

#### 等級構成案
```
【案1：シンプル4段階型】
- L1: エントリーレベル（新卒・未経験者）
- L2: ジュニア（1-3年目、基本的な業務を遂行）
- L3: シニア（3-5年目、自律的に業務を遂行、他者を支援）
- L4: リード（5年以上、チームを牽引、意思決定に参画）

【案2：5段階型（マネージャー層を含む）】
- L1: エントリーレベル
- L2: ジュニア
- L3: シニア
- L4: リード
- L5: マネージャー（チームマネジメント責任）
```

**推奨**: 初期は案1の4段階型。50名体制を超えたら案2に拡張。

#### 等級定義の要素
- **役割**: 期待される役割・責任
- **能力**: 必要なスキル・知識・経験
- **成果**: 期待される成果のレベル
- **影響範囲**: 個人→チーム→組織への影響度

#### 等級設計のプロセス
1. 現職員の等級仮振り（現状把握）
2. 等級定義のドラフト作成
3. 経営陣・マネージャーでのレビュー
4. 社員への説明とフィードバック収集
5. 最終確定

### 1-2. 評価制度設計の詳細

#### 評価軸の設計

**2軸評価モデル（推奨）**
```
【軸1：成果（What）】50-60%
- 目標達成度（MBO/OKR）
- 定量的成果（KPI達成）
- プロジェクト成果

【軸2：行動（How）】40-50%
- バリュー体現度
- 協働性・チームワーク
- 成長マインドセット
- リーダーシップ（該当者のみ）
```

#### 評価手法の選択

**推奨：ハイブリッド型**
- **目標設定**: OKR（全社アラインメント重視）
- **評価**: MBO的アプローチ（達成度評価）+ バリュー評価
- **報酬連動**: 成果軸と行動軸の総合評価

**評価頻度**
- **目標設定**: 半期ごと（4月・10月）
- **中間レビュー**: 四半期ごと（7月・1月）
- **評価実施**: 半期ごと（9月・3月）

#### 評価プロセス
1. **目標設定フェーズ**（4月・10月）
   - 全社OKRの共有
   - 個人OKR/MBOの設定
   - マネージャーとの合意形成

2. **中間レビューフェーズ**（7月・1月）
   - 進捗確認
   - 必要に応じた目標調整
   - フィードバック

3. **評価フェーズ**（9月・3月）
   - 自己評価
   - 上司評価
   - 評価面談
   - 評価結果の確定

#### 評価シートの構成
- 基本情報（氏名、部署、期間）
- 目標設定（OKR/MBO）
- 成果評価（達成度、成果の質）
- 行動評価（バリュー体現度、協働性など）
- 総合評価（S/A/B/C/D または 5段階）
- フィードバック（強み・改善点）
- 次期目標への示唆

---

### 1-3. 給与テーブル設計の詳細

#### 給与レンジの設定方法

**ステップ1：現状給与の整理**
- 現職員の給与を等級ごとに集計
- 中央値・平均値を算出

**ステップ2：市場水準の把握**（簡易版）
- 業界・職種別の給与水準をリサーチ
- スタートアップ向けの給与レポートを参照

**ステップ3：給与レンジの設定**
```
例：L2（ジュニア）の場合
- 最低額: 現状の最低給与
- 中央値: 市場水準を参考に設定
- 最高額: 中央値の1.3-1.5倍程度
```

#### 昇給ルール
- **定期昇給**: 年1回（評価結果に応じて）
  - S評価: 5-7%
  - A評価: 3-5%
  - B評価: 1-3%
  - C評価: 0%
  - D評価: 改善計画

- **昇格昇給**: 等級が上がった場合
  - 昇格時の給与調整（現給与の10-20%アップ）

#### 賞与の考え方
**推奨：全社業績連動型 + 個人評価反映**
- 基本賞与 = 全社業績 × 個人評価係数
- 全社業績: 売上目標達成度など
- 個人評価係数: S=1.2, A=1.0, B=0.8, C=0.6, D=0

---

### 1-4. 職務記述書（JD）の詳細

#### JDに含める要素
1. **基本情報**
   - 職種名
   - 所属部署
   - 等級

2. **役割・責任**
   - 主要な業務内容
   - 期待される成果
   - 意思決定権限

3. **必要なスキル・経験**
   - 必須スキル
   - 歓迎スキル
   - 必要な経験年数

4. **キャリアパス**
   - 前の等級での役割
   - 次の等級への道筋

#### JD整備の優先順位
1. **最優先**: マネージャー層、リード層
2. **高優先**: 各部署の代表的なポジション
3. **中優先**: その他のポジション（段階的に整備）

---

### 1-5. キャリアパス設計の詳細

#### キャリアパスの基本構造

**シンプルな2コース型**（初期）
```
【専門職コース】
L1 → L2 → L3 → L4（専門性を深める）

【マネジメントコース】
L1 → L2 → L3 → L4 → L5（マネジメント責任）
```

**将来的な複線化**（2027年以降）
- 専門職コースの細分化（エンジニア、営業、マーケなど）
- マネジメントコースの階層化（チームリード、部門マネージャーなど）

#### 昇格要件の明確化
- **在籍期間**: 各等級での最低在籍期間（例：L2→L3は1年以上）
- **成果要件**: 評価結果（例：直近2回の評価でA以上）
- **能力要件**: 次の等級に必要なスキルの習得
- **昇格プロセス**: 自己申告 + マネージャー推薦 + 面接

---

### 1-6. 評価者研修の詳細

#### 研修内容
1. **制度理解**（1時間）
   - 人事制度の全体像
   - 評価制度の目的と意義
   - 評価軸・評価基準の説明

2. **評価スキル**（2時間）
   - 目標設定の支援方法
   - フィードバックの方法
   - 評価の公平性を保つポイント
   - 評価面談の進め方

3. **実践演習**（1時間）
   - ケーススタディ
   - ロールプレイング

#### 研修の実施方法
- **集合研修**: 全マネージャーを集めて実施
- **オンライン研修**: 録画視聴 + 質疑応答
- **個別フォロー**: 必要に応じて1on1で補足説明

- 年次: 制度全体の見直し

**モニタリング項目**
- 評価結果の分布（偏りがないか）

## 4. リスクと対策

### 4-1. 想定されるリスク

| リスク | 影響度 | 発生確率 | 対策 |
|--------|--------|---------|------|
| 制度の形骸化 | 高 | 中 | 定期的な見直し、現場の声を反映 |
| 評価の不公平感 | 高 | 中 | 評価者研修の徹底、360度評価の導入 |
| 運用負荷の増大 | 中 | 高 | シンプルな設計、ツールの活用 |
| 制度変更による混乱 | 中 | 低 | 段階的な導入、十分な説明 |
| 文化とのミスマッチ | 高 | 低 | バリューとの整合性確認、対話重視 |

### 4-2. 緊急時の対応

**評価プロセスが機能しない場合**
- 原因分析（評価者スキル不足？制度設計の問題？）
- 緊急の評価者研修実施
- 必要に応じて制度の簡素化

**社員からの不満が噴出した場合**
- 全社員ヒアリングの実施
- 課題の可視化と優先順位付け
- 経営陣による説明会の開催
- 制度の見直し（必要に応じて）
