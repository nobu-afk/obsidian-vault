# SSOT ── LP コピー言語ガイドライン v1.0

> **位置付け：** GrowthFix 全 LP（CODE / Scan / Recruit / Cultivate / Coaching / Orbit / Shift / WhitePaper）のコピーで使用する言語の **3 階層分類 SSOT**。社内用語の対外露出を機械的に防ぎ、世界観語と具体性訴求を最大化する。
> **作成：** 2026-05-08（v1.0 確立）
> **連動：** `SSOT_用語と定義.md`（参謀名・タグライン）／`商品_5サービス詳細.md`（仕様詳細・社内用語の SSOT）／`lint_consistency.sh`（機械チェック）

---

## 0. 設計原則（不可侵）

### 0.1 LP の役割定義（260508 確立）

```
LP = 世界観への引き込み装置（最低限の情報・営業 30 分商談で口頭補足）
営業資料 = 受注クロージングの場（社内用語含む詳細仕様の説明）
```

→ LP では社内用語を **ゼロにする**。営業 30 分商談で **1 つずつ口頭で世界観に引き込みながら**説明する。

### 0.2 顧客動線の現実

```
コーポレート TOP → CODE / Scan / Coaching 申込（LP 経由可・体験商品のみ）
                ↓ Scan 4 型判定確定
                ↓ 石井 30 分個別商談 ★ ここで社内用語を口頭で世界観に翻訳
                R / C / Shift 提案 → 受注
```

→ R/C/Shift/Coaching の LP は **受注クロージングの場ではなく、信頼形成 + 商談予約の入口**。

### 0.3 「世界観 ≠ 社内用語」の区別

- **世界観語**：独自表現でブランディングコアになる語（残す）
- **社内用語**：独自分類・固有名詞・社内日付付き原則（LP では削る）

## 5. 削除パターン → 言い換えテンプレート

### Before / After 例

| 社内用語（Before）| 世界観 + 業界一般用語（After）|
|---|---|
| 「思想 3 + 設計 4 + 運用 5 の 12 要素を体系化」| **「採用基盤を体系的に組み立てる」** |
| 「3 ヶ月予言の書（A/B/C 気づき）を Week 1-2 で納品」| **「3 ヶ月後の経営者の状態を Week 1-2 で事前確定する設計図を納品」** |
| 「9 マス診断（Why × 才能 × 偏愛 × 整合 / ズレ / 漏れ）」| **「経営者の引力が現場で機能しているかを立体可視化」** |
| 「動く採用基盤 KPI（社長依存度 100% → 50%）」| **「社長が面接に出続けなくても採れる構造を 3 ヶ月で立ち上げる」** |
| 「C-5 ネガティブ・ケイパビリティ判定」| **「経営者の覚悟確信度（答えのない時間を待てる力）」** |
| 「Coaching is not frontline 原則（260429 確立）」| **「関係構築前提のサービス設計」** |
| 「Week 0 体験設計書 5 モジュール（30 分アライン → 296 時間 → 入社初日 → 1-3-6 ヶ月）」| **「内定承諾〜入社後 6 ヶ月の離脱予防設計を別冊で納品」** |
| 「組織 OS 診断書 6 章（B-21 / B-22 / B-1 / B-2 / B-10 / B-11）」| **「組織の引力を 6 つの観点で診断する設計図を納品」** |
| 「思想 3：AI 時代の育成型採用論／リスニングコンパス／インタビュー 5 原則」| **「採用判断軸の体系化（独自フレーム）」** |
| 「7 特徴：問題の再定義力／フレームワーク地図的提供／3 モード切替…」| **「3 モード深掘り対話 ＋ 経営判断軸のコード化」**（3 つに圧縮）|

### 言い換え原則

1. **数字付きカテゴリ**（「思想 3」「12 要素」「6 章」）→ 「体系的な〜」「複数の観点で〜」「複合的な〜」
2. **社内固有名詞**（「リスニングコンパス」）→ 業界一般語（「面接の聴き方フレーム」）or 削除
3. **社内日付付き原則**（「260429 確立」）→ 完全削除（年月日は対外公開不要）
4. **章番号**（「B-21」「B-22」）→ 完全削除（中身を業界一般語で表現）
5. **メソッド名**（「TDD 経営」）→ 一般化（「事前設計の検証可能化」）

## 9. lint_consistency.sh への組込候補（5/9 weekly-close）

```bash
# 章番号検出（HIGH）
grep -E "B-[0-9]+|A-[0-9]+|C-[0-9]+"

# 社内日付付き原則検出（HIGH）
grep -E "26[0-9]{4} 確立|26[0-9]{4} 確定|26[0-9]{4} シャープ化"
```

### 警告レベル

- **HIGH**：上記パターンが LP に検出された場合
- **MEDIUM**：FAQ で解説している社内用語（本文には出ていないが FAQ で出ている）
- **LOW**：曖昧な独自固有名詞（中央値判定）
