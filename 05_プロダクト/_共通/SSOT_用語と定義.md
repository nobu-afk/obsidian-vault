## 0.5. 命名二層運用ルール（260503 B 案確定・最重要）

Shift シリーズは **外的命名（顧客接点）** と **内的命名（社内設計・本 SSOT）** を **二層運用**する。CODE / Scan / Coaching / Orbit は単一命名（変更なし）。

- **顧客接点アウトプット**（LP / WP / Note / セミナー / メール / 見積書 / DM / SNS / 提案書）は **外的命名（Gravity Recruit / Activate / Shift）** で統一

### Gravity Shift（R+A 複合）の minimal LP 運用ルール（260503 確定・Orbit パターン適用）

> **戦略：** Gravity Shift（R+A 複合パッケージ・150 万・6 ヶ月）は **minimal LP 運用**（Orbit パターン）。顧客ジャーニーとして「採用 or 躍動」のピン痛で来店する経営者が大半なので、**フルパッケージは商談時に提案する**運用に統一。

> - 各 LP / コーポレート / Hub / 各 LP のフッター：**Gravity Shift カード/リンクは載せない**（Orbit と同じ運用）
> - `/gravity-shift/` は minimal LP（1 ページ・概要 + 価格 + Scan/Recruit/Activate への誘導）として直接 URL アクセス時のみ到達
> - **Recruit / Activate LP の pricing 欄の Shift 言及は保持**（10 万割引メリットの明示・営業前段の心理形成）
> - **Scan の 4 型「不毛型」推奨は「Recruit + Activate 順次（商談時に Shift 提案）」**に統一
> - サービス内容詳細・契約条件は商談時に口頭説明＋提案書記載
