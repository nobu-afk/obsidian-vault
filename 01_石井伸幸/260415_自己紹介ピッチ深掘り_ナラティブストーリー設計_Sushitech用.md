## 🎨 ナラティブアーク設計（ピッチの構造）

### フレーム候補A：偏愛の時間対比型（堀田スタイル）

```
[導入] 10歳で〇〇 → [現在] 30年後の今 → [現在地]何をしているか
```

例：
> 10歳で児童会長になった時、組織には重力がなかった。
> それから30年、組織の重力だけを研究してきた。
> 今、経営者の組織に引力を実装している。

### フレーム候補B：傷→再生→使命型（英雄の旅）

```
[原体験] 灰色の瞬間 → [再生] 気づき → [使命] 今やっていること
```

例：
> 高校3年間、世界が灰色に見えていた。
> でも大学で気づいた。灰色は構造が引力を失った状態だと。
> 今、経営者を灰色から引きずり出す仕事をしている。

### フレーム候補C：2層Why貫通型（自己主権）

```
[深層Why] 自己主権への渇望 → [関心の広さ] 金本位制／組織／自分 → [焦点] 経営者
```

例：
> 僕の人生のテーマは「自己主権」です。
> 価値の主権（金本位制）、時間の主権（やりたい時にやりたいことを）、そして組織における社長の主権。
> 社長の自己主権を事業から食わせない、それが仕事です。

### フレーム候補D：観察者型（幼稚園の関数）

```
[発現] 幼稚園のお誕生日カード → [軌跡] 観察の30年 → [到達] Gravity
```

例：
> 幼稚園のお誕生日カードに「せんせいのおはなしをさいごまできいてくれるのぶちゃん」って書かれてたんです。
> あの頃から今まで、僕はずっと観察者でした。
> 観察し続けて、50社を診断して、辿り着いたのがGravityです。

## 🔄 深掘りセッションで確定した真のWhy

> **「才能ある人が、本来の自分を取り戻せていない状態」への違和感**

### 行動原理（10歳から30年一貫）
```
[1] 自発じゃない（周りから推薦される）
 ↓
[2] でも「誰もやらないなら、自分が」という使命感で引き受ける
 ↓
[3] 引き受けたら本気で穴を埋めに行く（立ち上げ＝カラフル）
 ↓
[4] 埋め終わる前に次の景色を見に行く（去る）
```

### Gravity の真の定義（4/15確定）
> **経営者が「本来の自分の使い方」を理解し、その通りに生きる状態を作る装置**
>
> 使い方がカチッとはまった経営者の組織は、
> ブラックホールのような吸引力で、人を引き寄せる。

### サービス体系の意味
- Coaching だけでは経営者は現場で組織の鎖に引き戻される
- **Gravityシリーズ（Scan/Gravity/Shift/Orbit）は「本来の自分で居続けられる環境」を作る装置**
- 全サービスが「経営者を本来の自分に戻す」というゴールに収束

## 📇 名刺コピーへの還元（ナラティブから派生）

### サービス方針（260415確定）：CODE + Scan に変更

**判断根拠：**
- Sushitech は一期一会の場＝**次会う口実**が最重要
- CODE（10万・個人診断）＝「隠れた才能を丸裸にする」→ 個人の好奇心に刺さる
- Scan（10万・組織診断）＝「隠された伸びしろを丸裸にする」→ 経営者の組織課題に刺さる
- 両方とも**敷居が低く・体験価値が高い**入口
- Gravity（210万）は裏面から外す（Sushitechでいきなりは重い）

### コピーの方向性（深掘り後に確定）

- CODE：「**隠れた才能を丸裸にする**」をベースに、石井のナラティブと紐付いた1行へ
- Scan：「**隠された伸びしろを丸裸にする**」をベースに、同上
- 両方とも **「丸裸にする」** という過激語で統一 → 堀田「個性・誇張」に整合
