# 石井伸幸のコーチング特性分析

> 2026年4月4日作成。02_セッション記録の全セッション分析と01_石井伸幸のプロファイリングを統合した考察。

---

## コーチングの7つの特徴

### 1. 問題の再定義力
クライアントが持ち込むフレームを壊し、本質的な構造問題に置き換える。長島さんの「フォロワーシップ問題」→「完璧主義中毒」、長谷さんの「8:2トレードオフ」→「構造問題」など。全セッションで一貫した最大の武器。

### 2. フレームワークの地図的提供
免疫マップ、キャリアアンカー、Jobs vs 宮崎モデル、Will vs Can等を「処方箋」ではなく「地図」として提供。1セッション内で複数フレームワークを展開する。

### 3. コーチング×コンサル×ソマティックのモード切替
1セッション内で3モードを行き来する。頭の理解→身体の変容→組織の変化の3層を1人でカバーできる点は市場にほぼ類似がない。

### 4. セッション前の緻密な分析設計
認知エラーの階層化、セッションシナリオの分単位設計、事前レポート作成。即興より「構造を読み解いてから介入する」スタイル。

### 5. 90分×深掘り型
堀田さん（60分で6-9個の意思決定）に対し、90分で2-3個の深い変容を目指す。理論構築型・身体介入ありの「翻訳型コーチング」。

### 6. クライアントの認知構造をコード化
長島さんに対して12個の認知エラーを特定・階層化・相互依存分析。通常のコーチングでは行わないエンジニアリング的アプローチ。

### 7. 一貫した弱点：契約と収束
開始時のゴール合意が曖昧→中盤で知的探索が広がる→収束せずに時間切れ、というパターンが繰り返される（得点：64〜88/100、収束が常にボトルネック）。

## プロファイリングとの関連

| コーチングの特徴 | ソースコードv5との対応 |
|----------------|---------------------|
| 問題の再定義力 | ASICが人間ドメインで発火。市場の構造変化検知と同じ関数 |
| フレームワーク多層展開 | 「フレームワーク中毒」の直接的発現。強みだが感情の「間」を奪うリスクも |
| モード切替 | Layer 3の実行パス切替（org/person/market）と同構造 |
| 事前分析の緻密さ | 流動性知能65-70%＋「理解が先、行動が後」の認知スタイル |
| 90分深掘り型 | 空（Void）での構造吸収→結晶化プロセスに時間を要する設計 |
| 認知構造のコード化 | 再帰的メタ認知 f(f(x)) の対クライアント適用 |
| **契約・収束の弱さ** | **「触媒不在」パターン。構造を把握した時点で自分の中では完了しており、固化フェーズへの関心が薄い** |

### 最大の実用的示唆

収束の弱さはソースコードv5の「構造が固まる前に去る」ドライブの表出。セッション最後10分を意識的に「固化フェーズ」として設計することで改善可能。

### 権威性に関するジレンマ

権威を拒否する認知特性（Layer 2のREJECT条件）が、権威を獲得しなければ集客できないというコールドスタート問題の一因。肩書きではなく「証拠的権威」（ケーススタディ、フレームワークの流通、出版）で突破する戦略が整合する。
