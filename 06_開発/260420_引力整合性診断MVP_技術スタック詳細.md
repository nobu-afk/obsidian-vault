## 🎯 MVP スコープ確認（骨子から引用）

**最小機能：** 1on1録音 → 幹部の引力タイプ診断 ＋ 熱量4象限マッピング ＋ 思考OS診断

**技術的に実装が必要な3層：**
1. **Input層**：音声アップロード＋組織情報入力
2. **Process層**：文字起こし → AI診断 → HTMLレポート生成
3. **Output層**：レポート表示・ダウンロード

---

## 📐 技術スタック（MVP版）

### Layer 1：フロントエンド／UI

| 項目 | 選定 | 理由 |
|---|---|---|
| **言語** | HTML5 + vanilla JS | 既存CODE/Blueprint と同じスタックで統一・保守容易 |
| **スタイリング** | Vanilla CSS（既存 `site-base.css` 流用） | 既存LP群との一貫性 |
| **フォーム** | HTML `<form>` + `<input type="file">` + `<textarea>` | フレームワーク不要・軽量 |
| **状態管理** | localStorage（既存診断_本番/app.js パターン踏襲） | ページ遷移時にデータ保持 |
| **プログレスバー** | vanilla JS（既存 診断_本番/app.js 流用） | 処理時間可視化 |

### Layer 2：音声文字起こし

**選定肢：**

| 選択肢 | メリット | デメリット | 料金 |
|---|---|---|---|
| **A. OpenAI Whisper API** | 高精度・日本語強・業界標準 | OpenAI依存・別API key要 | $0.006/分 |
| **B. Claude API（音声対応）** | 既存Claude一本化・キャッシング効く | 音声対応は2025年後半〜・実装確認要 | Claude料金と統合 |
| **C. Google Cloud Speech-to-Text** | 多言語・話者分離 | GCP設定要・日本語はWhisper劣勢 | $0.024/分 |
| **D. Local Whisper（whisper.cpp）** | 無料・データ外部送信なし（セキュリティ◎） | 処理時間大・サーバー負荷 | 無料（計算資源のみ） |

**💡 MVP採用：A（OpenAI Whisper API）**

理由：
- 60分録音 = $0.36 ≒ 54円（格安）
- 日本語精度が最高（業界標準）
- 実装コストが最低（API key + HTTP POST のみ）
- Claude API は音声対応が2025年末以降の予定・不確実

**将来候補：D（Local Whisper）**
- データ外部送信なしがクライアント信頼を得る
- Shift本体で扱う組織データは機密性高い
- 6ヶ月後（2026年Q4）に移行検討

### Layer 3：AI診断（Claude API）

| 項目 | 選定 |
|---|---|
| **モデル** | Claude Opus 4.7 または Sonnet 4.6 |
| **プロンプト** | 既存CODE Executive `generate.php` の system_prompt を流用・幹部版に改訂 |
| **プロンプトキャッシング** | ✅ 必ず使用（同じ system_prompt は 5分キャッシュで90%コスト削減） |
| **出力形式** | HTML（既存CODE Executive と同じ page 構造） |
| **料金想定** | 60分トランスクリプト ≒ 20,000トークン → Opus 4.7で約$0.5/回 |

**プロンプト設計方針：**

```
<system>
あなたは「引力の整合性診断」のアナリストです。
経営者と幹部の1on1録音から、幹部の引力タイプ、思考OS、熱量マッピングを可視化します。

【診断フレーム】
1. 引力タイプ（CODE 4キャラ型応用）
   - ジャイアン型／スネ夫型／出木杉型／のび太型
2. 熱量4象限マッピング
   - X軸：本人の熱量（高↔低）／ Y軸：組織への貢献度（高↔低）
   - 宝象限／人材象限／罪象限／無象限
3. 思考OS
   - 意思決定パターン（即断／熟考／合議／依存）
   - 反応パターン（責任回避／担当化／委譲／自分事化）
4. 経営者との認知ギャップ
   - 言葉のズレ／前提のズレ

【絶対ルール】
- 録音トランスクリプトに実際に含まれる発言のみ使用
- 決めつけ禁止（データから支持される場合のみ判定）
- 経営者視点と幹部視点の両方を公平に扱う

【出力形式】
HTMLレポート（4セクション構成）
</system>
```

### Layer 4：バックエンド API

| 項目 | 選定 |
|---|---|
| **言語** | PHP（既存 Xserver + PHP環境の継続） |
| **ファイル構成** | `/integrity-diagnosis/generate.php`（既存 診断_本番/app.js パターン踏襲） |
| **API呼び出し** | cURL で OpenAI Whisper + Anthropic Claude へ |
| **認証** | APIキーは `config.php`（非公開・.gitignore） |
| **セッション管理** | `bin2hex(random_bytes(8))` でレポートID生成（既存パターン） |

### Layer 5：ホスティング・インフラ

| 項目 | 選定 |
|---|---|
| **ホスティング** | Xserver（既存 growthfix.jp と同一） |
| **URL** | `https://growthfix.jp/integrity-diagnosis/` |
| **SSL** | Xserver標準 |
| **PHP設定** | `upload_max_filesize = 100M`（60分録音で最大60MB想定） |
| **`post_max_size`** | `= 120M`（filesize の 2倍目安） |
| **`max_execution_time`** | `= 600`（10分・Whisper+Claude の処理完了まで待つ） |

**⚠️ Xserver制限確認事項：**
- PHP `max_execution_time` のサーバー側制限
- 大容量ファイルアップロードの挙動
- Claude API／OpenAI API への外部通信の確認

### Layer 6：セキュリティ

| 項目 | 対策 |
|---|---|
| **録音ファイル** | アップロード → 診断完了 → **自動削除**（`unlink()`） |
| **レポート保存** | `/integrity-diagnosis/reports/report_XXX.html`（16桁ランダムID） |
| **アクセス制御** | URL推測困難（16桁 hex = 2^64）・インデックス無効化 |
| **APIキー** | `config.php` に保存・`.gitignore` 済み |
| **レート制限** | 同一IPからの連続アクセス制限（中期課題・MVP では未実装） |
| **利用規約** | オプトイン必須：「本人承諾前提で録音をアップロードしたこと」を明示 |

## ⚠️ 技術リスクと対応

| リスク | 影響度 | 対応 |
|---|---|---|
| **OpenAI API停止／料金改定** | 高 | Whisper.cpp（Local）への移行パス確保 |

| **セキュリティ事故（録音漏洩）** | 極高 | 自動削除・アクセス制御・利用規約明示／事故時は速やか通知 |
