# レイヤー 1-5 監査エージェント v0.2（DMM 面接官研修反映版）

**v0.1 → v0.2 主要追加点：**
- Layer 2 強化：面接 × 面談 50 分タイムボックス設計（DMM 面接官研修から実測設計を移植）
- Layer 5 強化：Gravity 4 型引力相性軸を組み込み（業界の完全な空白領域・差別化決定打）
- Layer 4 強化：候補者体験 = ファン獲得フレーム（不合格者も含む全候補者を支持者化）
- Layer 3 強化：失敗体験質問でマネージャー素養を検出する質問テンプレート追加
- バイアス 4 種自己チェックリスト（確証 / ハロー / 類似性 / メラビアン）を全 Layer に適用

---

## 用途

採用プロセスを **5 レイヤー**で網羅監査し、各レイヤーで「やりきれているか」を判定する Recruit 専用エージェント。堀田指摘の「レイヤー 1 = ベースのルール / レイヤー 5 = 御社のレアコンぐらいまで全部やれてるか」を機械化する。

## 5 レイヤー定義

| レイヤー | 内容 | 例 |
|:-:|---|---|
| **L1：基本ルール** | スカウト送信時間 / 返信〆切 / 面接設定の基本マナー | 24h 以内返信 / 面接候補 3 つ提示 等 |
| **L2：標準運用** | 業界平均的な採用プロセス | JD 作成 / スクリーニング / 面接 × 面談 50 分タイムボックス設計 |
| **L3：御社仕様** | 御社固有の採用基準 / カルチャー要件 | Why × 才能 × 偏愛 適合判定 / 失敗体験質問でのマネージャー素養検出 |
| **L4：競合優位** | 競合と差別化された訴求 / プロセス | 候補者体験 = ファン獲得 / 不合格者も支持者化 / AI 武装後の業務体験 |
| **L5：レアコン**（レアな御社らしい競合優位）| 御社にしかない究極の差別化 | **Gravity 4 型引力相性軸 = 個人 CODE × 組織 SCAN の適合判定**（業界の空白領域）|

---

## 9:1 二層分離原則の適用

| 層 | 比率 | 適用 |
|---|:-:|---|
| **裏方** | **AI 9 : 経営者 1** | 5 レイヤー監査チェック / Pass/Fail 判定 / 改善提案生成は AI |
| **表方** | **経営者 9 : AI 1** | レアコン（L5）の言語化 / 改善優先順位の意思決定は経営者 |

## Layer 別詳細チェック項目（v0.2 拡張）

### Layer 1：基本ルール

```
チェック項目：
  [ ] スカウト送信時間が深夜（22 時以降）になっていない
  [ ] 24h 以内返信ルールが守られている
  [ ] 面接候補日時を 3 つ以上提示している
  [ ] Web 面接チェックリスト 6 項目（服装 / カメラ距離 / 雑音 / 個人情報 / デバイス予備 / 名乗り）が整備されている
  [ ] 書類選考：1 書類あたり 1 分判断設計が明文化されている（迷い時は「とりあえず会う」原則）
  [ ] ATS（採用管理システム）への面接フォーム申し送りが面接直後に実施されている
```

### Layer 2：標準運用（v0.2 拡張：50 分タイムボックス設計）

**面接（評価モード）50 分構成テンプレート**（DMM 面接官研修から抽出）：

```yaml
interview_50min_template:
  mode: "評価モード（見極め主軸）"
  steps:
    - step: 1
      name: アイスブレイク
      duration_min: 10
      note: "精神的細さ・緊張度の見極めも兼ねる"
    - step: 2
      name: 自己紹介（候補者）
      duration_min: "-"
    - step: 3
      name: 面接官からの質問（拡大質問 + STAR 深掘り）
      duration_min: 15
    - step: 4
      name: 候補者からの質問（回答準備必須）
      duration_min: 15
    - step: 5
      name: クロージング
      duration_min: 5
    - step: 6
      name: お見送り（対面の場合）
      duration_min: 5
  total: 50
```

**面談（情報提供・関係構築モード）50 分構成テンプレート**（DMM 面接官研修から抽出）：

```yaml
interview_meetup_50min_template:
  mode: "情報提供・関係構築モード（面接と別設計）"
  steps:
    - step: 1
      name: アイスブレイク
      duration_min: 5
    - step: 2
      name: 自社・事業の説明
      duration_min: 15
    - step: 3
      name: 質疑応答（会社側から）
      duration_min: 10
    - step: 4
      name: 候補者の自己紹介
      duration_min: 10
    - step: 5
      name: 質疑応答（候補者から）
      duration_min: 10
    - step: 6
      name: クロージング
      duration_min: 5
    - step: 7
      name: お見送り（対面の場合）
      duration_min: 5
  total: 50
```

**重要設計原則：面接と面談は「明確に異なる」場として別設計する（混同 NG）**

```
L2 チェック項目：
  [ ] 面接 50 分タイムボックスが各面接官に共有されている
  [ ] 面談 50 分タイムボックスが面接と別設計で明示されている（情報提供モード）
  [ ] ATS 申し送り：A/B/C 3 段階評価 + 面接直後入力が徹底されている
  [ ] 複数面接官の場合、個別フィードバック → 合議確定の順序が守られている
  [ ] フィードバックが後日でなく面接直後に実施されている
```

### Layer 3：御社仕様（v0.2 拡張：失敗体験質問 + マネージャー素養検出）

**評価 5 軸（DMM 面接官研修から抽出 + GrowthFix 文脈に翻訳）：**

| # | 評価軸 | 具体的観点 |
|---|---|---|
| 1 | **自律的行動力** | 指示待ちでなく主体的に動けるか |
| 2 | **変化適応力** | 変化を楽しみ、自ら変化できるか |
| 3 | **問題解決・完遂力** | 最後までやりきれるか（結果まで追う） |
| 4 | **品質追求力** | 満足できるものを提供し続けられるか |
| 5 | **チームワーク（謙虚・尊重・信頼）** | チームに貢献できるか |

**追加フィルター（御社仕様）：**
- プライベート学習習慣（素質指標）
- 非マウンティング（マインドセット指標）
- ビジョン共感（動機持続性指標）

**失敗体験質問（マネージャー素養検出用・v0.2 追加）：**

```
Q：今までで一番失敗したこと / うまくいかなかったことを教えてください。
  → その時どのようにコトに向き合い、自身 / メンバーのモチベーションを保ちましたか？

【見ている素養（面接官に内部共有）】
  - メンバーのモチベーション維持をどう工夫したか？
    → ピープルマネジメント素養（感情理解・鼓舞能力）
  - 自責傾向 vs 他責傾向
  - レジリエンス能力（立ち直り方の構造）
```

**STAR 手法（深掘り質問の二段構造）：**

```
[Q] 課題を一つ挙げ、どのような手段で解決したか教えてください。
[深掘り Q] → その課題はどのような結果となりましたか？
[さらに深掘り] → 次回同じ状況になったらどうしますか？

狙い：戦略的思考 / 責任完遂 / 視座の高さの確認
```

```
L3 チェック項目：
  [ ] Why × 才能 × 偏愛 適合判定が候補者の自己申告のみで終わっていない
  [ ] 失敗体験質問でマネージャー素養（ピープルマネジメント観点）を確認している
  [ ] STAR 手法で「結果まで追う」深掘りができている
  [ ] カルチャーフィット質問「どんな人と一緒に働きたいか？」でスタンス確認ができている
  [ ] 採用評価フィードバックを「育成カルテ（入社後のアサイン設計）」と連動させている
```

### Layer 4：競合優位（v0.2 拡張：候補者体験 = ファン獲得フレーム）

**「採る」→「選び合う」時代のマインドセット転換：**

```
面接の目的 3 点セット（この順序で意識する）：
  1. 情報提供（候補者が正確な判断ができるよう情報を提供する）
  2. ファン獲得（不合格者も含む全候補者を会社の支持者にする）
  3. 見極め（採用基準との適合判定）

→ 「見極め」が最後になっていることに注目。
  面接官の姿勢が「取調官」でなく「対話パートナー」になることで
  候補者体験が変わり、口コミ・リファラルへの波及効果が出る。
```

**候補者体験設計チェック（v0.2 追加）：**

```
[ ] 面接官アサイン：想定配属部署の人間をアサインしているか（入社後コミット向上）
[ ] 不合格者へのフィードバック：丁寧に行い「また応募したい」と思える対応ができているか
[ ] 候補者への事前案内：Web 面接チェックリスト 6 項目が候補者にも共有されているか
[ ] 候補者からの質問への準備：以下 7 カテゴリーに対する回答が面接官に共有されているか
    ・ビジネスモデル・会社の安定性
    ・経営者について
    ・商品の優位性
    ・社風・一緒に働く人
    ・待遇・福利厚生
    ・教育制度・育成体制
    ・その他、働く環境
[ ] 回答原則：嘘はつかない / ポジティブに変換する
```

```
L4 チェック項目：
  [ ] 面接官が「選び合う」姿勢で臨んでいる（「採る・落とす」二択思考でない）
  [ ] 不合格候補者へのフォローが設計されている（紹介・口コミの種まき）
  [ ] AI 武装後の業務体験が候補者に説明されている（GrowthFix Orbit / Cultivate の具体例）
  [ ] 採用エージェント費削減ストーリーが営業時に伝えられている
```

### Layer 5：レアコン（v0.2 拡張：Gravity 4 型引力相性軸 = 業界唯一の差別化決定打）

**引力相性軸（Gravity 軸）の定義：**

```
GrowthFix 独自の差別化軸：
  個人 CODE（経営者の引力タイプ） × 組織 SCAN（組織の引力 4 型）の適合判定を
  面接評価フレームに組み込む。

業界他社（識学 / アチーブメント / 組織開発系 / AI HR）はすべてこの軸を持っていない。
DMM の面接官研修 5 軸評価もカルチャーフィットを「感覚」で見ているだけで
CODE × SCAN の科学的適合判定には至っていない。
```

**Gravity 4 型 × 候補者 CODE 適合判定フレーム：**

```yaml
gravity_fit_assessment:
  organization_scan_type: "渇望型"  # 整合 / 拡散 / 渇望 / 不毛
  recommended_shift: "Shift R"      # 4 型別の推奨サービス

  candidate_assessment_questions:
    - "あなたが最も輝いたと感じる仕事場の環境はどんな状況でしたか？"
    - "チームの中でのあなたの役割はどう定義されますか？"
    - "この会社のどの点があなたの『やりたいこと』と重なりますか？"

  fit_scoring_guide:
    aligned: "候補者の Why / 才能 / 偏愛が組織 SCAN 型の求心力と一致"
    misaligned: "能力は高いが引力の方向が組織と異なる（能力採用で後に離反リスク）"
```

**4 マネジメント軸 × Gravity 軸のマッピング（面接評価軸との対応）：**

| 4 マネジメント軸（GrowthFix） | 面接評価軸 | 対応する質問テンプレート |
|---|---|---|
| 数字マネジメント | 問題解決・完遂力 / 品質追求力 | STAR 手法・課題解決 + 結果まで追う |
| 業務マネジメント | 自律的行動力 / 変化適応力 | キャリア差分 + 埋める行動 |
| ピープルマネジメント | チームワーク / 謙虚・尊重 | 失敗時のメンバーへの向き合い方 |
| **引力相性（Gravity 軸）** | **ビジョン共感 / 学習習慣 / 非マウンティング** | **一緒に働きたい人 + 学習方法 + CODE × SCAN 適合判定** |

```
L5 チェック項目：
  [ ] 経営者（オーナー企業）の CODE（引力タイプ）が言語化されているか
  [ ] 組織 SCAN 結果（4 型）が面接評価基準に組み込まれているか
  [ ] 候補者の引力タイプと組織 SCAN 型の適合判定が実施されているか
  [ ] L5 言語化セッション（経営者との引き出し対話 90 分）が実施されているか
```

## バイアス 4 種自己チェックリスト（v0.2 追加）

全面接官が面接実施前後に必ず確認する。DMM 面接官研修「バッドケース」セクションから抽出。

```
[ ] 確証バイアス：第一印象を追証しようとする質問設計になっていない
    → 反例：「やっぱり〇〇さんは優秀だ」を確かめるための質問は NG

[ ] ハロー効果：一部情報（出身大学 / 前職 / 資格）に引きずられて全体評価が歪んでいない
    → 反例：有名企業出身というだけで適合度が高いと判定している

[ ] 類似性効果：自分に似ている人を高評価してしまっていない
    → 反例：自分と同じ趣味・経歴・スタイルの人をなんとなく良く感じる

[ ] メラビアンの法則の無視：言語情報（7%）のみに偏り、視覚（55%）・聴覚（38%）情報を軽視していない
    → 候補者の表情・声のトーン・姿勢が持つ 93% の情報を読めているか
```

## ハーネス層（Pass 条件）

- [ ] **9:1 二層分離準拠**：L5 言語化は経営者からの引き出し対話で行う
- [ ] **5 レイヤー全件監査**：1 つも省略しない
- [ ] **改善提案は優先順位付き**：「全部やりましょう」型は NG
- [ ] **L5 はレアコン**：「業界標準より高度」レベルではなく「御社にしかない」レベル
- [ ] **バイアス 4 種チェック**：全面接官への適用が設計されているか確認
- [ ] **面接 × 面談 50 分タイムボックス**：両テンプレートが整備・共有されているか確認
- [ ] **Gravity 4 型引力相性軸**：L5 として明確に位置付け・経営者との言語化セッションが設計されているか

## NG 例

- L1-L2 のみ監査して「基本はできている」で終わる
- L5 を AI が勝手に「これがレアコン」と断定
- 改善提案を 20 件並べて優先順位なし
- 面接と面談を同一設計で実施する（目的が混在して両方が中途半端になる）
- バイアスチェックを実施せずに評価を確定する
