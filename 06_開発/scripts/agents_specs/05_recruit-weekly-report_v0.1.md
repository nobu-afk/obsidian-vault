# 週次レポート生成エージェント v0.1

## 用途

Recruit 専用の **4 つの分析エージェント出力**（議事録分析 / スカウト分析 / JD 仮説 / レイヤー監査）を集約し、**週次レポート A4 3p を自動生成** → 石井 1 推敲（5-10 分） → 顧客納品。

## 9:1 二層分離原則の適用

| 層 | 比率 | 適用 |
|---|:-:|---|
| **裏方** | **AI 9 : 経営者 1** | 4 エージェント出力の集約 / 数字算出 / グラフ生成 / 報告書フォーマット出力は AI |
| **表方** | **経営者 9 : AI 1** | 推敲 / 御社固有の解釈追加 / 次週アクション意思決定は石井（5-10 分） |

## 入力

- 議事録分析エージェント出力（過去 1 週間分）
- スカウト分析エージェント出力（過去 1 週間分）
- JD 仮説エージェント出力（月次のため当該週の進捗）
- レイヤー 1-5 監査エージェント出力（隔週のため当該週の進捗）
- 顧客プロファイル（プロジェクト memory）

## 出力

A4 3p（PDF + Notion）：

```
# Gravity Recruit 週次レポート（YYYY-MM-DD 〜 YYYY-MM-DD）

## 1. 今週のサマリー（A4 1p）
- スカウト送信数 / 返信率 / 推移グラフ
- 面接実施数 / 通過率 / 推移グラフ
- 採用決定数（累計）
- 主要トピック 3 件

## 2. 5 エージェント分析サマリー（A4 1p）
- ① 議事録分析：今週のペインポイント 5 件
- ② スカウト分析：返信率 22%（先週 +2pt）/ 高反応パターン 1 件抽出
- ③ JD 仮説：今月の JD A/B/C 3 案進捗
- ④ レイヤー監査：L4 改善 3 件 / L5 言語化進捗
- ⑤ Pre-Action（次週推奨アクション 3 件）

## 3. 次週アクション（A4 1p）
- アクション 1：[内容] / 担当 / 期日 / 期待効果
- アクション 2：...
- アクション 3：...

- 経営者意思決定が必要なポイント 1-3 件（**経営者アウトプット待ち**）
```

## ハーネス層（Pass 条件）

- [ ] **9:1 二層分離準拠**：「経営者意思決定が必要なポイント」セクションを必ず明示
- [ ] **A4 3p 厳守**：膨らませない（1 ページ = 約 400 字）
- [ ] **数字根拠ベース**：仮説や推測ではなく数字を引用
- [ ] **次週アクションは 3 件まで**：絞らないと優先度が崩れる
- [ ] **石井 1 推敲が前提**：「最終答え」型でなく「素材提示型」で出力

## NG 例

- ❌ A4 5p 以上に膨らませる（経営者の読む時間を奪う）
- ❌ 次週アクションを 10 件並べる（優先度ぼやける）
- ❌ 「経営者意思決定が必要なポイント」セクションを省略
- ❌ 数字なしの抽象的な推奨
