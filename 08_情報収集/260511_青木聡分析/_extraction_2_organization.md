### 差別化軸（GrowthFix が上書き・拡張する部分）

| 軸 | アチーブメントの限界 | GrowthFix が上書きするもの |
|---|---|---|
| **人格 vs 引力タイプ** | 「人格者を育てる」が育成の目標。人格の普遍的な定義を前提 | GrowthFix は「引力タイプの多様性」を前提とする。正解の人格型はない。Why × 才能 × 偏愛の違いを組織が引力として活かすのが目標 |

| **時間をかけて育てる vs 引力による自己選択** | 「3年・8年の育成時間論」。忍耐を前提とした長期育成 | GrowthFix は「Emotional Contagion（感情伝染）」が起きている職場では育成速度が加速するという構造を持つ。引力ある職場では自律的な成長曲線が変わる |

| **上位下達・理念浸透型 vs 引力設計型** | 理念を「浸透させる」「伝える」方向性。経営者からメンバーへの伝達モデル | GrowthFix の Scan / Shift C は「引力を設計する」。伝達ではなく、集まりたくなる構造を組織に実装するという設計思想の違い |

## 5. Shift C 中間経営職フレームへの強化提案（特設セクション）

### 現状の Shift C フレーム（4マネジメント）
```
数字マネジメント / 業務マネジメント / ピープルマネジメント / 引力相性マネジメント
```

### 青木フレームによる強化案

**強化点 1：業務マネジメントに「現役度」を追加**
```
業務マネジメント
  → 業務設計力（仕組み化・SOP構築）
  → 現役度（自分がYesを見せられるか。口先指導への崩壊を防ぐ）
```
根拠：[almoGkM3UJQ]「口先で農書で指導し始めた時に組織は衰退する」

**強化点 2：ピープルマネジメントを3サブ軸に分解**
```
ピープルマネジメント
  → 目標設計・達成支援（部下の目標達成を通して自分の目標を達成する）
  → 承認・心理的安全（6割合格文化の設計）
  → 意味付け・意義付け（仕事の社会性・個人の成長との接続）
```
根拠：[kh37cMdmlU8][q4nf6wfDc10][ZpIhxSLVuVM]の複数動画から構成

**強化点 3：「委任の条件整備」を Week 2-3 の独立ワークに昇格**
現状の Shift C ではおそらく「委任」が主テーマになっていない可能性がある。青木が徹底して語るのは「判断軸の明文化なしに委任すると必ず失敗する」という構造的真実。

提案するワーク設計：
```
Week 2：経営者の判断軸の言語化（判断軸カード A4 1枚）
  - この仕事をどうすれば「良し」と判断するか
  - 何をされると「NG」か
  - 自分と部下で判断が違う場面はどこか
Week 3：委任グレード設計
  - 完全委任 / 報告委任 / 共同実施 / 直接実施 の4分類
  - 各中間経営職への委任グレードマッピング
```
根拠：[5n6FFRsm2DA]「方針・ビジョン・判断軸は私が出す。実務は任せる」

**強化点 4：採用と躍動の連動構造を Week 6 に明示**
青木の組織論の根底にある「理念共官型採用 → 文化形成 → 躍動」という流れは GrowthFix の Shift R → Shift C の動線と完全に対応する。

Shift C Week 6 の「振り返りと次フェーズ設計」で「Orbit（月次）への移行」と同時に「Shift R との連携設計（躍動する組織が次の採用引力になる）」を明示することで Shift C 単体の完結感を高めつつ、継続サービスへの自然な橋渡しができる。
