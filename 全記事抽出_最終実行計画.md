# 全記事抽出 最終実行計画

## 📊 **確定情報**

### **対象データ**
- **サイト**: https://terminalhill.jugem.jp/
- **総記事数**: **5,652記事** （確定）
- **期間**: 2010年1月〜2025年7月（15年間）
- **記事ID範囲**: 137 〜 52,887
- **平均記事長**: 推定2,000-3,000文字

### **予想総データ量**
- **総文字数**: 約11,000,000〜17,000,000文字（11〜17MB）
- **実行時間**: 約3〜4時間（2秒間隔）
- **出力ファイル**: テキスト形式（UTF-8）

## 🛠️ **実行方法**

### **全記事抽出コマンド**
```bash
# 全記事を抽出（推奨設定）
python blog_scraper_improved.py --url https://terminalhill.jugem.jp/ --output terminalhill_all_articles.txt --delay 2

# またはより高速（サーバー負荷軽減のため1秒間隔）
python blog_scraper_improved.py --url https://terminalhill.jugem.jp/ --output terminalhill_all_articles.txt --delay 1
```

### **段階的実行（推奨）**
より安全な段階的実行も可能です：

1. **1000記事ずつテスト**:
```bash
python blog_scraper_improved.py --url https://terminalhill.jugem.jp/ --output terminalhill_1000.txt --max-articles 1000
```

2. **全記事抽出**:
```bash
python blog_scraper_improved.py --url https://terminalhill.jugem.jp/ --output terminalhill_all_articles.txt
```

## 📋 **実行時の注意点**

### **技術的考慮事項**
1. **ネットワーク安定性**: 長時間実行のため安定した接続が必要
2. **ディスク容量**: 最低20MB以上の空き容量確保
3. **中断対応**: 実行中断時は `--max-articles` で再開範囲を調整

### **エラー対処**
- **429エラー**: `--delay` を3秒以上に増加
- **タイムアウト**: 自動で次の記事に進行
- **文字化け**: 後処理で修正可能
