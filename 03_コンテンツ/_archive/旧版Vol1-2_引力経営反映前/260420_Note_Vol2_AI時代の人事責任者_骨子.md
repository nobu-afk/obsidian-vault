## 📐 記事構造（4層ハイブリッド）

### 🅰 導入層：C要素（問題提起・800字）

**見出し案：「AI時代、人事が丸裸にされる」**

- AI（特にLLM）が1on1録音・Slackログ・メールから**組織の実態を解析できる時代**が来た
- 経営者の人間性も、社員の熱量も、幹部の思考OSも、**データから可視化される**
- これまで「感覚」「経験」で見ていた組織の影が、**定量化される**
- → 人事の役割は"何"になるのか？

### 🅱 本文層：A要素（実装の権威性・1500字）

**見出し案：「私はClaude Codeで組織診断AIを作った」**

- 2026年3月〜：石井が Claude Code を使って Gravity CODE Executive を開発
- 6セクション構成（サマリー／才能のエンジン／本来の使い方／have to／次のステップ／取扱説明書）
- 堀田フレーム全搭載＋7問ゼロベース再設計
- 2026年4月の1ヶ月で、441タスク・手作業換算1,057時間分をAI実績129時間で完遂
- **AIで、チーム8人分の仕事を1人で回している**
- 実装プロセスの透明性：**「これは人事責任者なら作れる」**と言える範囲まで公開

**読者への主張：**
- AI実装は特別なエンジニアスキルではない
- **組織を見る目** × **AIの増幅力** で、誰でもこの位置に到達できる
- ただし**"組織を見る目"が先**。AIが先ではない

### 🅲 クライマックス層：D要素（Scan の具体アウトプット・1500字）

**見出し案：「Scan で幹部の思考OSが丸裸になる」**

- Gravity **Scan**（CEO＋幹部2-3名・30-50万）の実装プロセス公開
- **Scan の3レイヤー診断：**
  1. **対面セッション**：CEO＋幹部の同時ヒアリング
  2. **事前アンケート**：CEO／幹部個別の認識ギャップ炙り出し
  3. **AIガラス張り診断**：1on1録音解析（オプション）＋公開情報推論
- **仮想ケース：** "XX社（匿名化）" の幹部 Aさん
  - 入力：1on1録音60分＋事前アンケート
  - 出力：
    - 幹部Aの引力タイプ（スネ夫型×実は忠義厚い）
    - 熱量4象限マッピング（高熱量×中貢献 → "人材"象限）
    - 思考OS（即断×依存パターン）
    - 経営者との認知ギャップ（"任せる"の解釈相違）
    - 硬直化4要因のうちどれに引っかかっているか
- **具体的な抽出文（AI出力サンプル）を3-4つ引用**
- "知ってた"から"そういうことだったのか"への転換

**この記事の読者に伝わるべき感覚：**
- 「これは感覚ではなく、データから出てきた」という説得力
- 「自分の組織でも **Scan を受けたら**これが出せるのか」という実感

### 🅳 クローズ層：Cに回収（意味付け・700字）

**見出し案：「AI時代の人事責任者は、"組織の引力を設計する側"へ」**

- 人事はもう「制度を運用する側」ではない
- AI時代の人事責任者は：
  1. **経営者と組織の引力の整合性を観察する**
  2. **引力のズレを診断し、ズレを言語化する**
  3. **組織の引力を意図的に設計する**（制度は手段）
- **これは"翻訳"の仕事**。経営者の引力→組織の仕組みへの翻訳
- 石井の3層ブランディング：存在層＝組織の引力研究の人／機能層＝社長の翻訳者（組織側）＋引力の参謀（個人側）

**最後のメッセージ：**
> AI時代に残るのは、**AIを使う側**ではなく **AIが映し出した組織の"引力の歪み"を設計し直せる側**。人事責任者の仕事は、AI時代に初めて本領を発揮する。
