## 構成（骨子）

### 1. 外的フック（リード・約 400 字）

**伝える主旨：** 採用に金をかけているのに優秀層が採れない経営者の「痛みの言語」を 3 行で言い切る。読者の自己認識を先取りして「あ、俺の話だ」と思わせる。

**伝える順序：**

1. 採用エージェントに年 300 万払っているのに、採れる人の上限が変わっていない経営者へ
2. 「うちの中堅は数字を持たない」と感じているのに何年も変わっていない経営者へ
3. 面接で候補者の目に火が点かない──理由がわからないままエージェントを責め続けている経営者へ

**キーフレーズ：** 「原因はあなたでも候補者でもない。中間層のあり方が、採用の天井を決めている。」

**禁則：** 「優秀人材が」「引力が」「組織の」等の内的語彙を冒頭に出さない。外的な採用ペインの言語に徹する。

### 2. 問題提起（約 800 字）

**伝える主旨：** 「中間管理職」という概念そのものが AI 時代の事業速度に合っていないことを示す。識学的な「上位下達 × 結果責任」の限界を、攻撃せず構造として描く。

**伝える順序：**

1. 管理職という言葉の語源──「管理する」ことが仕事になった人間という設計思想の時代背景（高度成長期・工場モデル）
2. その設計が今も組織に残っているという事実。「評価する / 报告を受ける / 承認する」だけの中間層が生まれている構造
3. 識学対立軸の出現：上位下達 + 結果責任で整えた組織では中間層が「管理者」に収束しやすい。GrowthFix は「管理者」ではなく「経営職」という軸で中間層を設計する（批判ではなく役割分担として）
4. 「これは識学が悪いではなく、管理 OS の上に求心 OS が欠けているという話だ」という転換ブリッジ

**キーフレーズ：** 「管理職という言葉を使い続けている限り、その人は管理者にしかなれない。」

**識学対立軸の扱い：** 直接批判ではなく「管理 OS は機能する・その上に中間経営職という求心 OS を乗せると採用が回り始める」という並立構造で記述（feedback_edge_path_43.md 批判対応 3 ルール整合）

---

### 3. 中間経営職とは何か（約 1,200 字）

**伝える主旨：** GrowthFix 独自定義「中間経営職 = ミニ CEO」の 4 マネジメント（数字 / 業務 / ピープル / 引力相性）を、既存マネジメント論との違いを交えて解説する。

**伝える順序：**

1. 定義を 1 行で打つ：「中間経営職とは、事業を動かすミドルであり、経営者の引力を翻訳して現場に届ける経営代行レイヤーだ。」
2. 既存論との違い：「マネジメントの壁」「マネージャー = ミニ CEO」という視点は前からあった（JAFCO 動画など）。GrowthFix はそこに 4 つ目の軸として**引力相性マネジメント**を加えることで採用・定着と直結させた
3. 4 マネジメントの解説（1 軸ずつ 2-3 行）：
   - 数字マネジメント：採用 KPI・エンゲージメント KPI を経営者と同じ粒度で握る
   - 業務マネジメント：プロセスを設計・改善できる。現場に仕組みを残す
   - ピープルマネジメント：1on1 の質、評価の公正さ、育成の意図設計
   - 引力相性マネジメント：「この人はうちの引力に合っているか」を採用基準として運用し、既存メンバーの躍動土壌を維持する。ここが従来のマネジメント論にない軸
4. 「4 つ目がない組織は、採用基準が人事部や社長の感覚で決まり続ける」という具体的な弊害

**えげつない具体（本文用メモ）：** 「優秀なプレイヤーだった人が管理職になった瞬間、採用面接に来なくなる。来ても空気が変わらない。引力相性マネジメントは、その現象に名前をつける試みだ。」

### 4. 採用が回り始める構造（約 1,200 字・最重要）

**伝える主旨：** 中間経営職が機能することで採用 PMF の 2 段階（プロファイル確定 + 市場獲得動線構築）が初めて現場に降りてくる構造を一気通貫で描く。

**伝える順序：**

1. 採用が止まっている会社の共通構造：経営者が「いい人材が来ない」と言い、中間層は「採用は社長の仕事だ」と思っている。この分断が問題の本質
2. **採用 PMF 第 1 段階**の話：「うちはどんな人材が欲しいか」は実は曖昧なまま進んでいる。「あったらいいよなレベルでは採用できない」── 欲しい人材プロファイルを中間経営職が経営者と一緒に握れるかどうかが分岐点（LP 主軸には昇格しない・ここは本文限定の使い方）
3. **採用 PMF 第 2 段階**の話：プロファイルが固まっても、獲得動線（採用接続資産）が棚卸しされていない。4 ストック（既存社員リファラル / エージェント / 候補者 SNS / アライアンス）を中間経営職が責任を持って動かせるかが問われる
4. 中間経営職が 4 マネジメントの引力相性軸を持つことで「打席に立つ数と質が変わる」── 「打席に立つ数が足りないと PMF は分からない」という構造
5. 採用が回り始めるとはどういう状態か：候補者に会う前から「この人は合うか」の解像度が上がり、面接辞退率が下がり、入社後 3 ヶ月の離脱率が下がる。エージェント費が減る前に、まず「採れる上限」が変わる

**えげつない具体（本文用メモ）：** 「採用エージェントに月 30 万払って 3 年変わらないなら、それはエージェントの問題ではない。獲得動線を持って動ける中間経営職がいないという組織の問題だ。」

**伝える主旨：** Shift R/C の Week 11-12 で「中間経営職育成設計」を共通骨格として実装することを公開する。JAFCO 動画（トリニティ磨社長・大谷副社長）の引用は最小限・独自語彙に変換して使う。

**伝える順序：**

1. GrowthFix では、3 ヶ月の Shift R（採用基盤実装）または Shift C（躍動組織実装）の Week 11-12 に「中間経営職育成設計」を組み込んでいること
2. なぜ Week 11-12 なのか：3 ヶ月かけて引力相性軸の採用基準が固まり、オンボーディング設計が動き始めたタイミングで初めて「誰が担うか」の設計が意味を持つ
3. JAFCO チャンネルの動画（磨社長・大谷副社長）から得たフレームを GrowthFix がどう翻訳したか：4 マネジメントの「リスク軸 → 引力相性軸への置換」が核心（出典を脚注レベルで言及）
4. 「中間経営職の育成は人事部の仕事ではなく、経営者が 3 ヶ月かけて設計する仕事だ」という位置づけ

**引用指針（本文で使う際）：**

## 本文執筆フェーズで深堀りが必要な論点

### 論点 A：§ 3「引力相性マネジメント」の具体エピソード
骨子段階では「採用基準の引力相性軸を運用する」という定義だけで止まっている。本文執筆時に「引力相性がある / ないマネジメントの現場の違い」を石井自身の経験・クライアント事例で肉付けすること。ここが「えげつない具体」の核心になる。

### 論点 B：§ 4「採用 PMF 第 2 段階（獲得動線）」の描き方
採用接続資産 4 ストックを「棚卸し」する行為が中間経営職の仕事になる、という因果構造の説明が骨子では薄い。「なぜ社長や人事部ではなく中間経営職が動線を持つと採用が変わるのか」の一歩踏み込んだ解説が本文の差分になる。
