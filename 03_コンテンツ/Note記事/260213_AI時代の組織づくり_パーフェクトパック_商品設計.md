# AI時代の組織づくり パーフェクトパック 商品設計

## コンセプト

**正社員・業務委託・パート・AIで数億円を作る組織のための、標準化された設計マニュアル。**

3冊のマニュアル（モジュール）で「誰に何を任せ、どこで線を引き、成果をどう見るか」が一通り揃う。代行・コンサルではなく、**マニュアル＝自社の資産として残る売り切り型**。

- 個別カスタマイズではなく、**標準化されたテンプレ＋運用台本＋チェックリスト**で構成
- 「読めば分かる、すぐ回せる」をゴールにした実装マニュアル
- コンサルの属人デリバリーを排し、**マニュアル商材として再現性100%でスケール**する設計

## 月額サポート（オプション）

パック購入後、運用を回し続けるための定額サポート。マニュアルだけでは手が止まる層への救済策であり、LTVの柱。

| プラン | 月額（税別） | 内容 |
|--------|--------------|------|
| **Lite** | 3万円 | テキスト壁打ち（月3往復）＋月1回の運用Audit（チェックリストベース） |
| **Standard** | 5万円 | 壁打ち＋隔週Audit＋月1回の簡易レポート（回っている所/詰まっている所） |
| **Pro** | 10万円 | 壁打ち＋週次Audit＋優先対応＋投資家向けの組織説明整理 |

- **代行はしない**。Audit（設計通りに回っているか？）＋壁打ち（次の一手を決める）に限定
- 縛り：初月〜3ヶ月の最低契約を推奨（短すぎると効果が見えない）
- 解約：月単位。日割りなし

## 3つのモジュール

### モジュール1：協働設計マニュアル（10万円）

**「誰に何を任せ、どこで線を引くか」の設計書**

正社員・業務委託・パート・AIを「並列のリソース」として扱うための線引きと構成の型。組織図を「役職のツリー」から「成果オーナーと線引きのマップ」に書き換える土台。

| マニュアルの章立て | 含まれるテンプレ・ツール |
|-------------------|------------------------|
| 第1章：構成員マップの作り方 | 成果領域×担い手マップ（テンプレ）。正社員/業務委託/パート/AIの役割を1枚で可視化 |
| 第2章：線引き表の設計 | 「この判断は人」「この出力はAI」「この実行は業務委託」を領域別に整理するテンプレ。AI品質のガードレール付き |
| 第3章：スコープ定義の型 | 業務委託に渡すときの4点セット（成果物定義/期限/受入基準/禁止事項）。依頼書テンプレ |
| 第4章：意思決定プロトコル | 「何を誰が決めるか」のRACI＋決裁ルール。CEO/現場/AIの3層 |
| 第5章：運用チェックリスト | 月1回の「線引きが守られているか」セルフAuditシート |

**このモジュールで解決する痛み**：
- 全部CEOが決めている（ボトルネック）
- 業務委託に何をどこまで任せていいか分からない
- AIと人の役割分担が曖昧
- 「誰の仕事でもない」タスクが放置される

---

### モジュール2：採用・オンボーディングマニュアル（10万円）

**「誰を迎え入れ、最速で戦力化するか」の実装手順書**

正社員はキーパーソンだけ厳選。それ以外は業務委託・パート・AIで補う前提の採用設計と、雇用形態別のオンボーディング。

| マニュアルの章立て | 含まれるテンプレ・ツール |
|-------------------|------------------------|
| 第1章：人材調達の意思決定フロー | 「正社員を採る前にAI/業務委託で回せないか」を先に検討する判断フロー |
| 第2章：キーパーソン採用の型 | 採用要件定義→JD→面接設計→選考フロー→オファーの一式テンプレ |
| 第3章：業務委託・パートの切り出し方 | 何を業務委託に渡すか。スコープの切り方・契約・更新の考え方。依頼書テンプレ |
| 第4章：正社員オンボーディング（90日） | 30日/60日/90日のゴール定義＋マイルストーン＋チェックリスト＋週次レビューの型 |
| 第5章：業務委託・パート・AIオンボーディング（最速） | 役割/成果の見え方/判断基準を最短で揃える1ページキット。AIの使い方ガイド付き |

**このモジュールで解決する痛み**：
- 人が足りないとすぐ「正社員を採ろう」となる（他の選択肢を検討しない）
- 採用要件がブレる、面接で何を見るか統一されていない
- 業務委託が入るたびに同じ説明を繰り返す
- 入社/契約開始後の立ち上がりが遅い

---

### モジュール3：成果・カルチャーマニュアル（10万円）

**「成果をどう見て、どう報い、チームをどう維持するか」の運用台本**

等級制度ではなく「成果の可視化」で評価を回す。業務委託・パートも含めた報酬の考え方と、雇用形態を超えたカルチャーの型。

| マニュアルの章立て | 含まれるテンプレ・ツール |
|-------------------|------------------------|
| 第1章：成果定義とフィードバックの型 | 何をいつまでに出すか。週次10分レビューの型（正社員/業務委託共通） |
| 第2章：報酬設計の原則 | 固定×成果×混合の使い分け。業務委託・パートの報酬設計テンプレ。正社員とのバランスの考え方 |
| 第3章：レビュー会議の設計 | 会議を減らしプロトコルで回す前提の、最小レビュー会議設計 |
| 第4章：バリュー運用化 | バリューを「判断基準」に落とす。1on1/フィードバックの型。雇用形態を超えた共有の仕組み |
| 第5章：チームの維持と修復 | 混合チームの温度差解消、コンフリクト対処、オフボーディング（円満終了→リピート）の型 |

**このモジュールで解決する痛み**：
- 評価・報酬の考え方が雇用形態ごとにバラバラ
- 業務委託が「言われたことだけやる」状態（コミット不足）
- バリューを掲げているが運用されていない
- 正社員と業務委託の間で温度差がある

## 3冊の一貫性（なぜパックで買うべきか）

```
モジュール1（協働設計）    モジュール2（採用・オンボ）    モジュール3（成果・カルチャー）
   誰に何を任せるか    →    どう迎え入れるか      →     成果をどう見て報いるか
   線引き・構成設計          採用・立ち上げ                定着・運用・維持
       ───────────────── 一本の線でつながる ─────────────────
```

- 1冊目で「誰が何をやるか」を決めないと、2冊目（採用・オンボ）の要件が定まらない
- 2冊目で人を揃えても、3冊目（成果・カルチャー）がないと「任せっぱなし」で回らない
- **3冊セットで初めて「設計→調達→運用」が一本の線になる**

## 「標準化マニュアル」の設計原則

本パックは**個別コンサルではなく、標準化されたマニュアル商材**として設計する。

| 原則 | 内容 |
|------|------|
| **再現性** | 誰が読んでも同じ手順で実装できる。属人的なアドバイスは含まない |
| **テンプレ完備** | 各章に「そのまま使えるテンプレ」を付属。穴埋め式で自社用に落とせる |
| **チェックリスト駆動** | 「やったか/やっていないか」で進捗が分かるチェックリストで運用 |
| **読了目安** | 各モジュール30〜50ページ。1冊2〜3時間で読了、1〜2週間で実装開始できる分量 |
| **代行不要** | マニュアルの中に「次にやること」が明記されており、コンサルなしで回せる設計 |
| **月額は補助** | 月額サポートは「マニュアルだけでは手が止まる」層向けの救済策。マニュアル自体は自走前提 |

## ファネル設計（LP→オプトイン→ナーチャリング→販売）

### 1. LP（ランディングページ）

**入口**：AI時代の組織づくりのバリュープロポジション（VP100から選定したTop20）をLPの切り口に使用。

**LP構成案**：
1. **ヘッドライン**：「正社員・業務委託・パート・AIで数億円を作る。その"設計書"、ありますか？」
2. **痛みの提示**：全部CEOが決めている / 業務委託の成果が読めない / AIの使い方がバラバラ / 入社後が属人化
3. **解決策の提示**：3冊のマニュアルで「設計→調達→運用」が一通り揃う
4. **オプトインオファー**：無料テンプレ or 診断（後述）でリスト獲得
5. **CTA**：「無料で受け取る」

### 2. オプトインオファー（リスト獲得）

VP100のうち、衝動トリガーが強いものを無料テンプレとして提供し、リストを獲得。

**オプトイン候補**（テスト用に3〜5案）：
- VP-001「業務×担い手マトリクス」テンプレ無料DL
- VP-051「CEO/現場/AI 意思決定マトリクス」テンプレ無料DL
- VP-011「正社員/業務委託/パート/AI 最適配置マップ」テンプレ無料DL
- VP-081「偽装請負チェックリスト」テンプレ無料DL
- 「AI時代の組織づくり 5分診断」（設問5〜7問→結果レポート送付）

### 3. ナーチャリング（ステップメール / LINE）

**目的**：「痛みの自覚→解決策の理解→信頼構築→購入」の段階を踏む。

**ステップメール構成（7〜10通、2〜3週間）**：

| 通数 | テーマ | 狙い |
|------|--------|------|
| 1通目 | お礼＋テンプレの使い方ガイド | 開封率の維持。即座に価値提供 |
| 2通目 | 「なぜ今、協働の設計が必要なのか」 | 痛みの深堀り。問題の本質を理解させる |
| 3通目 | AI時代の組織づくり 3つの変化 | 「正社員だけで回す時代は終わった」の認知転換 |
| 4通目 | 事例：線引きがないとこうなる（失敗談） | 恐怖訴求。放置のリスクを具体化 |
| 5通目 | 事例：マニュアルを入れたらこう変わった（成功談） | 希望の提示。ビフォー/アフター |
| 6通目 | パーフェクトパックの中身紹介 | 商品の理解。3冊の構成と一貫性を解説 |
| 7通目 | よくある質問＋反論処理 | 購入障壁の除去。「高い」「自分でできる」への回答 |
| 8通目 | 限定オファー or 特典の案内 | 購入の後押し。期間限定特典 or 早期購入割引 |

### 4. 販売

**販売ページ**：ナーチャリング最終段階で誘導。パーフェクトパック30万円の販売ページ。

**購入後の導線**：
- 即時：マニュアル3冊のダウンロードリンク送付
- 1週間後：「実装は進んでいますか？」のフォローメール＋月額サポートの案内
- 2週間後：月額サポートの具体的な事例紹介
- 1ヶ月後：「詰まっていませんか？」のAudit体験（30分無料）→月額サポートへの誘導

## 対象顧客

### ICP（理想的な顧客像）

- **規模**：正社員1〜15名。業務委託3〜10名。パート1〜5名。AIツール複数利用
- **フェーズ**：プレシード〜シード。PMF前後。売上0〜数億円
- **意思決定者**：CEO / COO / ファウンダー
- **状態**：人事専任なし or 兼務1名。制度未整備。属人化が進行中
- **意識**：「正社員だけでは回せない」「業務委託・AIを使っているが設計がない」「このままではまずい」

### 刺さるシーン

- 業務委託が3人を超えた（管理コスト増）
- 「誰がどこまで責任を持つか」で揉めた
- CEOが全部の採用面接とオンボをやっている
- 投資家に「チームは？」と聞かれて答えられなかった
- AIを導入したが品質がバラバラ

### 刺さらないシーン

- 「全部やってほしい」（代行ニーズ）→ 月額サポートでも代行はしない
- 100人超の組織で制度の全面刷新をしたい → 本パックはスタートアップ向け
- まだ創業者1人で誰も巻き込んでいない → 時期尚早

## 競合との差別化

| 競合タイプ | 彼らの提供 | パーフェクトパックとの違い |
|------------|-----------|---------------------------|
| 採用代行（RPO） | 人を連れてくる | 本パックは「型を渡す」。採用代行を使う場合もこの型があれば歩留まり向上 |
| 人事コンサル（ASHR・事業人等） | 伴走・代行・制度設計 | 本パックは「売り切りマニュアル」。属人デリバリーではなく標準化された商材 |
| HR SaaS（SmartHR・HRMOS等） | ツールで効率化 | 本パックは「設計」。ツールの前に「何をどう回すか」を決める土台 |
| 人材紹介（フォースタ等） | 経営人材紹介 | 本パックは「仕組み」。人を入れた後に回す運用の型 |
| 書籍・オンライン講座 | 知識を渡す | 本パックは「テンプレ＋チェックリスト」で即実装。知識→実装のギャップを埋める |

**唯一の隙間**：「正社員・業務委託・パート・AIの協働の設計」を**標準化マニュアル＋テンプレで売り切る**プレイヤーはほぼいない。

## 反論処理（FAQ）

**Q. 30万円は高くないですか？**
A. 人事コンサルは月額30〜100万円が相場です。本パックは売り切り30万円で3冊のマニュアル＋15〜20個のテンプレが手に入り、自社の資産として何度でも使えます。正社員1人の採用ミス（推定コスト300〜500万円）を1回防ぐだけで、10倍以上のリターンです。

**Q. マニュアルだけで本当に実装できますか？**
A. はい。穴埋め式テンプレとチェックリストで、読んだその日から実装を開始できる設計です。それでも手が止まる場合は、月額サポート（月3万円〜）で壁打ち＋Auditを付けられます。

**Q. うちはまだAIを使っていませんが対象ですか？**
A. はい。今は「人と業務委託の線引き」だけでも、組織の生産性は上がります。AIを入れる際の土台にもなります。

**Q. 1冊だけ買えますか？**
A. 単体購入（各10万円）も可能です。ただし3冊セットで「設計→調達→運用」が一本の線になる設計なので、パック購入を推奨します。

**Q. カスタマイズは可能ですか？**
A. マニュアル自体は標準化されたものをお渡しします。自社に合わせた調整は、テンプレの穴埋めで対応できる設計です。個別のカスタマイズが必要な場合は、月額サポート（Standard以上）をご利用ください。
