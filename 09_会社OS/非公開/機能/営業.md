## Part 1: 思想層

> **★ 上位フレーム：接続装置（260430 確定）：** 営業会話・商談・適合判定は **接続装置.md** の 5 層モデルと二段運用に基づく。30 分相談での判定問い 5 つ（260430・スイートスポット詳細解像 G セクション）は接続装置の外的→内的転換を会話レベルで実装する装置。詳細：`09_会社OS/公開/経営思想/接続装置.md` Part 2 ペルソナ解像度。

### Why（なぜ営業が必要か・どう向き合うか）

セールス・マーケティング・需要創造は石井さんの**能力の輪の最大弱点**である（マンガー定義）。

知っている輪は広い（フレームワーク・マクロ経済・コーチング理論）。できる輪は中程度（翻訳力・組織診断・コーチング介入）。**稼げる輪は狭い**（Coaching 3 名のみ・Gravity 未販売）。

→ 営業を「能力の輪の中」に持ち込もうとしない。**能力の輪の外にある営業を、別の方法で機能させる**：

1. 業務委託パートナーで補完するのではなく（採用.md でも明示）
2. **石井さん自身が「客の前に立つ」経験を増やす**（マンガー処方箋）
3. ただし**売り込む営業ではなく、適合判定をする営業**として再定義する

### 定義

**営業：**
> 引力経営に共鳴する経営者と出会い、適合判定をする活動。**サービスを売り込む活動ではない。**

「成約率を上げる」より「適合率を上げる」を目指す。スイートスポット外の経営者は、たとえ成約しても長期的に GrowthFix を毀損する。

### 中核原則

#### ★ 本命：適合判定をする（売り込まない）

「この経営者は GrowthFix のスイートスポットに入っているか？」を判定するのが営業の仕事。

入っていなければ、適合外として丁寧に断る。CF が苦しい時期でも断る。**業績ではなく、適合率を最大化する。**

これが信用装置論の延長：規模最小・痛みの質最大・日次 4 指標チェック。短期コンバートを追わない（信用装置として中長期構築）。

「俺の時間を奪う案件」は引き受けない。フレームワーク（コンバージョン 3 条件）を満たしていても、自己主権侵害の感覚があれば断る。

#### サテライト 1：コンバージョン 3 条件で点検

商談・LP コピーは以下 3 条件をすべて満たすように設計：

| 条件 | 内容 |
|---|---|
| 痛みの質 | 顧客が「今すぐ」と感じる痛みか |
| タイミング | 顧客が買える状態か（予算・決裁権・緊急度） |
| 代替不可性 | この提供者でなければ駄目という理由があるか |

1 つでも欠けるとコンバートしない。3 条件不足が見えた時点で離脱する（押し込まない）。

経営者が究極に興味を持つのは「**人材獲得＝採用と離脱**」。商談・DM・自己紹介で必ずこの軸に接地する。

#### サテライト 3：信用装置として中長期構築

- マネタイズ装置：Coaching・Shift（規模最小・高単価・痛みの質最大）
- 信用供給：Note・WhitePaper・X・FB（無償の知見公開）

短期売上を追わない。**「広告ゼロ・紹介と求心力で年 5-10 社」**が最終形。中期はそこへの磁化過程として広告・LP・セミナーを運用する。

#### サテライト 4：Sushitech は名刺交換ではなく適合判定の場

2026 年 4 月 27-28 日 Sushitech に出展。50 件の接点獲得が目標だが、**獲得した接点のうち 5-10 件が適合判定パスする**ことが本当の成果。

接点数より適合率を見る。WP DL → セミナー誘導 → CODE → Scan の動線で、自然に適合外が脱落する設計。

### 適合外（謝絶ライン）の判定

| 区分 | 内容 | 対応 |
|---|---|---|
| 適用外（即断謝絶） | 倒産寸前・燃え尽き／属人維持の意思／即効性のみ要求 | その場で「他の選択肢を提案」と断る |
| 要応用（即決しない） | 1000 名超／完全コモディティ業種／完全労働集約型／共同経営体制 | 書面で適合外条件を返す・再検討は半年後 |
| **スイートスポット** | 10-100 名・単独経営者・個性依存業種・中長期視点 | 即受け |

**書面謝絶の理由：** 口頭で断ると「次回」「別の方法で」と話が長引き、自己主権が削られる。書面 1 通で完結させる。

### スイートスポット詳細解像（260430・堀田 4/29 ＋ 言語マップ統合）

> **位置付け：** 切り口 3「ターゲットペルソナの解像度向上」の最初の実装（memory `project_internal_external_concept_260430.md`）。本セクションは LP・WP・Note・営業会話の **ペルソナ判定基準**として機能する。

#### A. 基本属性

| 項目 | 内容 |
|---|---|
| 規模 | 従業員 10-100 名（中堅ベンチャー） |
| 経営形態 | **単独創業者・単独経営**（共同経営は要応用） |
| 業種 | 個性依存業種（プロダクト／クリエイティブ／コンサル／士業／ニッチ B2B 等） |
| 視点 | 中長期視点（5-10 年スパンで組織を作りたい） |
| AI 受容性 | AI 時代を脅威ではなく機会として捉える素地あり |

#### B. 経営者の人格特徴（堀田 4/29 セッション #4 で言語化）

| 軸 | 説明 |
|---|---|
| **シャイ** | 自己主張が苦手・前に出るのが恥ずかしい・自分の言葉を疑っている |
| **淡々** | カリスマ型ではない・声を張らない・静かなタイプ・冷静で内省的 |
| **雇い方が分からない** | 採用基準が言語化できていない・面接で何を聞けばいいか曖昧・口説き文句がない |

→ **「シャイ × 淡々 × 雇い方分からない」経営者** が GrowthFix の最深スイートスポット。**「ジャイアニズム × バキバキ × カリスマ採用」型は対象外**（識学・他コンサルの領域）。

#### C. 採用 MAX の天井（260429 堀田指摘）

| ライン | 状態 |
|---|---|
| 現状の採用 MAX | 明治大学レベル以下で頭打ち |
| 採用したい層 | 早慶上智 MARCH 上位・地頭の良い実務家・元大企業リード |
| 構造的問題 | 経営者個人の引力が言語化されていない → 候補者が**一段上の人材を口説けない** |

→ GrowthFix の中核ベネフィット = **「採用 MAX を 1 段引き上げる」**（明治レベル → 早慶 MARCH 上位レベル）

#### D. 組織内の構造的非対称（堀田 4/29 由来）

| 既存幹部の典型 | 説明 |
|---|---|
| **もじもじエンジニア** | 技術力高・対人苦手・経営者と少し似たタイプ・面接で口説けない |
| **バキバキ営業** | 数字に強・人当たり良し・候補者の口説きはできるが組織の引力を語れない |

→ 経営者が **「もじもじ側」**だが、採用は **「バキバキ側」**に依存している構造的非対称。**Shift R Week 1-2 の納品物（3 ヶ月予言の書 + 採用基盤実装書 12 要素 = 思想 3 / 設計 4 / 運用 5・A4 45p）**で「経営者の引力を、もじもじ系幹部にも口説けるフォーマットに翻訳」することが Shift R 受注の決め手になる（260430 Blueprint 廃止に伴い、翻訳機能は Shift R Week 1-2 に移管・260501 12 要素化＋予言の書追加）。

#### E. 言語マップとの紐付け（主訴別の典型像）

| 主訴カテゴリ | 典型ペルソナ | 推奨入口 |
|---|---|---|
| **A 採用関連の痛み**（A1-A7） | エージェント費用に疲弊・最終面接辞退に悔しさ | Scan → Shift R |
| **B 離脱関連の痛み**（B1-B6） | 優秀な幹部から先に去られた経験・本音不在の退職 | Scan → Shift R / Shift A |
| **C 組織の冷えの痛み**（C1-C6） | 会議で意見が出ない・経営者の言葉が空洞化 | Scan → Shift A |
| **D 経営者個人のコンプレックス**（D1-D7） | 演じる疲れ・自分の限界 = 事業の限界 | CODE → Coaching 並走 |
| **E AI 時代の不安**（E1-E4） | AI に代替されない経営の核を探している | セミナー → CODE → Scan |

#### F. 適合外（明示的に避けるペルソナ）

| 不適合タイプ | 理由 |
|---|---|
| ジャイアニズム・カリスマ型経営者 | 引力が既に強い・GrowthFix の翻訳器が不要 |
| 共同経営体制（複数創業者対等） | 引力源の特定が困難・意思決定が分散 |
| 1000 名超の大企業 | 既存組織慣性が強すぎる（要応用判定） |
| 即効性のみ要求 | 中長期視点との不一致 |
| 属人維持の意思 | 組織への翻訳が原理的に成立しない |
| 完全コモディティ業種 | 個性依存度が低い（要応用判定） |

#### G. 営業会話での判定問い（30 分相談で必ず聞く 6 つ・260501 C-5 追加）

1. 「過去 2 年で **採用エージェントに払った費用** はいくらですか？決定数は？」（A1 直撃）
2. 「**最終面接で辞退連絡**が来た経験はありますか？頻度は？」（A2 直撃）
3. 「**優秀な幹部から先に辞めていく**感覚はありますか？」（B1 直撃）
4. 「ご自身が **面接で何を口説いているか**、1 行で言えますか？」（A5 直撃 ＋ シャイ × 淡々 判定）
5. 「**識学やコンサルを試した経験**はありますか？効果はどうでしたか？」（D6 直撃 ＋ コンプレックス度合い判定）

**採用 → 離脱の二段階導線設計（堀田 #30/#35 連動・260429 追加）：**

堀田指摘：「離脱防止と採用の話って一緒（引力って意味で）」「痛みを伴う相談（離脱）は心理的安全性を求めるので初回に来にくい」「採用は拡大路線の人全員のニーズ」。

→ 営業ファネルでは **採用 = 初手フック ／ 離脱 = 信頼形成後の二段目** の役割分離が必要：

| ファネル段階 | 入口メッセージ軸 | 役割 |
|---|---|---|
| LP / 広告 / セミナー初接触 | **採用ペイン**（応募が来ない・口説けない・最終面接で辞退） | 全経営者に届く・心理的敷居低い |

| Coaching / 商談中盤 | **離脱・幹部問題**（信頼形成後）| 痛みを抱えた経営者の本音引き出し |
| Shift Week 1-2 診断 | 離脱と採用を統合分析 | 経営者の引力欠損の構造化 |

**コピー作成時のルール：**
- ❌ 初接触 LP / 広告で「離脱」を主軸に置かない（心理的安全性不足で離脱）
- ❌ Coaching セッションで「採用」だけを主軸に置く（拡大路線にミスフィット）
- ✅ 二段階を意識的に設計：採用で来てもらい・離脱で深まる

**チェンジマネジメント 3 層判定（Shift / Coaching 展開時）：**

| 段階 | 評価項目 | 該当時のアクション |
|---|---|---|
