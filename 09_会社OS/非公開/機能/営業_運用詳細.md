### カンファレンス参加の判断軸（260427 確定）

| 参加形態 | 判定 |

| **一般参加のみ** | **原則不参加**（撤退） |

### マッチングプラットフォーム DM チャネル（260428 検証開始）

Sushitech のオンラインマッチング機能を活用したコールド DM チャネル：

| 観点 | 内容 |
|---|---|
| **対象** | Sushitech 出展企業＋来場者から Gravity 適合度で厳選した 25 名（日本企業の経営層中心） |
| **送信方法** | DevTools Network から候補者 JSON 取得 → 適合度スコアリング → カスタマイズメッセージ生成 → mypage の chat 画面でコピペ送信 |
| **メッセージ構造** | タグライン（「組織に、引力を。」）+ Sushitech 経由の挨拶 + 候補プロフィール参照のフック + 経歴 + WP リンク + 30 分壁打ち CTA |
| **業界目安返信率** | 12-20%（コールド DM 標準） |
| **返信パターン** | パターン A（興味あり → Calendly 共有）／パターン B（質問 → 個別回答）／パターン C（5/15 セミナー絡み再接触）|

**判断ロジック（5/3 時点）：**

| 返信件数 | 判定 | 次のアクション |
|---|---|---|
| 5+ 件 | ✅ チャネル機能 | Tier C 10 名追加送信／他マッチングプラットフォームへ展開 |
| 3-4 件 | 🟡 検証継続 | データ蓄積・パターン整備 |
| 1-2 件 | ⚠️ 戦略要見直し | DM チャネル休止・広告／セミナーに集中 |
| 0 件 | ❌ チャネル不適合 | 撤退・Sushitech 系イベントに DM チャネルは投じない |

**今後の判定軸：**

- 同種マッチングイベント（IVS／TechCrunch Disrupt 等）への横展開可否は本検証結果次第
- 文化的にコールド DM が嫌われる日本市場でも、**ブランド磁化が完成した状態（タグライン＋ WP）+ 個別カスタマイズ**で許容ラインは越えられる仮説
- 関連ファイル：`04_GrowthFix/00_営業/260428_Sushitech_25候補_送付用メッセージ完成版.md` ／ `260428_Sushitech_返信管理.md`

### データ運用基盤（260507 確定・TASK-19）

> **実装着手タイミング：5/16 以降**。ブランディングロードマップ 7 軸完走・長谷さんモニター対応後。

**実装方式：自作 Python + daily skill 拡張ハイブリッド**

| レイヤー | 担当 | 概要 |
|---|---|---|
| Meta 日次 | `meta_ads_daily.py`（既存） + daily skill 自動取得 | 広告パフォーマンス（CTR / CPC / リーチ）を日次ログに自動反映 |
| Note 週次 | daily skill 拡張（5/16 着手） | PV / いいね / フォロワー増減を週次で反応率ダッシュボードに連携 |
| SNS 月次 | 手動集計（月末 weekly-close 内） | FB / X エンゲージメント・フォロワー推移 |

**優先取り込み順（Phase 1）：**
1. **広告層**：Meta 日次（既存・稼働中）→ 継続強化
2. **営業層**：パイプラインマスター（リード stage × 紹介経路タグ）→ 5/16 以降拡張
3. **コンテンツ層**（Note / SNS）：週次・月次で段階投入

**禁則：** 5 層全統合を一気に実装しない。広告 + 営業層の安定後に順次追加（段階実装原則）

（260507 確定・出典：TASK-19）

**[運用層]**

```
朝のリーチルーティン（10 分・毎朝最初）
  [1] パイプ更新（3 分）── 動いてないリード（最終接触日 > 7 日）3 件のステータス更新
  [2] DM／フォロー（5 分）── 5 件（新規 or 返信フォロー）
  [3] 投稿 1 本（2 分）── Note Vol／X／LinkedIn のいずれか（草稿は前日夜 or 朝の思考タイムで準備済）

週次反応率レビュー（金曜・weekly-close 内）
  [1] 11 指標を反応率ダッシュボードに転記
  [2] 「最も目標から離れている指標 1 つ」を特定（複数同時改善は無効化する）
  [3] 仮説立て→翌週の打ち手を 1 点だけ決める
  [4] 翌週末に効果測定
```

**[思想層]**

- **集中思考（壁打ち・概念化）はゼロにしない** ── 尖り消失リスク。完全に営業 100% にすると個性希薄化（memory `feedback_ai_brainstorm_individuality.md` 警告と同種）
- **「思考と発信を分けない」連鎖サイクル** ── 朝の思考（壁打ち・堀田セッション・概念化）を即その日の発信物（Note 草稿・DM 文面・X 投稿）に転換する設計
- **適合判定の本命は維持** ── フェーズ転換後も「売り込まない・適合判定をする」原則は不変。リーチ量は増やすが押し込みはしない

#### 管理（3 階層）

| 階層 | 何を | どこで | いつ |
|---|---|---|---|
| 日次 | パイプ更新・リーチ実施 | パイプラインマスター + daily.md | 毎朝 10 分 |
| 週次 | 反応率指標更新・ボトルネック特定 | 反応率ダッシュボード + weekly-close | 金曜（土曜の週次レビューでも可） |
| 月次 | 受注額集計・ファネル累計・運用見直し | 反応率ダッシュボード「累計サマリー」 | 月末 |

#### フェーズ転換後の禁則

- ❌ 朝に制作タスクから始めない（リーチを後回しにすると尖りに逃げる）
- ❌ 反応率が出るまで 2-3 週間の不確実性を「完成度の問題」と誤認しない（ボトルネックは時系列で観測する）
- ❌ 11 指標を全部追わない（1 点ボトルネック以外は据え置き）
- ❌ 反応率が低い時に「もっと作れば」で逃げない（リーチ手数の問題か仮説検証する）
