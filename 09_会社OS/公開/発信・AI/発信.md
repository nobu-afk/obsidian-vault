## Part 1: 思想層

> **★ 上位フレーム：接続装置（260430 確定）：** 発信のすべては **接続装置.md**（思想層内サブ層）の 5 層モデルと二段運用原則（外的で集めて内的で磁化）に基づく。本 MD は「**接続装置の主たる実装層**」として位置付けられる。LP / WP / Note / セミナー設計時は必ず `09_会社OS/公開/経営思想/接続装置.md` Part 1-3 を先に Read する。

### Why（なぜ発信するのか）

GrowthFix にとって発信は **「集客装置」ではなく「磁化装置」**である。

事業モデル三位一体（理論・実装・普及）の「普及層」として、引力経営の認知を広げ、**石井伸幸という重心に経営者を引き寄せる**ための重力場の形成活動である。

**接続装置との関係（260430）：** 発信＝磁化装置という思想は、接続装置の「外的で集めて内的で磁化」二段運用と完全整合。発信物は外的コンセプト（経営者のコンプレックス）で集める入口装置でありながら、最終的に内的世界観（引力経営・石井伸幸という重心）に磁化させる出口装置でもある。詳細：`接続装置.md` Part 1 中核原則。

### 定義（信用装置論）

発信は2つの機能で成立する：

| 機能 | 内容 |
|---|---|
| **マネタイズ装置** | Coaching・Shift（規模最小・高単価・痛みの質最大） |
| **信用供給** | Note・WhitePaper・X・FB（無償の知見公開） |

**運用基準：**
- 規模最小・痛みの質最大・日次 4 指標チェック
- ビジネス話だけでも、痛みの話だけでもバランスを欠く

### LP 変更時運用フロー（モバイル監査必須・260428 確定）

**ルール：`05_プロダクト/` 配下の LP/HTML を変更したら、必ずモバイル監査を実行してから完了とする。** モバイル最適化忘れによる事故を機械側にオフロードする。

**Why：** 260428 LIVE WEBINAR バー追加時、19 LP に新規セクションを追加したがモバイル最適化を忘れ、後追いで 2 回修正（`.fv_bg1/.fv_bg2` 試行 → `.fv_en` で確定）。LP は対外発信物の中核であり、SP 表示崩れは即離脱・受注機会損失に直結する。**人間の意識頼みでは再発する**ため、Claude 側の自動監査と手順書の二層防御を仕組みに組み込む。

**運用フロー：**

```
[1] LP/HTML 編集（Edit/Write）
        ↓
[2] Claude 自動でモバイル監査（言われなくても実行）
    ├─ 新規クラス名・新規インラインスタイル要素・新規構造の有無を点検
    ├─ チェックリスト参照：05_プロダクト/_共通/_LP変更時_モバイル確認チェックリスト.md
    └─ mobile.css 更新の必要性を判定
        ↓
[3] 必要なら mobile.css 編集案を提示
    └─ 石井さん承認後に適用
        ↓
[4] cache buster bump（?v=YYYYMMDDx）
    └─ 全 LP の mobile.css?v=... を一括更新
        ↓
[5] FTP デプロイ（mobile.css + 該当 HTML）
        ↓
[6] スマホ実機 or DevTools 375px で検証
```

**監査対象観点（チェックリスト 8 項目）：**

- font-size（PC 値 → clamp で再定義）
- padding / margin（mobile では詰める）
- horizontal layout（縦積みに反転）
- width 固定値（max-width: 100% に強制）
- white-space: nowrap（解除）
- 装飾画像／装飾テキスト（被るリスクなら display: none）
- 横並び区切り（mobile で改行する場合は非表示）
- タッチ要素（min-height: 44px・タップ領域確保）

**禁則：**
- ❌ HTML 編集だけして「完了」と報告しモバイル監査をスキップする
- ❌ 「変更が小さいから大丈夫」と独断する
- ❌ デプロイ後に「念のため確認」と後出しする

### 「光を当てるブランド」コンセプト（260427 SHIRO 動画分析から借用）

- **光を当てられた経験：** 1-2 年目・命の森水林で「社会不適合者が目をキラキラさせて生きる姿」を見て涙、「俺、変わろう」と決めた

**発信での使い方：**
- LP・Note・セミナー導入で「**光を当てるブランド**」フレームを活用
- 「**自分が光を当ててもらった人は、光を当てる人になる**」という反転構造で読者・視聴者の感情の上昇を生む
- 命の森水林エピソードを「光を当てられた経験」フレームで語る

### 良いアウトプットの合格基準（Pass条件）

- [ ] **広報 5 軸 or ナレッジ 3 テーマに紐付く**：軸不明な発信は撤回
- [ ] **命名二層運用準拠**（260503・対外発信は外的呼称必須）：Note / FB / WP / セミナータイトル / SNS 投稿で「Shift R / C / Full」内的呼称が混入していないか grep 確認。**外的呼称は Gravity Recruit / Cultivate / Shift**

- [ ] **人事系決め文句で着地**（堀田フレーム #35・260429）：人事・組織系コピーはベネフィット表現を必ず「採用◯倍」「離脱率◯%減」「優秀人材殺到」等に着地。マーケ系なら「売上◯億」。どちらでもない（思想だけ）はNG
- [ ] **えげつない具体**：AI で量産可能な抽象論レベルではなく「言っちゃいけないやつの言葉」レベル（`feedback_egetsunai_specific_over_abstract`）

- [ ] **三位一体普及層との整合**：理論・実装・普及 のどこで戦っているかが明示できる
