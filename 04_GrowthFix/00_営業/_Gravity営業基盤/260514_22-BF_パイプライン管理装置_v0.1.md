## 1. ダッシュボード全体構造（3 層 + 1 補助）

```
[層 1] 朝の即断ビュー（毎朝 3 分・朝のリーチルーティン #1）
    └─ 「動いてないリード」リスト = 商談以上のステージで最終接触日 > 7 日
    └─ 「今日の必須アクション」リスト = 締切が今日 or 過去のリード

[層 2] 週次ファネルビュー（金曜 weekly_close 内・10 分）
    └─ ステージ別件数推移（W17 → 今週）
    └─ 流入経路別の認知 → 興味転換率
    └─ 5 サービス別パイプライン金額（提案 + 商談ステージで暫定積上）

[層 3] 月次受注ビュー（月初 1 日・月次レビュー連動・20 分）
    └─ 当月受注件数 × 金額 vs 月次目標
    └─ 累計売上 vs Q 末 / 年末目標との距離
    └─ サービス別売上構成比（受注実績ベース）

[補助層] シナリオヒット集計（月次・22-BE §4 と連動）
    └─ 100 シナリオ × 実戦ヒット度マトリクス
    └─ ヒット率＞70% シナリオ：継続使用
    └─ ヒット率＜30% シナリオ：見直し or 廃止候補
```

## 2. 層 1：朝の即断ビュー（毎朝 3 分）

### 2.1 「動いてないリード」自動抽出ロジック

| 抽出条件 | 元データ | 期待アクション |
|---|---|---|
| ステージ ≥ 商談 かつ 最終接触日 > 7 日 | パイプラインマスター §2 | フォロー DM / 次面談打診 |
| ステージ = 提案 かつ 締切が 3 日以内 | パイプラインマスター §2.5 | クロージング電話 |
| ステージ = 興味 かつ 最終接触日 > 14 日 | パイプラインマスター §2.4 | 失注判定 or 認知に戻す |
| 締切 < 本日 のリード（全ステージ） | パイプラインマスター 締切列 | 即時アクション |

### 2.2 朝の即断ビュー表示形式（Notion / スプレッドシート）

```
🔴 アクション必須（今日中）：N 件
  - {ID} {名前略称} | {ステージ} | 締切 {日付} | {必要アクション}

🟡 動いてないリード（7-14 日経過）：N 件
  - {ID} {名前略称} | {ステージ} | 経過 {日数} 日 | {推奨アクション}

🟢 OK（動いている / 経過 < 7 日）：N 件
  - 一覧省略

📅 今週中の次接触予定：N 件
  - {ID} {名前略称} | {日付} | {予定アクション}
```

### 2.3 朝のリーチルーティンとの統合

- daily skill STEP 8 の本日 TODO 一覧 #1「パイプ更新（3 分）」と完全連動
- 朝のビューで赤 = 必須アクション件数を見て daily.md 重点アクションに反映
- 反応率ダッシュボード（260428）とは別運用：反応率は接点 KPI / 22-BF はファネル KPI
