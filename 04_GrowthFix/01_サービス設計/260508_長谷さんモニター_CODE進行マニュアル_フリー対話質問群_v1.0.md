## 0. このマニュアルの設計思想

### なぜ定型 5 問ではなく、フリー対話なのか

CODE の 5 問（Q1 立場 / Q2 才能の仮説 / Q3 Why の許せなさ / Q4 偏愛 / Q5 言行ギャップ）は **トリガー素材**であって、本質ではない。本質は **5 観点の枠を借りた 45 分のフリー対話で、Sonnet が「矛盾」と「キャラ命名素材」を抽出できる証言データを集めること**。

長谷さんは既に 2 年 Coaching クライアント。**定型 5 問の選択肢を機械的に消化する意味はゼロ**。むしろ 2 年の蓄積を 1 点に磁化する所作が必要。

### 石井さんの強み・弱みを踏まえた運用

**強み（260404 コーチング 7 特徴より）：**
- 構造把握・関係性デザイン・モード切替（傾聴／思考／言語化）
- 5 観点を聞きながら同時に「矛盾」を検出する解像度

**弱み：**
- **契約と収束（過去 64-88/100）** ── 構造把握した時点で完了感が出て、固化への関心が薄くなる
- 触媒不在パターン

→ **対策：各観点の最後に「Sonnet が拾える形での 1 行固化」を必ず実施**（収束ハーネスを各セクションに組み込む）

### 「抜く」コミュニケーション原則（カスタマー.md Part 1）

- ❌ 押し付けない（「こうすべき」「これが正解」）
- ❌ 情報を盛らない（フレーム解説・新概念導入）
- ✅ **問いを返して、長谷さんの中の答えを引き出す**
- ✅ **石井さんの解釈は最後の「読み合わせ T+48〜60」で投入**（CODE 対話 45 分中は基本ニュートラル）

| 時間 | 観点 | 配分 | 目的 |
|:-:|---|:-:|---|
| T+0〜3 | オープニング | 3 分 | CODE の意味付け・進め方確認・録画 ON |

### タイムキープのルール

- **各観点の終了 1 分前にチャイム（脳内タイマー）**
- 観点が早く収束したら次へ進めて OK（45 分使い切る必要なし）
- 観点が伸びそうなら **「もう 2 分だけ」延長 → それ以上は次回 Coaching へ持ち越し**を即決

### 収束のハーネス（各観点末で必ず実施）

> **石井さんの弱点「収束」をハーネス化。各観点の最後に必ず以下を実施する。**

```
「今のお話、私の理解でこう言ってよさそうですか？」
→ 1 文で要約して返す
→ 長谷さんの「はい」or「ちょっと違って…」を待つ
→ 違いがあれば修正して再要約
```

この 1 文が **Sonnet が後でレポート生成時に最重要証言データとして拾う**。

## 2. ① 才能 ── 10 分（T+3〜13）

### 設計意図

Sonnet が拾うべきデータ：
- **動詞 3 つ**（自然にできてしまう動き・3 番目は無意識の行動）
- **発火環境**（その動詞が起動する条件）
- **キャラ命名素材**（矛盾を含む名前）

長谷さん固有の事前仮説：「先回りして構造を完成させる」「建築家としての構造提示力」

### メイン問い（1-2 個まで・最初に投げる）

**A.** 「最近 1 ヶ月の経営の中で、**気づいたら勝手にやっていて、後から考えたらそれが意味ある仕事だった**ことって、何かありましたか？」

> ★ 設計：「勝手にやっていた」=無意識の動詞を引き出す問い。直接「あなたの才能は？」と聞くと社会的回答になるので回避。

**B.**（A が出にくい場合）「**他の経営者を見ていて『なぜそんな簡単なことができないんだろう』と感じた瞬間**、最近ありましたか？」

> ★ 設計：他人と比較した時の差分が才能の輪郭。

### 深掘り問い（行き詰まり時に投入・順序自由）

1. **「それを始めたきっかけは何でしたか？」**（無意識のスイッチを言語化）
2. **「それをやらないでいることって、できますか？」**（強制的に発火する才能か検証）
3. **「同じことを別の人がやろうとしたら、どう違いますか？」**（差分の言語化）
4. **「小学生の頃も、同じパターンの行動してませんでしたか？」**（時間軸接続・観点 ④ への伏線）
5. **「採用面接でその才能、出てますか？」**（事前データ＝採用 1,000 万の構造接続）

### 観察ポイント

- [ ] 動詞 3 つは抽出できたか（できなければ **発火環境を聞いて逆算**）
- [ ] 「先回りして構造を完成させる」が出るか／別の動詞が出るか
- [ ] 自慢気か・自嘲か・困惑か（**自嘲が最重要シグナル**＝矛盾検出材料）

### 長谷さん固有トリガー（事前情報からの個別問い）

> 既存 Coaching 2 年の蓄積から、もし沈黙が続いたら投入：

- **「採用面接で 3-4 名内定取れて辞退ほぼゼロ。あれって長谷さん何やってるんですか？」**（建築家・構造提示力を本人言語で）
- **「逆に、創業時メンバーが離脱した時、長谷さんはどう動きましたか？」**（フェーズ移行型離脱の文脈・観点 ⑤ への伏線）

### 収束ハーネス（必須・1 分・T+12 頃）

```
「今のお話だと、長谷さんの自然にできてしまう動きは、
 ① ○○○○ ／ ② ○○○○ ／ ③ ○○○○（無意識）
 で、それが発火するのは ○○○○ という環境の時、
 という理解で良さそうですか？」
```

→ 修正があれば 30 秒で。**「だいたい合ってます」で OK・100 点を狙わない**。

## 3. ② Why（許せなさ） ── 10 分（T+13〜23）

### 設計意図

Sonnet が拾うべきデータ：
- **Why の宣言文**：「[誰の][何] を、[動詞] するために、[何] を作り続ける」型 1 行
- **許せなさの源**（怒り・違和感・痛み）
- **灰色 vs カラフル**（社会的期待で塗り替えられた灰色 → 本来のカラフルへ）

長谷さん固有：建築家としてのキャリア・構造への偏愛・アトラクトコーチング 2 年の自己理解蓄積

### メイン問い（1-2 個まで）

**A.** 「最近、**経営の中で『これは絶対許せない』と感情が動いた瞬間**、ありましたか？小さくても良いので。」

> ★ 設計：「許せなさ」は Why の最も鋭い切り口（自己主権・カラフルの裏側）。

**B.**（A が出にくい場合）「**他の社長と話していて『この人はこういう経営してるけど、自分は絶対そうしない』と感じた瞬間**、最近ありませんでしたか？」

> ★ 設計：負の差別化＝偏愛・Why の輪郭が同時に出る。

### 深掘り問い（順序自由）

1. **「その『許せない』感覚って、いつから持ってますか？」**（時間軸接続）
2. **「許せない感覚に従って動いた時、誰に何が起きましたか？」**（影響の言語化＝経営インパクトへの伏線）
3. **「逆に、許せなさを飲み込んで動いた時、その後どうなりましたか？」**（自己主権の喪失パターン検出）
4. **「『これがやりたかったわけじゃない』と思いながらやっている仕事、ありますか？」**（have to 検出）
5. **「もし明日、社会の評価が全部消えたら、それでも続けたいことは何ですか？」**（カラフル抽出）

### Job/Career/Calling 判別問い（v1.1・260510 v0.4 追加・Wrzesniewski 1997）

> **目的：** Sonnet がレポート Block A で `.job-career-calling` 副次表示を生成するための素材集め。仕事への向き合い方を 3 型で判別：
> - **Job 型**：仕事は生計手段（自己実現の場ではない）
> - **Career 型**：仕事は階段上昇の手段（成功・地位・報酬）
> - **Calling 型**：仕事は使命の表現（社会的意義・自己同一化）

**判別問い（メイン問い・Job/Career/Calling 型）：**

「**もし明日、いま経営している会社が消えたら、何を一番喪失感じますか？　お金？　ポジション？　それとも『これをやれていたこと』そのもの？**」

**深掘り：**
- 「**仕事を辞めて、お金が一生不要だとしたら**、それでも今の経営を続けたいですか？」（Calling 型なら即答 Yes）
- 「**仕事の成功**と**仕事を通じた自己表現**、どっちが長谷さんを動かしてますか？」
- 「**社会から『よくやった』と評価される**ことと、**自分の中で『これは間違いなく自分の仕事だった』と感じる**こと、どちらが満足度高いですか？」

**判定基準：**
- **Calling 型**：「経営してることそのもの」「社会的意義」「自己表現」を即答
- **Career 型**：「階段の途中」「成功・地位・報酬」を強調
- **Job 型**：「お金がなくなったら困る」「収入手段」を強調

**長谷さん固有仮説：** Calling 型寄り（建築家からの転身・構造への偏愛・才能ズレ型を抱えていても続けている）。

### 観察ポイント

- [ ] **許せなさが具体エピソードで出るか／抽象論になるか**（抽象化は社会的期待の混入シグナル）
- [ ] 「うまく言えない」と詰まる瞬間 = **無意識のカラフルが浮上している兆候**（沈黙を待つ）
- [ ] 灰色（社会的期待で塗り替えられた言語）が混入していないか
- [ ] **Job/Career/Calling 型**のいずれかの素材が拾えたか（v1.1 追加）

### 長谷さん固有トリガー

- **「2 年前にコーチング始めた時、何がきっかけでしたか？あの時の『これじゃダメだ』感覚、まだ残ってますか？」**
- **「『先回りして構造を完成させる』のは、何を許せないからやってるんですか？**」（事前仮説への直球質問・建築家性の Why 接続）

### 収束ハーネス（必須・1 分・T+22 頃）

```
「長谷さんの Why、今日の対話だと
 『○○の○○を、○○するために、○○を作り続ける』
 という構造で読めますが、これでしっくり来ますか？」
```

> ★ Why 文は **必ずこの 1 行型で固化**（v5.3.4 の Why 構造指示厳守）。能力と才能を混在させない。動作主と対象を明確に。

## 4. ③ 偏愛 ── 8 分（T+23〜31）

### 設計意図

Sonnet が拾うべきデータ：
- **譲れない好み 2 つ**
- **絶対に選ばない嫌い 1 つ**
- 好み・嫌いは **セットで聞く**（コントラストが偏愛の輪郭）

長谷さん固有：構造美・建築家・型化への偏愛 ／ 曖昧・形式・属人化への嫌悪（仮説）

### メイン問い

**A.** 「経営の中で、**『これがあると楽しい』『これがあるから続けられる』** と感じる瞬間、何ですか？」

**B.** 「逆に、**『これだけは絶対に自分の会社で起こしたくない』** ことは何ですか？」

> ★ 設計：A と B をセットで投げる（ペアで聞くと偏愛の解像度が一気に上がる）。

### 深掘り問い

1. **「その『楽しい』感覚は、子どもの頃の何の遊びと似てますか？」**（時間軸接続）
2. **「逆に、その嫌いなものに今でも巻き込まれている瞬間、ありますか？」**（have to 検出）
3. **「好きなものと嫌いなもの、その間にあるグラデーションには、長谷さんの軸は何で決まりますか？」**（判断基準の言語化）

### HP/OP 判別問い（v1.1・260510 v0.4 追加・Vallerand 2003）

> **目的：** Sonnet がレポート Block A で `.hp-op-distinction` 副次表示を生成するための素材集め。偏愛の質を 2 型で判別：
> - **HP（Harmonious Passion・調和的情熱）**：偏愛が他の生活領域（家族・健康・趣味）と統合的に共存
> - **OP（Obsessive Passion・強迫的情熱）**：偏愛が他領域を侵食し、強迫的に従事しないと不安

**判別問い（メイン問い・HP/OP 判別）：**

「**経営の偏愛、家族や健康や他の人生領域と『心地よく共存』していますか？　それとも、**他のことを犠牲にしてでも経営に没頭しないと落ち着かない**感覚ありますか？」

**深掘り：**
- 「**1 週間まったく経営から離れる**ことができたら、どんな感覚ですか？　解放感？　不安？　罪悪感？」
- 「**家族と一緒の時間、頭の半分が経営**ということないですか？　その状態、本人として楽ですか？　苦しいですか？」
- 「**経営をやってない時の自分**、好きですか？　経営してない自分はもう自分じゃないという感覚ある？」

**判定基準：**
- **HP（調和的情熱）**：他領域とのバランス自覚 / 1 週間離れても OK / 家族時間に経営思考が混入しない
- **OP（強迫的情熱）**：他領域犠牲を自覚 / 1 週間離れると不安・罪悪感 / 家族時間に経営思考が常時混入 / 「経営してない自分はもう自分じゃない」

**長谷さん固有仮説：** OP 寄り（先回り構造完成型は強迫的情熱に転じやすい）。**Calling 型 + OP の組み合わせは「使命に飲み込まれる」リスクが高い**。レポートで `.hp-op-distinction` 副次表示で明示。

### 観察ポイント

- [ ] **「好き」「嫌い」を即答できるか／詰まるか**（即答できる = 偏愛が立ち上がっている）
- [ ] 好きと嫌いのコントラスト（例：構造 vs 曖昧／本音 vs 建前／創造 vs 維持）
- [ ] **HP / OP**のいずれかの素材が拾えたか（v1.1 追加・1 週間離れる感覚 / 家族時間の経営思考混入度）

### 長谷さん固有トリガー

- **「建築家から経営者になって、設計図と組織って、長谷さんの中で何が同じで何が違いますか？」**（建築家性 = 構造偏愛の核確認）

### 収束ハーネス（必須・30 秒・T+30 頃）

```
「長谷さんの偏愛、
 譲れない好み = ① ○○○○ ／ ② ○○○○
 絶対に選ばない嫌い = ○○○○
 という理解で良さそうですか？」
```

## 5. ④ 時間軸の一貫テーマ ── 7 分（T+31〜38）

### 設計意図

Sonnet が拾うべきデータ：
- **時間軸の一撃（対話内の表現をそのまま使う）**
- 30 年の符号（幼少期 → 学生 → 社会人 → 経営者の連続性）
- **無意識に繰り返してきたパターン**（才能の最古の発露）

長谷さん固有：建築家キャリア → 経営者 → コーチング 2 年。**「先回りして構造を完成させる」は人生最古からあるはず**。

### メイン問い

**A.** 「子どもの頃、**気づいたらいつも同じ役割をやっていた** こと、ありませんか？クラスでも、家でも、習い事でも。」

**B.** 「学生時代から今まで、**形を変えながら、ずっと同じことを続けている** と感じることはありますか？」

> ★ 設計：「形を変えながら」が鍵。表面の仕事は変わっても核は同じ＝才能の連続性。

### 深掘り問い

1. **「それを最初にやったのは、いつ・どんな状況？」**（最古の発露）
2. **「親や先生から『お前は○○な子だ』と言われたこと、覚えてますか？」**（外部からの認識）
3. **「逆に『お前は○○じゃない』と否定されたこと、ありますか？」**（社会的期待による塗り替えの初期）

### 観察ポイント

- [ ] 30 年通底するキーワードが 1-2 個出るか
- [ ] **才能観点（①）で出た動詞と、時間軸の最古エピソードが整合するか**（整合 = 整合型／不整合 = ズレ型のシグナル）

### 長谷さん固有トリガー

- **「建築家を選んだのは何故でしたか？建築家を選ぶ前から、その種ってありましたか？」**

### 収束ハーネス（必須・30 秒・T+37 頃）

```
「長谷さん、子どもの頃から今までずっと
 『○○し続けている』ように見えますが、
 この読み、合ってますか？」
```

## 6. ⑤ 矛盾と have to ── 5 分（T+38〜43）

## 7.5. STEP 5 読み合わせフェーズ 命名前後スクリプト（v1.1・260510 v0.4 追加）

> **目的：** レポート生成後の読み合わせフェーズで、3 つの一撃の **命名前 Externalization**（White & Epston）と **命名後 Somatic Resonance**（Lakoff × Gendlin）を口頭補完する標準スクリプト。CODE generate.php Block A の `.externalization-statement` と `.somatic-resonance-prompt` に対応。

### 命名前スクリプト（Externalization・外在化）── 各「一撃」読み上げ前に投入

> **思想：** 才能・Why・偏愛の言語化は「あなた = ◯◯な人」と同一視するのではなく、「◯◯」を本人から **外側に置いて観察する** ことで、本人が自分の才能を変革可能な対象として捉えられる。

**スクリプト（才能の一撃読み上げ前）：**

```
「これから 1 つ目、長谷さんの **才能の一撃** を読みます。
 これは長谷さんの中にある **道具** だと思って聞いてください。
 長谷さん = この才能、ではなくて、**長谷さんが手にしている道具** です。
 道具なので、状況によって **使う / 置く / 別の道具を使う** ことができます。

 では読みます。
 『◯◯◯◯◯◯◯◯◯◯』」
```

**スクリプト（Why の一撃読み上げ前）：**

```
「2 つ目、長谷さんの **Why の一撃**。
 これは長谷さんが **抱えている源** で、長谷さんそのものではない。
 源だから、表現の仕方は色々ある。今は経営という形で表現してるけど、
 この源自体を **観察対象** として捉えてみてください。

 では読みます。
 『◯◯の◯◯を、◯◯するために、◯◯を作り続ける』」
```

**スクリプト（偏愛の一撃読み上げ前）：**

```
「3 つ目、長谷さんの **偏愛の一撃**。
 これは長谷さんの **エネルギー源** だと思ってください。
 エネルギー源だから、家族・健康・趣味とのバランスは **本人が選べる**。
 ここでも一旦、外に置いて観察してみてください。

 では読みます。
 『◯◯◯◯◯◯◯◯◯◯』」
```

### 命名後スクリプト（Somatic Resonance・身体的共鳴）── 各「一撃」読み上げ後に投入

> **思想：** 言語化した才能・Why・偏愛が本人の **身体感覚** と一致しているかを 1 問だけ確認する。Lakoff の身体化された認知と Gendlin のフォーカシングを統合した手法。

**スクリプト（才能の一撃読み上げ後・才能のキャラ命名後）：**

```
「いま読んだ **『◯◯な◯◯』** という命名、
 長谷さんの **身体のどこか** に当たってる感覚ありますか？
 例えば **ここ（胸 / 腹 / 喉）に来た** みたいな身体感覚です。
 もし当たってたら、**どこに、どんな感じで**当たったか、教えてください。
 当たってなかったら、命名を一緒に修正します。」
```

**スクリプト（Why の一撃読み上げ後）：**

```
「Why の宣言文、いま読んだ **『◯◯の◯◯を、◯◯するために、◯◯を作り続ける』**、
 これ、声に出して 1 回読んでみていただけますか？
 読んだ後、**身体の感覚** どうですか？
 力が抜ける？　逆に力が入る？　涙が出そう？　冷める感じ？
 1 つだけ教えてください。」
```

**スクリプト（偏愛の一撃読み上げ後）：**

```
「偏愛、**『◯◯ vs ◯◯』**、これ、長谷さんの **身体が即座に Yes と言う** 感じですか？
 それとも頭で『そうだな』と納得してる感じ？
 身体が即 Yes = ばっちり当たってる ／ 頭で納得 = 修正余地ありです。」
```

### 命名前後スクリプトの運用ルール

- **必須実行**：3 つの一撃すべてで命名前 Externalization + 命名後 Somatic Resonance の両方を投入
- **タイムボックス**：1 撃あたり Externalization 30 秒 + 一撃読み上げ 1 分 + Somatic Resonance 1 分 = 計 2.5 分 × 3 撃 = 7.5 分（STEP 5 読み合わせ 12-13 分に収まる）
- **修正発生時**：身体的共鳴で「当たってない」が出たら 1-2 分で命名修正（Sonnet レポート修正は不要・口頭で経営者本人と決める）
- **NG 行動**：
  - ❌ Externalization なしでいきなり「あなたは◯◯な人だ」と読み上げる（同一視 → 変革不能感）
  - ❌ Somatic Resonance を省略して「どうですか？」だけで終える（頭での納得 / 身体での共鳴の区別がつかない）
  - ❌ 身体感覚が出るまで沈黙を待たない（フォーカシングは 5-10 秒の沈黙が必須）
