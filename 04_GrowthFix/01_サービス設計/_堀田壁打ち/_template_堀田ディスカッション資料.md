## 📐 このテンプレの設計思想

**堀田セッションの効果を最大化する 5 要素：**

1. **🔴 セッション目的の冒頭明記（必須）**：60 分という制約を踏まえ、何を持って帰るかを最初に決める
2. **重大決定の抽出**：今回の目的に関連する戦略転換点のみに絞る（最大 3-5 件）
3. **数値ダッシュボード**：今回の目的に関連する指標のみ Before/After で可視化
4. **問いの構造化**：3-5 問に絞り、各問いに前提条件を添えて壁打ち効率化
5. **当日見てもらう URL リスト**：壁打ち対象がページ・LP の場合は共有用 URL を一覧化

**最重要原則：**
> **60 分しかない。目的を決めてからコンテンツを作る。**
> 全活動の網羅報告は不要。**今回のセッションで何を持ち帰りたいかを最初に決める**。
> 目的に関係ない「タイムライン」「全戦略決定」「全数値」「全課題」は意図的に削る。

これらを定型化することで、毎回ゼロから資料準備する負担を排除しつつ、**セッションの集中度を最大化**する。

## 📝 テンプレ運用ルール

### 作成タイミング

- 堀田セッション**前日夕方**または**当日朝**に作成
- 直前すぎると焦って核心を絞り込めない／余裕すぎると最新動向が反映されない

### Claude Code への依頼パターン

```
「堀田セッション準備、定型フォーマットで作成。
前回 {前回日}、本日 {本日}。
今回の目的は『{セッション目的を 1 行で}』。
目的に関連する重大決定 3-5 件・指標 3-5 個・問い 3-5 問だけに絞って。
それ以外の論点は今回は外す。」
```

### 各セクションの記入優先順位

1. **🔴 セッション目的（最重要・最初に書く）**：これが決まらないと他のセクションが膨張する
2. **重大決定（目的に関連するものだけ）**：3-5 件以内に絞り込む
3. **数値ダッシュボード**：目的に関連する指標 3-5 個のみ
4. **問い**：核心・**目的に紐づくものだけ**・3-5 問で止める
5. **見てもらう URL リスト**：壁打ち対象が LP・ページの場合のみ追加

### よくある記入ミス・回避策

| ミス | 対策 |
|---|---|
| **目的を決めずに書き始める** | **セッション目的を最初に 1 行で書く**・これが決まらないと膨張する |
| 全部書こうとして長くなる | 「目的に関連するか？」を各セクションで自問・関係ないものは削る |
| 結論を書いてしまう | 「結論を出さず問いを持っていく」原則を冒頭に明記 |
| 数値が古い | 作成当日のデータで集計・前回数値は前回資料から正確に引用 |
| 60 分の制約を意識しない | **問いは 3-5 問まで**・1 問あたり 10-15 分の議論時間を見込む |
| 全活動の網羅報告をしてしまう | タイムライン・全戦略決定・全課題は**省略可**・目的に関連するもののみ |

### セッション後の運用

1. **当日夜：** セッション記録（書き起こし or 要点まとめ）を `02_セッション記録/{本日}_堀田コーチング_書き起こし.md` に保存
2. **翌日：** `{本日}_堀田コーチング_示唆と改善アクション.md`（次回テンプレで参照する記録）を作成
3. **2-3 日以内：** `{本日}_堀田セッション残課題_棚卸し.md`（次回までの中期 TODO リスト）を作成
4. **次回セッション準備時：** 上記 3 ファイルをこのテンプレで参照→ 次回ディスカッション資料を作成

## 🎓 堀田 5 原則（資料設計時の検証軸）

新規施策・コピー・サービス設計を堀田さんに壁打ちする際、以下 5 原則で自己検証する：

| 原則 | 検証質問 |
|---|---|
| **意外性** | 経営者が「えっ？」と立ち止まる驚きがあるか |
| **心理的動線** | 興味→共感→行動の心理が自然に流れるか |
| **権威付け** | 信頼装置（実績・経歴・第三者証言）が組み込まれているか |
| **点数積み上げ** | 数値・事例・ストーリーで具体性が積まれているか |
| **ブランディング** | 独自の世界観・思想が一貫して表現されているか |

→ 各問いを上記 5 原則で自己検証してから持っていくと、セッション効率が 1.5-2 倍上がる。
