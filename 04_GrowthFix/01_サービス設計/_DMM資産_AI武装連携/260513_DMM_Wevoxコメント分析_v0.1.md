1. **業務量過多パターン**：業務量・残業・キャパに言及しつつ「自分だけ」「不公平感」が重なる
2. **評価不満パターン**：頑張りと報酬・昇格の乖離感を訴え、比較対象（他者・他部署）への言及あり
3. **キャリア閉塞パターン**：現職での将来像が描けない、学べる機会がないという成長欲求の不充足
4. **人間関係悪化パターン**：特定の関係性への言及（マスキング済）+ 孤立・相談困難の組み合わせ
5. **組織不信パターン**：経営方針・制度変更への不満 + 説明がないという透明性欠如の組み合わせ

## 5. 個別フォロー判別ロジック（DMM 精査 SS から抽象化）

### フォロー優先度判定フレーム（3 段階）

```
if 離反シグナル >= 2種類 AND ネガティブ判定:          → Priority A（即時フォロー）
elif 離反シグナル == 1種類 OR 強ネガ（-2以上）:       → Priority B（今月中フォロー）
elif ポジティブ AND 建設的提案含む:                   → Priority C（次回 1on1 で確認）
else:                                                  → Priority D（モニタリング継続）
```

### フォロー種別マッピング

| コメントカテゴリー | 推奨フォロー種別 | 担当者 |
|---|---|---|
| 業務量・負荷 | 業務棚卸し + 優先度再設定 | 直属マネージャー |
| 上司・マネジメント | 匿名化した意見の 1on1 への持ち込み | HRBP |
| 評価・報酬 | 評価制度の説明 + 目標設定面談 | マネージャー + HR |
| キャリア・成長 | キャリア面談 + 社内公募情報提供 | HRBP |
| 人間関係・チーム | 心理的安全性ワーク or 個別相談 | HRBP（匿名性確保）|
| 組織・文化 | 経営層への意見還元 + フィードバック | HRBP → 経営 |

## 6. HR 精査フロー実態（スプレッドシート構造から逆算）

**スプレッドシート構造から確認された精査観点（個別テキスト不掲載・構造抽象化のみ）：**

SS の列・セル構造から以下の精査フローが確認された（担当者名・固有名詞はすべて除去済）：

- 精査ステップが番号付きで明文化されており、「チェック → 転記 → 連絡」の 3 段階構造
- コメントのカラー分類（掲載要検討 / 人事総務向け）による初期振り分け
- 育成側 HRBP と人事・総務側の役割分担が列単位で管理
- 最終決定の記録欄が独立して設けられている
- コメントへの対応メモを担当者が自由記述で付記する運用

### 推定精査フロー

1. **コメント収集**：Wevox から CSV / API エクスポート
2. **初期スクリーニング**：カテゴリーラベル付け（手動 or 半自動）
3. **緊急度スコアリング**：複数シグナル重複 → 高優先度フラグ
4. **担当振り分け**：HRBP 担当者別に割り当て
5. **アクション記録**：フォロー実施状況・結果を同一 SS に記録
6. **集計・報告**：月次 / 四半期でスコア集計 → 経営報告

### 精査の実態的限界（AI 武装で解決すべきボトルネック）

- コメント数が多い場合、目視精査に数時間〜数日かかる
- 精査担当者の主観・疲弊でバイアスが入る
- 離反予兆の見落とし（複合シグナルの検出が困難）
- フォロー後の効果追跡が SS 管理では属人的

### ポジ・ネガ判定 + 緊急度判定ロジック（v0.2）

```python
def classify_urgency(comment_text, category_matches, churn_signals):
    """
    Returns: (urgency: 'A'|'B'|'C'|'D', rationale: str)
    """
    if len(churn_signals) >= 2:
        return 'A', f'離反シグナル複合: {churn_signals}'
    if len(churn_signals) == 1:
        return 'B', f'離反シグナル単発: {churn_signals[0]}'
    if '評価・報酬' in category_matches or '人間関係・チーム' in category_matches:
        return 'B', 'ハイリスクカテゴリー検出'
    if any(pos in comment_text for pos in ['感謝', '満足', 'やりがい']):
        return 'C', 'ポジティブ + 建設的提案'
    return 'D', 'モニタリング継続'
```
