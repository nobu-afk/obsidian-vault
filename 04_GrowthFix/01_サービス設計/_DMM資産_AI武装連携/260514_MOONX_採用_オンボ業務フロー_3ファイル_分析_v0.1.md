## 1. 業務フロー：採用広報〜採用（PDF）構造抽出

### 1.1 全体設計の位置づけ

採用を「候補者の意識変容フロー」として設計した 1 枚のマトリクス図。横軸を候補者フェーズ（準備 → 認知 → 興味/検討 → 応募 → 選考 → 内定 → オンボーディング）、縦軸をタッチポイント種別に分解している。

**構造上の特徴：採用プロセスの「完全可視化・1 枚俯瞰」設計**
単なる選考フロー図ではなく、採用広報（SNS / 自社チャネル / 社内イベント）から内定後の「ラブレター（Offer 状）」まで全工程を連続したユーザージャーニーとして設計している点が際立つ。

### 1.2 チャネル設計の構造

**SNS チャネル層（認知〜興味形成）**

| チャネル種別 | 機能 |
|---|---|
| Wantedly | 採用文脈の認知獲得・ストーリー訴求 |
| Twitter（現 X） | 経営者・社員の個人発信 |
| LinkedIn | ミドル〜シニア層へのアプローチ |
| Facebook | 幅広い年代への認知 |
| Instagram | カルチャー・雰囲気訴求 |

**自社チャネル層（興味〜検討深化）**

| コンテンツ種別 | 内容 |
|---|---|
| 会社 HP | 基本情報 |
| 経営者ブログ | 代表の思想・スタンス訴求 |
| 社員ブログ | 現場リアリティ提供 |
| 人事ブログ | 採用プロセス・文化情報 |
| 採用ページ | 要件・期待役割・キャリアパス・必須/歓迎スキル・求める人物像・評価基準を掲載 |
| カルチャーデッキ | 価値観の詳細訴求 |
| 採用ピッチデッキ | 総合訴求資料（選考中配布） |

**社内イベント層（カルチャー浸透・温度感醸成）**

| 頻度 | イベント種別 |
|---|---|
| 毎週 | 社内交流会（飲み会形式・フランクな場） |
| 毎月 | 業界勉強会・人事主催イベント |
| 四半期 | 全社ミーティング（オールハンズ） |
| 毎年 | 年次総会（忘年会等） |
| 不定期 | 業界ニュース共有・オフサイトミーティング |

### 1.3 求人要件設計（ポジション設計の 7 要素）

採用ポジション 1 件あたり以下の 7 要素を定義する構造：

1. ペルソナ（新規作成）
2. 業務詳細（JD）
3. 期待される役割
4. ポジションの魅力（やりがい）
5. 上長の魅力・雰囲気
6. 部署の魅力・雰囲気
7. 入社後のキャリアパス

### 1.4 応募チャネル設計

**有料チャネル**：大手求人サービス（リクナビ・DODA）/ 求人広告系（indeed）/ 転職フェア

**無料チャネル**：求人サービス（ハローワーク）/ エージェント / リファラル（社員紹介）/ ビジネス特化型 SNS（ビズリーチ系）

**注目点：リファラルを正式チャネルとして設計に組み込み**（無料チャネルの中核）

### 1.5 選考フローの構造

```
一次面接 → 二次面接 → 最終面接 → 内定
                                    ↓
                            ラブレター（Offer 状）
                                    ↓
                                  入社
```

**ラブレター（Offer 状）** が単なる内定通知ではなく、独立した工程として明示されている点が特徴的。温度感維持・競合他社への意思決定優位性の確保が目的と読み取れる。

### 2.2 Coffee シリーズ設計（関係構築の構造化）

**「Networking Coffee」** という形式で、社内の関係構築を明示的にスケジュール化している点が際立つ。

| Coffee 種別 | 対象 | タイミング |
|---|---|---|
| Welcome Coffee | チーム全体 | Day1 前後 |
| 代表 Coffee（4 回設計） | 代表 × 入社者 | Pre / Week2 / 1 ヶ月 / 3 ヶ月 |
| マネジャー Coffee | 自部門マネジャー・他部門マネジャー | 各フェーズ |
| 社員 Coffee | 自部門社員・他部門社員 | 各フェーズ |

**特徴：代表と入社者の 1on1 を 4 回（Pre〜3 ヶ月）定期設計**。30〜60 名規模でも代表が全員に直接接触する構造を意図的に維持している。

### 2.3 チェックインと面談の設計

**人事面談の 2 段構成：**

- 1 ヶ月後面談：Goal 設定（初期期待値のすり合わせ）
- 3 ヶ月後面談：Goal 設定（中期成果レビュー・次フェーズ設定）

両方「Goal 設定」と明記されており、振り返りよりも前向きな期待設定を目的とした設計。

## 3. 採用広報（PPTX 24 スライド）構造抽出

### 3.1 企画全体の目的と KPI 設計

**背景と目的（Slide 7・9 より）：**

| ゴール | 指標 | 状況 |
|---|---|---|
| ターゲットへのアプローチ手段確立 | コンテンツ経由面接進捗率（●%以上） | 測定手段なし → 一次面接でヒアリング開始 |
| 会社の認知拡大 | 月間応募数（直近 3 ヶ月平均 33 名/月 → 目標値 TBD） | 自主応募:スカウト = 71:21 |
| 採用コスト削減 | 自主応募エントリー数の増加 | 自主応募 71 件→2 名採用 / スカウト 21 件→採用 0 |
| 選考中離脱防止 | 選考中辞退率（全国平均 15.7%・当時の実態 16.7%） | 平均より 1% 悪い状態 |

**選考中辞退率の管理**が採用広報の KPI に明示的に組み込まれている点が特徴的。コンテンツ施策を「ファネル全段（無関心〜活躍）」と繋げて設計している。

### 3.2 候補者ペルソナ設計（Slide 3・4・8 より）

**4P フレームワークで候補者の欲求と不安を構造化：**

| 軸 | 候補者の欲求 | 候補者の不安 |
|---|---|---|
| Philosophy（企業理念） | 社会的意義・自分の原体験との近さ・目的意識 | Vision との矛盾・浸透度・変更リスク |
| People（人・文化） | レベルの高い人との仕事・働きやすい風土 | チームの雰囲気実態・配属先メンバーの人柄 |
| Profession（事業） | 新スキル獲得・事業内容へのこだわり | 事業実態・競合優位性・今後の展開 |
| Privilege（待遇） | ワークライフバランス・給与・勤務地 | 昇給/昇格の実態・リモートの実態・転勤の有無 |

**ペルソナストーリー設計の特徴（Slide 8）：**

社会人 2〜5 年目（24〜27 歳）・レガシー大企業出身・「年功序列への違和感」「成長鈍化への不安」「裁量不足のモヤモヤ」を軸に、内面の変化を時系列で記述したナラティブ型ペルソナ。スペック定義ではなく、**心境変化のストーリーとして設計**している点が際立つ。

**刺さるメッセージ軸（Slide 8 より抽象化）：**

1. 成果主義・キャリアステップの可視化（ロールモデル明示）
2. 個人の裁量と組織の急成長の両立
3. 「会社をつくるフェーズ」のジョインワクワク感
4. メンバーが楽しそうに働いているリアリティ
5. 意思決定の速さ
6. 代表のフランクさ（距離感の適切さ）

### 3.3 差別化ポイント（PoD）設計（Slide 5 より）

**「候補者が求めていて、かつ競合が実現できない機能上の強み」として 4P 別に定義：**

| 軸 | PoD コンテンツ項目 |
|---|---|
| 企業理念 | Vision / Mission / 創業経緯 / 社名の由来 / 沿革 |
| 人・文化 | 経営陣・メンバー紹介 / チーム別雰囲気 / 企業文化 / Value / 組織図 |
| 事業 | 事業内容 / ビジネスモデル / 競合優位性 / 今後の戦略 |
| 待遇・働き方 | 働く環境 / 働き方詳細 / 福利厚生 / **オンボーディング内容** / 給与テーブル・評価制度 |

**「オンボーディング内容」が待遇・PoD の一要素として採用広報に組み込まれている**点は、採用〜定着を一体設計している証左。

### 3.4 コンテンツ戦略の 4P × ファネル設計（Slide 19 より）

採用ファネル（無関心 / 認知〜応募 / 選考〜入社）× コンテンツ軸（Philosophy / People / Profession / Privilege）のマトリクスで、**各チャネルの有効フェーズを定義**：

| ファネル段階 | 有効チャネル（代表例） |
|---|---|
| 無関心層 | Facebook / note / Instagram / YouTube / Wantedly / Twitter |
| 認知〜応募 | Facebook / LINE 公式 / note / YouTube / Wantedly / Twitter / Instagram |
| 選考〜入社 | Facebook / note / YouTube |

**YouTube と note が全フェーズに有効**とされており、作成優先度を高く設定。

### 3.6 KPI 管理体系（Slide 23 より）

チャネルごとに以下の 7 指標を並列管理する設計：

1. フォロワー数
2. 投稿数
3. FF 比（フォロワー/フォロー比率）
4. 選考中辞退率
5. 候補者プール数
6. 応募数
7. 入社数

**SNS 運用指標と採用結果指標を同一管理表で管理**する設計は、採用広報の施策と採用成果の因果関係を可視化する意図。

## 4. 3 ファイル横断パターン（スタートアップの採用設計に見える特徴）

### 4.1 採用〜定着の一体設計思想

3 ファイルを通じて最も際立つ特徴は、**採用広報・選考・内定・オンボーディングを分断せず 1 本のユーザージャーニーとして設計している点**。

- 採用広報 PDF：「準備 → 認知 → 興味/検討 → 応募 → 選考 → 内定 → オンボーディング」を 1 枚に収める
- PPTX：採用広報の KPI に「選考中辞退率」を組み込む（採用広報 = 内定受諾率にも効く）
- オンボーディング PDF：「採用時の期待値」と「入社後 Goal 設定」をリンクさせる設計（1 ヶ月・3 ヶ月面談）

### 4.2 代表の直接関与を構造化（30〜60 名規模の武器）

- オンボーディング PDF：代表 Coffee を 4 回（全フェーズ）スケジュール設計
- PPTX ペルソナ：「代表のフランクさ」を刺さるメッセージ軸の 1 つに明示
- 採用 PDF：代表ブログ・経営者ブログを自社チャネルの 1 柱として定義

→ 大企業では不可能な「代表の人格を採用エンジンに組み込む」構造を意図的に設計。

### 4.3 コンテンツによる候補者の意識変容設計（温度感管理）

PPTX の採用広報設計は一貫して「**コンテンツによる候補者の温度感上げ**」を目的として設計：

- 無関心層へのリーチ → SNS 広告・note・YouTube
- 温度感の高い層の維持 → 音声コンテンツ・インタビュー記事・社員 1 日ルーティン
- 選考中の離脱防止 → 選考期間中の継続的コンテンツ接触
- 内定後の競合対策 → ラブレター（Offer 状）

### 4.4 KPI の連鎖設計（SNS 指標 → 採用成果指標）

PPTX の KPI 管理が「フォロワー数・投稿数・FF 比」（SNS 運用指標）と「候補者プール数・応募数・入社数」（採用成果指標）を同一テーブルで管理している。

→ **採用広報の活動量と採用成果の因果関係を日次〜月次で可視化できる設計**。

### 4.5 4P フレームで採用メッセージを体系化

ポジショニング設計に使う 4P（Philosophy / People / Profession / Privilege）を採用広報設計に転用。候補者の欲求・不安・PoD・コンテンツ種別・チャネル割り当てのすべてを同一フレームで統一している。

## 5. R/C/Orbit AI 武装サービス v0.3 反映候補（Shift R 最優先）

### P0（Shift R Week 1-2 納品物への即時転用候補）

**P0-a：採用ポジション設計 7 要素テンプレ（Shift R Week 1-2 実装）**

採用 PDF が定義した 7 要素（ペルソナ / JD / 期待役割 / ポジション魅力・やりがい / 上長の魅力 / 部署の魅力 / キャリアパス）を、Shift R の JD 設計テンプレとして構造転用可能。

現行 Shift R Week 1-2 納品物は採用基盤設計フレームとして「JD 作成」を含むが、**7 要素テンプレの明示的な提供**はまだ未整備の可能性が高い。即時転用価値が高い。

**P0-b：オンボーディング 4 軸 × 3 フェーズ設計テンプレ（Shift R Week 2 納品物候補）**

「How to live / How to learn / How to work / How to influence」× 「Pre〜Day1 / 1 ヶ月 / 3 ヶ月」のマトリクステンプレは、Shift R の採用基盤設計の「定着設計」パーツとして転用可能。

特に以下の 3 要素は汎用性が高い：

1. 代表 Coffee（4 回スケジュール設計）= 社長の関与度設計
2. 入社初期資材 7 点（歓迎グッズ / 用語集 / Notion / リンク集 / 事業 History / KPI Tree / 入社後計画）
3. 1 ヶ月・3 ヶ月 Goal 設定面談（振り返りではなく期待値設定として設計）

**P0-c：選考中離脱率の KPI 組み込み設計**

「採用広報の効果 = コンテンツ経由面接進捗率 + 選考中辞退率」として設計している構造は、Shift R の採用 KPI 設計のフレームとして即転用可能。

現行 Shift R の KPI 設計が「応募数・内定数」にとどまっている場合、**選考中辞退率を第 3 の KPI として追加**することで採用広報との接続設計が完成する。

### P1（Shift C / Gravity Orbit 反映候補）

**P1-a：入社後成果設計（Shift C 反映）**

オンボーディングの「How to influence（他者への影響力発揮）」軸は、Shift C（躍動組織設計）の初期期待値設定フレームに転用可能。特に「3 ヶ月振り返り = Goal 設定（次フェーズ）」という形式は Shift C の月次レビュー設計に組み込める。

**P1-b：SNS 採用広報 × コンテンツ KPI 管理（Orbit AI 武装）**

チャネル別 7 指標（フォロワー / 投稿数 / FF 比 / 辞退率 / プール数 / 応募数 / 入社数）の一元管理設計は、Orbit の AI 自動化ダッシュボード設計の参考モデルになる。Zoom 録音 → 月次レポートと同様に「採用広報活動量 → 採用成果」の因果可視化を AI 自動化できる。

### P2（コンテンツ設計・営業広報への参考）

**P2-a：音声コンテンツ対談設計**

代表 × 新メンバー / 先輩 × 新メンバー / 代表の 1 日密着 という音声コンテンツラインナップは、GrowthFix の採用広報（GrowthFix 自身の採用・Shift R クライアントへの事例提示）に転用可能。

## 6. 仮説 E v0.2 営業資料への組込候補（3-5 件）

### 組込候補 1：「採用は JD ではなくユーザージャーニーで設計する」訴求

PPTX の採用広報設計全体が示す「候補者の意識変容フロー（無関心 → 活躍）」として採用を設計するアプローチは、Gravity の「採用基盤 = 引力を設計する」思想と直結。

営業資料への組込案：

> 多くの会社の採用設計は「JD を書いて媒体に出す」で止まっている。私たちが設計するのは、候補者が「無関心」から「この会社で働く」と決意するまでの全工程だ。

### 組込候補 2：「代表の人格を採用エンジンに組み込む」設計思想

30〜60 名規模スタートアップで代表 Coffee を 4 フェーズ設計した実例は、「代表の引力を採用に使いきれていない経営者」への具体的な処方箋として機能する。

営業資料への組込案：

> 30〜60 名規模では「代表の人格・スタンス」が最大の採用差別化要因になる。Shift R では代表の引力を組織的に設計し直す。

### 組込候補 3：「選考中辞退率を採用広報の KPI に組み込む」設計

採用広報の KPI に「選考中辞退率（業界平均 15.7%・改善目標値を設定）」を明示している設計は、採用広報 = 採用成果への影響を数値で語れる構造を示す具体例。

営業資料への組込案：

> 採用広報の失敗は「応募が来ない」だけではない。選考中の離脱率が 1% 改善するだけで、年間採用コストが大幅に変わる。Shift R では採用広報設計から辞退率までを一体で管理する。

### 組込候補 4：「オンボーディングを採用広報の PoD として使う」訴求

PPTX Slide 5 で「オンボーディング内容」が採用差別化ポイント（PoD）として候補者に訴求される設計は、採用〜定着の一体設計が採用競争力に直結することを示す実例。

営業資料への組込案：

> 採用が強い会社のオンボーディングは、選考前から設計されている。「入社後どうなるか」をコンテンツで見せることが、今最も効く採用広報の武器になっている。

### 組込候補 5：「採用 KPI を 7 指標で管理する」構造提示

フォロワー数〜入社数まで 7 指標をチャネル別に並列管理する設計は、「採用活動の可視化・AI ダッシュボード化」のニーズを示すリアルな事例。Orbit の AI 武装訴求と連動可能。
