本資料は DGホールディングス（DMM グループの持株会社）が 2021 年に策定した「新規事業バックオフィスサポート」改善提案（改善 16）の成果報告書。スコープは M&A / カーブアウト型新規事業の PMI 場面で、バックオフィス各部（法務・経理・情報システム等）が縦割りになっていた課題を解消する「新規事業バックサポートチーム」設置提案が核心。

## 1. 業務分解構造（バックオフィス機能）

本資料で示された関与機能（部署）：

| 機能 | 部署名（マスキング済・組織種別として記載） |
|---|---|
| グループ財務統括 | CFO 室 |
| 法的リスク管理 | 法務部 |
| コーポレートサポート全般 | コーポレート室 |
| 中長期経営計画・事業評価 | 経営企画室 |
| 会計・財務報告 | 主計部 |
| IT インフラ・システム整備 | 情報システム部 |

**構造上の特徴（課題として顕在化していた点）：**
1. バックオフィス各部の窓口が**分散**しており、事業部は各部に個別説明が必要
2. バックオフィス部門間の**横連携がない** → 差し戻しが多発
3. 事業部からバックオフィスへの情報伝達タイミングが**遅い**（スキーム固まってからの相談 → 前工程アドバイス不可）

**改善の方向性（提案要旨）：**
- 「新規事業バックサポートチーム」を設置し、事業企画**初期段階**から事業部に同行
- 0→1 立上げ + 1→10 利益創出フェーズの両方をカバー
- M&A / カーブアウト型新規事業を主対象とし、統一フロー + チェックリストで意思決定の**属人性を排除**

## 2. 改善 16 提案の論点抽出

本スライドは「改善 16」という社内連番のプロジェクトの成果報告。核心論点は 2 点：

### 論点 A：M&A フローの標準化（プロセス設計）

- 当初対象：エンタメ本部のみ → **全社展開に拡大**
- 成果：COO 室と協力して M&A フロー策定 / DD（デューデリジェンス）手法を各部で統一 / チェックリスト整備
- 2022 年度中に全社ルール適用を目標とした（2021 年提案時点）

### 論点 B：「忖度なし」検討体制（ガバナンス設計）

- チェックリストで感覚・忖度に依存しない判断フローを実現
- 事業部への「アドバイス型関与」（受身 → 能動）へのシフト

**提案の本質構造：**
```
Before：事業部→各バックオフィスに個別説明（縦割り・後工程相談）
After ：バックサポートチームが初期から伴走（横連携・前工程介入）
```

## 3. Cultivate AI 武装エージェント反映候補

優先度は低（Shift S 凍結中・具体的業務フローの資料がない）。ただし以下 2 点は思考素材として価値あり。

### 3.1 「前工程介入型 HR」の設計原則

本資料の最大の示唆は「HR（バックオフィス）が事業の後工程でなく**企画初期から介入する**」ことで価値が変わるという構造。

→ Cultivate（業務分解 AI）への反映候補：
- **マネージャー支援のタイミング設計**：問題発生後のフォローではなく、組織設計・採用計画の**初期段階から AI アシスト**が入るフローを GrowthFix サービス設計に組み込む
- Orbit「軸 15 個ゲート」の G 軸（ガバナンス系）に「意思決定プロセスの早期介入ポイント設計」を追記候補とするか検討可

### 3.2 チェックリスト × 属人性排除のフレーム

- 「チェックリストで忖度なし検討」= Orbit の定例月次フォーマットや Scan 診断の定型評価軸と同じ設計思想
- **既存 Orbit / Scan 設計と整合しており、追加実装なし**で OK。既存方向性の傍証として保存

### 3.3 バックオフィス 6 機能の統合フロー

6 機能（CFO / 法務 / コーポレート / 経営企画 / 主計 / IT）が PMI 場面で一体化する必要があるという構造は、**Shift C（組織躍動化）の設計コンテキスト**として有用。Shift C がターゲットとする「部門間連携の崩壊」課題の具体ケースとして参照可。
