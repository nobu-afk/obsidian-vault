## 1. リファラル制度の運用フロー構造

### 1.1 全体フェーズ構成（4 フェーズ）

PDF 画像から視覚的に確認した構造。左列 = 担当者・フェーズ名、右列 = 具体手順・コミュニケーションテンプレ。

```
Phase 1：エントリー（紹介者 → 被紹介者 → HR）
  ├─ 紹介者が社内フォームから被紹介者情報を登録
  ├─ HR が被紹介者にコンタクト（メールテンプレ）
  └─ カジュアル面談の設定

Phase 2：選抜 / 面談管理
  ├─ 書類確認（通常選考と同一基準・明示）
  ├─ 1次面接 → 最終面接の設計
  ├─ 面接官向けコミュニケーション指示（紹介者への途中報告禁止ルール等）
  └─ 不合格通知フロー（紹介者へのフォロー含む）

Phase 3：金券管理（インセンティブ処理）
  ├─ 入社決定時の確定処理（HR → 担当部門 → 経理ルート）
  ├─ 支払タイミングの明示（入社後一定期間後の在籍確認ゲート）
  └─ 金券種別・申請フォーム管理

Phase 4：入社後管理（オンボーディング連動）
  ├─ 入社後フォロー面談のタイミング指定
  ├─ 在籍確認時期と紹介者への完了通知
  └─ 紹介者へのフィードバックループ（定性）
```

### 1.2 フロー設計の核心構造的特徴

**特徴 1：紹介者と被紹介者を完全に分離したコミュニケーション管理**
- 面接進捗を紹介者に逐一報告することを明示的に禁止
- 理由推定：紹介者経由の「圧力」による不本意な選考結果を回避 / 評価の公正性担保
- Gravity 解釈：候補者体験設計（Layer 4：ファン獲得）との親和性 = 不合格でも候補者・紹介者を支持者として保持するための分離設計

**特徴 2：通常選考と同一基準の明示**
- 「リファラルだから通過しやすい」という誤解を紹介者・被紹介者双方に払拭
- 基準の明示によって紹介者の「人脈消費リスク」を軽減 → リファラル参加障壁を下げる
- Gravity 解釈：ASA Theory（Schneider 1987）の機能維持 = 引力で集まる人材の質を守る装置

**特徴 3：在籍確認ゲートによる二段階インセンティブ**
- 入社決定 + 一定期間在籍を確認してから紹介者にインセンティブ支払
- 短期離職を経てのインセンティブ成立を防ぐ設計
- Gravity 解釈：P-O Fit が実際に機能したか（定着）を確認してから価値を確定する思想と一致

**特徴 4：不合格時の紹介者フォロー設計**
- 不合格通知後に紹介者へのフォローコミュニケーションが別途設計されている（推定）
- 紹介者の「恥をかかせてしまった」感情を HR が吸収する仕組み
- Gravity 解釈：リファラル参加のリピート率を維持する最重要設計。整合型組織ではこれが自然発生しているが、構造化することで再現性が生まれる

## 2. インセンティブ設計

### 2.1 設計構造（抽象化）

| 要素 | 構造 |
|---|---|
| 受取者 | 紹介者（既存社員）のみ / 被紹介者への支払なし（推定） |
| 支払タイミング | 二段階：入社決定時の権利確定 + 在籍確認後の実支払 |
| 金券種別 | 現金相当の金券（Amazonギフト等相当と推定） |
| 申請経路 | 紹介者 → 専用申請フォーム → HR → 経理 |
| 金額水準 | DMM 固有数値のため非掲載。業界相場観として：正社員採用 1 件あたり 5-30 万円水準 |
| 確認ゲート | 在籍X ヶ月以上（具体期間は非掲載） |

### 2.2 インセンティブ設計の構造的示唆

- 「入社決定だけで払う」設計は短期離職リスクを紹介者が負わない半面、質より速度の紹介を誘発する
- DMM の二段階ゲートは「紹介者も在籍を応援する」インセンティブ構造 = 定着率向上への間接効果あり
- Gravity 解釈：インセンティブ金額より「紹介する文化」の醸成が本質。整合型組織では金銭インセンティブなしでも社員リファラルが 40% 超（第 7 章 Case1 実証）

## 3. 体験設計

### 3.1 紹介者（既存社員）の体験設計

| フェーズ | 体験設計要素 |
|---|---|
| 登録時 | フォーム完結・HR 担当者明示での受付確認連絡（安心感） |
| 選考中 | 進捗非開示の明示（逆に「あなたの人脈を傷つけない」安心設計） |
| 不合格時 | HR からの紹介者向け個別フォロー（詳細非掲載だが設計存在を確認） |
| 合格・入社時 | 節目での通知 + インセンティブ申請案内 |
| 在籍確認後 | 完了通知 + 謝意メッセージ（推定） |

**体験設計の要点**：紹介者が「次も紹介したい」と思う状態を維持するための不安除去設計が体系化されている。

### 3.2 被紹介者の体験設計

| フェーズ | 体験設計要素 |
|---|---|
| 最初のコンタクト | HR 担当者から個別メール（テンプレあり） |
| カジュアル面談 | 通常選考とは別軸の「まずは話を聞く」設計 |
| 選考参加 | 通常選考と同一基準を明示（特別扱いなし → 入社後の公正感担保） |
| オファー | 通常選考と同一プロセス（推定） |
| 入社後フォロー | 在籍確認フェーズに連動した HR フォロー面談 |

**体験設計の要点**：「知人の会社だから甘い選考」ではなく「知人が胸を張って紹介できる会社の水準を通過した」という自己効力感の付与。Bandura（1977）の Self-Efficacy 設計として機能している。

## 5. 書籍補強素材

### 5.1 第 7 章 Case1（整合型・ペット業界・リファラル 40% → 55%）への補強

**現状の Case1 記述**（v0.1）：社員リファラルが採用チャネルの 40% 以上・6 ヶ月後 55% に上昇・採用コスト 40% 削減

**DMM 資産で補強できる内容**：

| 補強ポイント | 内容 |
|---|---|
| 「なぜリファラルが機能するか」の制度的裏付け | DMM の 4 フェーズフロー = 整合型組織で自然発生しているリファラルを「意識的に維持・拡大する装置」として提示可能 |
| 「紹介者体験設計がリファラル持続の鍵」 | 不合格時フォロー / 進捗非開示ルールが紹介者の継続参加を生む構造を、Case1 の 6 ヶ月後 55% 上昇の理由として後付け説明できる |
| インセンティブ設計の比較軸 | Case1 は金銭インセンティブなしで 55% 達成 = 整合型の CE 高さが金銭不要のリファラルを生む。DMM は金銭インセンティブあり = 拡散型・渇望型でもリファラルを機能させる補助装置 |

**追記案（Case1 補強パラグラフ）**：
> 社員リファラルが 55% に達したこの会社には、リファラル制度が明文化されていなかった。フォームもインセンティブもなく、ただ「この会社に来たいか聞いてみて」という口頭の文化だけがあった。整合型の CE 高い組織では、Schneider（1987）の ASA Attraction が内側から自然に機能する。対照的に、大手企業のリファラル50制度が示すのは、整合型でない組織でリファラルを機能させるには「紹介者体験設計 × 二段階インセンティブ × 不合格フォロー」という精密な装置が必要だということだ。整合型の中堅企業にとって、この対比は強力なメッセージになる。「あなたの会社が整合型に近づくほど、このコストが要らなくなる」。

### 5.2 第 9 章 §4（採用接続資産 / ストック 1：既存社員リファラルストック）への補強

**現状の §4 記述**（v0.1）：リファラル ROI = エージェント採用の 3-5 倍 / 手数料不要・文化 Fit 高い・早期離脱率低い / eNPS 確認が前提

**DMM 資産で補強できる内容**：

| 補強ポイント | 内容 |
|---|---|
| 「リファラルが機能しない会社の構造」の具体化 | DMM の 4 フェーズは整備してあるが、中堅 SMB の多くはフェーズ 1（フォームのみ）で止まっている。フェーズ 2-4 がないと「登録したが何も聞こえてこない」で紹介者体験が死ぬ |
| リファラル比率の目安 | 整合型組織 40%超（Case1）が自然発生の上限。大手でも精密設計で20-30%が到達目標水準（DMM 参考） |
| Week 11-12 棚卸し項目の追加 | リファラルフロー 4 フェーズ整備状況 / 不合格フォロー有無 / eNPS × リファラル比率相関 を具体的チェック項目として追加 |

## 9. 主要発見サマリー（トップ 3）

**発見 1【最重要】リファラル制度は「フォームを作る」ではなく「4 フェーズを全部設計する」が機能の条件**

中堅 SMB の多くはフェーズ 1（登録フォーム）で止まる。フェーズ 2（進捗非開示ルール）・フェーズ 3（二段階インセンティブ）・フェーズ 4（入社後フォロー）がなければ紹介者体験が死に、リファラルは 1-2 回で機能停止する。recruit-layer-audit Layer 1 の診断軸として最重要。

**発見 2【重要】不合格時の紹介者フォローが「リファラル文化の持続装置」**

不合格が出た時に HR が紹介者の感情を吸収する設計がある会社だけが、リファラルをチャネルとして持続できる。大手が持つ精密設計の最大の差別化ポイント。中堅 SMB への営業で「あなたの会社にこれはあるか」を即座に効く診断軸として活用可。

**発見 3【書籍補強】整合型 vs 非整合型のリファラルROI対比が第 7 章 Case1 を立体化する**

整合型（Case1・金銭インセンティブなし・リファラル 55%）vs 非整合型（精密 4 フェーズ設計あり・金銭インセンティブあり）の対比が、「Gravity で整合型に近づくほど採用コストが下がる」という仮説 E の中核訴求と完全一致。書籍第 7 章と営業資料の双方を同時に強化できる素材。
