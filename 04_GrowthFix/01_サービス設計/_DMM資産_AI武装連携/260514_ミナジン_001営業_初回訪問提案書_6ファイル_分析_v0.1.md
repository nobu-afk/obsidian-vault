## 1. 初回訪問提案書のバージョン進化から見える営業戦略変遷

### 1.1 ファイルサイズの推移と版歴

| バージョン | ファイル名 | サイズ | 更新日 | 特徴 |
|---|---|---|---|---|
| 旧版（2016年） | 人事評価システム提案書（単体30）_1612.ppt | 8,046 KB | 2017-01-19 | 最大サイズ・詳細仕様版 |
| 中版（2018年1月） | 人事評価システム提案書_20180123.ppt | 6,918 KB | 2018-04-11 | 軽量化・訴求重視 |
| 中版（2018年5月） | 【人事評価システム】提案書_20180524.pptx | 2,379 KB | 2019-04-01 | 大幅軽量化（.pptx化） |
| 主版（2019年4月）| 【人事評価システム】提案書_20190401_料金なし.pptx | 2,553 KB | 2019-04-01 | 「料金なし」版 |
| 最終版（2020年） | 【人事評価システム】提案書_20200302.pptx | 2,498 KB | 2020-03-07 | COVID 直前最終版 |

**発見 2：提案書は 8MB（2016）→ 2.5MB（2019）へ 68% 軽量化されていた**

これは「詳細仕様説明 → 価値訴求」への戦略転換を示す。
初期は「システムができること」を詳しく説明（機能仕様型提案書）。
後期は「導入後に何が変わるか」を伝える（価値訴求型提案書）。

→ GrowthFix Shift R/C LP 設計への教訓：**LP は機能説明から価値変容の物語へ**。

### 1.2 「料金なし」バージョンの存在が示す営業戦略

「提案書_20190401_**料金なし**.pptx」が初回訪問時の原本として設計されている。

**なぜ初回訪問時に料金を見せないのか：**
- 先に「価値の理解」を作ってから「価格対話」に移行する 2 段階営業
- 社長・経営幹部への初回面談では「痛みの共感 → 解決策の仮説」で終わらせる
- 料金は「見積書（b_見積もり作成時）」フェーズで提示

→ ミナジンの標準営業フローが「初回提案（料金なし）→ 見積提示 → 申込」の 3 ステップであることが構造化できる。

**GrowthFix への転用**：Shift R/C の初商談では「80 万 / 150 万」の数字より先に「3 ヶ月後の組織像」を提示する設計が有効。Gravity が LP 前面メッセージに料金を出さないのは同じ原則。

### 1.3 「みんなの人事評価」と「人事制度サポート」の二軸提案体制

フォルダ構造から 2 種類の初回提案書が存在することが確認できる：

**軸 1：みんなの人事評価（SaaS 訴求型）**
- サイズ：2,937 KB（訪問時）/ 2,096 KB（パンフ）
- 対象：「評価システムを探している」顕在層
- 訴求：SaaS の使いやすさ・操作画面・月額費用対効果

**軸 2：人事制度サポート（コンサル訴求型）**
- サイズ：2,945 KB（2018年）
- 対象：「制度から整備したい」潜在層
- 訴求：10 年の中小企業支援実績・3 フェーズ（設計→構築→運用）

**発見 3：ミナジンは「SaaS 起点」と「制度設計起点」の 2 つの入口を整備していた**

どちらの入口で顧客が来ても最終的にオーダーコンサル（制度設計）+ みんなの人事評価（SaaS）+ 運用サポート（月次）のフルパッケージに誘導する設計。

→ GrowthFix への対応：
- 「Web 診断（無料）」= ミナジンの「みんなの人事評価 SaaS 体験」入口
- 「CODE 診断（5 万）」= ミナジンの「人事制度サポート初回アセスメント」入口
- どちらから入っても「Shift R/C → Orbit」フルパッケージへ誘導

## 2. カスタムメイド（資料）フォルダの意義

### 2.1 対象ファイル

- ジョブマッピングを作ってみよう！.pptx（152 KB・2017-10）
- 基本仕様の作り方_20170606.ppt（183 KB・2017-06）
- 評価制度構築ご提案書.pptx（523 KB・2017-06）

### 2.2 「カスタムメイド」という分類の意味

サイズが標準提案書（2,500 KB）の 1/5 以下で、汎用化された軽量資料として設計されている。

**ジョブマッピングを作ってみよう！（152 KB）**：
ワークショップ形式で顧客企業に「ジョブ定義」を自分で作らせるためのフォーム資料と推測。
→ GrowthFix Shift R 「職務定義書ワーク」の前身として参照可能。

**基本仕様の作り方（183 KB）**：
システム設定に必要な「基本仕様（評価軸・評価者・被評価者）」を顧客に記入させるフォーム。
→ GrowthFix Shift R/C 「現状ヒアリングシート」として転用可能。

**評価制度構築ご提案書（523 KB）**：
標準提案書の軽量カスタム版。業界 / 規模に合わせた提案書をカスタムメイドする際のベース。

**発見 4：初回訪問の提案書は「標準版（重い）」と「カスタム軽量版」の 2 形態を使い分けていた**

→ GrowthFix の初商談設計：Shift R/C の提案書は「標準版 LP PDF」+「ヒアリング用ワークシート（軽量）」の 2 点セットとして設計する。

## 3. 初回訪問フロー全体の構造化

JSON から判明したサブフォルダ構成から営業フローを逆算する：

```
a_初回訪問時/
├─ 原本/              ← メインの提案書（担当者が持ち運ぶ）
│   └─ 旧版/          ← バージョン管理（4 版確認済）
├─ みんなの人事評価（訪問時）/   ← SaaS 訴求型
├─ みんなの人事評価（資料送付用）/ ← 訪問後フォロー
├─ みんなの人事評価_20190618/   ← 特定時点の最新版セット
├─ カスタムメイド（資料）/       ← 業種別・規模別カスタム
└─ 旧資料/
    └─ ご提案書_人事評価システム/
        └─ 新版）送付用/         ← 訪問後メール送付版
```

**「訪問時」と「資料送付用」が分かれている意味**：
- 訪問時：大きなパンフ / 詳細版（担当者が説明しながら使う）
- 資料送付用：軽量 PDF 版（商談後にメールで送る・社長が一人で読める）

**発見 5：ミナジンは「商談中に使う資料」と「商談後のフォロー資料」を意図的に分けて設計していた**

→ GrowthFix への転用：
- 商談時：LP URL + SCAN/CODE 診断レポートを画面共有
- 商談後フォロー：自動送信の「引力タイプ診断 PDF（30 分説明セッション案内付き）」
- **Gravity の Web 診断 + 自動 PDF 送付フローはこれに対応している（デジタル版として優位）**

## 4. GrowthFix への直接反映候補

### 4.1 Tier 1（即時反映）

**[T1-A] 初商談提案書の 2 形態設計**
- 商談中：引力タイプ診断レポート PDF（A4 × 4 ページ）
- 商談後フォロー：Shift R/C サービス詳細 PDF（自動送信）
→ `04_GrowthFix/00_営業/_Scan商談/` 配下の提案書フォーマットに反映。

**[T1-B] 「料金なし版」の分離設計**
初商談提案書に価格を載せない。「次回お見積提示します」で商談を終わらせる。
→ Shift R 80 万 / Shift C 150 万 は提案書本体から分離。見積書は `b_見積もり作成時` 相当の別フェーズで提示。

### 4.2 Tier 2（中期参照）

**[T2-A] ジョブマッピングワークシート設計**
Shift R Week 1-2「現状ヒアリング」の際に顧客自身が記入するワークシート化。
GrowthFix 版：「あなたの会社の引力マップ」ワークシート（CODE × SCAN 診断補完版）。

**[T2-B] 「新版）送付用」フォルダ概念の DX 化**
ミナジンは紙フォルダで管理。GrowthFix は Notion + 自動メール配信（CRM 化）。
