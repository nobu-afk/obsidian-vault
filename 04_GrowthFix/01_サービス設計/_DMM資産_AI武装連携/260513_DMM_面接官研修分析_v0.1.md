## 5. 質問テンプレート（抽象化済・10 例）

### キャリア・志向系

```
Q1：転職活動に至った理由は何ですか？
    → [自社]を志望した理由は何ですか？
    狙い：転職理由のネガティブ確認・批判的思考傾向の検出

Q2：将来どのようなキャリアを考えていますか？
    → 将来像と現状との差分は何だと思いますか？
    → その差分を埋めるために何をしていますか？
    狙い：キャリア思考の一貫性・自頭の良さ・戦略的思考の確認

Q3：普段、最新の技術やトレンドについて、どのように情報収集・学習していますか？
    狙い：キャッチアップ速度・調査能力・成長意欲・積極性 vs 受動性の判定
```

### 問題解決・実績系（STAR 手法）

```
Q4：これまでの業務で発生した課題を一つ挙げ、どのような手段で解決したか教えてください。
    → その課題はどのような結果となりましたか？
    狙い：戦略的思考・コミュニケーション能力・視座の高さ・責任感の確認
    ポイント：解決方法だけでなく結果まで追う（責任完遂の確認）

Q5：今までで一番失敗したこと / うまくいかなかったことを教えてください。
    → その時どのようにコトに向き合い、自身 / メンバーのモチベーションを保ちましたか？
    狙い：素直さ・自責 vs 他責傾向・レジリエンス能力・マネージャー素養の確認
```

### カルチャーフィット系

```
Q6：どんな人と一緒に働きたいですか？
    狙い：カルチャーマッチ確認・仕事スタンス・指向性の把握

Q7（アイスブレイク文例）：
    「面接という場ですが、お互いに理解を深める場にしたいので、なんでも聞いてくださいね」
    「緊張していると思うので、簡単に自己紹介をしながら少しずつ○○さんのことを教えてください」
    「いまどのようなお仕事をされているのですか？」
    狙い：緊張解除・話しやすい空気づくり（面接官の役目として明示）
```

### 候補者からの質問への準備必須カテゴリ

```
・ビジネスモデル・会社の安定性
・経営者について
・商品の優位性
・社風・一緒に働く人
・待遇・福利厚生（[MASKED]）
・教育制度・育成体制
・その他、働く環境
```

**候補者質問への回答原則：嘘はつかない / ポジティブに変換する**

## 6. NG 行動 / 失敗パターン

資料内「バッドケース」セクション（Slide 46-48）から抽象化：

| # | NG パターン | カテゴリ |
|---|---|---|
| 1 | **確証バイアス**：第一印象を追証しようとする質問設計 | 評価歪み |
| 2 | **ハロー効果**：一部情報に引きずられて全体評価が歪む | 評価歪み |
| 3 | **類似性効果**：自分に似ている人を高評価してしまう | 評価歪み |

## 7. 評価基準・スコアリング設計

### ATS（採用管理システム）への申し送り

- 面接後は ATS（[MASKED]社では [MASKED] を利用）に面接フォームまたは直接入力
- **判断迷い時はスコア「C」をつける**（A/B/C 3 段階想定。C = 保留・判断留保）
- 2 名以上の面接官が入った場合は個別にフィードバック → 合議でスコア確定

### 書類選考スコアリング設計 2 点

```
POINT 1：ハイパフォーマー特性の抽出
- 現在自社でパフォーマンス高い人材の共通要素を書類選考チェック軸に変換
- 現チーム構成のギャップから採りたい人物像を定める

POINT 2：1 分判断設計
- 1 書類あたり 1 分以内で判断できるよう評価軸を明文化
- ツール（ATS / スコアシート等）で支援

迷い時の原則：「とりあえず会ってみる」
理由：①書類では測れない暗黙知の可能性 ②会社のファン獲得機会 ③エージェント信頼向上
```

## 補足：分析者コメント（GrowthFix 文脈）

### 本資料の時代的位置づけ
2020 年作成。「選び合う時代」への転換を明確に認識した設計で、候補者体験（Candidate Experience）の重要性を正面から取り上げている。この文脈は現在の Gravity Scan（Pre-Shift 適合診断）の「双方向で見極める」設計と方向性が一致する。

### GrowthFix との差分（独自付加価値）
本資料は優れた「面接運用の型」を提供しているが、以下は空白領域：
1. **引力相性（個人の CODE × 組織 SCAN の適合判定）** - 面接でカルチャーフィットを「感覚」で見ているが、Gravity の 4 型分類を面接評価フレームに組み込むことで科学的に代替可能
2. **マネージャー育成への体系的接続** - 面接フィードバックを育成カルテに変換する仕組みが未設計。Shift C の躍動組織設計で補完可能
3. **データ蓄積 → ハイパフォーマー特性の継続更新** - 書類選考 POINT 1 で言及しているが、Gravity Scan の診断データと紐付けた定量的ハイパフォーマープロファイリングは未実装
