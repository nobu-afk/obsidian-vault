## 1. 人事考課制度提案書テンプレ（汎用フォーマット抽出）

### 1-1. 提案書の章立て構造（汎用テンプレとして抽出）

大手鉄道事業者（A 社）向けに 2004 年に作成された人事考課制度改革提案書の構造。クライアント名は A 社に統一。

```
第 1 章: 背景と目的の確認
  1-1. 本プロジェクトの背景（コンサル会社の理解・要約）
  1-2. 本プロジェクトの目的と範囲
  1-3. 今回提案フェーズの目的と範囲（素案作成フェーズ）

第 2 章: 貴社の現状と提案ポイント
  2-1. 人事考課制度の現状（問題点と影響の構造図）
  2-2. 経営ミッションの理解（3 ミッション × 評価軸の整合）
  2-3. 今後求められる人材像

第 3 章: 人事考課制度設計の考え方
  3-1. 改革方針（評価対象の具体化・ミッション反映・フィードバック促進）
  3-2. 評価対象の 3 分類（成果 / 応用行動 / 基本行動）
  3-3. 評価項目の体系化（フレームワーク）
  3-4. フレームワークの活用方法

第 4 章: 新制度導入で何が変わるか
  4-1. 社員の意識改革（印象評価 → 具体的事実評価）
  4-2. 管理職の行動変革

第 5 章: 素案作成フェーズのアプローチ（8 ステップ）
  Step 1: 現状把握
  Step 2: あるべき姿の設定
  Step 3: 問題点の整理
  Step 4: フレームワーク完成（評価単位設定 → 成功要因識別）
  Step 5: パイロット評価単位への適用（ツール設計 3 種）
  Step 6: 運用イメージの検討（賃金反映ロジック）
  Step 7: フレームワークとツールの最終化
  Step 8: 次フェーズ以降の実施計画策定

第 6 章: 作業計画・スケジュール・費用
  6-1. 共同成果物一覧
  6-2. ガントチャート（13 週間）
  6-3. 詳細設計以降のロードマップ
  6-4. プロジェクト体制（役割分担）
  6-5. 前提条件
```

### 1-2. 評価フレームワークの核心構造

**3 ミッション × 3 評価対象 × 3 運用軸 のマトリクス**が提案書の骨格：

| 評価対象 | 定義 | 主対象者 | 賃金反映 |
|---|---|---|---|
| **成果** | 結果指標（管理職目標 KPI）| 主に管理職 | 賞与・加点評価 |
| **応用行動** | 状況対応能力（2-3 年かけて習得）| 全社員 | 昇格・職種間異動 |
| **基本行動** | マニュアル・タスク遂行（数日〜数ヶ月で習得）| 全社員 | 定期昇給・減点評価 |
| **性格** | 人格的特性 | — | **評価対象外**（主観的になるため）|

**重要原則**：性格は評価対象から除外。「成果との関係が明らかでない限り、評価すべきでない」と明示。

### 1-3. 評価ツール設計の 3 種類

- **成果指標管理シート**：定量 KPI + 課題設定
- **応用行動評価シート**：コンピテンシーレベル 1-5 段階尺度
- **基本行動チェックシート**：現場マニュアルベース・部門別設計

### 1-4. プロジェクト進行のポイント（コンサル側の役割）

- **ファシリテーター役**に徹する（成果物は顧客側が作る）
- ワークショップ形式による参加型設計
- 他社事例・モデルを「たたき台」として提供
- コンサル側は「議論の進捗管理 / 仮説提示 / 他社事例提供」に限定

## 2. JIN-G 評価者研修（評価者教育の体系構造）

### 2-1. 研修の 3 層構造（Why-What-How）

講師：JIN-G 代表取締役（組織人事戦略コンサルタント）。旧ベリングポイント（現 PwC）出身でコンサル事例 1 と同一人物。

```
Why（なぜ評価をするのか）
  → 評価の 4 目的：人材育成 / 組織目標達成 / 納得性の高い処遇 / 部下の動機付け
  → まとめ：「評価はマネジメントの責任そのもの」「査定の作業ではなく"マネジメント"そのもの」

What（何を評価すべきか）
  → 4 項目：目標設定 / 行動観察 / 評価（査定） / フィードバック

How（どう評価するか）
  → 目標設定：SMART 原則（Specific / Measurable / Achievable / Relevant / Time-Bound）
  → 行動観察：育成メッセージを焦点に / 「回数」で育成 / 1 on 1 作成
  → フィードバック：GROW アプローチ（Goal → Reality → Option → Will の 6 ステップ面談）
  → スキル 3 種：傾聴（うなずき・反復・要約）/ 質問（拡大/限定/肯定/未来質問）/ フィードバック（I メッセージ・事実と意見を分ける）
```

### 2-2. 研修で使われる「悪い例 / 良い例」対比手法

スライドで「こんなこと言ってませんか？」→「これでは責任を果たしていません」→「こうしましょう！」の 3 枚 1 セットで 4 回繰り返す構造。受講者の自己認識ギャップを引き出す体験型設計。

**4 つの責任 × 悪い例の類型**：
1. 人材育成責任：「どうせダメな部下はダメ」→ 固定マインドセットの例示
2. 組織目標達成責任：目標設定シートを人事提出のみで終わらせる
3. 納得性責任：業績が悪い時に根拠なく評価を下げる
4. 動機付け責任：部下の不満を第三者に向けるリーダー

### 2-3. 評価面談の 6 ステップ（GROW アプローチ）

```
1. 面談の事前準備（期中コミュニケーションの蓄積 / 市場情報・部門目標の整理 / 能力課題メモ）
2. 話し合いの雰囲気作り（30 分〜1 時間 / ねぎらい → 身近な話題から入る）
3. 目標像の共有（G: Goal）
4. 現状認識（R: Reality / Resource）
5. ギャップ対策（O: Option）
6. 意思確認と激励（W: Will）
```

### 2-4. 評価根拠の具体性基準（Aさん vs Bさん例）

- A さん「プログラミングの基本作業は独力でできるが、応用作業は上位者サポートが必要」→ **正しい**
- B さん「努力が足りないから」→ **不正**
- 原則：**客観的な事実に基づく評価**（主観的感想は不可）

## 3. クライアント E 社 ダイバーシティ案件（外資 D&I 設計）

### 3-1. クライアント E 社のプロファイル

化学系製造業（国内大手）。男女比 9:1。女性管理職比率 1.8%。次世代育成支援対策推進法の第 3 期行動計画（2015〜2020 年）を推進中。産後復職率 100%（育児休業後の復職率が高い）という強みを持つ一方、女性管理職比率が極端に低い。

（機密マスキング：社名 → E 社）

### 3-2. 前回ミーティングで特定された課題構造

**制度面（整備済み）**：
- 復職率 100%（制度的には機能）
- 子ども 1 歳まで育休 / 3 歳まで時短勤務可

**運用・意識面（問題あり）**：
- 「子育て中の女性が管理職になれない / ならない」
- 未婚女性のみが管理職に就いている実態
- 女性本人が「現在の職責に満足している」→ キャリアアップ意欲の欠如
- 産休復職後「私が管理職と言われても？」という反応
- 上司側の「女性の使い方への遠慮」

### 3-3. 「ありたい姿」の設定方法

議論を経て定量目標を設定するプロセスを使用：

```
現状: 女性管理職 1.8% / 女性社員比率 10%
中期目標: XX%（ワークショップで設定）
ありたい姿: 女性社員が結婚後も育児中も育児休暇復職後も意欲的に能力を発揮し続ける職場
```

**Diversity の価値命題**：「人口減社会が生み出す人材不足を補う」「職場の動機付けと活気向上」

### 3-4. 課題特定方法：グループインタビューの設計

```
対象者設計:
  ・女性社員（未婚 / 既婚 / 子ども有無でミックス）: 10 名 × 2 回
  ・女性部下を持つ上司: 10 名 × 1 回

プロセス:
  事前アンケート → グループインタビュー → 社内公表（トップへのレポート + 社内誌一部公開）

インタビュー内容:
  ・現状に対する問題意識
  ・女性活躍への意識の程度
  ・制度の浸透具合
```

**本音引き出しのポイント**：個別ではなくグループインタビュー形式で本音を引き出しやすい雰囲気を作る。

### 3-5. 外部データ活用（説得材料として提示）

- 女性管理職の割合 6.6%（厚生労働省 2013 年）→ E 社 1.8% は業界平均以下
- 「管理職になりたいですか？」: なりたくない 82%（PRESIDENT WOMAN）
- 「管理職になるのに必要なもの」: 実力・スキルへの自信（女性 60% / 男性 17%）
- **ロールモデルの罠**：現存の女性管理職が「スーパーウーマン」すぎて共感できない問題

### 3-6. 対策パッケージ（提案内容）

```
既存社員向け:
  ① 女性意識改革セミナー（JIN-G 主催「人事の女子大学」）
  ② 上司意識改革研修
  ③ 社内雰囲気改革
  ④ 上司と女性交流促進

未来の社員向け（採用側）:
  ⑤ 新卒採用 PR 方針変更（競合他社事例：同業 B 社・C 社の採用広報比較）
```

## 4. クライアント F 社 + 教育機関 G 海外研修事前研修（グローバル人材育成）

### 4-1. 連携構造

教育機関 G（国内大学）が JIN-G と連携し、クライアント企業（F 社）向けに「海外実践型研修プログラム」を提供。研修名は「ミッションコンプリート」。

（機密マスキング：大学名 → 教育機関 G。MC = ミッションコンプリートの略）

### 4-2. プログラムの思想的骨格

**問題意識**：従来のグローバル研修（語学留学 / 海外スタディツアー / MBA 留学）は「理論・知識重視 / 教養重視」に偏り、「ビジネス成果重視 / 実行重視」が欠けている。

**グローバル人材の 5 レベル定義**：
```
Level 1: 海外業務に興味を持つ
Level 2: 海外業務に必要な知識を身につける
Level 3: 海外業務に必要なスキルを身につける
Level 4: 海外で業務を遂行する  ← ビジネスはここが必要
Level 5: 成果をあげる
```

「英語ができればグローバル人材」ではなく「国境や国籍の違いを障壁ではなく機会と捉える人材」が定義。

### 4-3. ミッションコンプリートの特徴

```
① アウェイな環境での業務遂行: MBA ケーススタディ・フィールドワークと異なり、成果を求められる実践型
② マルチタスクで与えられるミッション: 同時に複数ミッションをこなす（通常業務と同様）
③ 挑戦意欲を奮いたたせるミッション: 頭だけで完了できない内容（行動力が必要）
```

**ファイナルミッション + 日々のミッションの 2 層構造**で研修期間全体を設計。

### 4-4. Why（Golden Circle）の活用

Simon Sinek「Start with Why」動画を事前研修に組み込み、参加者自身の「なぜ参加するか」を言語化させる。

```
京都の魅力を Why → How → What で説明するワーク
自分自身の参加動機を Why → How → What で組み立てるワーク
```

これにより「目的意識を持った渡航」を研修成果として定義。

### 4-5. リスク対応トレーニング（想定問題 8 パターン）

海外で発生しうる実際のトラブルをシナリオ形式で事前に考えさせる：
1. WiFi 未接続（インフラ障害）
2. ひったくり被害（パスポート + PC 紛失）
3. 体調急変（腹痛・高熱）
4. ホテルで水が出ない
5. タクシー不正請求
6. 犬に噛まれる（医療対応）
7. ミッション達成できない気がする（メンタル）
8. 日本に帰りたい（離脱衝動）

## 5. コンサル事例横断パターン（提案 → 受注 → 設計 → 研修実施のサイクル）

### 5-1. JIN-G のコンサルスタイルの一貫した特徴

**思想的一貫性**（2004 年〜2015 年を通じて）：
- 「制度を作ることが目的ではない。人が動くことが目的」
- 「評価はマネジメントの責任そのもの」
- 「上司の行動変容なしに制度改革は機能しない」

**提案フェーズの構造**（人事考課制度提案書から抽出）：

```
Phase A: 現状把握ヒアリング（事前ヒアリングの理解を冒頭で確認）
Phase B: 問題の構造化（「問題現象 → 原因 → 影響」の連鎖図）
Phase C: あるべき姿の設定（経営ミッションとの整合）
Phase D: フレームワーク設計（たたき台提示 → ワークショップで完成）
Phase E: パイロット適用（1-2 部門で試験設計）
Phase F: 運用設計（賃金反映 / 評価者訓練 / 社員周知）
Phase G: 次フェーズ実施計画（詳細設計への橋渡し）
```

### 5-2. ワークショップ方式の役割分担モデル

```
コンサル側（ファシリテーター）:
  - 議論の進捗管理
  - 仮説（たたき台）の提示
  - 検討用ツールの提供
  - 他社事例の紹介
  - 検討内容の整理

クライアント側（意思決定者）:
  - 活発な意見提示
  - 必要な情報の提供
  - 成果物の作成（主体はクライアント）
  - 建設的な議論
  - 社内調整
```

**原則**：「成果物はクライアントが作る。コンサルは支援する」（アドバイザリー契約）

### 5-3. 研修教材設計の 3 つの共通パターン

**パターン 1: 悪い例 / 良い例 対比**（評価者研修）
- 「こんなこと言ってませんか？」で受講者の自己認識を揺さぶる
- 「これでは責任を果たしていない」で問題化
- 「こうしましょう！」で行動処方を提示

**パターン 2: 外部データ活用による説得**（ダイバーシティ案件）
- 法令・政府統計 → 「やらざるを得ない」圧力を先に示す
- アンケートデータ → 現場の実態と照合させる
- 競合比較 → 「他社はここまでやっている」刺激

**パターン 3: Why から始める設計**（グローバル研修）
- Golden Circle（Why → How → What）を参加者に内面化させる
- ファイナルミッションに向けた意味付け
- リスク想定による心理的準備

### 5-4. 失敗パターンの明示化（教育コンテンツ）

4 事例に共通して「陥りやすい失敗」を先に見せてから「正解」を提示する設計を使用。これは GrowthFix の Shift R「課題発見 → 変容促進」の構造と同型。

## 6. R/C/Orbit AI 武装サービスへの反映候補（5 件）

- **適用場面**: Shift R Week 6 の評価制度設計ワークショップ事前準備

### R2: 評価者研修コンテンツの即時活用

JIN-G 評価者研修の「Why-What-How 3 層構造 × 悪い例・良い例対比 × GROW 面談 6 ステップ」は、Shift R の Week 7-8 マネジャー研修コンテンツとして直接活用できる品質。
- **AI 武装案**: 「悪い例・良い例」シナリオ生成エージェント（クライアント業種を入力すると業種固有の評価者行動悪例を生成）
- **適用場面**: Shift R フォローアップセッション / Orbit 月次研修コンテンツ

### R3: Gravity CODE への「性格評価除外原則」の補強

「性格は評価対象外（成果との関係が明らかでない限り）」という 2004 年時点のコンサル知見は、CODE 診断が「キャラクター・人格評価ではなく Authentic Self（行動パターン）の評価」であることと整合する。
- **LP コピー補強**: 「引力診断は性格テストではありません。日常の行動パターンから組織の引力を測定します」という差別化訴求に活用可
- **適用場面**: CODE LP / Scan LP の FAQ または競合差別化セクション
