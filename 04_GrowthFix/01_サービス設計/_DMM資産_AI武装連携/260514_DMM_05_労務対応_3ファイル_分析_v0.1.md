### 1.2 労務トラブル対応カテゴリ体系（スライド 1）

HRBP が実際に対応する労務事案は以下 3 軸で分類されていた：

**事案種別（3 分類）：**
- 勤怠不良：遅刻・欠勤・不規則な出勤パターン
- 問題行動：ハラスメント・規律違反・コンプライアンス事案
- 低業績（Low Performer）：業務成果が基準を下回る継続状態

**対応措置フロー（難易度・リスク低 → 高 の順序）：**

```
段階 1：継続的な指導（口頭→書面→面談記録）
  ↓ 改善困難と判断
段階 2：以下 4 択の選択（並列・組み合わせ可）
  A. 休職（規定通り対応・休職明けの調整負荷大）
  B. 出社しないで転職活動（在籍中給与支払・配慮措置）
  C. 再就職支援（外部エージェント費用 会社負担）
  D. 退職一時金（即日退職・最大半年分支給）
  ↓ 最終手段
段階 3：懲戒処分（規定内での処分適用）
```

**PIP（Performance Improvement Program）への言及：**
- 「労務リスクおよび運用工数がタフだが、PIP を通じた実質的な退職勧奨も手段」と明記
- Low Performer への PIP 設計 = HRBP の高度技術業務として位置づけ

### 1.3 人員整理・雇用調整の判断基準（スライド 2）

「組織の解体・縮小を伴う意思決定を素早く実行すること自体が競争力の源泉」という認識のもと、人員整理時の方針として「労使双方が幸せになるようにベストチョイスを探る」アプローチを採用。

背景認識：組織縮小決定後の従業員個別対応では「経歴とスキルを活かす配属が難しいケースが生じる」という構造的課題を前提としていた。

### 1.4 キャリア自立支援フレーム（スライド 3-9）

**支援対象ケース：**
- 所属部署でキャリア形成ができない社員
- スキルと業務がミスマッチしている社員

**支援の方向性（2 択）：**
1. 社内残留支援：異動・職種転換・育成プログラム
2. 社外転職支援：前向きな外部機会探索の促進

**プログラム設計（年代別頻度）：**

| 年代 | 推奨頻度 | 目的 |
|---|:-:|---|
| 20 代 | 社内スキル検討 1 回 | 社内での活躍軸の確認 |
| 30 代 | マネジメント含め 1 回 | キャリアの方向性確定 |
| 40 代 | 2 回 | 活躍継続 or 社外転身の検討 |
| 50 代 | 1 回程度 | 残余キャリアの最適化 |

**他社事例（事例 A・B 社・IT 大手）からの学び：**
- 上司と部下が同じワークシートを用いた「期待の明示化」が効果的（2 か月に 1 回実施）
- キャリアシート運用 + 人材開発会議には人事担当 3-5 名が必要（2,000 名規模）
- 集合研修規模：個別対話あり 24 名 / 個別対話なし 50 名が目安

**外部サービス連携（リクルートキャリアコンサルティング）：**
- 再就職支援：成功報酬 20% 以下（企業負担）
- キャリアカウンセリング：個別 1 時間 × レポート付き（オプション）
- サーベイ実施との組み合わせが推奨

### 1.5 若手リテンション施策の課題認識（スライド 10・MTG メモ 2020.10.21）

リクルートキャリアコンサルティングとの協議で確認された当時の課題認識：

**課題の優先順位：**
1. 3 年で辞める層のリテンション（20-30 代の定着率）が最優先
2. 幹部候補育成（0→1 人材の不足・中途採用では対応困難）
3. 「会社にぶら下がる人材」の活性化が緊急度高い

**印象的な課題認識の発言（MTG メモ原文）：**
- 「既に弊社で行っている 1on1 や目標決め等である程度は回せていそう」
- 「制度としてはあるが、ビジネス職は使いきれてない印象」
- 「新卒というより、経営・マネジメント層を育てたいのは課題」
- 「優秀な方は取れている。0→1 不足している（中途採用ではカバー不可）」

### 2.9 退職アンケートとの「裏返し」対称性

入社アンケートの設問と退職アンケートの設問は、ほぼ完全な対称構造になっている：

| 入社アンケート（期待） | 退職アンケート（失望） |
|---|---|
| やりたい仕事ができそう | やりたい仕事ができなかった・できなくなったから |
| やりがい・達成感を感じられそう | やりがい・達成感を感じないから |
| チャレンジできる環境がありそう | チャレンジできる環境がないから |
| 裁量が与えられそう | 裁量が与えられていないから |
| 給与が高い | 給与が低かったから |
| 評価・人事制度に納得感がありそう | 評価・人事制度に不満があったから |
| スキルアップできそう | スキルアップできないから |
| 成長実感が得られそう | 自分が成長している実感が得られないから |
| 育成してもらえる環境がありそう | 育成してもらえる環境がないから |
| キャリア形成ができそう | キャリア形成ができないから |
| 上長とコミュニケーションが円滑にとれそう | 上長とコミュニケーションがとれないから |
| 上長のマネジメント力がありそう | 上長のマネジメント力がないから |
| 人間関係がよさそう | 人間関係が悪いから |
| 企業に将来性を感じる | 企業の将来性に疑問を感じたから |
| 社風や風土が合いそう | 社風や風土が合わなかったから |
| 経営層の方針が明瞭・合う | 経営層の方針が不明瞭・合わないから |
| チームで助け合う環境がありそう | チームで助け合う環境がなかったから |
| 残業・休日出勤など拘束時間が短そう | 残業・休日出勤など拘束時間が長かったから |

**設計の意図：** 入社前の「期待値」と退職時の「失望ポイント」を同一軸で比較できる完全対称設計。「どこで期待が剥落したか」を組織的に追跡するための仕組み。

### 4.1 大企業 HRBP の労務管理サイクル（全体像）

### 4.2 「期待値設計の失敗」という核心構造

入社アンケートと退職アンケートの対比から見えるパターン：

**入社時の上位期待（4+5 点率 85%+）：**
1. チャレンジ環境（91.5%）
2. 将来性（89.3%）
3. スキルアップ（88.7%）
4. やりがい・達成感（88.4%）
5. 成長実感（86.2%）

**退職時の上位失望（4+5 点率 44%+）：**
1. やりがい・達成感なし（48.0%）
2. やりたい仕事できなくなった（47.1%）
3. キャリア形成できない（47.0%）
4. 成長実感なし（44.8%）

→ **入社時の上位期待項目がそのまま退職時の上位失望項目になっている。**「チャレンジ・成長・やりがい」という期待で人を集め、在籍中に同じ軸で失望させて排出する循環構造。

### 4.3 v0.3 §1 発見 3-4 への追加裏付け

**発見 3「Wevox vs 退職アンケートのズレ」の裏付け：**
- 220824 退職者分析に Wevox スコア × 退職者 LIST の突合設計が内包されていた（シート 9）
- Wevox は「組織単位の平均」で測定 → 個別退職者の「個人の失望理由」は Wevox では捕捉できない
- 本 3 ファイルのシステムは「個人レベルの Exit Survey」と「組織レベルの Wevox」の両方を用意していたが、**統合分析する仕組みは HRBP の手動作業に依存していた**

**発見 4「5-7 年層の離反」の定量補強：**
在籍年数別 × 主要退職理由（やりがいなし）のクロス集計：

| 在籍年数 | やりがいなし 4+5 点率 | やりたい仕事なし 4+5 点率 | 成長実感なし 4+5 点率 |
|---|:-:|:-:|:-:|
| 1 年未満 | 39.4% | 31.0% | 33.8% |
| 1-3 年 | 49.4% | 49.4% | 47.2% |
| 3-5 年 | 46.2% | 47.6% | 45.3% |
| **5-7 年** | **50.4%** | **56.2%** | **50.4%** |
| 7-9 年 | 64.5% | 51.6% | 45.2% |
| 9 年以上 | 66.7% | 61.9% | 52.4% |

**読み取り：**
- 「やりたい仕事なし」は 5-7 年層（56.2%）で急増し、最も深刻
- 「やりがいなし」は 7 年以上で急増（64.5%・66.7%）
- 3 年未満は「キャリア形成できない（49.4%）」が特に高い → 早期離職の主因はキャリア見通し
- 5 年以降は「やりたい仕事が実際にできなくなった」が主因に変化 → 業務アサインの問題

**新発見（クロス分析から）：** 退職理由の「主因」は在籍年数で変化する：
- 1-3 年：キャリア形成できない（将来不安）
- 3-5 年：スキルアップ・成長実感の伸び悩み
- 5-7 年：やりたい仕事へのアサイン失敗

### 4.4 上長アンケートが示す「見える化されない退職」の構造

上長の 69.2% が退職を「想定していた・ある程度想定していた」にもかかわらず、引き止め意向「4+5」は 31.3% に留まる。

これは以下を示唆する：
- 上長は退職の予兆を把握していても「引き止め行動を取らない」ケースが大多数
- 引き止めない理由の推定（MTG メモとの接続）：「会社にぶら下がる人材が優先」「0→1 人材不足の方が問題」という組織認識
- つまり「優秀な人材の退職」は黙認・想定内として処理されていた

→ **「退職の可視化」と「介入タイミング」の設計失敗** = GrowthFix Shift R の文脈では採用基盤の問題だけでなく「定着設計の欠如」としてフレームできる。

> 「退職者の 57.8% が転職先で『やりたい仕事ができる（前職より良い）』と回答。組織の側ではなく、マーケットの側に問題の答えがある場合が多い。」

### 6.2 「HRBP の仕事量」の実態として

pptx・xlsx の規模感から見えた HRBP 実務の負荷：
- 入社面談ログ：3,238 行（1 人あたり 35 列の面談記録）
- 退職者 LIST：1,000 行（75 列 × 3 種類のアンケートデータ）
- 退職アンケート：上長・社員の 2 面からの記録（447 + 805 件）

この量の HRBP 業務を AI が補助する価値は「時間の節約」ではなく「見落としの防止」。個別退職者データを HRBP が手動でチェックしていた時代から、AI が異常値・離脱シグナルを自動検知する時代へ。

### 6.3 「上長が退職を知っているのに対処しない」問題の提起

営業トークの論点として活用できる：
> 「上長の 69% は部下の退職を事前に予見していた。しかし引き止め行動は 31% しか取っていなかった。これはマネジメントの問題か？仕組みの問題か？ GrowthFix では仕組みを設計します。」

## 7. 機密マスキングログ

| 除外・抽象化した情報 | 処理方法 |
|---|---|
| 退職者個別氏名（退職者 LIST・QA 面談_imp） | 全件非掲載・統計値のみ使用 |
| 社員コード・メールアドレス | 全件非掲載 |
| 個別退職理由の自由記述（QA 面談_imp） | 本 MD 不掲載（構造抽出のみ） |
| 入社面談ログの個人記録 | 設問テンプレート構造のみ抽出 |
| Wevox の部門スコア（具体的部門名） | 「事業部 A / B」等の抽象表記に変換済 |
| 高フラグ退職者の等級・処遇情報 | 「高年俸層」の集計分布のみ使用 |
| 内定競合企業名（入社アンケート列 54） | 全件非掲載 |
| 退職一時金・再就職支援の具体的費用（pptx） | 範囲表記に留める（金額言及済の情報は公知範囲内） |
| 上長アンケートの個別回答 | 集計・分布のみ使用 |
| 「松本案件」等の固有名詞が入ったカテゴリ | カテゴリ名のみ記載・件数を含む集計表に留める |
