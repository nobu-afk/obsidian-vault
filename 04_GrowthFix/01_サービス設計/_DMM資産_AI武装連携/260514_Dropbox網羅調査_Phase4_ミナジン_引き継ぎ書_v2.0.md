## 2. 機密マスキング方針（最重要・必須遵守）

### 2.1 絶対除外（出力に1件でも含めれば重大コンプラ違反）

- 個人氏名（漢字 / カタカナ / 英字）：**窪さん / 平川さん / 三城さん / 河本さん / 横江さん / 田大学さん 等の社内関係者** すべて除外
- **クライアント実名（マスキング超厳格）**：
  - 既出社名：富士総業 / 丸七住宅 / コミュニケーターズ / インシュアランスサービス / f4samurai / ジャパンアーツ / YRKand / 東北芸術工科大学 / 大洞印刷 / 森商事 / フナミズ刃型 / シシドモータース / ティアンドエス / エス・エム・アイ トラベルジャパン / 光通信 等
  - すべて **「クライアント A 社」「クライアント B 社」...** に番号統一抽象化
- 業務委託費・契約金額・売上数字
- 個人在籍社員データ（給料台帳 / 人事評価対象者名 / 退職者名）
- メールアドレス・電話番号・住所
- 内部 URL / パスワード / Slack 通知設定値

### 2.2 抽象化処理

- 「○○本部」→「事業部 A」
- 「年商 ○ 億」→「○ 億規模」「中堅規模」「SMB」
- 「正社員 ○ 名」→「○○ 名規模」
- クライアント業界 → 「製造業 / 建築業 / 旅行業 / ゲーム」など業界レベルまで（具体企業名 NG）
- 「窪さん人事制度」→「ベテランコンサル A」または「ミナジン社内ロール A」

### 2.3 抽出 OK（公開情報・業務プロセス構造）

- 業務プロセス・ワークフロー
- 人事評価制度の設計フレーム（評価軸・配点ロジック・グレード階層）
- 目標管理添削の構造（添削ポイント・典型エラーパターン）
- セミナー設計の章立て・流れ
- 営業提案書のフレーム（章立て・訴求軸）
- 中小企業の組織課題マップ（業界レベルで抽象化）

### 2.4 user 在籍 10 年の特殊配慮（最重要・Phase 4 固有）

user が在籍した会社のため、Phase 1-3（外部企業）とは機密性が異なる：

1. **個人エピソード・社内政治情報は完全除外**：「○○さんが××と言った」「あの時の判断は△△だった」等は除外
2. **業務プロセス構造のみ抽出**：「人事評価制度の設計プロセス」「セミナー集客フロー」など、属人化を排した構造のみ
3. **公開済情報（書籍・公開セミナー資料）は積極引用 OK**：人事の大学の公開研修テキストなど
4. **ミナジン社の戦略機密情報は完全除外**：事業計画 / 売上目標 / 競合戦略 / M&A 検討等
5. **「窪さん打ち合わせ」フォルダは原則スキップ**（個人記録の機密性が極高）
