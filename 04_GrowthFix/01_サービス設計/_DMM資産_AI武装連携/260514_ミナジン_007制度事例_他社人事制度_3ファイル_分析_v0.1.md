## 1. 各ファイルの人事制度構造（等級・評価・報酬・運用の 4 軸）

### 1.1 ファイル A：光通信 コンシューマー事業部 2013 年人事制度（63 ページ）

#### 制度の全体像

第 27 期（2013 年）の営業現場特化型の人事制度一式。フレッツ営業部（NTT フレッツ光の代理店営業部門）の事業部別制度編。63 ページにわたり、等級・給与・評価・報奨金・昇降格・運用フローを完全体系化している。

#### 1.1-A 等級構造

**二軸構造（職群 × 職格）：**

| 職群 | 職格グループ | 呼称 |
|---|---|---|
| 総合職（営業系）| 総合職（S） / サブマネージャー（Smgr）/ ゼネラルマネージャー（GM）/ 統轄 / 執行役員 | 課長代理 → 課長 → 部長 → 執行役員 |
| 総合職（事務系）| TL 系 / TM 系 / TN 系 / TO 系 / TP 系 | 同上 |
| 一般職（B）| 地域限定社員（KMA 系 / KMB 系） | なし |
| 専門職（SS）| スーパープレイヤー（KMSA 系） | スーパープレイヤー |
| クルー（非正規）| アポインター / クルーTR / SW（スウィングマネージャー）| ブルー / シルバー / ゴールド / プラチナ |

**構造的特徴：管理職ルートとプレイヤールートの完全二分化**
- 総合職（管理者目指す）⇔ 専門職 SS（スーパープレイヤー・業績トップ目指す）が相互転換可能
- クルー（業務委託に近い非正規）にも独立したステータス体系（ブルー〜プラチナ）と昇格基準を設けている
- 管理職（Smgr・GM）の職級は「管轄人数規模」で決定する（部下数が多いほど上位職級）

**保護観察職級（99 職級）：**
- 懲戒案件・ライセンス未取得・事業所監査ペナルティで発動する降格の「バッファ層」
- Smgr の KMB99・GM の KMC99 等が対象
- 昇降格の最低職級としての常用は禁止。問題解消後は元の職級に復帰

#### 1.1-B 評価構造

**ポイント制評価（粗利ポイント × 完了率 - コストポイント = 利益ポイント）：**

```
粗利ポイント = 一時金粗利（現金収入）+ ストック粗利（将来収益見込）
               ※ 商材ごとに P（ポイント）定義（例：フレッツ光ファミリー = 4.2P）
               ※ キャッシュバック等の施策は「マイナスポイント」として減算

コストポイント = 社員コスト（固定給・交通費・事務所家賃等の合算）
               ※ 個人当たりコストを四半期ごとに算定

利益ポイント = 粗利ポイント（×完了率補正）- コストポイント
```

**完了率補正（P 定義の核心ルール）：**
- エントリーから工事完了まで完了した件数の割合を四半期ごとに算出
- 「入金されて初めて売上」思想：完了率が低いプレイヤーはポイント単価が下がる設計
- 完了率 50% の場合、NTT フレッツ 4.0P → 2.0P に半減する

**評価サイクル：**
- 四半期（年 4 回）査定：各階層の利益ポイント達成率で昇降格・昇降級を判定
- 管理職（GM）は管轄エリアの利益ポイント → 部下成績が直接 GM 自身の評価に連動

**役職別評価指標の分離：**

| 役職 | 評価指標 |
|---|---|
| クルー / 一般社員 | 個人の粗利ポイント達成率 |
| クルーTR / 社員 TR | 管轄部署の利益ポイント達成率 |
| Smgr / SW | 管轄部署の利益ポイント達成率 |
| GM | 管轄エリアの利益ポイント達成率 |

→ 役職が上がるにつれ「個人業績」から「チーム業績」へ評価軸がシフトする設計。

#### 1.1-C 報酬構造

**構成要素（4 層）：**

1. **基本給（固定部分）**：職群・職格・地域（第一地域 / 第二地域）で決定。
   - 構造として：総合職（S）の基本給は一定（役職手当で差をつける設計）
2. **職級手当（職内格差）**：同一職格内での在籍年数・実績に応じたステップ昇給
3. **管理職手当（管理職専用固定手当）**：GM 以上に適用。固定額。
4. **変動報酬（3 種類）**：
   - 達成金規定：利益ポイント目標 100%以上達成時に支給
   - 歩合規定：利益ポイントの利益額に対する還元率（役職ごとに異なる還元率）
   - 社長賞 / 事業部長賞：ランキング競争上位者への表彰型報酬

**特徴的な報酬設計：**
- クルー（非正規）にも月次達成金・部署達成連動インセンティブを設計。「クルーも利益責任者」として動機づけ
- 人材育成報奨金：Smgr 昇格させた上位者に昇格後 2 ヶ月目・7 ヶ月目の 2 回払い（合計・規模感として「十数万〜数十万円」水準）
- 蔦出評価益：優秀人材を他部門に輩出した GM へ評価ポイント付与。「人材の囲い込み」ではなく「輩出を評価」する設計

#### 1.1-D 運用構造

**昇格の自動判定ゲート（4 条件同時充足で原則自動昇格）：**
1. 四半期累計利益ポイント達成率 100% 以上
2. 事業部長へのコミットメント表明
3. 過去 1 年以内に重度の懲戒がないこと
4. 所定ライセンスの取得・合格

→ 条件を満たしても昇格枠（事業計画ベース）が埋まれば翌期に繰り越し。

**降格の自動トリガー（3 条件同時で原則降格）：**
1. 四半期累計利益ポイント達成率 50% 未満
2. 高い目標へのコミット不可
3. 不正・モラル違反等

**クルーの昇降級（時給制）：**
- 3 ヶ月ごとの稼働ポイント平均値（1 時間当たり粗利）で昇降級
- ステータス（ブルー〜プラチナ）は月次粗利ポイント水準で定義
- 時給は 3 ヶ月ごとの査定で上下。最高時給・最低時給の上下限あり

**承認フロー（制度変更 / 例外対応）：**
- 管理職系：GM 以上の申請 → 事業部長 OK → 人事本部長 → 社長確認後承認
- 保護観察職級の適用は社長 / 人事本部長承認が必要

## 2. R/C/Orbit AI 武装サービスへの反映候補

### 2.1 「利益ポイント制」評価 AI → Gravity Scan / Shift R・C への組込

**抽出構造：**
光通信の評価は「粗利ポイント - コストポイント = 利益ポイント」という原価計算型評価。各従業員の人件費コストを「1 人あたりのコスト」として可視化し、評価に組み込む設計。

**Gravity AI 武装への反映：**

| 反映候補 | 優先度 | 根拠 |
|---|:-:|---|
| **Orbit 月次レポートに「1 人あたり人件費コスト vs 粗利貢献」計算式を追加**。採用後の「貢献 BEP（損益分岐）到達月数」を可視化 | **P0** | Orbit G07（キャリアパス言語化）との接続。クライアント経営者が「この社員はいつから利益に転換したか」を把握できる指標として |
| **Shift C のあるべき組織診断軸に「管理職（GM / Smgr）の部下数と評価指標のズレ」を追加**。管理職の評価が「個人業績」から「部下成績」に切り替わっているかを診断 | **P0** | SCAN 組織引力診断の「躍動軸」Collective Efficacy 評価と直接接続 |
| **Gravity Scan（旧 CODE Scan）の 4 型診断に「評価とキャリアパスの整合性」軸を追加**。整合型組織では評価軸が役職ごとに明確に分離しているか | P1 | 整合型 / 拡散型の判定基準として機能 |
| **Shift R の Week 11-12「等級・評価設計」に「職群二軸（管理職 vs スーパープレイヤー）設計テンプレ」を追加** | P1 | 採用後のキャリアパス設計として光通信の総合職 ⇔ 専門職 SS 相互転換モデルを中小 SMB 向けに抽象化 |

### 2.2 「絶対評価 × 行動目標 × 運用サポート」の三位一体モデル → Orbit コア設計への転用

**抽出構造：**
あしたのチームが提示した「制度構築 + システム + 運用サポート のワンストップ」という競合優位の根拠は、「中小企業の人事評価は制度と運用が分断されると死ぬ」という現実認識に基づく。Gravity Orbit は「継続伴走」がコアバリューであり、この構造と完全に一致する。

| 反映候補 | 優先度 | 根拠 |

## 3. 仮説 E v0.2 営業資料 v0.4 への組込候補

### 3.1 「評価制度と運用支援のワンストップ」が圧倒的競合優位になるという構造証明

**あしたのチームの競合分析は、そのまま Gravity のポジショニングに転用できる：**

| 競合タイプ | あしたのチームの指摘 | Gravity に置き換えると |
|---|---|---|
| 大手コンサル | 制度設計のみ・運用なし | 「Shift C（制度設計 6 ヶ月）だけで終わる競合」 |
| SaaS ベンダー | システムのみ・伴走なし | 「Wevox 等のツール単体導入で終わる競合」 |
| 社労士 | 運用のみ・制度設計なし | 「顧問社労士のみで人事制度が機能すると思っている経営者」 |
| **Gravity Orbit** | **制度設計（Shift C）+ 定着支援（Orbit）の一気通貫** | **「制度を作って終わりにしない唯一の伴走」** |

**営業資料 v0.4 組込案（スライド案）：**

> 「人事制度が形骸化する会社の共通構造：制度設計（6 ヶ月）→ 運用不在（残り 10 年）。Gravity は Shift で設計し、Orbit で定着させる。制度の賞味期限は Shift 単独なら 6 ヶ月。Orbit と組み合わせると 3 年以上持続する。」

→ 競合の「制度設計のみ」提供との差別化を「運用支援の有無」という軸で可視化。

### 3.2 「評価制度の目的は業績向上であり査定ではない」という思想訴求

**あしたのチーム資料の核心訴求：**

「きのうの人事評価 = 査定のための目標。あしたの人事評価 = 成果を出すための行動目標。」

この対比は Gravity の「引力経営」の思想と接続できる：

> 「評価制度が社員の引力を破壊している会社が多い。引力を生む評価とは何か。それは成果を出すプロセスを本人が設計し、上司が伴走する評価だ。Gravity Shift C では、この『引力型評価』を 6 ヶ月かけて一緒に設計する。」

**組込箇所：** 営業資料 v0.4 の「Shift C の差別化ポイント」スライドの「きのうの評価 vs 引力型評価」対比として活用可。

### 3.3 「役職別評価指標の切り替え」が Shift C の診断フレームを補強

**光通信制度の示す構造：**
- 一般社員・クルー = 個人粗利ポイント
- 管理職（TR / Smgr / GM）= 管轄部署 / エリアの利益ポイント

この「昇格に伴う評価指標の切り替え」が設計されていない会社は、管理職が「プレイング個人業績」に引きずられ Layer-3 不在を招く（Phase 3 発見 3 との直接接続）。

**営業トーク転用案：**
> 「御社の管理職は、個人業績で評価されていますか？それとも部下の成長・チームの成績で評価されていますか？前者のままでは、優秀なプレイヤーが管理職になっても、管理職として機能しません。Gravity Shift C では、このスイッチを設計します。」

## 4. 書籍『引力経営』v0.3 への補強素材

### 4.1 第 5 章「評価制度が引力を破壊する構造」への補強（P0）

**現状の第 5 章想定構成**（未完・推定）：人事評価制度の形骸化が組織の引力を下げる構造。

**ミナジン 007 資料で補強できる内容：**

「きのうの人事評価 = 相対評価 × 過去成果 × 業務日誌と同じ査定」が SMB の実態であることを、あしたのチームの 2016 年時点の競合分析で実証できる。

**補強パラグラフ案（第 5 章 §1 に追加）：**

> あるコンサル会社の 2016 年時点の顧客調査では、中小企業の人事評価の大半が「査定のための目標」「自分で数値目標を設定できない」「ほとんど差がつかない」「結果だけで評価」という 4 つの特徴を持っていた。これを「きのうの人事評価」と呼ぶ。きのうの人事評価が問題なのは、社員が「頑張ってもどうせ評価が変わらない」と感じる点だ。これは Bandura（1977）の Self-Efficacy の破壊装置として機能する。引力経営において評価制度の役割は査定ではない。「自分の行動が成果につながる」という確信を持たせる装置だ。

### 4.2 第 8 章「管理職の役割設計が組織の天井を決める」への補強（P1）

光通信制度の「役職別評価指標の完全分離」（一般社員 = 個人業績 / 管理職 = チーム業績）は、Phase 3 発見 3「Layer-3 不在が組織崩壊の最大共通リスク」の解決設計として機能する。

**補強ポイント：**
- 「管理職の評価指標が個人業績のままである会社では、優秀なプレイヤーが管理職になっても Layer-3 としての機能を発揮しない」という主張の実例として光通信の二軸設計（個人指標 → チーム指標の切り替え）を引用可

## 8. 主要発見サマリー（トップ 5）

**発見 1【最重要・光通信】「評価指標は役職によって個人業績からチーム業績へ完全切替」する設計が Layer-3 機能不全の解決策**

Phase 3 発見 3「Layer-3（ミドルマネジメント）不在」の構造的解決策として、光通信制度が実例を提供している。管理職の評価指標を「個人粗利」から「管轄チームの利益ポイント」に切り替える設計が Shift C の Week 6-8（評価制度設計）に直接組込可能。これが設計されていない会社は管理職が「プレイングマネージャー固定」になり組織の天井が低くなる。

**発見 2【最重要・あしたのチーム】「制度 + システム + 運用サポートのワンストップ」は既に 2016 年時点で確立された競合優位の枠組み**

Gravity Orbit の「継続伴走」を経営者に説明する際、「制度設計のみ提供する競合」「システムのみ提供する競合」「運用のみの競合」との三角比較は説得力が高い。あしたのチームが中小企業向けに使っていたフレームをそのまま Gravity の営業資料 v0.4 に転用できる。v1.0 既収録パターン「制度設計の事業フェーズ依存」と組合わせることで「なぜ今この制度が必要か」→「なぜ Gravity でなければならないか」の 2 段論法が完成する。

**発見 3【重要・光通信】「管理職ルート ⇔ スーパープレイヤールートの相互転換」設計が採用・定着の引力を高める**

光通信は総合職（管理者を目指す）⇔ 専門職 SS（スーパープレイヤーを目指す）の相互転換を制度化。中小 SMB でこれを設計すると「管理職になれなくても評価される」インセンティブが生まれ、Gravity CODE（個人引力）の「才能 × 偏愛 × Why」が活きるキャリアパスを組織が用意できる。Shift R の Week 11-12 成果物の厚みを増やす素材として有効。

**発見 4【重要・あしたのチーム】「相対評価 → 絶対評価」の切り替えが SMB 人事制度の最大改革点**

「ほとんど差がつかない」（相対評価 + 曖昧目標）から「差がつく」（絶対評価 + 行動目標 + 達成水準事前定義）への移行が、評価制度改革の核心。あしたのチーム資料はこの構造を SMB 営業提案の形式で整理済。Gravity Shift C の診断軸「現在の評価制度が相対か絶対か」を P0 チェックポイントとして Gravity Scan 診断に組込可能。

**発見 5【参考・職業能力評価基準 XLSX】製造業向け 6 段階等級体系のコンピテンシー記述スタイル**

112,929 行のデータは製造業特化のため直接転用不可だが、「各等級で何を期待するか」を「職務遂行のための基準（行動指標）」として文章化する粒度・記述スタイルは参考になる。Shift R Week 11-12 の等級コンピテンシー設計時の「書き方テンプレ」として参照するにとどめる（v1.0 既収録パターン「等級グレードと役割定義」の SMB 補強事例として追記）。
