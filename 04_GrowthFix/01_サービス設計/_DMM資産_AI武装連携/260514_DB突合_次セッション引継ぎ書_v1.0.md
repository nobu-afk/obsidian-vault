## 6. 注意事項・落とし穴

### 6.1 同じファイルを 2 回 Read しない

CLAUDE.md ルール「同セッション再 Read 不要」原則。ただし**別 Node では新規 Read 必要**：
- 別 Node では `260514_DMM_Wevoxコメント分析_v0.1.md` は新規 Read 必須（H2 + H3 同時利用なので一度 Read すれば OK）
- `260514_DMM_02_オンボーディング_入社後面談_分析_v0.1.md` も H1 で新規 Read 必要

### 6.2 機密マスキング方針厳守

DMM 個人情報・給与金額の具体数値は一切記載しない。制度の設計構造・集計分布・運用パターン・NPS スコア集計値のみを対象とする（既存 MD 群はマスキング済）。

### 6.3 仮説修正の可能性を排除しない

H5 で「仮説の前提が誤り」と判明した実績がある。H1+H2+H3 でも**期待結果（仮説）に合わない実証データが出る可能性**を前提に検証する。「仮説 ⊃ 実証 / 仮説 ≠ 実証 / 仮説外 ⊂ 実証」3 分類で振り分けるのが堀田フレーム認知価格設計の最強根拠。

### 6.4 業務系タスク Part 3 参照

CLAUDE.md ルール：このタスクは**サービス設計**に該当するため、`09_会社OS/公開/ガイドライン/商品.md` Part 3 + `260429_堀田フレームワーク体系_最新版.md`（堀田フレーム）の参照を推奨。ただし、H1+H2+H3 は検証作業中心であり、設計の意思決定はせずに「検証結果のマトリクス化」までに留める方針。意思決定は P3 v1.0 確定書作成時にメインスレッド Opus が実施する。
