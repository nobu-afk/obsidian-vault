## 1. 営業検討シート（215 KB xlsx）：ミナジンの営業プロセス設計

### 1.1 サイズから推測される内容

215 KB の xlsx = 複数シート・グラフ・マクロを含む本格的な管理シート。
「営業検討」という名称から「案件の検討・判断に使う営業管理ツール」と推測。

**内容推測（中小 SaaS 営業管理 Excel の標準構成）**：
1. **案件管理シート**（商談 ID / 企業名 / 担当 / 商談ステータス / 見込み金額 / 確度）
2. **営業フロー定義シート**（ステップ 1〜N の標準フロー・各ステップの判断基準）
3. **月次実績集計シート**（コール数 / アポ数 / 商談数 / 見積数 / 受注数・月別推移）
4. **営業ロールプレイ素材**（よくある質問 / 反論対応）

### 1.2 営業フロー 7 段階の構造化（推論）

Phase 4 で判明した知見 + JSON 構造 + ミナジン営業実態から「7 段階営業フロー」を構造化する：

```
【ミナジン営業 7 段階フロー（推論）】

STEP 1：リスト獲得
　├─ 00_リスト系フォルダ（顧客リスト Excel・アントレペディア等）
　└─ セミナー参加者アンケート → CRM 登録

STEP 2：アプローチ（初回接触）
　├─ FAX DM（FAXDM_MINAGINE_OSAKA.pdf 168KB 確認済）
　├─ 電話コール（Salesforce メール文面.txt で CRM 管理）
　└─ セミナー後フォロー（翌日メール等）

STEP 3：初回訪問（提案）
　└─ a_初回訪問時フォルダ一式
　　├─ 標準提案書（料金なし版）
　　└─ カスタムメイド（資料）

STEP 4：ヒアリング・現状診断
　└─ d_オーダー提案 / 組織診断（素材）

STEP 5：見積提示
　└─ b_見積もり作成時フォルダ
　　├─ 人事評価見積（xlsx）
　　└─ コンサル見積（xlsx）+ スケジュール表

STEP 6：申込・契約
　├─ c_お申込書フォルダ（利用規約 + 申込書）
　└─ ｆ_契約書関連（NDA + 業務委託契約書）

STEP 7：納品・運用開始
　└─ d_オーダー提案 / 運用フォローサービス → Phase 4 MD11（003 運用支援）へ接続
```

**発見 10：ミナジンの 001_営業資料フォルダ全体が「営業 7 段階フロー」の物理的な実装になっていた**

フォルダ名が営業フローのステップそのものに対応している。
GrowthFix の営業フローとの対比：

| ミナジン STEP | GrowthFix 対応 | デジタル化レベル |
|---|---|---|
| STEP 1 リスト獲得 | Note/FB フォロワー / DM 5 件 / 日 / Web 診断流入 | ◎ |
| STEP 2 アプローチ | DM / Lステップ シーケンス / 無料診断誘導 | ◎ |
| STEP 3 初回訪問 | Web 診断レポート + 30 分説明セッション | ◎（非訪問化） |
| STEP 4 現状診断 | SCAN 診断 / CODE 診断 | ◎（自動化） |
| STEP 5 見積提示 | 標準見積 PDF（未整備） | △（要設計） |
| STEP 6 申込・契約 | 申込書フォーム（要設計） | △（要設計） |
| STEP 7 運用開始 | Orbit 月次 SOP（v0.1 設計済み） | ○（設計済み・要精度向上） |

**STEP 5 見積提示 と STEP 6 申込・契約 が GrowthFix の弱点として特定された**。

## 2. Salesforce メール文面（1.7 KB txt）：CRM 運用の実態

### 2.1 テキストファイルの意味

1.7 KB の txt は「Salesforce から送信するメールのテンプレート文面」を保管したメモ。
**Salesforce で CRM を管理していた**ことが確認できる。

→ ミナジン 2019 年時点では Salesforce（有料 CRM）を導入していた。GrowthFix は現在 Lステップ + スプレッドシートで代替しているが、クライアント 5 社超えたタイミングでの CRM 移行を検討材料とする。

### 2.2 フォルダが「01_営業フロー構築」に配置されている意味

営業フロー構築フォルダに「メール文面（CRM テンプレ）」が置かれていることは、ミナジンが「メール文面の標準化 = 営業フロー設計の一部」として捉えていたことを示す。

→ GrowthFix での展開：Lステップのメッセージシーケンスを「営業フロー設計」の一部として明文化する。

## 3. 競合分析表（8,059 KB xlsx）：2019年12月時点の業界地図

### 3.1 ファイルサイズから推測できる内容

8MB の xlsx は非常に大きい。内容として「比較対象ツール / 各機能の有無 / 価格帯 / 強み弱み / 実績社数」を多軸で比較する大型スプレッドシート＋スクリーンショット画像が含まれると推測。

**競合分析対象（フォルダ名「人事評価_競合分析表」から）：**
人事評価システムの競合を対象とした分析。

引き継ぎ書 §3.5 によると「識学 / Wevox / マルゴト人事 vs ミナジン」の対比が含まれる。

### 3.2 2019年12月時点の業界競合構造（推測）

| 競合カテゴリ | 代表企業（推定）| 訴求軸 | ミナジン vs |
|---|---|---|---|
| 評価管理 SaaS（機能特化） | 複数社（HR テック） | 操作性・機能数・価格 | コンサル × SaaS 融合 |
| 識学系（規律・評価） | 識学株式会社 | マネジメント規律・明確な評価基準 | コンテンツ深度 |
| エンゲージメント系 | Wevox / Sansan等 | サーベイ・エンゲージメントスコア | 制度設計伴走 |
| フルコンサル | マルゴト人事等 | ワンストップ制度設計 | SaaS × コンサル融合 |
| 社労士パートナー | 地域社労士法人 | 法律対応・労務管理 | 人事制度設計力 |

**発見 11：2019 年時点の競合地図は「識学 / Wevox / マルゴト人事」が 3 大競合軸だった**

これは Phase 4 MD2 で判明した「HiManager（1on1 SaaS 競合）」が 2019-2020 年以降に台頭したことを示す。
GrowthFix 2026 年の競合地図では：識学・Wevox・HiManager・採用 BPO・外資コンサルが 5 大軸（Phase 14 確定）。

→ 2019 年ミナジン視点での「3 大競合（識学 / Wevox / マルゴト人事）」は、GrowthFix 営業資料 v0.5 の「業界の進化史」として使える（2019 年の 3 軸 → 2026 年の 5 軸へ）。

### 3.3 マーケコンテンツ Excel（10.3 KB）の意味

軽量（10KB）の Excel = マーケティングコンテンツの企画管理シートと推測。
「何を・誰に・いつ・どこで」発信するかのコンテンツカレンダー。

**2019 年 6 月作成**：セミナー集客を本格化した時期と一致（2019年2月・4月開催実績あり）。

## 5. 横断発見：GrowthFix 営業フローの弱点マップ

本 MD（営業フロー＋マーケ）と前 MD（初回訪問・オーダー提案）を統合すると、**GrowthFix 営業の整備状況**が浮き彫りになる：

| 営業ステップ | ミナジン整備状況 | GrowthFix 整備状況 | 優先度 |
|---|:-:|:-:|:-:|
| リスト獲得 | 整備済（00_リスト系） | DM 5 件 / 日 ✅ | - |
| 集客セミナー | 整備済（00_集客セミナー） | Web 診断 ✅ | - |
| 初回訪問 | 整備済（a_初回訪問時） | 30 分説明セッション ✅ | - |
| 現状診断 | 整備済（d_組織診断素材） | SCAN/CODE ✅ | - |
| **見積提示** | **整備済（b_見積もり）** | **未整備（3 種見積テンプレ不在）** | **P1** |
| **申込・契約** | **整備済（c_お申込書 / ｆ_契約書）** | **未整備（申込フォーム・契約書テンプレ不在）** | **P1** |

| 解約防止 | 整備済（解約書フォルダあり） | 未整備 | P2 |

**発見 12：GrowthFix が最急に整備すべきは「見積書テンプレ 3 種」「申込フォーム」「契約書テンプレ」の 3 点**

これは Phase 4 の「Shift → Orbit 接続 SOP 未整備」に加えて発見した**新規の構造的弱点**。
