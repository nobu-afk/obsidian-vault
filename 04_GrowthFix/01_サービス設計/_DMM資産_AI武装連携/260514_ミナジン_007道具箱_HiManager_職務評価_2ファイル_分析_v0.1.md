## 1. HiManager提案書（1on1マネジメントTech 他社プロダクト）の構造

### 1-1. サービスコンセプトと対象課題

**コンセプト:** 「マネージャーのマネジメント力を強化し、従業員エンゲージメントを向上させることで、組織を活性化するサービス」

**提案先の典型課題（スライド4から）:**
- 組織に活気がない
- 離職がある
- リファラル採用がない
- 従業員エンゲージメント（関係性）に課題がある

**エンゲージメント理論（スライド5-9）:**
- 「仕事が好き・メンバーが好き・会社が好き」= 愛社心がある状態
- エンゲージメントが高い → 組織活性化 / 離職低下 / リファラル採用発生 / 生産性UP
- 低下原因のNo.1 = **直属上司との関係性**（Google社調査として引用）
- Googleの5因子（心理的安全性・相互信頼・構造と明確さ・仕事の意味・インパクト）を援用
- **心理的安全性 = マネージャーが部下の自己開示・弱みを許容することで発動**

### 1-2. 改善手法の4軸構造（スライド10）

```
改善サイクル = サーベイ（把握）× OKR（目標）× 1on1（面談）× フィードバック/称賛
```

スライドでは「CFR」（Conversation / Feedback / Recognition）+ OKR の組み合わせをパフォーマンスサイクルとして提示。両軸（把握と改善）を同時に回す構造。

### 1-3. ツール機能一覧（スライド15-21）

| 機能 | 内容 |
|---|---|
| サーベイ | パルスアンケート / エンゲージメント状況把握 |
| OKR管理 | 目標設定・進捗管理 |
| 1on1管理 | 定期面談の議題・記録管理 |
| フィードバック | 上下・360度フィードバック機能 |
| 称賛 | ピアツーピアの称賛機能（Slackライク） |

### 1-4. コンサルティング（伴走）モデル（スライド22・26）

- **月2回、コーチがマネージャーを直接コーチング**
- 導入フロー: 無料トライアル → 状況把握サーベイ → アドバイス → 運用設計 → 実運用
- コーチが2週間に1回マネージャーをフォロー（ツール + 人的伴走の二層構造）

[機密処理済 - 月額料金は分析対象から除外]

### 1-6. 競合分析メモ（Gravity Orbit との差異）

| 比較軸 | HiManager（ピアリー） | Gravity Orbit（GrowthFix）|
|---|---|---|
| アプローチ起点 | エンゲージメント向上（愛社心） | 引力設計（Authentic Org / 留まる引力）|
| 伴走対象 | マネージャー | 経営者・人事責任者 |
| 理論背景 | 心理的安全性・Google5因子 | Embedded Self × Continuous EC（Tomprou 2015等）|
| AI活用 | 記載なし（2023年以前の資料と推定）| AI武装 11エージェント（Orbit v3.1）|
| 測定軸 | エンゲージメントサーベイ（単一尺度系） | 軸15個ゲート（Phase 11-14反映・3引力対称）|
| ターゲット規模 | 明示なし（SMB〜中堅）| SMB経営者（10-100名）|

**差別化の核心:** HiManagerは「マネージャーを鍛えてエンゲージメントを上げる」という手段論。Gravity Orbit は「なぜその会社に留まりたいのかの引力構造を設計する」という哲学論から始まる。**「会社に引き留める仕組み」vs「留まりたい会社の設計」** という根本差異が言語化できる。

## 2. パート社員職務評価ツール（厚労省ガイド系・SMB適用）の構造

### 2-1. 文書概要

- **発行:** 厚生労働省（委託：株式会社ヘイ コンサルティンググループ）
- **発行年:** 平成23年（2011年）第2版
- **全体構成:** 第1章（問題提起）/ 第2章（実施手法）/ 第3章（導入事例）/ 参考（職務評価3手法 + 様式）

### 2-2. 問題提起（第1章）

**課題設定の3点:**
1. 働きや貢献に見合った待遇にする（パートタイム労働法 ねらい）
2. パート社員と正社員を均衡待遇とする
3. パート社員への説明責任を果たす

**パートタイム労働法の均衡待遇 3要件:**
1. 職務の内容（業務内容と責任の程度）
2. 人材活用の仕組みや運用（転勤・配置変更の有無・範囲）
3. 契約期間

### 2-3. 職務分析・職務評価の2ステップ（第2章）

**ステップ1 職務分析（情報収集・整理）:**

情報収集軸 = **業務の内容** × **責任の程度**

業務の内容の3要素:
- 主な業務（特徴を表し・なくてはならない・時間占有率が高い業務）
- 取り扱う対象・範囲
- 必要な知識や技能の水準（4段階：当日習得 → 数日 → 数ヶ月 → 専門知識要）

責任の程度の4要素:
- 権限（部下の有無・権限の範囲）
- 役割の範囲（リーダー的役割の有無）
- トラブル発生時・緊急時の対応（自己判断範囲）
- 成果への期待の程度（ノルマ・数値目標の有無）

情報収集手法 = **インタビュー法が推奨**（本人 + 共通上司の2者から聴取）

**ステップ2 職務評価（職務の比較）:**

ステップ2-1 業務の内容を比較:
- 「主な業務」を比較 → 同じ・一見異なる・異なる の2分岐
- 「取り扱う対象・範囲」が違うだけで業務性質が同じなら「実質的に同じ」と判断可

ステップ2-2 責任の程度を比較:
- 4要素（権限・役割・トラブル対応・成果期待）を総合判断
- 著しく異なる → 職務の内容は異なる
- 著しく異ならない → 職務の内容は同じ

### 2-4. 職務評価の3手法（参考コラム）

| 手法 | 概要 | 適用 |
|---|---|---|
| 単純比較法 | 職務を1対1で全体比較 | 小規模・シンプル |
| 分類法 | 職務レベル定義書と照合 | 中規模・標準化 |
| 要素比較法 | 職務を構成要素（知識・問題解決難度・成果の大きさ）に分解 → 要素ごとにレベル判定 | 大規模・客観性高 |
| 要素別点数法 | 要素比較法 + 点数化（重み付け可能）| 大規模・精密 |

**要素比較法の3構成要素:**
- 必要な知識や技能の水準（能力）
- 問題解決の難しさ（思考・行動）
- 求められる成果の大きさ（成果）

レベル定義 = A〜D（A=最高：専門知識必要 / D=最低：マニュアルで即日習得）

### 2-5. 導入事例（第3章）

**事例1（自動車整備・保険・顧客サービス業）:**
- 正社員7名・パート社員5名の小規模企業
- 保有資格 × 業務の種類 × 業務の内容 で職務等級（1〜4）を定義
- 等級ごとに2〜4段階の号俸（時給）設定
- **制度導入効果:** 目標意識・向上意欲の向上 / 新規採用パートへの期待値明示が可能に
- **パート社員の声:** 「単なる労働力として見ていないということなので、やり甲斐があります」

**事例2（東日本本社 / 全国チェーン小売業）:**
- 商品知識・接客熟練度でランク分け（1〜5段階）/ 正社員・パート共通ランク表
- ランクが上がるほど幅広い仕事 → 時給UP
- **制度導入効果:** 定着率向上 / 前向きなスキル習得意欲の向上 / 正社員との役割差が明確化

### 2-6. 職務説明書の位置づけ

職務分析の結果を「職務説明書」にまとめることで:
- 待遇への説明責任を果たせる
- 採用基準として活用できる（どのスキルを持つ人に・どの責任水準で・何をやってもらうか）

## 3. R/C/Orbit AI武装サービスへの反映候補（5件）

### [候補1] ★ Gravity Orbit の競合差別化言語を強化

**根拠:** HiManager は「エンゲージメント = 愛社心」と定義し、マネージャー訓練で解こうとしている。Gravity Orbit は「留まる引力 = Embedded Self × Continuous EC」という構造論から入る。この差異は営業トークで最重要。

**具体反映案:**
- Orbit LP Hero コピー候補: 「マネージャーを鍛えても離職は止まらない。問題は、なぜその会社が居心地いいのか、その設計ができていないことにある。」
- 営業スクリプト追加: HiManager/Wevox/SmartHR等を導入済みのクライアントへの差別化トーク（「ツールはある。でも引力の設計はまだですよね」軸）

### [候補2] ★ Orbit の1on1サーベイ設計に HiManager の測定軸を逆用

**根拠:** HiManager の測定軸（心理的安全性・相互信頼・構造と明確さ・仕事の意味・インパクト = Google5因子）は広く認知されている。Orbit の Phase 14 軸15個ゲートと対比させることで「既存ツールでは測れていない軸」を可視化できる。

**具体反映案:**
- Orbit 初回診断の問診票に「現在使っているエンゲージメントツール」を選択肢化
- 「Google5因子ではカバーできない留まる引力の3要素」をページ6（運用テンプレ）に追記

### [候補3] ★ Gravity Shift R の「採用前診断」に職務評価2ステップを組み込む

**根拠:** 厚労省職務評価ツールの「業務の内容 × 責任の程度」2軸 + インタビュー3点（日々の業務・業務目的・知識技能）は、Shift R Week1 現状診断の構造と整合する。現状は石井が経験則でやっている診断を、このフレームで標準化できる。

**具体反映案:**
- Shift R Week1-2 納品物テンプレートに「職務比較表（パート / 正社員 / 将来採用ポジション）」を追加
- AI武装候補エージェントとして「職務分析インタビュー自動化エージェント」を設計（日々の業務・業務目的・知識技能の3点を構造化質問で聴取 → 職務説明書ドラフト自動生成）

### [候補4] ⚪ Gravity Coaching の「組織現状診断」フェーズに職務評価の責任軸を援用

**根拠:** 職務評価の「責任の程度 4要素（権限・役割・トラブル対応・成果期待）」はコーチング初期の組織診断で使いやすい構造的質問セット。「あなたの会社のパートと正社員、責任の差が言語化できますか？」という問いが組織の不透明さを炙り出す。

**具体反映案:**
- Gravity Coaching 初回ヒアリングシートに「責任軸の4要素 × 職種別」の簡易診断を追加
- 「職務説明書がない = 組織引力を設計できない状態」というフレームをコーチングトークに組み込む

### [候補5] ⚪ 1on1 SaaS 競合マップを整備（Orbit 営業資料用）

**根拠:** HiManager の構造（ツール + 人的伴走 + 月額）はOrbit と外形的に似ているため、クライアントが比較検討する。競合マップに位置づけを明示することで、営業段階の混乱を防ぐ。

**具体反映案:**
- 競合比較表 v1.1（既存：識学/Wevox/採用BPO/外資コンサル）に HiManager / SmartHR / カオナビ等のSaaS軸を追加する列を新設
- 比較軸: 「アプローチ（手段論 vs 哲学論）」「伴走対象（マネージャー vs 経営者）」「理論背景（エンゲージメント系 vs 引力設計系）」

## 4. 仮説E v0.2 営業資料v0.4への組込候補（3件）

### [組込1] ★ 「ツール完備 = 組織完成ではない」というフレームの強化

**背景:** 仮説Eは「Gravity統合 + AI武装でR/C/Orbit内で実務レイヤーも拾う」方針。HiManager資料は「ツール（サーベイ/OKR/1on1/FB/称賛）を入れたが組織は変わらなかった」という典型的な失敗パターンの言語化として活用できる。

**組込案:** 営業資料v0.4の「なぜ既存ツールでは不十分か」セクションに、HiManagerが解こうとしている問題（マネージャー訓練 × エンゲージメント測定）を「手段層の解決策」として位置づけ、Gravity が「構造層の設計」として差別化するスライドを1枚追加。

### [組込2] ★ SMBオーナーへの「職務の言語化 = 採用引力の設計」訴求

**背景:** 厚労省ツールの「職務説明書がないと待遇説明できない」問題は、Gravity の文脈では「採用段階で何を求めているか言語化できていない = 引き寄せる力がない」と翻訳できる。

**組込案:** 営業資料v0.4に「あなたの会社の職務説明書はありますか？」の問いを導入診断スライドに追加。「職務が曖昧 = 採用引力もコーチング効果も出ない」という流れを1スライドで示す。

### [組込3] ⚪ HiManager型サービスからの乗り換えクライアント向けトーク

**背景:** ミナジンは HiManager を「道具箱」として検討していた。同様に「1on1 SaaS + コーチング伴走をすでに試みた企業」はOrbit の主要見込み客層。

**組込案:** 営業資料v0.4に「HiManager / Wevox等の1on1 SaaSを試みた会社が次に何を求めるか」の具体シナリオ（「測定はできた。でも何が変わったか言えない」）を Want段落に追加。

## 5. 書籍『引力経営』v0.3への補強素材（2件）

### [書籍候補1] ★ 「エンゲージメント疲れ」の時代背景としての1on1 SaaS乱立

**概要:** HiManagerの提案書が示す「エンゲージメント = 愛社心 = マネージャーが高める」という論理は、2010年代後半の日本HR業界の支配的解釈。この解釈を前提に構築されたSaaSが多数存在するが、それらが「何を測り・何を設計できていないか」を整理することで、書籍の「引力とは何か」の章で使える反証素材になる。

**書籍への配置案:** 第1章または第2章「既存の組織論が答えられなかったこと」節に挿入。「エンゲージメントサーベイでは測れない『留まりたい理由』」というサブテーマで、HiManagerのような手段論的アプローチが解決できない問いを提示し、引力概念への橋渡しにする。

### [書籍候補2] ⚪ 職務の「責任の程度 4要素」を引力の構造的証拠として援用

**概要:** 厚労省フレームの「責任の程度 = 権限 × 役割の範囲 × トラブル対応 × 成果への期待」は、留まる引力の「Embedded Self（自分の役割・人間関係・資産の蓄積）」と構造的に接続できる。「役割が言語化されていること」「自分の判断範囲が明確であること」は Embedded Selfの中核要素である。

**書籍への配置案:** Orbit / 留まる引力の章で「なぜ職務の明確化が引力になるのか」の補論として1段落引用可。出典：厚生労働省「職務分析・職務評価実施マニュアル」（平成23年第2版）。
