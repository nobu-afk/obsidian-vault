## 1. 構造抽出（業務プロセス）

### 1.1 フォームの基本設計

この面談ファイルは**採用選考〜内定決断〜入社後フォローを 1 行に収める統合型追跡シート**として設計されている。

```
列グループ構成（左 → 右）：

【管理基本情報】
  担当 / 実施日 / 社員番号 / 名前 / 入社日 / 配属 / 前職

【活動プロセス（入社前：転職行動）】
  転職のきっかけ / 企業選びの軸 / 媒体 / DMMの第一印象

【選考フェーズ別（入社前：面接体験）】
  １次面接：面接官 / 面接内容感想 / 志望度変化 / 他社動向
  ２次面接：同上

【内定意思決定フェーズ（入社前）】
  他内定企業 / 即決したか / 懸念点や不安点 / 決断理由

【選考全体評価（入社前）】
  感想・意見 / 要望・改善

【入社後フォロー（入社後）】
  入社後ガイダンス感想 / 入社後フォロー状況 /
  入社書類の郵送 / DMMへの要望 / 制度・福利厚生への希望

【リファラル・メディア活用調査（入社後）】
  周囲の転職者情報 / 紹介可能か / SNS取り扱い /
  DMMmeetup 認知・参加歴 / DMMmeetup が採用きっかけになったか
```

**設計の特徴**：入社前（転職動機・選考体験）と入社後（オンボーディング体験）を単一行に持つことで、**入社前の期待値と入社後の実体験を同一人物ベースで比較可能**な構造。これは HRBP が「採用→定着」を一気通貫で追跡していた証拠。

## 3. 仮説 E v0.2 営業資料への組込候補

### 3.1 営業資料 v0.3 への追加候補テーマ

**A：「入社から2週間が勝負」の科学的根拠（P0 優先）**

647 件の実データから、入社後フォロー面談の 65% が入社後 14 日以内に集中。DMM 規模の企業でもこの設計が確立されている → 「入社後2週間の受け入れ体験が定着を決める」という Orbit 月次サイクルの起点設計を実証数値で語れる。

**B：「業務ロール不明」が入社前懸念のトップ2（雇用形態と並列）**

懸念点回答 699 件中、「業務内容・ロールの不明瞭さ」が 9%（60 件）= 雇用形態の懸念（62 件）と同率。これは採用フェーズのロール合意が不十分なまま入社している実態を示す → **Gravity Scan の Pre-Shift 適合診断「採用前のロール期待値合意」の必要性**を数値で訴求可能。

**C：「誰と働くか」が企業選択の隠れた最大軸**

企業選びの軸で「人・文化への共感」（32 件）と「安定性」（45 件）は近似するが、転職後フォロー状況の定性コメントでは「チームメンバーに助けられた」「気軽に聞ける環境」が最多頻出 → **Phase 1 発見「人間関係・チームが エンゲージメント 47.7% の最大規定因」（Wevox）との整合**が完全。

**D：入社前懸念解消が「即決」の分岐点**

「即決したか否か」列と「懸念点・不安点」列のクロスでは、懸念が解消された（「人事の方と話して大丈夫とわかった」「やることが決まっていた」）ケースで即決が多い → **Gravity Scan の「入社前 1on1 設計（Shift R Week 1-2 納品物）」の価値**を裏付ける実例。

## 6. v0.2 マスター（260513）との差分・追加事項

### 既存 v0.2 で P2 扱いだった項目の P0 格上げ候補

v0.2 の「2.1 Recruit AI 武装版」にて：

> 入社後面談議事録分析（02_オンボーディング/04_入社後面談）| P2 | サブフォルダ確認済

→ 今回の詳細分析により、以下の理由で **P1 格上げを推奨**：

1. 「入社後2週間集中（65%）」という実装根拠が確定した
2. シート7の「期待値ギャップ × 1ヶ月困りごと」設計が Orbit 月次サイクルの直接原型と判明
3. 懸念点未解消（業務ロール不明・年収変化）が離反先行指標となる連動ロジックが確認できた

### minutes-analyzer への新規追加仕様候補

現行 minutes-analyzer は「1on1 議事録分析」が主目的。今回の発見から「**入社後面談モード**」の追加が妥当：

```yaml
onboarding_interview_mode:
  phase_detection:
    - pre_entry_expectation  # 入社前の期待値（転職動機・企業選択軸）
    - selection_experience   # 選考体験スコア
    - concern_resolution     # 懸念点解消確認（内定〜入社後）
    - day30_gap_check        # 30日時点の期待値ギャップ
  alert_flags:
    - role_ambiguity         # 業務ロール不明続行
    - team_isolation         # チームサポート不足
    - tool_overload          # ツール過多・情報消化不良
  output:
    - onboarding_risk_score  # 0-100（離反リスク先行指標）
    - referral_intent        # リファラル意向
    - followup_timing        # 次回フォロータイミング推奨
```
