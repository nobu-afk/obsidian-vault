## 1. 7ステップコンサルメソッドの全体構造（第1回〜第7回の流れ）

### メソッドの4層構造

```
戦略レイヤー
  Step 1: Mission–Strategy Matching（理念・ビジョン・戦略の調和）
  Step 2: Grade Mission（職位ごとの役割）

役割レイヤー
  Step 3: Function Role（機能ごとの役割）

成長レイヤー
  Step 4: Job Learning Step（仕事を習得するステップ）
  Step 5: Career Development Vision（キャリア開発のビジョン）

制度レイヤー
  Step 6: Incentive Design（報酬設計）
  Step 7: Evaluation Design（評価設計）
```

### 成果物セット（7回終了後に揃うもの）

| 成果物 | 内容 |
|---|---|
| 人事制度ガイドブック | 目的・グレード・評価基準・運用ルール・賃金ルール |
| 部署ステートメント | 役割・ミッション・KPI・業務内容・CDV |
| 目標シート | 目標シート + 行動計画表 |
| 賃金テーブル | グレード別給与バンド + ピッチ |

### コンサルの思想的前提（Step 1 で明示）

- **育成 > 評価**：人事制度の目的は「給料を決めること」でなく「より大きな成果を上げること」
- **報酬 3種類**：金銭的報酬 / 機会報酬 / やりがい報酬（Step 1 冒頭で宣言）
- **一貫性原則**：理念→価値基準→ビジョン→事業戦略→人事戦略→人事制度の縦串整合

## 2. 第1回 理念ビジョン戦略の調和（業務プロセス構造化）

### Section 1: 人事戦略・制度の位置づけ

```
ミッション/理念 → バリュー/価値基準 → ビジョン
  ↓
事業戦略（商品・マーケ・営業・ロジ）
  ↓
人事戦略（採用・育成・教育）
  ↓
人事制度 / 人事労務実務
```

「組織と個人が理念でブリッジされている状態」がゴール。個人目標・ビジョン・行動計画が組織の目標・ビジョン・事業戦略と4層でブリッジされるモデル。

### Section 4: 理念・ビジョンの整理（ステークホルダー別3軸）

**整理の枠組み：** 顧客に対して / 社員に対して / 社会に対して、それぞれ
- 理念（存在意義）
- 価値基準（意思決定基準）：行動指針・クレドの上位概念
- ビジョン（領域 × 到達レベル）
- 目標（ビジョン実現への途中到達点）

**一貫性チェックポイント（コンサルが顧客に問う問い）：**
- 価値基準に反した行動でも評価されていないか？
- ビジョン実現に必要な人材を育成・採用していないか？

### Section 5: 未来の組織図作成

- 事業戦略遂行時の組織体制を描く（細部にこだわらず人事戦略策定に必要なレベル）
- 後日：グレード別人員構成・概算人件費の試算

### Section 6: 事業戦略の整理

**整理枠組み（7種）：** 事業構想 / 新規事業 / 既存事業（ビジモデル転換）/ 商品・サービス開発 / マーケ・販売 / 組織・IT / その他

→ 各項で「①戦略」「②不足機能（スキル不足 or 人員不足）」を分解

### Section 7: 人事戦略の整理

```
不足リソースの確保戦略（リソース戦略）
  ├── 社内で習得可能 → 育成（OJT / 社内研修 / 外部研修 / 出向）
  ├── 社内で習得不可 → 中途採用 / 新卒採用 / アウトソース
  └── 採用後 → CDV（必要人物像 = マインド・ナレッジ・スキル）
```

**コンサルの鍵問い：** 「その人事戦略で事業戦略の遂行が可能になるか？」

## 3. 第6回 報酬設計（業務プロセス構造化）

### 報酬3種類モデル（理論的基盤）

| 報酬種別 | 内容 | 会社の施策例 |
|---|---|---|
| 金銭的報酬 | 給与・賞与 | 賃金テーブル・バンド設計 |
| 機会報酬 | 昇格・チャレンジ仕事・教育機会 | CDV・ローテ設計 |
| やりがい報酬 | 達成感・貢献感（本人が感じるもの） | 社員表彰・サンクスメール・社内提案制度 |

### 給与設計の4工程

**工程1: 労働分配率の確認**
- 積極的投資企業 → 35%
- 事業投資を要さない企業 → 50%
- 他社比較は非推奨（ビジモデル違いで無意味）

**工程2: バンド設計（グレード別給与幅）**
- 論理的唯一解なし → 「経営者のビジョン感覚」が意思決定材料
- 人件費シミュレーション（平均 / 最低 / 最高）で労働分配率逆算

**工程3: ピッチ設計（1等級昇級時の昇給額）**
- 低グレード：ピッチ狭く（頻繁昇級・コストインパクト小）
- 高グレード：ピッチ広く（成果連動・メリハリ強調）
- 1評価期間に何段階昇級させるかで逆算設計

**工程4: 導入原則**
- 制度導入時は「給与据え置き」原則（下げも上げもしない）
- 運用しながら実態と制度を合わせていく

### 手当設計の基本姿勢

- 手当は「明確な意図を持って支給するもの以外は廃止を推奨」
- 問題点：不公平感・計算複雑化・種類肥大化
- 時間外割増賃金への算入判定（家族手当・住宅手当の注意）

### 生産性向上手当（特徴的設計）

「時間で働く」→「成果を上げるために働く」概念転換を意図した手当。成果にコミットし自ら勤務時間を決められる社員のみに適用。基本給を低く抑え（最低賃金ライン）、総額保証額との差を手当で補填する構造。

### 賞与設計の2方式

| 方式 | 内容 | 動機づけ効果 |
|---|---|---|
| 規定に基づく支給 | 基準額×評価係数 | 目標達成への動機づけに強い |
| 利益の分配 | 決算賞与・グレード別持ち点方式 | 全社利益把握が前提・情報共有必須 |

評価単位の選択：全社 / 部署・チーム / 個人（組み合わせ可）

## 4. 第7回 評価設計（業務プロセス構造化）

### 評価の3カテゴリ

| カテゴリ | 内容 | 根拠ステップ |
|---|---|---|
| 目標達成 | KPI の達成状況 | Step 3 Function Role |
| マインド | 価値基準・行動指針の体現度 + グレード別マインド | Step 1 + Step 2 |
| 業務能力 | 担当業務・マネジメント業務の遂行能力 | Step 4 Job Learning Step |

### グレード別評価重点の傾向

```
ディレクター（上位）: 目標達成（売上・利益）中心
マネージャー（中位）: 目標達成（売上・利益以外）＋マインド
スタッフ（下位）: 業務能力中心
```

### 評価→処遇の連動構造

```
各評価項目 → 項目ごと評点（1-5等） → 
  目標達成 総評
  マインド 総評     → 総評スコア → 昇降格・昇降級判定 → 処遇決定
  業務能力 総評
```

処遇パターン：昇格 / 2等級以上昇級 / 1等級昇級 / 現状維持 / 降級 / 降格

### 評価設計の実装原則

- 重みづけは「厳格化・機械化・成長より評価へ意識向き」のリスクを認識した上で設定
- **「制度を事前に作り込みすぎるな」** 原則：一旦実運用→不具合箇所を修正の反復が有効
- 評価期間中の高頻度進捗確認（期中運用）を「重要」と明示

### 昇降格・昇降級の2方式

| 方式 | 適合場面 |
|---|---|
| 昇降格・昇降級の「要件」を定める | マネージャー以上・成果測定可能な業種 |
| 等級に「該当する要件」を定める | 担当者レベル・業務習得に年月を要する業種 |

**AI 武装案：** 社員アンケートデータから機会報酬・やりがい報酬の充足度を自動スコアリングするシンプルな Python スクリプト。Wevox 等の外部ツールと差別化する「Gravity 独自軸」として機能。

### C-5-3: Gravity C「生産性向上手当」設計支援テンプレート

## 6. 仮説 E v0.2 営業資料 v0.4 への組込候補（3件・コンサル方法論軸の差別化）

### P-6-1: 「7ステップ人事コンサルと Gravity の違い」競合差別化スライド（核心）

SMB 人事コンサル方法論（Step 1-7 型）は「制度構築」に終始する。評価基準・賃金テーブル・目標シートを作るための 7 回セッションであり、「人が集まる・躍動する・留まる」引力構造そのものは設計しない。

**営業資料 v0.4 への組込文案（案）：**
> 「従来の人事コンサルは『制度の箱』を作ります。理念整理・評価設計・賃金テーブル。精緻な成果物は揃います。しかし、なぜその制度が"機能しない"のか。理由は一つです。人が集まる構造（引力）がないまま制度を入れても、器が引力を生みません。Gravity はその根本の引力を設計します。」

識学（管理強化）/ Wevox（指標可視化）/ 従来型人事コンサル（制度設計）の3者は、すべて「引力の設計不在」という共通の盲点を持つ。この3者に対して「Gravity 引力設計が先、制度は後」のポジション設計が成立する。

### P-6-2: 「報酬3種類モデル」＝ Gravity CODE×SCAN の言語化補強

金銭的報酬 / 機会報酬 / やりがい報酬の3種類は、Gravity の「引力の3軸（集まる × 躍動 × 留まる）」と以下のように対応する：

| 報酬種別 | Gravity 引力軸 | 学術的裏付け |
|---|---|---|
| やりがい報酬 | 個人引力（Authentic Self × Harmonious Passion） | Vallerand 2003 / SDT |
| 機会報酬 | 躍動引力（Collective Efficacy × Growth） | Bandura 2000 |
| 金銭的報酬 | 留まり引力の外発的側面（継続コスト） | Embedded Self との対比 |

営業資料で「Gravity は引力を3種類の報酬すべてで設計する。金銭だけに頼る組織は必ずガタつく」という主張の補強エビデンスとして使える。

### P-6-3: 「制度は事前に作り込むな」原則 → Orbit の継続価値証拠として使用可能

Step 7 で「評価設計は事前に完成させるな、実運用しながら修正せよ」と明言している。これは Orbit（月15万・継続伴走）のサービス価値「制度を生きたものにし続ける伴走」の根拠として活用できる。

**営業トーク案：**
> 「人事コンサルのプロも言っています。評価制度は作り込んで終わりではない。実運用しながら不具合を修正し続けることで機能する、と。Orbit はその継続的修正と引力メンテナンスを月次で提供します。」

## 7. 書籍『引力経営』v0.3 への補強素材（2件）

### B-7-1: 「報酬3種類」モデルとの接続（Part 3 躍動・留まり章の補強）

書籍の「留まる引力（Embedded Self × Continuous EC）」章で、従来型経営が「金銭的報酬のみで社員を留めようとし続けた帰結」を章立てする際の対比事例として使える。2014年当時の SMB 人事コンサルティングの現場リアルとして「金銭報酬 + 制度設計が主流だった時代」のスナップショットを提供。

機会報酬・やりがい報酬は SDT（自己決定理論）の外発的報酬 vs. 内発的報酬の翻訳であり、Gravity の Harmonious Passion（Vallerand 2003）/ Intrinsic Motivation（Deci-Ryan）との理論接続が既に可能。

### B-7-2: 「理念→戦略の一貫性設計」は Gravity の前提構造として引用価値あり

書籍「集まる引力」章での「なぜ人が集まるのか」の設計原理として、「理念・価値基準・ビジョン・戦略の縦串整合がないと引力が機能しない」という Step 1 の論点を補強引用できる。

Schneider（1987）ASA 理論（P-O Fit・引力-選択-摩耗）の「理念が組織を定義し、同質の人が集まる」メカニズムの具体的実装例として、SMB コンサル現場の「場の定義付け」概念を参照できる。
