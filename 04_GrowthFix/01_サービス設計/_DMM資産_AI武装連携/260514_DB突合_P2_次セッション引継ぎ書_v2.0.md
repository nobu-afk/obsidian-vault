## 3. P2 詳細実行手順

### 3.1 H6：Tomprou VRBs 3 次元実務妥当性検証（所要 4 時間）

**学術命題：** Tomprou 2015 Post-Violation Model（被引用 277・JOB）
- VRBs（Violation Resolution Beliefs）3 次元：
  - **Probability**（解決可能性 / 5 件法）
  - **Timing**（解決までの時間軸 / 1 ヶ月〜解決不可能）
  - **Agency**（解決の主体帰属 / 経営者主導〜誰も動けない）
- 4 outcomes 予測：
  - 🟢 Thriving（高 VRBs + cognitive regulation）
  - 🟡 Reactivation（中-高 VRBs + 元契約維持）
  - 🔴 Impairment（低 VRBs + 感情抑制兆候・静かな離職）
  - 🚨 Dissolution（極低 VRBs + 離脱意図）

**主要データソース：**

| ファイル | 用途 |
|---|---|
| `260513_DMM_Wevoxコメント分析_v0.1.md`（6,780 件・経年追跡可）| **H6 主軸**：退職者の退職前 3 ヶ月発言から VRBs 3 次元を逆推定 |
| `260514_DMM_05_労務対応_退職アンケート_分析_v0.1.md`（1,455 件）| **H6 補強**：自由記述（怒り / 裏切られ / 失望）から Agency 帰属を逆推定 |
| `260514_DMM_05_労務対応_3ファイル_分析_v0.1.md`| **H6 補強**：退職面談ログ 422 件のカテゴリ分類 |

**検証手順：**

1. **Wevox コメント分析を経年で追跡**（離脱予兆 5 シグナル検出件数 152+121+72+52+18 = 415 件のうち、実離脱者を特定可能か）
2. **退職前 3 ヶ月の発言パターンから VRBs 3 次元を逆推定**：
   - Probability：「解決できる」「もう無理」系キーワード
   - Timing：「いつか」「今すぐ」「もう手遅れ」系キーワード
   - Agency：「経営者次第」「自分次第」「誰も動かない」系キーワード
3. **離脱直前の VRBs 軌道を 4 outcomes と対応付け**：
   - Dissolution 軌道（極低 VRBs + 「もう無理」+ 「誰も動けない」）の 3 ヶ月以内離職率を検証
   - Impairment 軌道（低 VRBs + 感情抑制 + 静かな離職）の検出可能性を検証
4. **3 分類で振り分け**（論文 ⊃ 実証 / 論文 ≠ 実証 / 論文外 ⊂ 実証）

**期待アウトプット：**

   - 退職面談ログ 422 件のカテゴリ分類で Dissolution / Impairment / Reactivation / Thriving の分岐構造を逆推定
3. **静かな離職（Impairment 軌道）の検出可能性検証**：
   - Wevox ニュートラル 85.2% のうち、実際は感情抑制で Impairment 軌道にいる人材の検出
   - F-14 Affective 表現変容パターンとの接続

## 5. 注意事項・落とし穴

### 5.1 同じファイルを 2 回 Read しない

CLAUDE.md ルール「同セッション再 Read 不要」原則。ただし**別 Node では新規 Read 必要**：
- H6 では `260513_DMM_Wevoxコメント分析_v0.1.md` を新規 Read 必須
- H7 では全 DMM 個別分析 MD 22 ファイル + 統合業務マスター v2.0 を横断 Read

### 5.2 機密マスキング方針厳守

DMM 個人情報・給与金額の具体数値は一切記載しない。制度の設計構造・集計分布・運用パターン・NPS スコア集計値のみを対象とする。

### 5.3 仮説修正の可能性を排除しない

H5 で「仮説の前提が誤り」+ H3 で「Normative -10% / Affective +10%」と判明している。H6+H7 でも**期待結果（仮説）に合わない実証データが出る可能性**を前提に検証する。**論文 ⊃ 実証 / 論文 ≠ 実証 / 論文外 ⊂ 実証**の 3 分類で振り分けるのが堀田フレーム認知価格設計の最強根拠。

### 5.4 業務系タスク Part 3 参照

CLAUDE.md ルール：このタスクは**サービス設計**に該当するため、`09_会社OS/公開/ガイドライン/商品.md` Part 3 + `260429_堀田フレームワーク体系_最新版.md`（堀田フレーム）の参照を推奨。ただし、H6+H7 は検証作業中心であり、設計の意思決定はせずに「検証結果のマトリクス化」までに留める方針。意思決定は P3 v1.1 確定書作成時にメインスレッドが実施する。

### 5.5 H6 VRBs 逆推定の限界に注意

退職前 3 ヶ月発言から VRBs を逆推定する手法は推測ベース。実証データとして**完全に再現できない可能性**を前提に：
- 「再現可能」「部分再現可能」「再現不可能」の 3 段階で評価
- 「論文 ≠ 実証」と判定された場合も、それ自体が貴重な発見（実証データから VRBs を逆推定できないなら、月次セッションでの「直接 VRBs 測定」装置の必要性が裏付けられる）

### 5.6 v1.0 反映済 SSOT を破壊しない

v1.0 反映 5 Phase 完走済（3 軸完成設計書 v1.0 / 商談スクリプト v1.3 / Page 6 v1.2 / 書籍 第 4 章 v0.4 / Orbit LP +77 行 / SSOT_用語と定義.md v5.1）。P3 v1.1 化作業時は **既存 v1.0 構造を温存しながら H6+H7 結果を追記補強する** 形で進める（章番号維持・既存セクション削除禁止）。
