## 2. スカウト文面構造分析：テンプレート 7 パターン

実測データから確認されたパターン。返信率は実績値（送信数 5 件以上のもの）。

### Pattern 1 ── 担当者名乗り + 個別スキル引用型（返信率平均 10.4%）

大多数の文面が採用するベースパターン。エンジニア向けで最も多く使用された。

**構造：**
```
[挨拶] 突然のご連絡失礼いたします。
[名乗り] [会社名] [事業部名] [役職] の [担当者名] と申します。
[個別引用] [候補者 ID] 様のご経験を拝見し、[具体スキル X / Y] のご経験に魅力を感じました。
[接続] これらのご経験に加え、[志向性] から、弊社 [チーム名] にてご活躍いただけるのでは、と思いました。
[自社紹介] ————（セパレータ）以下に自社紹介
[CTA] 一度お話しいただけないでしょうか？
```

**特徴：**
- 候補者のスキルを箇条書きで 2-3 点引用（コピー感を減らす個別化装置として機能）
- 担当者が現場の責任者・マネージャーレベルであることを明示（採用担当ではなく事業部長・EM クラス）
- 自社紹介は「セパレータ（————）」で本文と分離するフォーマット

**最高実績：** 返信率 31.2%（送信 16 件）/ データインフラ・トラッキング系エンジニアポジション

---

### Pattern 2 ── 超短文・カジュアル接触型（返信率 23.5%）

最短クラス（233 字・6 行）で最高返信率を記録した例外パターン。

**構造：**
```
[挨拶] はじめまして、[会社名] の [担当者名] と申します。
[理由一言] ご経歴を拝見し、一度お話をしてみたくご連絡させていただきました。
[訴求] [自分が今探している人物像を一言で]
[ハードル下げ] 転職のご意思はなくても構いません。お互いの状況についてカジュアルに情報交換できれば。
[参考] 求人例を添付しておきます。他ポジション多数ございます。
[CTA] ご検討の程よろしくお願い申し上げます。
```

**特徴：**
- 個別化ゼロ・テンプレ度 100%でも返信率が高い（カジュアル接触ハードル低下が主因）
- 「転職意思なしでも OK」と明示することでガードを下げる
- 求人詳細は「添付参照」に委ねて本文をシンプルに保つ

**示唆：** 文面の長さと返信率は比例しない。情報量過多よりシンプル接触が上回る場合がある。

---

### Pattern 3 ── 候補者プロフィール先行引用型（返信率平均 8.2%）

文面冒頭に「評価ポイント」セクションを置き、候補者の職務経歴書を転記するパターン。

**構造：**
```
[プロフィール引用] 評価ポイント
  [スキル名]：[年数]年/[技術スタック]
  [強みの記述（職歴書より転記）]
[候補者 ID 宛名] [候補者 ID] 様
[以下、Pattern 1 と同構造]
```

**特徴：**
- 担当者の名乗りより先に候補者プロフィールを置く（「あなたのことを見ています」シグナル）
- 実測では Pattern 1 より返信率が低い（8.2% vs 10.4%）
- セキュリティエンジニア向け（返信率 15.8%）など職種によっては有効

**示唆：** プロフィール転記は「見てますよ感」を演出するが、機械的なコピペと識別されるリスクあり。

---

### Pattern 4 ── ハイレイヤー向け限定感・役職者接触型（返信率 42.9-71.4%）

事業開発・経営企画・ハイレイヤー候補者向けの高返信率パターン。A/B テスト比較シートで計測。

**構造：**
```
[タイトル] 【執行役員直接面接】役員直下新規事業/[具体ミッション名]まで一任
[冒頭] ・社員採用/業務委託にてスポット契約等、転職意向に合わせてご相談可能
       ・年商 XX 億以上の大型事業
[本文] ペルソナ特定要素（ハードウェア製品を扱う大企業の事業部長格）
[動機付け] 会社の方針で自身が動かしたいことが動かせないフラストレーション訴求
[CTA] 応募ハードルを最大限下げる（フォーマット自由・まずはカジュアルに）
```

**特徴：**
- タイトルに「役員直接面接」「一任」を入れることでビジネスチャンス感を演出
- 転職意向の有無に関わらず「業務委託」「スポット契約」を選択肢として提示
- 採用ペルソナを事前に詳細定義（年齢帯・経歴パターン・現状の不満要因）して文面に反映

**最高実績：** 返信率 71.4%（送信 7 件） / 事業開発室・事業開発ポジション

---

### Pattern 5 ── 衛生要因強調型（ローレイヤー向け）

職種レベルが低めのポジション（一般事務・法務事務等）向けのパターン。動機付け要因ではなく衛生要因（職場環境・待遇安定性・リモート）に特化。

**構造：**
```
[タイトル] "おしゃれなオフィス × 私服勤務 × 在宅勤務あり" + 安定した環境で長期キャリア
[冒頭] 候補者の受発注業務経験に魅力を持った旨（1 文）
[衛生要因列挙] 在宅勤務あり / 安定業界 / 成長率 XX%（具体数字）
[CTA] 面接日程を複数提示いただければ
```

**特徴：**
- キャリアアップや裁量系の訴求（動機付け要因）は意図的に排除
- 「安定」「コロナ禍でも業績右肩上がり」等の安全欲求に刺さるワードを優先
- 応募書類（履歴書）持参を最初からリクエストすることでハードルを意図的に上げる（書類精度確保目的）

---

### Pattern 6 ── 責任者自己紹介 + ミッション共有型（Biz 系・返信率 28-50%）

Biz 系ポジション（経営企画・コンテンツ編成・財務経理）で使われたパターン。採用担当ではなく「責任者本人が送る」体裁。

**構造：**
```
[挨拶] はじめまして。
[名乗り] [会社名] [事業部名] の責任者をしております [担当者名] と申します。
[個別化トリガー] [候補者 ID] 様のご経歴を拝見し是非一度お話したいと思いメッセージしました。
[自社文脈説明] [事業部が新設された背景 / 全社最重要事業であること等]
[担当者自己開示] 私自身も [X 社] から入社した経緯 / キャリアパス簡潔に
[未公開情報フック] まだ詳しい事業展開は世の中に発信していない状況 → 直接話したい
[CTA] ご多忙かと存じますがぜひご検討いただけますと
```

**特徴：**
- 担当者の略歴（前職 / キャリア転換理由）を 2-3 行で自己開示 → 人として共感を引く
- 「未公開情報」を武器にして返信の動機を作る（メールを開いても全情報を得られないため）
- ポジション複数の同一事業で同構造の文面を使い回し（コンテンツ制作 / 経営企画 / 編成と共通テンプレを各責任者にカスタマイズ）

---

### Pattern 7 ── 企業直接スカウト差別化型

エージェント経由スカウトが多い市場（エンジニア・Biz 上位層）への対抗パターン。「企業から直接あなたを選んだ」シグナルを前面に出す。

**構造：**
```
[タイトル] 【企業直接スカウト】 + 【年収 XX 万以上】 + 【フレックス / リモート可】
[冒頭] 企業直接スカウトであることを 1 行目で明示
[本文] 簡潔（Pattern 2 に近い）
```

**特徴：**
- 「エージェントからのスカウトと区別をつける」が設計意図として明記（設計シートより）
- 年収・働き方をタイトルに入れる衛生要因訴求と組み合わせ
- 内在的動機（キャリアパス）をタイトルに入れるとより効果的とコメントあり

## 4. A/B テストの痕跡

### 確認されたテストペア

**テスト 1 ── バックエンドエンジニア 修正前後**
- 修正前：汎用タイトル「新規事業立ち上げや事業活性化ができる人材を募集」/ 返信率 9.1%
- 修正後：未計測（テスト中断の可能性）
- 示唆：汎用タイトルの返信率が低く、改善を検討した形跡

**テスト 2 ── 企画営業 ハイレイヤー（同一ポジション 2 バリアント）**
- バリアント A：「【執行役員直接面接】役員直下新規事業 / 半導体分野で市場創り〜組織構築まで一任」/ 返信率 42.9%
- バリアント B：「【事業部長がお会いします】ライセンスビジネスについて一度お話しませんか」/ 返信率 71.4%（ただし送信 7 件と少サンプル）
- 示唆：「役員直接面接」より「事業部長がお会いします」の方が具体感・会話感で上回る可能性

**テスト 3 ── Biz 系 同一テンプレ × ポジション違い**
- コンテンツ制作：返信率 50.0% / 経営企画：41.7% / コンテンツ編成：28.6%
- 同一テンプレ構造（共通 80% + ポジション固有 20%）で返信率がポジションによって差異
- 示唆：テンプレ共通化は効率的だが、ポジションの魅力度・希少性自体が返信率に影響

### A/B テスト設計上の観察

- テストは「修正前後」単純比較が中心。体系的なランダム配信ではなく、送信済み文面の振り返り型
- 各シートに「制作意図 / 採用ペルソナ / 導線ポイント」行が設計フレームとして存在 → テンプレ設計は意図的に行われていた
- 返信率追跡は書類通過・1 次面接・最終面接・内定・決定まで全ファネルで管理

## 5. NG パターン（実測・コメントから抽出）

### NG-1：汎用タイトル

```
NG例（抽象化）: 「新規事業立ち上げや事業活性化ができる人材を募集しております。一度お話をしませんか。」
返信率: 9.1%（44件送信）
問題: 職種名も会社の具体性もなく、誰にでも送れる文面
```

### NG-2：スキルリスト転記型（コピペ感）

```
NG例: 「評価ポイント」セクションに職歴書の技術スタックをそのまま貼り付け
問題: 読んでいる個別感が出ない / 機械処理感が伝わる
実測返信率: 平均 8.2%（Pattern 1 の 10.4% を下回る）
```

### NG-3：求人ページ外部リンク貼り付け

```
NG例: 求人ページ以外の外部 URL（会社紹介記事等）を本文に複数貼り付け
問題: 離脱を促す（設計シートコメントより「離脱懸念あるため求人ページ以外のリンクは張らない」と明記）
```

### NG-4：同一文面を複数事業部から同一候補者に送信

```
問題: 同じ候補者 ID に対して複数事業部から同内容のスカウトが届く事例あり（気づきシートに記録）
実害: 候補者からの不信感 / 組織の採用一貫性の欠如
```

### NG-5：衛生要因のみの文面をハイレイヤー向けに使用

```
問題: 動機付け要因（裁量・ミッション・成長）を省いた衛生要因中心文面はハイレイヤーには効果薄
対策: ポジションレイヤーに応じて Pattern 5 ↔ Pattern 6 を切り替える
```

### NG-6：文面長過多（1,900 字超）のエンジニア向け文面

```
問題: データインフラ系で最高返信率 31.2% を記録した文面は 1,880 字
ただし別途、233 字の超短文で 23.5% を記録
示唆: 長文が悪いわけではないが、短文でも勝てる。コンテンツ質 > 分量
```

## 6. scout-analyzer エージェント v0.2 反映候補

### 6.1 テンプレートライブラリ（7 パターン × Gem 化）

v0.1 では「高返信パターン / 低返信パターン」の概念定義のみ。v0.2 では実測から抽出した 7 パターンを Gem として明文化する。

| Gem ID | パターン名 | 推奨対象 | 期待返信率 |
|---|---|---|:-:|
| gem-01 | 担当者名乗り + 個別スキル引用型 | エンジニア全般 | 10-31% |
| gem-02 | 超短文カジュアル接触型 | アクティブ候補者 | 20-25% |
| gem-03 | プロフィール先行引用型 | セキュリティ / スペシャリスト | 8-16% |
| gem-04 | ハイレイヤー限定感・役職者接触型 | 事業開発 / CxO 候補 | 40-70% |
| gem-05 | 衛生要因強調型 | 一般事務 / ローレイヤー | 計測未確定 |
| gem-06 | 責任者自己紹介 + ミッション共有型 | 経営企画 / Biz 系 | 28-50% |
| gem-07 | 企業直接スカウト差別化型 | エンジニア（エージェント競合市場）| 未計測 |

### 6.2 導線フレーム（設計シートから抽出）

v0.1 には「導線ポイント」概念がなかった。実際の設計シートには以下の導線設計が存在していた。これを v0.2 のテンプレート選択ロジックに追加する。

```
標準導線 ──
タイトル（一覧クリック誘引）
→ 冒頭数行（一番重要なポイント 3 点）
→ 候補者の経験に魅力を感じる旨（個別化シグナル）
→ 自社 / 事業部紹介（1-2 段落）
→ 面接喚起（CTA）

ハイレイヤー向け変形 ──
タイトルに「役職者が会う」「一任」を入れる
→ 冒頭でビジネスチャンス感を出す
→ 応募ハードルを最大限下げる（フォーマット自由 / 業務委託 OK 等）
→ ペルソナの「現状の不満」に刺さるワード
```

### 6.3 採用ペルソナ入力フィールドの追加（v0.2 仕様変更）

v0.1 の入力設計には「採用ペルソナ」フィールドがなかった。実際の設計シートでは文面作成前にペルソナ定義（年齢帯・経歴・衛生要因・動機付け要因）を行っていた。

**v0.2 入力追加候補：**
```yaml
persona:
  age_range: "20代後半-30代前半"
  experience_background: "受発注業務経験者"
  hygiene_factors:
    - "リモート勤務可"
    - "安定した業界"
  motivation_factors:
    - "新規事業への参画"
    - "裁量の大きさ"
  frustration_trigger: "現職での成長天井感"  # ハイレイヤー向け
```

### 6.4 A/B テスト構造の自動設計

v0.1 では A/B テストは言及のみ。v0.2 では以下の自動設計ロジックを追加する。

```
テスト変数（1 回のテストにつき 1 変数を変える）:
  - タイトルバリアント（A: 汎用型 / B: 役職者接触型 / C: 待遇明示型）
  - 冒頭個別化レベル（A: なし / B: スキル転記 / C: 具体志向コメント付き）
  - 文面長（A: 短文 < 400 字 / B: 標準 400-1,000 字 / C: 詳細 1,000 字超）
  - 担当者属性（A: 採用担当 / B: 現場 EM / C: 事業部長）

最低計測サンプル: 送信 15 件以上で返信率を比較
```

### 6.5 ファネル全体追跡（v0.2 出力追加）

v0.1 の出力は「返信率」止まり。実際の運用では書類通過・1 次面接・最終・内定・決定まで全ファネルを追跡していた。

**v0.2 出力追加候補：**
```yaml
funnel_tracking:
  sent: 100
  replied: 20        # 返信率 20%
  doc_passed: 18     # 書類通過率 90%（返信者中）
  first_interview: 5  # 1 次面接率 28%
  final_interview: 2  # 最終面接率 40%
  offer: 1           # 内定率 50%
  accepted: 1        # 内定承諾率 100%
  # → 100 送信からの最終決定率: 1%
```

### 6.6 外部リンク管理ルール（NG-3 対応）

v0.1 にはリンク管理の規定なし。v0.2 では以下を明文化する。

```
許可: 求人詳細ページへの直接リンク（1 件のみ）
禁止: 外部メディア記事（会社紹介 / 担当者紹介記事 / SNS 等）
禁止: 複数リンクの掲載（離脱リスク増）
```

## 7. 機密情報マスキング処理ログ

| カテゴリ | 件数 | 正規表現パターン |
