# 勝ち筋ループワークショップ 設計

> **目的：** 秋吉ルール7「反復トビ」を最速で回す。有料で買ってくれるかのファクト検証
> **位置づけ：** セミナー（無料）とScan（10万）の間の有料中間ステップ
> **起点：** 秋吉「失敗しないビジネスアイデア7ルール」＋堀田「中間導線設計」

---

## 概要

| 項目 | 内容 |
|---|---|
| **タイトル** | 事業の勝ち筋ループを描く──社長の天井が90分で見えるワークショップ |
| **形式** | Zoom・90分・少人数（5名限定） |
| **価格** | 3万円（税込） |
| **頻度** | 月2回開催 |
| **ターゲット** | セミナー参加者＋SNS経由の経営者（社員5-50名） |
| **成果物** | 自社の勝ち筋ループ図＋ボトルネック特定＋石井の「翻訳」1つ |

---

## ファネル上の位置づけ

```
セミナー（無料・60分）：4象限で「自分はどこにいるか」を知る。聞くだけ
  ↓「もっと具体的に知りたい」
勝ち筋ループWS（3万・90分）：自社のループを描いて天井を特定する。描く ← ★NEW
  ↓「もっと精密に深掘りたい」
Scan（10万・60分）：1対1でレポート納品。診断する
  ↓「構造を変えたい」
Gravity（月35万・6ヶ月）：仕組みに変える
```

**価格グラデーション：** 無料 → 3万 → 10万 → 月35万

---

## セミナー・WS・Scanの違い

| | セミナー | **WS** | Scan |
|---|---|---|---|
| 形式 | 1対多・聞く | **少人数・描く** | 1対1・深掘り |
| 時間 | 60分 | **90分** | 60分 |
| 価格 | 無料 | **3万円** | 10万円 |
| メインフレーム | 4象限 | **勝ち筋ループ** | 建物4フロア |
| 成果物 | 自己診断の気づき | **勝ち筋ループ図＋翻訳1つ** | レポート（6セクション） |
| 石井の関与 | 講義 | **個別翻訳（1人5分）** | 60分フルヒアリング |

## ワークショップ構成（90分・5名限定）

### 0-10分：イントロ＋5パターン説明

> 「今日は、あなたの事業の勝ち筋ループを実際に描いてもらいます」
>
> 「事業モデルには5つのパターンがあります。あなたの事業はどれに近いですか？」

| # | パターン | 一言 |
|---|---|---|
| 1 | 単循環型 | 営業→納品→満足→紹介で回す |
| 2 | ストック型 | 顧客が積み上がる（SaaS/D2C/サブスク） |
| 3 | M&A型 | 既存深化＋資金調達＋買収で拡張 |
| 4 | PF型 | 面を取って回遊させる |
| 5 | エキスパート型 | 専門知識→発信→信頼→案件 |

> 「チャットにどれに近いか書いてください。複合型もOKです」

---

### 10-25分：勝ち筋ループを描く（参加者ワーク）

**ワークシートを画面共有：**

```
┌──────┐   ┌──────┐   ┌──────┐   ┌──────┐
│      │→  │      │→  │      │→  │      │
│要素1  │   │要素2  │   │要素3  │   │要素4  │
│      │   │      │   │      │   │      │
└──────┘   └──────┘   └──────┘   └──────┘
     ↑                                │
     └────────────────────────────────┘
```

> 「あなたの事業は、何がどう回ると売上が伸びますか？要素を書き出してください」
>
> 「例：営業→サービス提供→顧客満足→紹介→営業…」
>
> 「正解はありません。今の頭の中にあるものをそのまま書いてください。15分でお願いします」

石井は各参加者の画面を見ながら声かけ。描けない人にはパターン別のヒントを出す。

---

### 25-40分：担い手を書き込む（参加者ワーク）

> 「ループの各要素に、"今、誰が回しているか"を書いてください」
>
> 「選択肢：社長 / マネージャー / メンバー / AI / 外部 / 仕組み」
>
> 「**"社長"と書いた要素にマーカーを引いてください。**」

> （15分後）
>
> 「社長にマーカーを引いた要素がいくつありますか？チャットに数字を書いてください」

---

### 40-65分：石井の「翻訳タイム」（★核心・1人5分×5名）

**参加者1人ずつ、ループを画面共有。石井が即座に「翻訳」する。**

> 「Aさん、見せてください。…なるほど。営業が社長ですね。」
>
> 「Aさんが"営業力が足りない"とおっしゃっていましたが、私の見立ては違います。**足りないのは営業力ではなく、案件の見極め基準が言語化されていないこと**です。Aさんの頭の中にある基準が、他の人に移転できていない。ここが天井です」

**これが「翻訳」のライブデモ。** 参加者全員が他の人の翻訳も聞ける。「あ、うちもそうだ」が起きる。

| 参加者 | 言ったこと | 石井の翻訳 |
|---|---|---|
| A社長 | 「営業力が足りない」 | 「案件見極め基準が属人的」 |
| B社長 | 「人が辞める」 | 「評価基準が不透明」 |
| C社長 | 「任せられない」 | 「判断基準が社長の頭の中」 |
| D社長 | 「1on1やっても変わらない」 | 「施策が勝ち筋ループに接続されていない」 |
| E社長 | 「自分がいないと回らない」 | 「ループの2要素以上が社長依存」 |

---

### 65-80分：グループ議論

> 「他の方のループと翻訳を聞いて、気づいたことはありますか？」

ここで参加者同士の対話が生まれる。
- 「あ、うちも同じ構造だ」
- 「営業の属人性、まさにそれ」
- 「施策が接続されてないって言われてドキッとした」

**この場の価値＝「自分だけじゃない」という安心感＋「石井の翻訳は的確」という信頼。**

---

### 80-88分：ネクストステップ

> 「今日90分で、勝ち筋ループの概形とボトルネックは見えました」
>
> 「でも"具体的にどの力が足りなくて、何を仕組みに変えるか"は、今日の90分では深掘りしきれません」

| 選択肢 | 内容 | 対象 |
|---|---|---|
| A. 自走 | 今日のループ図をベースに、自分でボトルネックを解消 | WSの気づきで十分な方 |
| B. Scan | 1対1・60分で「建物の何階が空洞か」まで深掘り。レポート納品（10万） | もっと精密に知りたい方 |
| C. 個別相談 | Gravity（6ヶ月）が合うか、30分無料で話す | すぐ動きたい方 |

---

### 88-90分：クロージング

> 「今日覚えてほしいことを1つだけ。**勝ち筋ループを描いて"社長"と書いた要素が、あなたの事業の天井です。**」

## 収益シミュレーション

| 指標 | 数値 |
|---|---|
| 価格 | 3万円 |
| 開催頻度 | 月2回 |
| 定員 | 5名/回 |
| 月間参加者 | 最大10名 |
| **月間売上** | **最大30万円** |
| 石井の稼働 | 月3時間（90分×2回） |
| **実効時給** | **10万円/時間** |

**ウニ丼モデル的に見ても効率的。** 90分で15万円。石井の「翻訳」が最大の付加価値で、それ以外の時間（ワーク・議論）は参加者が主体。

## ルール7「反復トビ」の実行計画

| 回 | 目的 | 検証ポイント |
|---|---|---|
| 第1回 | ワークシートの使いやすさ | ループを描けない人がいるか。5パターンの説明でカバーできるか |
| 第2回 | 翻訳の精度 | 参加者の反応。「それだ！」が出るか |
| 第3回 | Scanへのコンバージョン | WS後にScan申込が出るか。出ないならなぜか |
| 第4回 | 価格感 | 3万で妥当か。「安い」と言われたら5万に上げてみる |
| 第5回〜 | 定常運営 | 月2回を安定的に回す。集客チャネルの最適化 |

## 秋吉7ルールとの適合度

| ルール | 適合度 | 根拠 |
|---|---|---|
| 1. 自分がペイン | ◎ | MOON-Xで「ループのBASEが追いつかない」を体験 |
| 2. 既存への不満 | ◎ | 既存WSは「座学で終わる」。石井のWSは「翻訳」がある |
| 3. お客さんの声 | ◎ | 宋さん「構造が見えたのは初めて」 |
| 4. 伸び市場×ペイン | ◎ | AI時代の組織変革 × 天井問題 |
| 5. BMから考えない | ◎ | ペインから入っている |
| 6. 心からやりたい | ◎ | want to＝「変革人材を生み出す」 |
| 7. ファクト検証 | **★このWSで回す** | 月10名に直接ぶつけて反復改善 |
