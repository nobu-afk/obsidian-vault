# Gravity モニター標準フロー SOP v1.0

> **作成：** 2026-05-08（金）夜
> **目的：** CODE / SCAN モニター実施時の標準運用フロー。260508 長谷さんモニター（CODE+SCAN 同日 90 分）で得た学びを SOP 化。
> **対象：** Gravity Code / Scan の体験モニター・有償セッション両方
> **想定改訂：** 5/15 長谷さん SCAN 本番後の学びを反映 → v1.1 へ

---

## 0. このドキュメントの位置づけ

長谷さん 260508 モニターで以下が判明：

1. **CODE 60 分 + SCAN 60 分 = 90 分** は、経営者側の認知負荷が高く、実質 60 分強で CODE のみで終わる傾向
2. **同日中の連続実施は、レポート生成・読み合わせの時間を圧迫**して、サービス全体への深いフィードバック取得が困難
3. **過去ペイロードの混入リスク**（生成 AI が過去事例の語彙を空想補完する事故）が同日連続実施では検証困難

→ **CODE と SCAN は別日に実施することを標準とする。**

## 3. レポート品質管理ハーネス（260508 学び）

過去ペイロードの語彙が AI レポートに混入する事故（260508 長谷さんモニターで「釣り」「地図」「採用ベース」等が混入）の再発防止。

### 3.1 Layer 1（SYSTEM プロンプト）：入力データ厳守ハーネス

CODE / SCAN 両 generate.php の SYSTEM プロンプトに以下が追加済（260508 夜）：

> **transcript / freetext / choices / pre_info に明記されていない情報を一切推測・補完・空想しない。**

### 3.2 Layer 2（PHP 後処理）：社内用語強制置換ハーネス

CODE / SCAN 両 generate.php の AI 出力後に、80+ NG 用語を対外語へ機械的に置換（jargon_map）。

### 3.3 セッション本番前のダミーペイロードテスト（必須プロトコル）

**毎セッション前日に：**

```bash
# 1. ダミーペイロードで SCAN 生成テスト
curl -X POST "https://growthfix.jp/gravity-scan/diagnose/generate.php" \
  -H "Content-Type: application/json" \
  --data-binary @"05_プロダクト/GravityScan/レポート/260508_長谷さんダミー_payload.json"

# 2. 生成完了後、レポートに過去事例の固有語が混入していないか grep
curl "https://growthfix.jp/gravity-scan/diagnose/generate.php?report=$JOB_ID" | \
  grep -oE "(釣り|地図|採用ベース|採用コイン|建築家|完成恐怖症|薬機法|獣医|動物|林さん|ペット)"

# 3. 残存ゼロを実証 → 本番セッション実施可
```

★ この事前テストを **必ず実施**。混入が検出されたら本番セッション前に generate.php を修正。

## 4. レポート読み合わせフェーズの 4 観点（260508 長谷さん学び）

レポート生成後の読み合わせフェーズ（CODE: T+48〜60 / SCAN: T+68〜85）で **必ず確認する 4 観点**：

1. **引力定義（理論背景）**：「引力とは何か」をレポートで明示しているか / 不在なら口頭補完
2. **経営インパクト**：レポート内容が経営判断にどう影響するか / 不在なら口頭補完
3. **次の一手の納得感**：推奨パッケージの根拠が経営者の言葉で説明できているか
4. **Final Question への沈黙時間**：30 秒以上の沈黙は経営者の覚悟確認の重要シグナル

## 5. フィードバック取得の標準項目（モニター向け）

### CODE モニター後（D+1〜D+7 で Slack / メール）

- レポートの引力タイプ判定の納得感（◎/◯/△/×）
- キャラ命名の刺さり度（◎/◯/△/×）
- 経営判断への活用見込み（具体的にどう使えそうか）
- 5 万円の価格感覚（安い / 妥当 / 高い）

### SCAN モニター後（D+1〜D+7 で Slack / メール）

- 4 型判定の納得感（◎/◯/△/×）
- 3 ヶ月の重点アクション（α/β/γ/δ）の優先度感覚
- 推奨パッケージへの意思決定（受講予定 / 検討中 / 見送り）
- 10 万円の価格感覚（安い / 妥当 / 高い）

### Gravity シリーズ全体への気づき（必ず取りに行く）

- **理論背景・エビデンスの透明性**は十分か（260508 長谷さんからの直接フィードバック反映）
- 全体動線（CODE → SCAN → Recruit/Cultivate/Shift）への評価
- 競合（識学・エンゲージメント系・採用系）との対比でどう見えるか
