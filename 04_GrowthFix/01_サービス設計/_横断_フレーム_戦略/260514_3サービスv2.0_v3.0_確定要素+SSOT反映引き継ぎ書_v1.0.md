### §6.2 推奨進行順

1. **両 Node 同時起動**：script-writer + lp-implementer がそれぞれ作業開始
2. **lp-implementer は LP 編集を先に完了**（lint 更新を待たずに本書を SSOT として参照）
3. **script-writer の lint 更新完了後**、lp-implementer が最終 lint 検証を実行
4. 両完了通知後、user が確認

### §6.3 衝突回避ルール

- script-writer は LP ファイルを編集しない
- lp-implementer は SSOT / lint ファイルを編集しない
- 両 Node が読むだけのファイル（叩き台 3 つ + 本書）は OK
