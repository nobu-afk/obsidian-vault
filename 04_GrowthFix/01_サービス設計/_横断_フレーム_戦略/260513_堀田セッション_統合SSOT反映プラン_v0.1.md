## 8. 反映時の安全装置

### 8.1 既存 SSOT との非破壊原則
- **Coaching 関連 SSOT には触れない**（思想層温存）
- **値上げ見送りスタンス温存**（260511 確定）
- **学術武装（253 件論文 / 7 本柱 14,897 cites）は維持**（C 案二重武装方針）
- **既存運用パターン（CLAUDE.md / lint / verify / monitor）厳守**

### 8.2 ハーネス層チェック
- 各 SSOT MD の Part 3 ハーネス層を反映前後で確認
- 各 LP 修正後に lint_consistency.sh + audit_mobile_sync.py + verify_deployment.sh を実行
- スクリプト追加時に `06_開発/scripts/_README.md` 更新

### 8.3 ロールバック装置
- 各反映は **git commit を 1 反映 = 1 commit**で粒度細かく
- 問題発生時は git revert で個別ロールバック可能
- 大規模変更は別ブランチで作業 → main マージ前にレビュー
