## 1. 3 軸統合フレーム（思想層）

### 1.1 引力経営の 3 軸構造

```
[Why = 引力経営の起点]
    経営者の引力 = Authentic Self × Emotional Contagion
    （CODE：個人引力解読）
            ↓
[How = 組織の引力設計]
    集まる軸 = 採用基盤の世界一
    躍動する軸 = AI × 人事データ × エンゲージメント設計の世界一
    留まる軸 = 継続装置の世界一
            ↓
[What = 事業中心人事の成果]
    経営イシューを解決する「優秀人材が躍動する会社」
```

**重要な構造判定：**
- 「事業中心人事（事業イシュー AI 実装）」は **独立 STEP ではなく 3 軸すべての中の高解像度層**。Shift C の躍動軸実装 + Orbit の留まる軸継続伴走に内包させる
- 「パルスサーベイ系」は **3 軸横断の測定基盤レイヤー**。留まる軸（主）+ 躍動軸（副）に統合
- 「AI 武装」は **3 軸 × AI の交差軸**。11 AI 武装エージェント仕様 v0.1（260513 仮説 E 採用）を 3 軸内に分散配置

### 1.2 3 引力対称理論（書籍『引力経営』2027 Q1 中核）

| 構造要素 | CODE（個人引力）| SCAN（組織引力）| Orbit（継続引力）|
|---|---|---|---|
| 本体概念 | Authentic Self | 集合的 Authentic Org | Embedded Self |
| 発現メカニズム | Emotional Contagion | Collective EC | **Continuous EC（時間軸）**|
| 時間軸 | 瞬間（解読・1 回）| 場（診断・1 回）| 時間（月次拍動・5 年装置）|
| Bandura 系列 | SE（1977）| CE（2000）| Sustained CE |
| 学術主軸 | Calling / Flow / Passion / SDT | ASA / Engagement / CE / 心理的安全性 | Mitchell JE / Rousseau / Meyer-Allen |
| 顧客の Pain | 経営者が「自分の引力」を言語化できない | 採用がうまくいかない・抜ける | 育てた人材が辞める・組織が形骸化 |
| 商品 | 5 万 | 10 万 | 月 15 万 × 12 = 180 万 |

**戦略的意義：** Phase 11 までの 2 引力（個人・組織）が「瞬間 × 場」止まりだったのに対し、Phase 12 で **時間軸の引力（継続）を独立第 3 引力として確立**。これで識学（時間軸概念なし）/ 外資コンサル（瞬間最適化偏重）/ サーベイ会社（場の測定のみ）への構造的差別化が決定的に成立。

### 1.3 軸間動線（顧客導入順序）

**標準フロー（260430 確定 → 3 軸版に更新）：**
```
CODE（前提）→ Scan（場の状態判定）→ Shift R（集まる実装）→ Shift C（躍動実装）→ Orbit（留まる継続）
```

**4 型診断 → 軸推奨ロジック（Scan 出力）：**
- 整合型 → Orbit 直接（既に集まる × 躍動が整っている → 留まる継続装置のみ）
- 拡散型 → Shift C → Orbit（躍動が散漫 → 焦点化 → 継続）
- 渇望型 → Shift R → Orbit（採用が足りない → 基盤構築 → 継続）
- 不毛型 → Shift Full → Orbit（3 軸すべて欠落 → 統合実装 → 継続）
