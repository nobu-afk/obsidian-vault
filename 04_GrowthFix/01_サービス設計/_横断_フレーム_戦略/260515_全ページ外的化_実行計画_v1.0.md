## 5. リスク管理

### 5.1 想定リスクと対策

| リスク | 対策 |
|---|---|
| Agent が「実装しない」を破って勝手に sed 実行 | brief で「Read / grep のみ」明記・後で grep 検証 |
| 平易化案が世界観コアを毀損する | user 承認ゲート必須・SSOT §1 残置語リスト共有 |
| 8 ドラフトの形式バラバラで集約困難 | 統一フォーマット強制・報告 800 字以内圧縮 |
| 実装段階で参照リンクが切れる | sed は文字列単位置換のため URL 不変・ただし confirm 推奨 |

### 5.2 ロールバック準備

- Layer A 開始前 git commit（auto-commit に依存しない明示 commit 推奨）
- Layer C 実装は **承認単位で 1 ファイルずつ**（複数 file 一括実装は避ける）
- 本番デプロイは Layer 4 ガード + verify_deployment.sh で保護済
