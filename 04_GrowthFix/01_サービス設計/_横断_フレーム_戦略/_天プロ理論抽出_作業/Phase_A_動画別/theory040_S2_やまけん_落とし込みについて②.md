## 講師流派タグ
**やまけん流（メソドロジー体系・主軸）**

## 主軸主張（1-3 行）
読書・インプットの落とし込みとは「要点を空で言えるようにし、抽象から自分の具体に接続する」こと。「本の中の抽象 ↔ 本の中の具体」と「本の抽象 ↔ 自分の現実の具体」の2つの接続が落とし込みの実践。センターピンに合わせたインプット絞り込みと、課題をとどめをさすまでやりきることが最重要。

## 体系番号（あれば）
**落とし込みシリーズ ②**（前回「学習の落とし込みの技術」の続き・実践編）

## 章立て・流れ
1. 前回（落とし込み①）の要点：センターピンに合わせたインプット / インプットゲームを制限 / 要点を空で言える / 具体と抽象を照らし合わせる
2. 題材確認：「儲けを目指す喜びの方程式」2章（28ページ）
3. 本の抽象の要点を取り出す：仕組み＝商品を売るためではなくお客さんを動かすため
4. さらに深掘り：喜びが仕組みの動力源
5. 本の中での「抽象 ↔ 具体（事例）」照合：看板・建築会社・トナカイ・代行事例
6. 自分の現実への落とし込み（抽象から具体）：目前の集客ライブで何を言えば意図する行動をとってもらえるか
7. 具体から具体への転用：安城建設の看板→自分の発信のポイントを抽出
8. アイドルタイム活用事例：有料サービスをアイドルタイムに限定無料化するタイミング転用

## 主要概念リスト
| 概念名 | 一言定義 |
|---|---|
| センターピンに合わせたインプット | 自分の今の最大課題に合わせた本・学習だけを選ぶ。それ以外は趣味として分離 |
| インプットゲームの制限 | 内容を完全理解し自分に落とし込んで実践形になるまで次のインプットをしない |
| 要点を空で言える | 本が手元になくても重要ポイントを思い返せる状態まで反復学習する |
| 抽象と具体の照らし合わせ | 本の抽象（原理・定義）と本の具体（事例）を接続し、さらに自分の現実の具体に変換する |
| 仕組みの定義 | お客さんを動かす仕組み。商品を売るための仕組みではない |
| 喜びが動力源 | 仕組みの中心は「お客さんを喜ばせる」こと。喜んでくれたお客さんが行動する |
| 意図する行動を取ってもらう | セールスの焦点は「売る」ではなく「お客さんに意図した行動（フォロー/購入/申込）を取ってもらう」こと |
| 抽象→自分の具体 | 本の原理から自分の明日の行動に変換する実践的落とし込み |
| 具体→具体（転用） | 他の事例の「ポイント」を抽出し自分の状況に当てはめる |
| アイドルタイム転用 | 暇な時間を使った無料提供で見込みの高いタイミングのお客さんとの接点を作る |

## 使用された例・事例
- **安城建設の看板**：堀内公園近くで「ターゲット（お父さんお母さんへ）→懸念払拭（入場料不要）→ベネフィット（お子様が楽しく遊べる家）」の3要素の看板。抽象原理→具体事例の照合
- **代行のアイドルタイム事例**：車検代行でアイドルタイムに1年無料提供→車を買い替えるタイミングで購買確率が高い見込み客に接触
- **集客ライブの設計**：6月25日セミナー申込か/メルマガ登録、という意図した行動から逆算して「何を言えば喜ぶか」を設計する実践
