## 講師流派タグ
**しん流（コンセプト・セールス設計体系）**

## 主軸主張（1-3 行）
コンセプトが機能するための 2 要件（自分には必要・これじゃないとダメ）を未解決ニーズ × 差別化ロジックで実現する。セールス（個別相談）の場は「越境の瞬間に立ち会うコーチング + 診断」という意味付けで行う。セールスへの抵抗を解消し、クロージングを美しく設計する体系。

## 体系番号（あれば）
「しん研究会（たまけん）」第 2 回。内的/外的コンセプト・未解決ニーズ・差別化ロジック・個別相談クロージング を扱う。次回はファネル体系（第 3 回）へ連続。

## 章立て・流れ
1. 前回の質問への回答（説明量と制約率のトレードオフ）
2. コンセプトの定義（外的コンセプト：なぜそれがお客さんに必要かを言語化したもの）
3. コンセプトの 2 要件：①これが必要（欲しいと思ってもらう）②これじゃないとダメ（他と違う）
4. 未解決ニーズの概念と発見方法（GDT の法則 / ライフホイール）
5. 差別化ロジック：根本原因の言語化 × 他との違いを区別（下げない）
6. セールスの意味付け（コーチング + 診断）
7. 個別相談のフロー実践（信頼構築→ヒアリング→フックポイント→ポジティブイメージ構築）
8. 参加者個別質問へのフィードバック

## 主要概念リスト
| 概念名 | 一言定義 |
|---|---|
| 外的コンセプト | なぜそれがお客さんに必要なのかを分かりやすく言語化したもの |
| 内的コンセプト | 自分の動機・人生コンセプトが反映されたもの（前回定義）|
| コンセプトの 2 要件 | ①これが自分に必要（欲しい）②これじゃないとダメ（他と違う） |
| 未解決ニーズ | 気づいているが解決を諦めてしまっているニーズ。潜在ニーズより見つけやすい |
| GDTの法則 | Goal / Desire / Teaser。これに沿ったコンセプトは売れやすい |
| ライフホイール | コーチング等で用いる人生の 8-10 軸のバランス確認ツール。未解決ニーズ発見に活用 |
| 差別化ロジック | 競合との違いを出す際に「根本原因の解消」まで言語化すると強力な差別化になる |
| 直接競合 vs 間接競合 | 直接＝同じサービス形態。間接＝同じ結果を別方法で得る競合（例：研修 vs 採用） |
| 区別（下げない） | 他社を下げずに「こういう人にはうちが合う」と区別する方法 |
| セールス = コーチング | 越境の瞬間に立ち会う役割。「行くぞ」と決めた瞬間が変化の起点 |
| セールス = 診断 | 根本的な問題を特定し、自分の手持ちの薬が合うかどうかをジャッジする場 |
| 越境の瞬間 | コンフォートゾーンを出て行きたい方向に踏み出す瞬間。セールスの本質 |

## 字幕引用（理論の核心部分・1-2 段落）
> コンセプトがまず満たしたい要件としては 2 つあって、これが自分には必要だって思ってもらう、欲しいと思ってもらうっていう要素と、これじゃないとダメ、他じゃなくてこれがいいって思ってもらうっていう 2 つの要素を満たしていると良い。
>
> セールスクロージングっていう場は越境の瞬間に立ち会えるかもしれない、めちゃくちゃえい瞬間に立ち合うっていう感じ。「行くぞ」って本当に決めた人はもうその瞬間に変わってるんで、その後変わっていくんじゃなくて決めた瞬間に変わっていく。だからそこがえいし、めちゃくちゃサービスの一部だと僕は思ってます。
