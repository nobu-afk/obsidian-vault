## Block A：引力の本質（Q1-Q5）

### Q1. 引力の3層定義：物理層・構造層・思想層

**問い：** 引力を3層で定義するとしたら、何か？

**選択肢：**
- **A. 物理層のみ**：引力＝人が集まる/熱量が上がる/辞めない等の観測可能な現象
- **B. 構造層主導**：引力＝経営者の存在から自然発生する"場の歪み"（物理層はその結果）
- **C. 3層統合**：物理（現象）× 構造（源泉）× 思想（価値）の全層セットでしか引力は定義できない
- **D. 「引力」概念自体が曖昧なので、別の言葉に置き換える**

**推奨：C（3層統合）**
理由：石井の核心使命「才能解放」＋人生テーマ「組織の求心力」を統合するには、単層定義では不十分。3層セットでの定義が Gravity 全体（CODE/Blueprint/Scan/Shift/Coaching）を貫く軸になる。

**判断軸：** 定義の実用性＞簡潔性。レポート・LP・Note連載に全て使える定義になっているか。

---

### Q2. 引力の源泉：1つか、複数か

**問い：** 引力の源泉は「1つに収束する」のか、「複数要素の組み合わせ」か？

**選択肢：**
- **A. 1つに収束**：引力の源泉は「経営者の本音（Whyの一貫性）」で、他は全て派生
- **B. 2軸の組み合わせ**：源泉＝「本音の強度」×「本音と行動の一貫性」
- **C. 3要素の組み合わせ**：本音（Why）× 表現（動詞・行動）× 環境（文脈・場）の3要素が揃って初めて引力になる
- **D. 4要素以上**：もっと多層

**推奨：C（3要素組み合わせ）**
理由：Why単体だと「本音はあるが空回り」状態を説明できない。行動単体だと「動いているが空虚」を説明できない。環境単体だと「場によって引力変わる」問題が説明できない。3要素の相互作用で引力が生まれる。LP の「動詞・エンジン・環境条件」と自然に接続する。

**判断軸：** 経営者の実感と一致するか（「うちの引力、何が源泉だろう？」に答えられるか）。

---

### Q3. 引力の測定/可視化：何を見れば引力が分かるか

**問い：** 引力を測定・可視化するとしたら、何を指標にするか？

**選択肢：**
- **A. 定量指標のみ**：採用応募数・離職率・顧客継続率・紹介発生率等、数値で測る
- **B. 定性指標のみ**：会議の熱量・雑談の多さ・自発的提案の頻度・社長不在時の自走度
- **C. 定量×定性の両輪**：定量は結果指標・定性はプロセス指標。両方見ないと引力の実態は分からない
- **D. 測定自体が罠**：引力は測ろうとした瞬間に消える（量子的観測問題）

**推奨：B（定性指標のみ）**
理由：定量は結果であって引力そのものではない（引力がない企業でも給与で数字は作れる）。**引力の本質は「なぜ人が離れないか」「なぜ自発的に動くか」の**プロセス**にある。定性指標を敢えて前面に出すことで、CODE の世界観（解剖・翻訳）と整合する。ただし、ScanではCEO+幹部診断でこの定性をどう"見える化"するかが鍵。

**判断軸：** 経営者が「うちは引力ある/ない」を自己判定できる指標になっているか。

### Q4. 「引力」と類似語の使い分け

**問い：** 以下の類似語と「引力」をどう使い分けるか？

**類似語候補：** 重力 / 求心力 / 磁力 / 吸引力 / オーラ / カリスマ / 魅力 / 影響力 / リーダーシップ / 影響圏 / エネルギー場

**選択肢：**
- **A. 引力一本で統一**：他は使わない。Gravity の独自語彙として完全に切り分ける
- **B. 引力＋重力の2軸**：引力＝人を惹きつける力、重力＝現状を縛りつける力（李師匠の「重力を外す」との差別化）
- **C. 引力（=Gravity全体）の下に：求心力（組織視点）・動詞（個人発動）・エンジン（駆動源）・環境条件（発火場）の4子概念を置く**
- **D. 完全に新語を作る**：「引力」すら捨てて独自概念を立ち上げる

**推奨：B（引力＋重力の2軸）**
理由：李師匠の「重力を外す認知科学者」と石井の「組織引力を作る翻訳者」のポジション差を概念で明示する。**重力＝縛る（Do less）、引力＝引き寄せる（Pull more）**。経営者の課題は「重力を外して引力を立てる」と2段階で説明できる。求心力・オーラ等は引力の下位表現として位置付ける。

**判断軸：** Note連載・LP・コピーで言葉がブレずに使えるか。李師匠との差別化が明確か。

### Q5. 引力の思想的価値：なぜ5万円？

**問い：** 経営者が CODE に5万円払う理由は何か？

**選択肢：**
- **A. 自己理解の深化**：「自分の引力を知る」こと自体が価値
- **B. 経営判断の精度向上**：自分の引力を知ると、事業判断・人材配置・組織設計の精度が上がる（実利）
- **C. 事業の天井突破**：経営者の引力が組織の引力上限を規定する。天井を外すには引力の解剖が必須
- **D. 才能解放の起点**：才能ある経営者が社会的期待に埋もれている状態を自覚する「目覚めの1冊」

**推奨：C＋D統合**
理由：C（天井突破）は経営者の合理性（ROI）に訴求。D（才能解放）は石井の核心使命と接続。**「あなたの事業の天井は、あなたの引力の天井。それを解剖する」**がCODEのキャッチコピーになる。A（自己理解）は表層訴求で弱い。B（経営判断）は範囲が広く曖昧。

**判断軸：** LPの hero コピー・CTA直前の説得・レポートの冒頭3行がこの価値と整合するか。

## Block B：引力タイプの構成（Q6-Q11）

### Q6. 引力タイプの定義：何の「型」か

**問い：** 「引力タイプ」という言葉で、結局何を型化しているのか？

**選択肢：**
- **A. 引力の発動パターン**：動詞3連鎖＝引力がどう発動するかの癖
- **B. 引力の構造**：源泉（Why）・動作（動詞）・環境（場）の3要素の組み合わせパターン
- **C. 引力の質感**：熱量型／静音型／鋭角型／包容型 等、引力の肌感の分類
- **D. 引力の機能**：人を集める／人を離さない／人を育てる／人を超える の機能別分類

**推奨：B（引力の構造）**
理由：Q2で採用した「本音×表現×環境」の3要素組み合わせと整合。**引力タイプ＝この3要素の固有パターン**と定義すれば、LP の「動詞・エンジン・環境条件」が自然に収まる。A/C/D は部分的で、「引力とは何か」の答えにならない。

**判断軸：** 「引力タイプ = ○○」が1行で答えられるか。

### Q7. 動詞3連鎖の位置付け：引力の"何"か

**問い：** 動詞3連鎖（例：見抜く→翻訳する→手放す）は、引力の中で何を表しているか？

**選択肢：**
- **A. 引力そのもの**：動詞＝引力の本体
- **B. 引力の発動動作**：引力が場に働きかけるときの動き
- **C. 引力の表層**：目に見える部分。下に源泉（Why）と環境（場）がある
- **D. 引力の発動動作＋源泉の痕跡**：動詞の連鎖そのものに源泉の構造が埋め込まれている

**推奨：D（発動動作＋源泉の痕跡）**
理由：動詞は単なる行動癖ではなく、**その動詞を選び続ける背後に源泉（Why）がある**。例：石井の「翻訳する」は「才能解放」という源泉から発動されている動詞。動詞だけ取り出しても意味は半分しか伝わらない。動詞3連鎖は**source（根拠エピソード）とセット**で初めて引力タイプを構成する。

**判断軸：** verb-map の「source（根拠エピソード）」の役割がここで腹落ちするか。

### Q8. エンジン・環境条件との関係

**問い：** 動詞3連鎖と、エンジン（8型）、環境条件（発火/停止）の3要素の関係は？

**選択肢：**
- **A. 独立した3要素**：それぞれ別の側面を見ている
- **B. 階層構造**：エンジン（源泉）→ 動詞（発動）→ 環境条件（文脈）の順で引力が具現化
- **C. 三角形関係**：エンジンは引力の"駆動源"、動詞は"表現形式"、環境条件は"発火条件"。三者の整合性で引力の強度が決まる
- **D. 動詞だけで十分**：エンジン・環境条件は診断項目として冗長。動詞3連鎖に統合する

**推奨：C（三角形関係）**
理由：引力＝エンジン×動詞×環境 の積で強度が決まる。**整合＝引力強、不整合＝引力漏れ**。例：石井のエンジン「才能解放」×動詞「翻訳する」×環境「1対1の深い対話」は整合している。一方、現状「エンジン：翻訳×動詞：翻訳×環境：セールス場」という不整合が引力漏れを生んでいる。

**判断軸：** レポートでこの三角形の整合/不整合を可視化できるか。

### Q9. キャラ命名の意味：引力の"何"を名付けるか

**問い：** 「○○な○○」のキャラ命名は、引力の何を名付けているか？

**選択肢：**
- **A. 経営者の自己像**：「あなたはこういう人」の要約ラベル
- **B. 引力タイプの別名**：引力の構造（エンジン×動詞×環境）の総合表現
- **C. 引力の矛盾の可視化**：経営者の引力に内在する緊張構造を名前として固定する
- **D. 名付け不要**：動詞3連鎖があれば十分

**推奨：C（引力の矛盾の可視化）**
理由：「完成恐怖症の翻訳者」「逃げ足の速い建築家」等の良例は、**矛盾する2要素の緊張**を名前にしている。これが「え、そういうことだったのか」の衝撃を生む。**引力＝矛盾の緊張から生まれるエネルギー**という理解と整合。A/B は要約的で弱い。

**判断軸：** 読者が名前を聞いた瞬間に「うっ」と来るか（批判覚悟の堀田フレーム）。

### Q10. 8型マトリクスの再位置付け

**問い：** 8型マトリクス（一点突破型／専門特化型／精密設計型等）はどう扱うか？

**選択肢：**
- **A. 現状維持**：8型分類を前面に出す
- **B. 内部参照のみ**：AIの分析用に使うが、レポートには型名＋1行しか出さない
- **C. 廃止**：8型は汎用的すぎる。キャラ命名（Q9）で代替
- **D. 逆転発想**：8型ではなく、**「引力の強度」×「引力の方向」**の2軸マトリクスに置き換える

**推奨：B→D移行**
理由：v4 時点では B（内部参照のみ・既に実装済み）。思想が固まったら D（引力の強度×方向マトリクス）への置き換えを検討。**強度＝人を惹きつける力の大きさ、方向＝何に向けて引力が発動しているか**。8型よりも引力概念と整合し、経営者の実感と合う。ただし思想確定前は B を維持。

**判断軸：** 8型の型名を経営者が自分で覚えるか／「うちは○○型だ」と言えるか。

### Q11. 引力の源泉と死角の構造

**問い：** 引力の源泉と死角は、対称構造か非対称か？

**選択肢：**
- **A. 対称**：引力の源泉があれば必ず裏返しの死角がある（エネルギー保存則的）
- **B. 非対称**：源泉は単数、死角は複数（1つの源泉に複数の死角が付随）
- **C. 共振構造**：源泉と死角は同じ構造から生まれる2面性。分離不可
- **D. 源泉＝死角**：引力を強めている要素が、同時に人を離す要素にもなっている

**推奨：D（源泉＝死角）**
理由：例：石井の源泉「相手の未言語化を翻訳する」は、同時に「翻訳後に執着しない→人が定着しない」という死角でもある。**源泉と死角は同じコインの表裏**。これが「引力の矛盾」（Q9のキャラ命名）と接続する。A/B は分離しすぎで、経営者の実感と合わない。

**判断軸：** レポートで源泉と死角が同じ構造から生まれていると示せるか。

## Block C：動線とBlueprint/Shift接続（Q12-Q16）

### Q12. 個人引力→組織引力の伝播論理

**問い：** 個人（経営者）の引力が、組織の引力に変わるメカニズムは何か？

**選択肢：**
- **A. 模倣**：経営者の引力を部下が真似することで組織に拡がる
- **B. 共鳴**：経営者の引力に共鳴する人が集まり、似た引力を発する集団になる
- **C. 翻訳**：経営者の引力を幹部が組織の言語に翻訳することで、組織に実装される
- **D. 浸透**：引力は組織設計・制度・文化に埋め込まれて時間をかけて浸透する

**推奨：C（翻訳）**
理由：石井の肩書き「社長の翻訳者」と直結。**経営者の引力は、翻訳されないと組織には届かない**。この翻訳役が Blueprint（個人の引力構造化）→ Scan（組織への翻訳）→ Shift（組織実装）のサービスチェーンの論理的必然性を作る。A/B/D は現象的で、サービス動線と接続しない。

**判断軸：** CODE→Blueprint/Shift の動線理由として経営者が納得できるか。

### Q13. CODE→Blueprint/Shift の必然性

**問い：** CODE を受けた経営者が Blueprint/Shift に進む「必然性」はどこにあるか？

**選択肢：**
- **A. 自己理解→組織改革**：自分が分かったら次は組織（順序論）
- **B. 個人引力の限界**：経営者個人の引力だけでは組織の引力上限が決まる。その天井を突破するには組織引力の設計が必要
- **C. 翻訳の不完全性**：経営者は自分の引力を自分では翻訳しきれない。幹部視点を入れないと見えない（Scan の必然性）
- **D. B+C統合**：個人の天井＋翻訳の盲点の2つの理由

**推奨：D（B+C統合）**
理由：B単体だと「組織改革したい人だけが行く」になり、全員動線にならない。C単体だと「翻訳が不十分だから」で弱い。**両方の理由が揃って初めて「CODE→Blueprint→Scan→Shift」の階段が論理的必然になる**。レポートの path-card に両方の理由を書くことで、経営者の次行動が定まる。

**判断軸：** レポートを読み終えた経営者が「次は Blueprint だな」と自然に感じるか。

### Q14. レポートで「次」を促す方法

**問い：** レポート内で次のサービス（Blueprint/Coaching）に誘導する技法は何が最強か？

**選択肢：**
- **A. CTAボタン**：最後に「Coaching 30分無料」で誘導
- **B. 盲点の指摘**：「このレポートで見つけた矛盾には、もう1層深い理由がある。60分では辿り着けなかった」（オチを出さない）
- **C. 動線の論理構造**：レポート内で引力の構造を示し、「個人→組織への翻訳は別サービス」と位置付ける
- **D. B+C統合**：盲点（感情）と論理構造（思考）の両輪

**推奨：D（B+C統合）**
理由：感情だけだと一時的衝動、論理だけだと冷静に止まる。**感情（盲点）→思考（論理）→行動（CTA）の順で3層設計**すれば CTR が上がる。現状の v4 は B は入っているが C が弱い。

**判断軸：** レポート読後に「次に何をするか」が明確か／曖昧か。

### Q15. path-cards の説得構造

**問い：** path-cards（Coaching/Blueprint/セルフリファイン）の3択は、どう構造化すべきか？

**選択肢：**
- **A. 3択フラット**：現状通り、3つを並べて経営者が選ぶ
- **B. Coaching推奨のみ**：他2択は削除または1行だけ
- **C. 段階別提示**：「まずCoachingで自分を掘り下げる→次にBlueprintで組織に翻訳」の順序を明示
- **D. 判断フレーム付き**：「have toが外せない人はCoaching／事業に伸びしろを感じる人はBlueprint／自分で調整できる人はセルフ」と判断軸を先に提示

**推奨：D（判断フレーム付き）**
理由：3択の理由が経営者の自己判断と紐づく。**「私は have to が外せない側だ」と自認した瞬間に Coaching 申込が確定する**。前処理主義（RECODE 373行）の応用。A は選択負荷が高い、B は選択肢なし、C は順序強要で押しつけ感。

**判断軸：** 経営者が10秒で「自分はどれか」を自己判定できるか。

### Q16. CODE単体完結 vs 動線前提のスタンス

**問い：** CODE は単体で完結するサービスか、Blueprint/Shift への入り口か？

**選択肢：**
- **A. 単体完結**：CODE だけで経営者の引力理解は完了。Blueprint 以降は別サービス
- **B. 入り口専用**：CODE は Blueprint/Shift への導入ファネル。単体価値は最小化
- **C. 両立**：単体でも価値があり、かつ上位サービスへの論理橋もある（両輪）
- **D. 時限完結**：CODE は3ヶ月で効果が薄れる。継続には上位サービスが必要

**推奨：C（両立）**
理由：B にすると5万円の価格根拠が崩れる。A にすると上位サービスの受注が減る。**単体で完結する価値を持ちつつ、上位サービスへの必然性も内在させる**のが最適。具体的には「CODE＝個人引力の可視化、Blueprint＝それを組織言語に翻訳、Shift＝組織実装」の役割分担を明確にする。

**判断軸：** 経営者が「CODE だけでもいい」「でも Blueprint もやりたい」の両方を感じるか。

## 統合された引力の定義（単一情報源）

> **引力とは、経営者の本音（Why）×動詞×環境の3要素の整合から生まれ、人が離れない・自発的に動く等の定性的現象として観測される「場の力」である。**
>
> - **3層定義**：物理層（現象）× 構造層（源泉）× 思想層（価値）
> - **源泉構造**：Why×動詞×環境の三角形整合性（整合=引力強、不整合=引力漏れ）
> - **測定**：定性指標（会議の熱量・自発的提案・社長不在時の自走度等）
> - **類似概念**：重力（外す対象／李師匠）↔ 引力（立てる対象／石井）
> - **伝播メカニズム**：翻訳（経営者→幹部→組織→現場）
> - **価値命題**：事業の天井は経営者の引力の天井。社会的期待（＝重力）を外すことで才能（＝本来の引力）が立ち上がる
