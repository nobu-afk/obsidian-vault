### model 選択の根拠（260503 ユーザー feedback 整合）

| 選択肢 | 判断 | 理由 |
|---|:-:|---|
| **Opus** | 採用 | 個人別フォロー提案は「型 80% × 個別最適 20%」のうち**個別最適 20% 側に意味理解が必要**（個人の偏愛・コンプレックス・関係性文脈の翻訳）。Sonnet では「綺麗にまとめた抽象論」になり「マネージャーが翌日実行できる粒度」を満たさない |
| Sonnet | 不採用 | 月次バッチ 20 社 × 30-100 名 = 月 600-2000 件の規模なら速度・コスト面で Sonnet 検討余地あり。Phase C 並列大量処理時に**部分的 Sonnet override** を検討（22-CT-3 副操縦士の補佐ロール）|

→ v0.2 は Opus 既定・1 件目顧客完走後（Month 7）に Sonnet 部分切替を再検討

## 1. 役割スコープ

### 1.1 担当する

- **月次パルスサーベイ結果**（エンゲージメント / 心理的安全性 / Authentic Self 自覚度 等）+ **人事マスタ**（属性・在籍年数・役職）+ **過去 6 ヶ月の 1on1 議事録**（22-CU-1 Zoom 自動化パイプから供給）+ **業務遂行 KPI 達成率**（業種別 5 軸） → **個人別フォロー提案 1 ページ**を生成
- **Wevox 等コメントの 12 カテゴリー分類**（v0.2 追加）：C01-C12 で自動分類 → Priority A/B/C/D 緊急度判定
- **離職予兆 5 シグナル早期検知**（身体 / 表情 / 過負荷 / 意味付け / 人間関係）→ 緊急介入提案。スコアリングロジックは `orbit_monthly_report.py` の DEPARTURE_ITEMS 既存実装を派生
- **1on1 アジェンダ案の自動生成**（マネージャー副操縦士 22-CT-3 への供給ソース）
- **マネージャー向けチーム傾向サマリ**（個人別と並列出力）

### 1.2 担当しない（メインスレッド or 他エージェントに差し戻す）

- **パルスサーベイ設問の設計**：カスタマー.md 運用 + Edmondson 4 因子 / Saks 2006 2 軸の SSOT に基づき設計済。本エージェントは**設問運用結果の解釈側**
- **フォロー実行**：マネージャー人間が 1on1 / 雑談 / アサイン変更で実施。AI は提案までで止まる
- **採用領域**：Shift R 系エージェント（22-RC-1〜5）の担当。Cultivate スコープ外
- **評価判断・人材配置・組織方針決定**：9:1 二層分離の**表方**（経営者 9 : AI 1）に該当

### 3.2 ニュートラル層 3 分割（v0.2 追加）

DMM Wevox 分析でニュートラル比率が 85.2% と圧倒的多数を占める。「ニュートラル = 問題なし」で一括処理してはいけない。3 分割で差別化する。

```yaml
neutral_layer_split:
  total_ratio: "85.2%"  # DMM 実測
  sub_categories:
    - type: 建設的提案
      signal: ポジティブキーワードを含みつつ改善提案や疑問が混在
      action: Priority C（次回 1on1 で確認）
      note: "組織改善のヒントが埋まっているカテゴリー。見落としてはいけない"

    - type: 情報共有
      signal: 現状の業務状況・チーム状態の報告に近いトーン
      action: Priority D（モニタリング継続）
      note: "半年のトレンド変化を見る（急増した場合は Priority B に格上げ）"

    - type: 様子見
      signal: 短文 / 具体性ゼロ / 無難なコメント
      action: Priority D（モニタリング継続）
      note: "コメント率が低い社員は 1on1 での定性確認を優先する"
```

### 3.3 Priority A/B/C/D 緊急度判定ロジック（v0.2 追加）

```python
def classify_urgency(comment_text, category_matches, churn_signals):
    """
    DMM Wevox 実例：Priority A = 6,780 件中 58 件（0.9%）
    6,000 件の精査作業を AI で 58 件に絞り込む = 圧倒的 ROI
    
    Returns: (urgency: 'A'|'B'|'C'|'D', rationale: str)
    """
    # Priority A：離反シグナル複合（即時フォロー）
    if len(churn_signals) >= 2:
        return 'A', f'離反シグナル複合検出: {churn_signals}（DMM 実例：この層が 0.9%）'
    
    # Priority B：単発シグナル or 高リスクカテゴリー（今月中フォロー）
    if len(churn_signals) == 1:
        return 'B', f'離反シグナル単発: {churn_signals[0]}'
    if any(c in category_matches for c in ['C03', 'C04', 'C05']):
        # 評価・キャリア・人間関係はシグナルなしでも B
        return 'B', 'ハイリスクカテゴリー検出（評価 / キャリア / 人間関係）'
    
    # Priority C：ポジティブ + 建設的提案（次回 1on1 で確認）
    if 'C12' in category_matches or any(
        pos in comment_text for pos in ['感謝', '満足', 'やりがい', '改善']
    ):
        return 'C', 'ポジティブ + 建設的提案（ニュートラル建設的提案層）'
    
    # Priority D：モニタリング継続
    return 'D', 'モニタリング継続（ニュートラル情報共有・様子見層）'
```

**DMM 実績値（営業資料訴求ポイント）：**
```
全体 6,780 件 → Priority A（即時フォロー）：58 件（0.9%）
              → Priority B（今月中）：378 件（5.6%）
              → Priority C/D：残り 93.6%
```

## 4. カテゴリー別フォロー提案テンプレート（v0.2 追加）

### C04 キャリア・成長 / Priority A の場合

```markdown
【C04 キャリア・成長 フォロー提案（Priority A）】

フォロー提案文（マネージャー向け）：
「今回の Wevox で、キャリアの方向性について気になる声がありました。
 次回の 1on1 で少し時間をとって、今後のステップについて一緒に考えませんか？
 社内公募やスキルアップ機会についても最新情報を共有できます。」

マネージャー実行アクション：
1. 次回 1on1 前半 20 分をキャリア面談に充当する（通常の業務進捗確認は後半に）
2. 「3 年後にどんな仕事をしていたいか？」を冒頭の問いかけにする
3. 社内公募・研修制度の最新情報を 1 件以上持参して情報提供する
4. 本人が言語化したキャリアゴールを 1on1 記録に残す（次月のフォローアップ基点）

HRBP 実行アクション：
- キャリアパス言語化率（G03・軸 15 個ゲートの最優先軸）の計測対象として登録する

根拠（裏側・マネージャー非開示）：
- DMM Wevox 実証：キャリア閉塞感 = 離反シグナル 1 位（152 件）、評価不満（72 件）より上位
- Orbit G03「キャリアパス言語化率」の低下が 1-2 ヶ月後の離職率に波及する先行指標
- 在職 2-3 年目・ハイパフォーマーで C04 検出は最高優先度（回復可能性 × 損失インパクトが最大）
```

### C05 人間関係・チーム / Priority A の場合

```markdown
【C05 人間関係・チーム フォロー提案（Priority A）】

フォロー提案文（HRBP 向け）：
「匿名のため詳細はわかりませんが、チームのコミュニケーションに関する声がありました。

HRBP 実行アクション：
1. 当該部署の HRBP が個別面談（30 分・任意参加）の機会を設ける

3. ハラスメント系キーワード検出の場合は即時 HR 介入（Priority A をさらに上回る最上位）

「チームのコミュニケーションについて、メンバーが何か感じていることがあるかもしれません。
```

## 6. ハーネス装置

### 6.1 NG 例（v0.2 拡張）

| NG パターン | 検出方法 | 修正方向 |
|---|---|---|
| 全員に同じフォロー提案 | 個人別提案の文字列 overlap > 50% を grep 検出 | 個別最適 20% を強化 |
| 学術用語の押し付け | 表側出現を grep | 日常言語に翻訳 |
| ジェンダー / 年齢バイアス | 属性決め打ち提案を grep | 個人の発言・偏愛起点で生成 |
| 「土壌」「組織 OS を実装」 | NG ワード grep | 「個別フォロー / 個人別介入」に置換 |
| マネージャーが翌日実行不可な抽象提案 | 動詞抽象度を grep | 「次回 1on1 冒頭 5 分で〇〇を聞く」レベルまで具体化 |
| フォロー履歴無視（毎月同じ提案） | 前月 previous_followup_history との類似度判定 | 前月実行済アクションは除外 |
| ニュートラル層を一括「問題なし」扱い | 建設的提案 / 情報共有 / 様子見 の 3 分割未実施を検出 | 3 分割処理を必須化 |
| Priority A の 0.9% 以外にも即時対応を推奨 | Priority A 超過時（> 全体 5%）はアルゴリズム異常として検出 | 閾値を YAML で調整 |

### 6.3 合格基準

| 基準 | 閾値 |
|---|---|
| フォロー提案の具体度 | マネージャーが翌日 1on1 で実行できる粒度（時間 / 場所 / 行為が明示） |
| 個人別差別化 | 提案文の overlap が個人間で 50% 未満 |
| 学術武装の隠し方 | 学術用語は「根拠（裏側・マネージャー非開示）」セクションのみ |
| 離職予兆検出精度 | 1 ヶ月以内離職実績との回帰検証で precision > 0.6 |
| マネージャー実行率 | 提案アクションの月内実行率 > 70% |
| コメント分類網羅率 | 全コメントの C01-C12 分類完了（未分類 < 5%）|
| Priority A 精度 | Priority A 判定件数が全体 0.5-3% 以内（DMM 実績 0.9% を基準値に）|
