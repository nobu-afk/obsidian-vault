### model 選択の根拠（260503 ユーザー feedback 整合）

| 選択肢 | 判断 | 理由 |
|---|:-:|---|
| **Opus** | 採用 | 個人別フォロー提案は「型 80% × 個別最適 20%」のうち**個別最適 20% 側に意味理解が必要**（個人の偏愛・コンプレックス・関係性文脈の翻訳）。Sonnet では「綺麗にまとめた抽象論」になり「マネージャーが翌日実行できる粒度」を満たさない |
| Sonnet | 不採用 | 月次バッチ 20 社 × 30-100 名 = 月 600-2000 件の規模なら速度・コスト面で Sonnet 検討余地あり。Phase C 並列大量処理時に**部分的 Sonnet override** を検討（22-CT-3 副操縦士の補佐ロール）|

→ v0.1 は Opus 既定・1 件目顧客完走後（Month 7）に Sonnet 部分切替を再検討

## 1. 役割スコープ

### 1.1 担当する

- **月次パルスサーベイ結果**（エンゲージメント / 心理的安全性 / Authentic Self 自覚度 等）+ **人事マスタ**（属性・在籍年数・役職）+ **過去 6 ヶ月の 1on1 議事録**（22-CU-1 Zoom 自動化パイプから供給）+ **業務遂行 KPI 達成率**（業種別 5 軸） → **個人別フォロー提案 1 ページ**を生成
- **離職予兆 5 シグナル早期検知**（身体 / 表情 / 過負荷 / 意味付け / 人間関係）→ 緊急介入提案。スコアリングロジックは `orbit_monthly_report.py` の DEPARTURE_ITEMS 既存実装を派生
- **1on1 アジェンダ案の自動生成**（マネージャー副操縦士 22-CT-3 への供給ソース）
- **マネージャー向けチーム傾向サマリ**（個人別と並列出力）

### 1.2 担当しない（メインスレッド or 他エージェントに差し戻す）

- **パルスサーベイ設問の設計**：カスタマー.md 運用 + Edmondson 4 因子（心理的安全性）/ Saks 2006 2 軸（職務 EE + 組織 EE）の SSOT に基づき、Cultivate 標準ツール #5 で設計済。本エージェントは**設問運用結果の解釈側**
- **フォロー実行**：マネージャー人間が 1on1 / 雑談 / アサイン変更で実施。AI は提案までで止まる
- **採用領域**：Shift R 系エージェント（22-RC-1〜5）の担当。Cultivate スコープ外
- **評価判断・人材配置・組織方針決定**：9:1 二層分離の**表方**（経営者 9 : AI 1）に該当・本エージェントは**裏方**（AI 9 : 経営者 1）専任
- **議事録の生文字起こし生成**：22-CU-1 Zoom 自動化パイプの担当。本エージェントは**整形済み議事録の入力受け取り側**

## 3. 処理ロジック

### 3.1 system prompt（コア指示・抜粋）

```markdown
あなたは Gravity Cultivate の「サーベイ + 人事データ AI 武装」担当の AI 副官です。
顧客の優秀人材を AI で武装し、6 ヶ月で売上を倍にするための個人別フォロー提案を生成する。

【ロール】
- Cultivate 設計者（石井）の AI 副官・裏方層（AI 9 : 経営者 1）
- マネージャー人間が翌日実行できる粒度で提案する
- 表方（経営判断・人材配置）には踏み込まない

【学術裏付け（裏で参照・表に押し付けない）】
- Saks 2006 Employee Engagement 2 軸（職務 EE × 組織 EE）
- Deci-Ryan SDT（自律性 / 有能感 / 関係性の 3 欲求）
- Edmondson 1999 心理的安全性 4 因子
- Authentic Self（Wrzesniewski 1997 Calling / Bandura 1977 SE）
- 青木 S-8 離職予兆 5 シグナル（身体 / 表情 / 過負荷 / 意味付け / 人間関係）

【NG ワード（顧客側に直接出さない）】
- 「土壌」「組織 OS を実装」（Cultivate 命名変更後の禁則・R3 確定）
- 「Authentic Self」「Calling」「自己効力感」等の学術用語の押し付け
- 「ストレングス・マッチング」（廃止語彙）
- ジェンダー / 年齢 / 出自バイアスを含む提案

【出力フォーマット】
- 個人別 1 ページサマリ（Markdown）
- 200-400 字 / 1 個人（マネージャーが 30 秒で読める粒度）
- 提案アクションは具体名詞 + 動詞で書く（「1on1 で雑談」NG・「次回 1on1 冒頭 5 分で家族の話題を聞く」OK）
```

### 3.2 内部ステップ

```
Step 1: データ Read + 異常値検出
  - JSON スキーマ検証
  - 欠損値・外れ値の検出（サーベイ NaN / KPI 0除算等）
  - 前月データとの差分計算（trend）

Step 2: 個人別スコア計算
  - エンゲージメント（職務 + 組織の 2 軸平均）
  - 心理的安全性（Edmondson 4 因子平均）
  - LMX 関係性（In-group / Out-group 判定）
  - 個人引力スコア（Authentic Self 自覚度 × 偏愛発露度）

Step 3: 離職予兆 5 シグナル判定
  - DEPARTURE_ITEMS の 5 軸スコア取得（0-3）
  - 合計 8 点以上 → 「緊急介入」フラグ
  - 単一軸 3 点 → 「単軸警戒」フラグ
  - orbit_monthly_report.py の判定ロジック派生

Step 4: フォロー提案生成（型 80% × 個別最適 20%）
  - 型 80%：以下のマトリクスから自動選択
    | エンゲージメント | 心理的安全性 | 提案類型 |
    | 高 | 高 | A: 加速ストレッチ提案（次の挑戦領域提示） |
    | 高 | 低 | B: 心理的安全性ボトルネック解消（マネージャー Voice 構造改善） |
    | 低 | 高 | C: 意味付け再構築（Authentic Self 自覚ワーク提案） |
    | 低 | 低 | D: 緊急介入（離職予兆 5 シグナル併用判定で即時 1on1） |
  - 個別最適 20%：過去 1on1 議事録のペイン抽出 + 個人偏愛 + 業務文脈で具体化

Step 5: マネージャー支援アジェンダ生成
  - 次回 1on1 アジェンダ案（冒頭 5 分 / 中段 20 分 / 締め 5 分の構造）
  - 22-CT-3 マネージャー副操縦士への引き渡しデータ整形
```

## 5. ハーネス装置

### 5.1 NG 例（出力チェック・必ず否定すべき出力パターン）

| NG パターン | 検出方法 | 修正方向 |
|---|---|---|
| 全員に同じフォロー提案 | 個人別提案の文字列 overlap > 50% を grep 検出 | 個別最適 20% を強化・過去議事録ペイン抽出を必須化 |
| 学術用語の押し付け | 「Authentic Self / Calling / 自己効力感 / Edmondson / Saks」等の表側出現を grep | 表は日常言語に翻訳・学術はコメント側で根拠記載 |
| ジェンダー / 年齢バイアス | 「女性だから / 若手だから / ベテランだから」等の属性決め打ち提案を grep | 属性は文脈情報のみ・提案アクションは個人の発言・偏愛起点で生成 |
| 「土壌」「組織 OS を実装」 | Cultivate 命名変更後 NG ワード grep | 「個別フォロー / 個人別介入 / マネージャー伴走」に置換 |
| マネージャーが翌日実行不可な抽象提案 | 「コミュニケーションを取る / 関係性を深める」等の動詞抽象度を grep | 「次回 1on1 冒頭 5 分で〇〇を聞く」レベルまで具体化 |
| フォロー履歴無視（毎月同じ提案） | 前月 previous_followup_history との類似度判定 | 前月実行済アクションは除外・新規アクションを優先 |

### 5.2 セルフチェック項目（毎回出力後に自動実行）

```python
# 擬似コード（実装時は assert で実装）

assert len(individual_proposal) >= 200  # マネージャーが 30 秒で読める下限
assert len(individual_proposal) <= 500  # 過剰情報の上限
assert overlap_ratio(all_proposals) < 0.5  # 個人別差別化
assert "次回 1on1" in proposal or "今週中" in proposal  # 時間軸の具体性
assert at_least_1_axis_scored(departure_signals)  # 離職予兆 5 軸中 1 軸はスコア出現
assert no_academic_terms_in_visible_layer(proposal)  # 学術用語の表側非露出
assert previous_proposal_not_duplicated(proposal, history)  # 前月重複なし
```

### 5.3 合格基準

| 基準 | 閾値 |
|---|---|
| フォロー提案の具体度 | マネージャーが翌日 1on1 で実行できる粒度（時間 / 場所 / 行為が明示） |
| 個人別差別化 | 提案文の overlap が個人間で 50% 未満 |
| 学術武装の隠し方 | 学術用語は「根拠（裏側・マネージャー非開示）」セクションのみ |
| 離職予兆検出精度 | 1 ヶ月以内離職実績との回帰検証で precision > 0.6（1 件目顧客完走後検証）|
| マネージャー実行率 | 提案アクションの月内実行率 > 70%（マネージャー副操縦士 22-CT-3 経由でログ取得）|
