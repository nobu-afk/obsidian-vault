# 第 3 部：意思決定のための論点整理

## 3.1 基本的な意思決定軸

### 軸 1：独自性 vs 学術整合性
- 独自フレーム維持 → Gravity の唯一性最大化
- 学術整合性最大化 → 識学差別化最強・WhitePaper / 書籍の権威性最大化

### 軸 2：移行コスト vs 理論的純度
- 既存温存 → サービス・LP・診断ツール改修ゼロ
- 学術完全整合 → 全面改修必要（数日〜数週間の作業）

### 軸 3：石井さんの 50 社診断知見との整合性
- 既存 3 要素 / 4 型は 50 社診断の現場知見から構築
- 学術論文との整合は事後確認・本質は現場知見

## 3.2 推奨判断のための問い

### CODE 判断問い
1. 「Why × 才能 × 偏愛」の 3 要素は石井さん 50 社診断で**実証済**か？それとも**理論的構成**か？
2. 各要素を学術用語に置換すると、診断時の経営者の理解が**深まる**か**遠ざかる**か？
3. 既存 3 要素を温存して「補強引用」として学術裏付けするのと、学術完全整合で再構築するのと、**どちらが多くの人を救うか**？

### SCAN 判断問い
1. 4 型分類（整合 / 拡散 / 渇望 / 不毛）は石井さん 50 社診断で**実証済**か？
2. 既存ラベル（整合 / 拡散 / 渇望 / 不毛）vs 学術ラベル（Job 高 × Org 高 等）、**経営者にどちらが届くか**？
3. 4 型維持で軸定義を多層的にする（🅰 案）vs Saks Engagement 2 軸で完全置換（🅳 案）、**どちらが Gravity の本質か**？

## 3.3 段階的移行のシナリオ

### シナリオ X：完全独立路線（🅰 案）
- CODE / SCAN ともに既存維持 + 学術補強引用追加
- 移行コスト：ほぼゼロ
- 成果：本日の Phase 9-10 反映で完了済 = 追加作業なし

### シナリオ Y：完全学術整合路線（🅱/🅳/🅴 案組合せ）
- CODE → 🅱（Bandura SE 4 源泉）or 🅳（3 軸統合）
- SCAN → 🅴（3 層階層 Bandura CE）
- 移行コスト：数週間（LP / 診断ツール / 商談資料 / WhitePaper / SSOT 全件改修）
- 成果：学術整合性最大・WhitePaper / 書籍の権威性最強・識学差別化最強
