# 事業の勝ち筋ループ：パターン類型化

> **目的：** 「事業の勝ち筋ループ」を体系化する。単純な循環ではなく、事業モデルごとにループの構造が異なる。パターンを類型化し、経営者が自社のループを描けるツールにする
> **起点：** MOON-Xの「CORE→MORE→BASE」モデル＋50社の実務経験
> **ステータス：** 議論・構築中

---

## 課題：現状の勝ち筋ループは単純すぎる

現在のフレーム：
```
[顧客獲得] → [価値提供] → [顧客満足] → [拡大再生産] → ...
```

これはSaaS的な「売る→使う→続ける→広がる」のモデルしか表現できない。実際の事業はもっと複雑：

- **M&A型**（MOON-X）：COREの深化＋資金調達＋買収＋PMI＋基盤統合
- **SB型（宋さん）**：既存オペレーション維持＋営業で新規案件獲得＋将来M&A
- **受託型**：案件獲得→デリバリー→実績→リファラル
- **SaaS型**：マーケ→IS→FS→CS→拡張→MRR成長→マーケ投資
- **プラットフォーム型**：供給側拡大→需要側拡大→ネットワーク効果
- **コンテンツ型**：作る→集客→マネタイズ→再投資

## MOON-Xモデルの勝ち筋ループ（実例分析）

石井さんの実体験から描くと：

```
┌──────────────────────────────────────────────┐
│                                              │
│  ┌─────┐     ┌──────────┐     ┌─────┐       │
│  │CORE │ ──→ │ファイナンス│ ──→ │ MORE │      │
│  │既存の│     │資金調達   │     │買収・│      │
│  │売上  │     │（実績が   │     │チャネル│     │
│  │深化  │     │ 根拠）   │     │拡大  │      │
│  └──┬──┘     └──────────┘     └──┬──┘       │
│     │                              │         │
│     │         ┌──────┐             │         │
│     └────────→│ BASE │←────────────┘         │
│               │基盤  │                        │
│               │人材・│                        │
│               │ノウハウ│                       │
│               │集約  │                        │
│               └──────┘                        │
│                                              │
│  ★ MA候補の発掘・デューデリジェンス             │
│    （常時並行で走るサブループ）                  │
│                                              │
└──────────────────────────────────────────────┘
```

**このモデルの勝ち筋の「力」：**

| ループ要素 | 必要な「力」 | MOON-Xでの担い手 |
|---|---|---|
| CORE深化 | ブランドマーケ力・EC運用力 | 各ブランドチーム |
| ファイナンス | 資金調達力・IR力 | CEO＋CFO |
| MORE（買収） | M&A目利き力・交渉力 | CEO＋BD |
| MORE（チャネル拡大） | 新チャネル開拓力 | 各ブランド＋本部 |
| BASE | PMI力・バックオフィス統合力 | 人事・経理・法務 |
| MA候補発掘 | ネットワーク・案件ソーシング | CEO＋BD |

**MOON-Xの天井はどこだったか：**
- COREの深化は各ブランドチームで回っていた
- しかしBASE（PMI・人材統合）が買収スピードに追いつかなかった
- 石井さんが入ったのはまさにこのBASEの構築

## 勝ち筋ループのパターン類型（仮説）

50社の経験から、事業の勝ち筋ループは大きく**5つのパターン**に分類できるのではないか。

### パターン1：単循環型（最もシンプル）

```
[獲得] → [提供] → [満足] → [再生産] → [獲得]...
```

**該当する事業：** 受託コンサル、小売、飲食、個人サービス
**特徴：** ループが1つ。社長がループの全要素を回していることが多い（❶❷の典型）
**天井：** 社長の24時間 ＝ ループの回転数の上限
**宋さんはここ：** 営業→オペレーション→顧客満足→紹介の単循環

### パターン2：ストック型（SaaS・D2C・サブスク）

```
[認知・集客] → [獲得] → [初期体験] → [継続/リピート] → [拡張/LTV向上]
                                                              ↓
                                                        [収益→再投資]
                                                              ↓
                                                         [認知・集客]...
```

**該当する事業：** SaaS、D2C、サブスク、会員制サービス
**共通の本質：** 顧客が積み上がるモデル。1回の獲得が長期的な収益を生む
**パターン内の分類：**

| サブタイプ | 収益モデル | 勝ち筋の核 |
|---|---|---|
| SaaS | MRR（月額継続） | チャーン防止＋アップセル |
| D2C | LTV（リピート購入） | 商品力＋ブランド×マーケ |
| サブスク（メディア等） | 会費（継続） | コンテンツの質＋体験 |
| 会員制サービス | 月額・年額 | コミュニティ価値 |

**特徴：** 獲得→継続の各段階が独立した機能。分業が前提（マーケ/IS/FS/CS等）
**天井：** ファネルのどこかがボトルネック。SaaSならCS（解約防止）、D2Cならリピート率に社長が介入しがち
**GrowthFixもここに近い：** セミナー→Scan→Gravity→Orbitの積み上げ

### パターン3：CORE＋MORE型（M&A・多角化）

```
CORE（既存深化）──→ 実績 ──→ ファイナンス ──→ MORE（買収・拡張）
       ↑                                           │
       └──── BASE（基盤統合・人材・ノウハウ） ←─────┘
```

**該当する事業：** M&A型成長企業、多角化経営、ホールディングス
**特徴：** 複数のループが並走。COREとMOREで必要な「力」が全く異なる
**天井：** BASEの構築が買収スピードに追いつかない。PMIの失敗
**MOON-Xがここ。** SBもここを志向している

### パターン4：プラットフォーム型（DMMモデル）

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│   ┌───────────────────────────────┐                 │
│   │   PF事業領域（収益の中核）     │                 │
│   │                               │                 │
│   │  認知事業        収益事業       │     その他領域   │
│   │  （赤字OK）      （黒字必須）   │    （非PF/toB） │
│   │  ↓会員獲得       ↑課金        │    ニッチトップ  │
│   │     └→ PF回遊 →┘            │    を狙う       │
│   │                               │                 │
│   │  クラスタ拡大 + 新規拡張       │                 │
│   │  → 面を取って回遊させる       │                 │
│   └───────────────────────────────┘                 │
│                    ↑                                 │
│               BASE（基盤）                           │
│         会員基盤・決済・データ・技術                    │
│                                                     │
└─────────────────────────────────────────────────────┘

勝ち筋ループ：
認知事業で会員獲得 → PF内回遊 → 収益事業で課金
  → 収益でクラスタ拡大（面の拡張）→ 回遊が増える → さらに課金
  → 余剰キャッシュで「面白い領域」に進出 → ニッチトップ
```

**該当する事業：** プラットフォーム、マーケットプレイス、総合サービス
**特徴：**
- 認知事業（赤字OK）と収益事業（黒字必須）の役割分担がある
- 「面を取る→回遊させる」が勝ち筋の核
- PF内にコンテンツ制作・販売・金融等の異なるビジネスモデルが混在
- 「なんでもやってる」に見えるが、PF回遊という1つのロジックで繋がっている
**天井：**
- クラスタが拡大しても回遊しない（事業間の導線が設計されていない）
- 認知事業の赤字が膨らみすぎて収益事業でカバーできない
- 事業が増えすぎてBASE（基盤）の統合が追いつかない
**DMMがここ：** 50事業のHRBPを統括した石井の実体験。各事業の勝ち筋が異なるのに、PF全体としての勝ち筋ループで統合する必要があった

### パターン5：エキスパート型（個人の専門性×レバレッジ）

```
[専門知識の発信] → [信頼構築] → [案件獲得] → [デリバリー] → [実績・知見の蓄積]
                                                                    ↓
                                                              [発信に還元]
                                                                    ↓
                                                               [信頼構築]...

レバレッジの拡張：
  講座化 / 書籍化 / コミュニティ化 / パートナー委譲（ウニ丼モデル）
```

**該当する事業：** コンサル、コーチ、士業、教育、情報発信、専門家ビジネス
**特徴：**
- 社長の専門知識そのものが事業の源泉
- 発信→信頼→案件の循環が基本ループ
- スケールするにはレバレッジ（講座化・パートナー委譲）が必須
- ウニ丼モデル（自分の時間を圧縮し、低コスト労働でパッケージ）がスケールの鍵
**天井：**
- 社長の専門知識＝事業の天井（最も❶に陥りやすいパターン）
- デリバリーが属人的で、社長の稼働時間＝売上の上限
- レバレッジに移行できないと永遠に時間を売り続ける
**堀田さんのモデルがここ。GrowthFixもここからスタート**

## パターンごとの「社長依存しやすい要素」と処方箋

| パターン | 社長依存しやすい要素 | 天井の構造 | 処方箋 |
|---|---|---|---|
| 1. 単循環型 | 全て（特に営業） | 社長の24時間＝ループの回転数上限 | 営業プロセスの標準化。「上から採る」 |
| 2. ストック型 | FS（成約）とCS（解約防止） | ファネルの特定段階がボトルネック | 各段階の採用基準＋KPI設計 |
| 3. CORE＋MORE型 | MA候補発掘＋ファイナンス | BASEが買収スピードに追いつかない | BASE（PMI・人材統合）の先行構築 |
| 4. プラットフォーム型 | 新クラスタ拡張の意思決定＋事業間導線設計 | 面は取れたが回遊しない。事業間の接続が属人的 | 回遊を促す導線の仕組み化。認知/収益の役割分担の明文化 |
| 5. エキスパート型 | 専門知識の発信＋デリバリー | 社長の稼働時間＝売上の上限 | ウニ丼モデル（パートナー委譲）＋講座化・仕組み化 |

## Scanでの使い方（案）

ヒアリングの冒頭で：

> 「あなたの事業は、この5つのどれに近いですか？」
>
> 1. **単循環型**：営業→納品→満足→紹介の循環で回す
> 2. **ストック型**：マーケ→商談→成約→継続→拡張のファネルで回す
> 3. **M&A型**：既存事業の深化＋資金調達＋買収で拡張
> 4. **プラットフォーム型**：面を取って回遊させる。認知事業と収益事業がある
> 5. **エキスパート型**：社長の専門知識→発信→信頼→案件→実績の循環
>
> → パターンに応じて「社長依存しやすい要素」を重点的にヒアリング

### パターン×4象限の組み合わせ

| パターン | ❶に多い | ❷に多い | ❸に多い | ❹に行ける条件 |
|---|---|---|---|---|
| 1. 単循環型 | ◎ 最多 | ○ | △ | 営業の仕組み化 |
| 2. ストック型 | ○ | ◎ | ○ | ファネル各段階の分業 |
| 3. M&A型 | △ | ◎ | ○ | BASEの先行構築 |
| 4. PF型 | △ | ○ | ◎（面はあるが回遊しない） | 回遊の仕組み化 |
| 5. エキスパート型 | ◎ | ○ | △ | ウニ丼モデル |

### GrowthFixのターゲット分布（仮説）

Scanに来る経営者の分布予測：
- パターン1（単循環型）：**50%**（最多。5-30人の成長企業の大半）
- パターン5（エキスパート型）：**20%**（コンサル・士業・コーチ）
- パターン2（ストック型）：**15%**（SaaS・サブスク）
- パターン3（M&A型）：**10%**（多角化経営者）
- パターン4（PF型）：**5%**（稀。大手向け）

→ **パターン1と5の勝ち筋ループ設計が最も需要が高い**

## 議論メモ

### 単純ループでは不十分だった理由
- MOON-Xの場合、[獲得]→[提供]→[満足]→[再生産]では事業の本質が表現できない
- 実際は「COREの深化」「資金調達」「M&A」「BASE統合」が並行して走る
- ループが1つではなく複数あり、それぞれに異なる「力」が必要

### パターン分類のメリット
- 経営者が「自社はパターンXだ」と即座に認識できる
- パターンごとに「社長依存しやすい要素」が予測できる → ヒアリングの精度が上がる
- セミナーで「5パターンのうちどれですか？」と聞ける → 参加者の自分事化

### 検討が必要な点
- 5パターンで網羅できるか？他にないか？
- 事業フェーズによってパターンが変わる会社の扱い

### 複合型について（4/2追記）

実際の事業は「純粋な1パターン」ではなく、複数パターンの組み合わせであることが多い。

**ミナジンの例（石井の1社目）：**
- 勤怠SaaS → パターン2（ストック型）
- 給与計算BPO → パターン1（単循環型）単体販売あり
- コンサルティング → パターン1（単循環型）
- SaaS＋BPOのクロスセル → パターン1＋2のハイブリッド

→ 1社の中にパターン1と2が混在。クロスセル構造で繋がっている

**宋さんの例：**
- 既存事業 → パターン1（単循環型）
- 将来構想 → パターン3（M&A型）に移行したい
- 現状は「パターン1のまま、パターン3を夢見ている」状態

**Scanでの扱い方：**
- 「あなたの事業はどのパターンに近いですか？」は入口の質問
- 実際には「メインのパターンは何で、サブのパターンが混在しているか」を特定する
- **メインパターンの勝ち筋ループを先に描き、サブパターンとの接続点を見る**

### 石井の5パターン全網羅（4/2確認）

| パターン | 石井の実体験 | 立場 | 具体的な経験 |
|---|---|---|---|
| 1. 単循環型 | ミナジン コンサル＋BPO | 事業立ち上げ＋執行役員 | 受託→納品→紹介の循環を0→1で構築 |
| 2. ストック型 | ミナジン 勤怠SaaS | 事業部長 | マーケ→商談→成約→継続のファネル＋BPOクロスセル |
| 1+2 ハイブリッド | ミナジン SaaS×BPO | 事業部長 | 単体販売＋クロスセルの両方を経験 |
| 3. M&A型 | MOON-X | 1人目人事責任者 | CORE深化＋買収＋BASE統合を人事側で設計 |
| 4. PF型 | DMM | 50事業HRBP統括 | PF回遊＋認知/収益分離＋面の拡張を横断で見た |
| 5. エキスパート型 | GrowthFix | 創業者 | 専門知識→発信→案件→ウニ丼モデルを実践中 |

→ **5パターン＋ハイブリッドの全てを「中の人」として経験。これが石井の最大の独自性**
→ セミナーで「5パターンどれですか？」と聞いたとき、どのパターンの経営者が来ても石井の実体験で語れる
