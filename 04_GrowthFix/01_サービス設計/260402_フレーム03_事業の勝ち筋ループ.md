# フレーム03：事業の勝ち筋ループ

> **位置づけ：** 建物4フロアで「3F↔4Fの接続が空洞」と判明した場合、または❸「やったのに変わらない」の根本原因を特定するフレーム
> **導線：** 4象限 → 建物4フロア → **勝ち筋ループ**（事業起点で組織を設計する）
> **ソース：** ビジおたch vol.239 石井の発言「事業の勝ち筋ループが見えてないと基本採用しちゃダメ」＋50社の実務経験

---

## 核心の主張

> **「事業の勝ち筋ループが見えていない状態で人を採ると、人は増えるが事業は伸びない」**

ほとんどの経営者は「良い人を採れば事業が伸びる」と信じている（4象限の❷）。

しかし「良い人」の定義は**事業の勝ち筋ループ**から逆算しないと決まらない。ループが見えていない状態で人を採ると、何をさせるか不明確→採用ミス→人件費だけ増える→社長が楽にならない。

---

## フレームワーク

### STEP 1：勝ち筋ループを描く

あなたの事業は、何がどう回ると売上が伸びるのか。

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│          │    │          │    │          │    │          │
│ 顧客獲得  │ →  │ 価値提供  │ →  │ 顧客満足  │ →  │ 拡大再生産 │
│          │    │          │    │          │    │          │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
      ↑                                              │
      └──────────────────────────────────────────────┘
```

これは最もシンプルな形。実際の事業ではもっと複雑になる。

**例：SaaS企業の勝ち筋ループ**
```
マーケ（認知）→ インサイドセールス（商談化）→ フィールドセールス（成約）
    → オンボーディング（立ち上げ）→ CS（継続）→ アップセル/紹介
        → MRR成長 → マーケ投資拡大 → ...
```

**例：宋さん（資源循環事業）の勝ち筋ループ**
```
営業（新規案件獲得）→ 現場オペレーション（サービス提供）
    → 顧客満足（継続契約）→ 紹介/口コミ → 営業...
```
★ 「営業」の部分が宋さん1人。ここがループのボトルネック。

---

### STEP 2：各ループ要素に必要な「力」を定義する

ループの各要素を回すために、どんな「力」（ケイパビリティ）が必要か。

| ループ要素 | 必要な「力」 | 現在の担い手 | 理想の担い手 |
|---|---|---|---|
| 顧客獲得 | 営業力・マーケ力 | 社長1人 | 営業チーム＋仕組み |
| 価値提供 | オペレーション力 | リーダー＋メンバー | 仕組み化された現場 |
| 顧客満足 | CS力・品質管理 | 属人的対応 | 基準化されたCS |
| 拡大再生産 | 紹介の仕組み・マーケ | なし（偶発的） | リファラルの設計 |

**「現在の担い手」が社長や特定個人に集中している要素＝ループのボトルネック**

---

### STEP 3：ボトルネックを特定する

ループの中で**最も詰まっている要素**を1つ特定する。

```
     [顧客獲得] ← ★ここが社長1人。ボトルネック
         ↓
     [価値提供] ← 現場は回っている
         ↓
     [顧客満足] ← 品質は安定
         ↓
     [拡大再生産] ← 紹介はたまに来る程度
         ↓
     [顧客獲得] ← ★戻ってきた。やっぱりここが詰まっている
```

**ボトルネックの見つけ方：**
- 「この要素を2倍にしたら、事業全体が2倍になるか？」→ Yesなら、そこがボトルネック
- 「この要素が止まったら、ループ全体が止まるか？」→ Yesなら、そこがボトルネック
- 「この要素は社長（または特定の1人）に依存しているか？」→ Yesなら、属人ボトルネック

---

### STEP 4：ボトルネックに対して「力」を配置する

ボトルネックが特定されたら、そこに**人×AI×仕組み**を配置する。

| 手段 | やること | 例 |
|---|---|---|
| **人** | ボトルネックの「力」を持つ人を採用/配置 | 営業マネージャーを採用 |
| **AI** | ボトルネックの一部をAIで代替/効率化 | リード獲得をAI＋マーケオートメーションで |
| **仕組み** | ボトルネックの属人性を排除する設計 | 営業プロセスを標準化。誰でも再現可能に |

**ここで初めて「何の力を持つ人を、どこに配置するか」が決まる。**

これが「事業から逆算した採用」の正体。❹の会社がやっていることはこれ。

---

## なぜ❸「やったのに変わらない」が起きるのか

❸の会社は、勝ち筋ループを描かずに施策を入れている。

```
【❸の構造】

勝ち筋ループ（4F）：  ？？？（描けていない）
                         ↑
                    接続されていない
                         ↑
HR施策（3F）：     1on1 / サーベイ / 評価制度 / AI
```

1on1をやっている。でも**何のためにやっているかが事業ループに接続されていない。**

- 1on1の目的が「部下のケア」→ ループのどこに効くか不明
- サーベイを取っている → データは増えるがループは回らない
- 評価制度を作った → 事業KPIとの接続がない

**勝ち筋ループを先に描き、ループのボトルネックに施策を接続する。** これが❸→❹への唯一の道。

```
【❹の構造】

勝ち筋ループ（4F）：  [獲得]→[提供]→[満足]→[再生産]
                         ↑        ↑        ↑
                    接続されている
                         ↑        ↑        ↑
HR施策（3F）：     営業採用基準  OJT設計   CS評価基準
                   リード獲得AI  品質基準   紹介制度
```

## 4象限 × 勝ち筋ループ

| 4象限 | 勝ち筋ループの状態 | 処方箋 |
|---|---|---|
| **❶** | ループの全要素を社長が1人で回している | まずループを描いて、最大のボトルネックを1つ特定 |
| **❷** | 人を増やしたがループのどこに配置すべきか不明確 | ループを描き直し、各要素に必要な「力」を定義。「上から採る」 |
| **❸** | ループを描かずに施策を入れている。3F↔4F未接続 | **勝ち筋ループを描いてから施策を接続し直す** |
| **❹** | ループが描かれ、各要素に人×AI×仕組みが配置されている | Orbitでループの最適化を継続 |

## 宋さんで検証

```
宋さんの勝ち筋ループ：

  [営業：新規案件獲得] → [現場：サービス提供] → [顧客：継続契約] → [紹介]
        ↑                      ↑
     ★宋さん1人            リーダー＋メンバー
     ボトルネック！          （回っている）

ボトルネック：営業。宋さんが全部やっている。
リーダーは「営業の取り方がブラックボックス」と感じている。

処方箋：
1. 宋さんの営業プロセスを言語化する（仕組み化）
2. リーダーに営業スキルを移転する（人の配置転換）
3. → これでループの営業要素が宋さん依存から脱却
4. → 宋さんはM&A（次のステージ）に集中できる
```

**宋さん自身が「営業に興味あるって言ってくれた。教えやすくて助かる」と言っていた** → ボトルネックの解消が自然に始まっている。Scanがこの構造を可視化した。

## セミナー・対談での使い方

### 飲み会で（30秒）
「事業が伸びるかどうかって、ループなんですよ。顧客獲得→サービス提供→満足→拡大再生産。このループのどこが詰まってるかで打ち手が全然違う。人を採る前に、まずループのどこがボトルネックか特定した方がいい」

### セミナーで（4象限＋建物の後に10分）
1. 「事業の伸ばし方は4つしかない」（4象限）
2. 「あなたの会社は何階が空洞か」（建物4フロア）
3. 「では、あなたの事業は何がどう回ると売上が伸びますか？」（勝ち筋ループ）
4. 「このループのどこが社長に依存していますか？」
5. 「そこがボトルネック。そこに人×AI×仕組みを配置するのが❹への道」

### 対談で
「高野さん、投資先のスタートアップで、勝ち筋ループが見えないまま採用してる会社ってどのくらいありますか？」

### Scanでの活用
ヒアリングの冒頭で「あなたの事業は、何がどう回ると売上が伸びますか？」と聞く。
→ これが描けない経営者は❶❷❸のどれか。
→ 描けた上で「どこが社長依存か」を特定する。

## 石井である必然性

1. **「勝ち筋ループが見えてないと採用しちゃダメ」は石井のオリジナル発言**（ビジおたch）
2. **50社で勝ち筋ループのパターンを見てきた**。SaaS、物流、資源循環、教育、HR…業種横断のパターン認識
3. **DMMで60事業のHRBP**。各事業の勝ち筋ループが全く異なることを横断的に経験
4. **ムーンX（M&A型）**。買収した事業ごとに勝ち筋ループが違い、それぞれに合わせた組織設計が必要だった
5. **宋さんのScanモニターで実証済み**。ループのボトルネック（営業の属人性）が60分で特定された

## 3フレームの全体像

```
【診断フロー】

STEP 1：4象限
「事業の伸ばし方は4つしかない。あなたはどこにいるか？」
  → ❶❷❸❹のどこかを特定

STEP 2：建物4フロア
「あなたの会社は何階が空洞か？」
  → 1F（経営チーム）/ 2F（組織図）/ 3F（HR仕組み）/ 4F（事業）のどこが空洞かを特定

STEP 3：勝ち筋ループ（3F↔4Fの接続が問題の場合）
「あなたの事業は何がどう回ると売上が伸びるか？ そのどこが社長に依存しているか？」
  → ループのボトルネックを特定し、人×AI×仕組みの配置を設計

【サービス接続】

STEP 1-2 = Gravity Scan（60分・10万円）
STEP 3 = Gravity（6ヶ月・月35万円）
継続 = Gravity Orbit（月15万〜）
```
