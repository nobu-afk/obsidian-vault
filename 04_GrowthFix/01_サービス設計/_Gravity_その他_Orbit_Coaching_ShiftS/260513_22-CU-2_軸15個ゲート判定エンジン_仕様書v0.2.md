## 4. Wevox コメント → ゲート判定統合フロー（v0.2 追加）

22-CT-1 個別フォロー提案エンジンとの連動設計。

```
月次コメント分析 → ゲート判定フロー：

INPUT:  当月 Wevox コメント全件（22-CT-1 から分類済 JSON を受け取る）
STEP1:  カテゴリー分類結果受け取り（C01-C12・Priority A/B/C/D）
STEP2:  離反シグナル（S1-S5）検出 × 留まる軸（G_R1〜G_R5）照合
  S1 業務量過多 → G_R4（1on1 実施率）警告フラグ
  S2 評価不満 → G_R1（優秀人材離職率）警告フラグ
  S3 キャリア閉塞 → G_R3（キャリアパス言語化率）【最優先】警告フラグ
  S4 人間関係悪化 → G_R4（ドラムビート定着率）警告フラグ
  S5 組織不信 → G_E5（個人引力スコア）警告フラグ
STEP3:  15 軸ゲート照合（G_A1〜G_A5 / G_E1〜G_E5 / G_R1〜G_R5）
STEP4:  ゲート別スコア更新（コメント検出数 → スコア換算）
STEP5:  月次レポート生成（Priority A 件数の ROI 訴求含む）
```

## 5. 処理ロジック（v0.1 踏襲 + v0.2 拡張分のみ記載）

### 5.1 system prompt 追加（v0.2 拡張部分）

```
【v0.2 追加ルール：留まる軸の最優先処理】

G_R3（キャリアパス言語化率）は全 15 軸の中で最優先軸として扱う。

- Priority A が 1 件以上存在する軸（G_R1〜G_R3）には自動的に「warning」フラグ追加
- 月次レポートに「Priority A 件数 / 全体の 0.9% 基準値との比較」を必ず掲載

【v0.2 追加：営業資料訴求ポイントの自動生成】

月次レポートに以下のセクションを自動追加：
「今月の Wevox 精査効率化：XX 件 → Priority A XX 件（全体 X%）に絞込。
 従来の手作業精査 XX 時間 → AI 処理 XX 分（削減率 XX%）」
```

| ⑪ | 留まる軸（G_R1〜G_R5）を集まる軸・躍動軸と同等の優先度で処理する | v0.2 の設計哲学「留まる軸 = 定着基盤 = 5 年継続装置の核心」に反する | group_summary で留まる軸の danger 件数 >= 2 なのに warnings が 0 件の場合エラー |
| ⑫ | 22-CT-1 から Priority A 件数を受け取っていても wevox_integration フィールドを省略 | ROI 訴求の根拠データが欠落 | wevox_integration が null の場合エラー |

### 7.2 セルフチェック項目（v0.2 拡張）

```python
def self_check(output_json: dict) -> dict:
    checks = {
        "all_15_axes_evaluated": len(output_json["axes"]) == 15,
        "consecutive_counter_integrity_ok": _verify_sqlite_consistency(),
        "correlation_4_rules_evaluated": True,  # warnings の rule_id で確認
        "five_star_payload_complete": all("stars" in a for a in output_json["axes"]),
        "category_star_flags_present": "category_stars" in output_json,
        "gravity_full_crown_flag_present": "gravity_full_crown" in output_json,
        "next_focus_recommendation_present": "next_focus_recommendation" in output_json,
        # v0.2 追加
        "group_summary_present": "group_summary" in output_json,
        "wevox_integration_present": "wevox_integration" in output_json,
        "g_r3_priority_handled": _check_g_r3_priority(output_json),
        "churn_signals_all_5_linked": len(output_json.get("wevox_integration", {}).get("churn_signals", {})) == 5,
    }
    return checks

def _check_g_r3_priority(output_json):
    """G_R3 が danger の場合、priority='最優先' が設定されているか確認"""
    for axis in output_json["axes"]:
        if axis["axis_id"] == "G_R3" and axis["status"] == "danger":
            return axis.get("priority") == "最優先"
    return True  # danger でなければチェック不要
```
