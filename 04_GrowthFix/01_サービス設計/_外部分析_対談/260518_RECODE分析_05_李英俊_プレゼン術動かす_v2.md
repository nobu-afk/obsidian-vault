## 1. 主要ポイント（10 項目・濃縮）

### ① 「プレゼン = 動かす」3 段階定義（伝える / 伝わる / 動かす）

- 李英俊「**プレゼンって動かすっていうためにしかやらないんで**」「**動かす = 人の心を動かすとか場を動かすとか商品購入の気持ちにするとかもそうだけど、動いてないとダメ**」
- 「**プレゼンテーションが 10 分でも 30 分でも終わった後、徴収の状態がどうなってるか**」「**伝えるじゃダメだし、伝わるでもダメ。伝わっても動いてなければプレゼン失敗**」
- 「**そこ以外やる意味がないっていう。誤解のある言い方になる**」「**自分に対してそう思ってる**」
- 「**それなら手紙で渡せばいい**」「**何言いたいかわからないのはもう論外で、言いたいこと言ったみたいなのもダメ。スタイル能力って話だけで言うと実は徴収の理解力が全て**」
- 「**相手のゴールに引っかかってない話をしてもそれはプレゼンテーションって言わないし、よっしゃ一応やってみようってならない**」
- **GrowthFix 翻訳**：LP / 商談 / WP / Note すべてに「動かす = 商談化 / 申込 / 認識転換」のゴール定義を埋め込む。「読まれた」「シェアされた」では失敗。**動いていない発信は KPI 達成ではない**

### ② 「厳しいぐらい徴収分析をする」儀式（2 パターンの場面判別）

- 李「**厳しいぐらい徴収分析をする**」「**徴収のメリット 100% じゃないと、こちらの思惑に乗っかってもらえないから**」
- **(a) 他社主催のカンファレンス / 媒体イベント**：自社で集客してないので「**本当にどういう人が集まってるのか**」をめちゃめちゃ分析する。「**その媒体がどうやってイベントを過去やってきて、誰が今回来る確率が高くて**」を全部調べる
- **(b) 自社開催 or 企業選抜協同型**：「**もう対象者のあるべき論を決めて、その者たちを引っ張ってきてる**」「**ターゲットにしたい人しか着席させない**」= 集客で人を絞る = **採用業務に近い**
- 「**他者さんがやるとこに呼ばれて講演する時、ちょっと違うなと思ったら断る**」「**なんならリスナー連れて行って集客でやってしまうならやってでもやる**」
- 「**マインドセットの 10 年の歴史で言うとほぼ実写企画（自社主催）ばっかり。他人のイベントに行かないっていうのは徴収分析が究極までできないから**」
- **GrowthFix 翻訳**：商談前の **徴収分析プロトコル**を必ず通過させる（業界 / フェーズ / 直近 3 ヶ月の経営者言動 / 採用課題の言語化レベル / 内部参謀の有無 等 10 軸ヒアリング） → 通過しなければ初回商談を **断る**。「呼ばれたから行く」「リードがあるから提案する」を捨てる

### ③ 「スポットライトを 1 秒も自分に当てない」原則（自社情報・自己紹介の禁則）

- 李「**理解がなされてることがゆえにスポットライトが自分に当たってるけど、物理的には話のスポットライトを一切自分に当てないままプレゼンをやりきる**」
- アンチパターン：「**転職 2 社目で 3 年目の人事やってます誰ですみたいな**」「**就活してる学生にとって目の前の人事のキャリアとか知らんがな**」「**会社はこういう会社で、サマーインターンはこういうプログラムなんで是非参加したらどうぞみたいな**」
- 李の実装：「**新卒の例 / 部隊に行ったら、今座ってる学生さんが、これがね、6 月だったらとか 2 月だったらこういうことが就活生で悩むあるあるだよねとかって、同意できる人何人ぐらいいるとか**」「**学生さんの話しかしない**」
- 「**過去の就活生の話をしてるだけ。今目の前の就活生の話をしてるだけ。そこを課題解決しようと思ったら、こういう経験を 1 ヶ月やると成長するよね、人はって**」
- 「**会社の話もしない。ここに来ればいいから以上みたい**」「**会社が余番 / 本社住所 / 資本金とかビルの写真とか絶対いらないでしょ**」
- 「**スポットライトがずっと徴収にある状態**」「**リの個人情報喋ってないし会社概要喋ってないから、どうしても気になるからなぜこのおっちゃんはこのプログラムを学生のためにガチで作ったかって言うと、会社がそういう考え方をしててねっていう会社だってことだけ伝えるだけ**」
- 「**それで行くってなった企業を調べるよね。したらホームページ見れば書いてある**」
- **GrowthFix 翻訳**：260516 夜の認知シフト（自己紹介 2/3 短縮 / 経営者カード並列廃止 / 料金完全非公開）の **方法論的根拠**そのもの。商談・LP・セミナーすべてで「**顧客の現状の話だけして、自社情報は最小化、興味があれば調べてもらう**」を貫く

### ④ 「徴収のメリット 100%」 → 「相手の潜在人数を顕在化する」

- 李「**相手が聞きたかった以上の潜在人数を顕在化するために、めちゃくちゃ調べがついてて、その人たちのために生命時間を出す**」
- 「**徴収のメリット 100% じゃないと、こちらの思惑に乗っかってもらえない**」
- 「**人数を抽象化するだけに過ぎないので、何個かのクラスターを抽象化するだけ**」「**この専門系の人とか人事の人とか人事コンサル側の人とか全部こうまとめてあげて、じゃこの 3 つの人たちがクラスターでいるようだから、その観点で喋っていきます**」
- **GrowthFix 翻訳**：商談・セミナー設計で **「対象クラスター 3 つに事前抽象化 → 各クラスターのメリット 100% を顕在化する一文を冒頭に置く」**を標準プロトコル化。漠然と「経営者の皆さま」と言わない

### ⑤ 「他者主催の場合の繋ぎ役 = 乗り代の 15 分」

- 李「**大型カンファレンスで調公園とかだったら別ですけど、特別公園の間にちょっと時間もらって出るだとスポットライトがない可能性があるから、あの観客の皆さんはその次の次の人の話を聞くために座ってるみたいな**」
- 「**そういう場合はちゃんと自分がこのテーマに対して皆さんに価値が提供できる人であるっていうライトニングピッチぐらいはするよ。60 秒ぐらいは**」
- 「**それも自分のキャリアを喋るというよりは、調やった人とか特別講演やった人とか 1 番最後のレジェンドが最後喋るみたいな、控で全部お話しして名刺交換して、その人たちの情報喋る**」
- 「**今日の最初の人のあの話がめちゃくちゃ価値高かったですよねとかなると一緒に見たっていう風にラポールができる**」
- 「**最後の人が聞きたくて皆さんが座ってんのは確かで、僕もめっちゃ楽しみなんで、さっきちょっと挨拶させってもらったんですけど、めちゃくちゃ良かったっすよとかやっとく**」
- 「**今日の流れの話と後で出てくる人の話の溶接になる、1 番なんか乗り代になれるような時間が 15 分できたら最高ですみたいな話で喋り始める**」
- **GrowthFix 翻訳**：セミナー / Podcast ゲスト出演 / 招待講演で **「徴収 + 主催者 + 前後登壇者を喋る = 自分を喋らない」**プロトコル。「乗り代の 15 分」コンセプトはセミナー OP の設計フレームとして即運用可能

### ⑥ 「準備倍量 × 本番台本捨て × ひらめいた何かを必ず言う」3 重構造

- 李「**お題に対してその時間の倍ぐらいなコンテンツの大風呂敷を広げまくって、LP にそれ全部 まし（マシマシ）たよね**」
- 「**大風呂敷広げる人格のリーと、舞台に上がってなんでこんな大ボラ吹きになってしまったんだろうって閉じる時間の制限の中に収めなきゃいけないってもう 1 個の人格が舞台装置の上で泣きながら格闘してひねり出すものを出す**」
- 「**徴収とのやり取りの反応を見て、その場でひらめたことを必ず言う、っていうのは決めている**」
- 「**全ての台本はちゃんと作って練習はされてる、けど台本作って台本を本番で捨てる**」
- 「**脳みその圧プレッシャーによって出てきた何かっていうのと、目の前の人がいるから生まれた何かっていう、コ・クリエーションっていう話で、それを必ず入れるので、すごい緊張とストレスマックスみたいな形で舞台に行く**」
- **GrowthFix 翻訳**：Coaching セッション / Cultivate ファシリ / 商談 / セミナーすべてで「**台本を作り、本番で台本を捨てる**」を運用標準化。**準備量 = 本番時間の 2 倍**を最低基準として営業 PB / Coaching SOP に明記

### ⑦ 「PC 忘れた事件 = ラップトップは忘れ物ではなかった」

- 李「**ある時 1 回それをやろうと思ったら PC 忘れてたので**」「**大阪当に。しかも軽いから 890g みたいな**」「**気づかずに、そういう時に限ってクラウドのバックアップがないみたいな**」

- 「**つまりラップトップは忘れ物ではなかったっていうこと**」「**1 回経験してやっぱちょっと変わったかな。偶然の産物からこのスタイルがちょっと外れた**」

## 5. メタ観察 — 李英俊の対話技法 / コーチング構造から学べる Gravity Coaching 応用

### MO-1. 「異常シリーズ」の場づくり = 自虐先行 → 弟子認知差利用

- ホスト「**本人は普通だと思っているが、どう考えても異常なさんの悪まるシリーズ**」「**これ絶対ディスってるよ**」「**参考にはならないって**」
- 李「**バレるんか。バレるな**」「**面白いと思うんで。プレゼンね、これもプレゼン研修みたいに構造的に喋れないよ**」
- **構造**：自虐先行 → 弟子（ホスト）による「異常」レッテル → 李の本気度自然引き出し → 視聴者「常人とは違う何かを聞ける」期待形成
- **Gravity Coaching 応用**：初回セッションで「**私はあなたを変えに来てない。あなたを変えるのはあなた**」と先に **自分の役割を縮小宣言**することで、クライアントの本気度を引き出す構造

### MO-2. 「準備の段階から普通の人と違う」を主軸先行で先取り

- ホスト「**裸ても準備段階からなんか普通の人と違げえな**」「**多分何も今ピンときてないと思うんですけど。それが当たり前だと思ってるんで**」
- 李「**そうだね。自分はこうしてるっていうのはあって、それは別に人に何ていうかな求めてないというか、合ってるか合ってないかってのがあるから自分の性格にもう**」
- **構造**：弟子が「準備段階から異常」と先に枠を作り、李は「自分は普通」と謙遜しつつ「自分の性格に合ってる」と独自性を肯定 → **自然な独自性開示**
- **Gravity Coaching 応用**：Coach が「**あなたの中の異常さは、あなたの引力源です**」と最初に **異常性 = 引力源** の翻訳枠を提示すると、クライアントが自分の異常性を恥じずに開示できる

### MO-3. 「プレゼン研修みたいに構造的に喋れない」 = 構造論否定の冒頭スタンス

- 李「**プレゼンスがあって、コンテンツシナリオがあって、デバがあってドキュメントがあってセットポスチャーからやるみたいな、じゃない話でプレゼンか**」
- **構造**：既存の構造論（プレゼンスキル研修）を明確に否定して、自分の OS を語る前提を作る → **教科書的アプローチへの違和感を共有**することで、聴き手が「**じゃあ何を聞かせてくれるんだ**」と前のめりになる
- **Gravity Coaching 応用**：Coaching 初回で「**今日は MBA で習うようなフレームワーク話はしません。あなたの引力をホワイトボードに描き出します**」と既存枠を否定 → 新しい OS を期待させる導入

### MO-4. 「自分の話してない」の自己実況

- 李「**スポットライトがずっと徴収にある状態。だってリの個人情報喋ってないし、会社概要喋ってないから**」「**それで行くってなった企業を調べるよね**」
- **構造**：自分のやってる手法を **その場で実況解説** = メタ認知を共有することで、聞き手は「自分も同じ構造を運用できそう」と思える → **教えてないのに教えてる**
- **Gravity Coaching 応用**：セッション中 30 分くらいで「**今、私はあなたに対してこの問いを置きました。なぜかというと…**」と **介入の意図を実況開示**すると、クライアントが「自分が変えられている儀式」を客観視できる = 主体性転換装置

### MO-5. 「もっと高い次元の目的を持って立つ」 = 場の影響力変容メカニズム

- 李「**徴収と主催者と喋ってる私を超えた目的を 1 個置く**」「**祈りに近いもの**」「**情報と物理の楽さがでかいほど人の心ってのは書き変わる**」
- **構造**：個人の意図 < 場の意図 < 社会の意図 と階層化し、**最上位階層を内包して立つ**ことで、自分自身のエネルギー状態を変える → 場が動く
- **Gravity Coaching 応用**：Coach がセッション前に「**このクライアントの 3 年後の家族 / 社員 / 顧客が幸せになりますように**」と心の中で唱える **裏儀式** = Coach 自身のエネルギー状態が変わる → クライアントが「**この人は私個人を超えた何かを見ている**」と直感する → 信頼形成
- **学術接続候補**：Goleman & Boyatzis「Resonant Leadership」/ Senge「Presencing」/ Otto Scharmer「Theory U」/ Mindfulness in Organizations 系研究

### MO-6. 「テイクワン縛り」自己実況 = メタ規律の可視化

- 李「**この YouTube も咲（さくたる）例じゃないですか。15 分ぐらいっていう制限をこっち側の都合で勝手に作ってる。本来 2、3 時間やるテーマを全然圧縮してやってたりとか、尺とあとテイクワン縛りとかを勝手にやってるのもなんか自分にその場でプレッシャー与えて出してる**」
- ホスト「**1 年半 1 回もないですね**（取り直しゼロ）」
- **構造**：**自分に課したメタ規律をその場で開示**することで、視聴者は「これは編集された動画ではなく、生の即興だ」と認知転換 → 動画への信頼度が一段上がる
- **Gravity Coaching 応用**：Coach が「**今日は 60 分のうち、私は答えを一切提示しません。すべてあなたから引き出します**」と **自己規律を冒頭宣言** → クライアントの主体性引き出しが構造化される

### 🎯 3 軸横断の現場知見 3 件

#### 横断-1. 「動かす KPI」を 3 軸すべてで標準化

- **集まる軸**：採用説明会 → 応募者の「来週中に応募するかどうかの意思決定」が動いた率
- **躍動する軸**：1on1 / マネジメント会議 → 部下 / 参加者の「来週からの動きの言語化」が動いた率
- **留まる軸**：経営者 Coaching → 経営者の「次月の経営判断の言語化」が動いた率
- **3 軸共通 KPI** = 「動いた率」を組織の引力設計プログラム月次レポート標準項目として採用

#### 横断-2. 「徴収分析が終わってない場所には出ない」儀式の 3 軸現場展開

- **集まる軸**：求人媒体 / 採用説明会 / カジュアル面談前に、対象求職者ペルソナ 3 タイプを言語化通過 → 通過しないと出さない
- **躍動する軸**：マネジメント会議 / 全社会議前に、参加者のテーマ理解度 3 階層を分析 → 通過しないと開催しない
- **留まる軸**：経営者 Coaching セッション前に、経営者の直近 2 週間の意思決定 3 件を Coach が事前分析 → 通過しないとセッション開始しない
- **3 軸共通プロトコル** = 「徴収分析 → 通過 → 出る」を組織の引力設計プログラム標準 SOP として明記

#### 横断-3. 「自分を超えた高次の祈り」儀式の 3 軸現場展開

- **集まる軸**：採用説明会前、人事担当者が「今日参加した求職者 1 人 1 人の 3 年後のキャリアが幸せになりますように」と心の中で唱える
- **躍動する軸**：1on1 前、マネージャーが「この部下の 3 年後の家族 / 友人 / 部下が幸せになりますように」と心の中で唱える
- **留まる軸**：Coaching セッション前、Coach が「このクライアントの 3 年後の家族 / 社員 / 顧客が幸せになりますように」と心の中で唱える
- **3 軸共通裏儀式** = 「高次の祈り」を組織の引力設計プログラム裏 SOP として運用化（口に出さない）

## 8. 副次：規模別 1 件（70 名以上プラン差別化候補・1 件のみ・モリモリ化反転原則準拠）

### 70 名以上プラン差別化候補：「動かす儀式 OS」企業文化埋め込み深度差

- **30 名以下プラン（月 20 万）**：採用説明会 OP テンプレ + マネージャー 1on1 SOP 提供（標準納品物）
