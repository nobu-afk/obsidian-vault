# §3. 🟡 躍動軸（17 本横串集約）

## 3.1 現場の核心仮説（17 本横串）

> **「組織の躍動施策は『社員の内的動機 × AI に渡さない領域 × 期限付きプロジェクト × 役割解除装置』の四点セットで設計すべき。外的動機（KPI / 売上目標）ベースの施策は中長期的に社員の自信を蝕み、AI ぽい仕事（整った文書・整ったプレゼン・整った会議）の量産は社員の本能性を殺す。期限付き解散プロジェクト + コネクティブ AI（人と人の間に介在する AI）+ 全員ポスター発表（個別性 × 全社）+ アンラーニング装置（レゴ城バラし）の組み合わせが躍動を最大化する」**

1. **「学習 = 9:1 でアウトプットが 9」原理**（上野山 03）→ 小さなアウトプット機会を AI に渡さない設計
2. **「コネクティブ AI（人と人の間に介在する AI）」**（上野山 03）→ 西洋型エージェント AI ではなく日本型関係性介在 AI
3. **「期限付きプロジェクト + 解散の祝祭」**（安斎 + 本田 + 神本 + 高木）→ だらだら中庸を避け、極振り × 戻すリズム

> **「組織の留まる力は『離脱可能性が常に開いている × 経営者の人生体重伝達 × 卒業を歓迎する装置 × 関係資本マップ × dual キャリア許容』の五点セットで設計する。給与・福利厚生で人を縛る『恥型留まる』は中長期的に崩壊する。リクルートが想像的組織として機能しているのは卒業を歓迎する装置（博士号方式）を制度化しているから。卒業 = 入学一体化（第 2 成人式）で離脱を新陳代謝点に変える。経営者個人 ≠ 組織人格の制度化が、留まる力の本質」**

1. **「やめ方のデザイン = 留まる力の本物」**（17 本中 9 本通奏低音 #3）

3. **「光を当てる連鎖 = 健全留まる」**（SHIRO 今井 04）

## 4.3 留まる軸の現場ツール集（7 件）

### ツール 1：**「離脱型 × 留まる質」マトリクス**（月次レポート定型・拡張版）
```
                 留まっている強度
                  ▲
                  │
   健全留まる     │  依存留まる
   （祝祭型）     │  （恥型）
─────────────────┼─────────────────▶ 離脱可能性
   健全離脱       │  抑圧離脱
   （博士号型＋   │  （バーンアウト型・
    握手型）      │   炎上型）
                  │
                  ▼
```
→ 「**握手率 KPI**」= 全離脱者のうち握手型・博士号型の比率を計測
→ 出典：MIMIGURI 安斎（260517）+ 田中渓（02）

### ツール 2：**「光を当てる連鎖マップ」**（半期社員アンケート定型）
- 「光を受けた回数」× 「光を当てた回数」の 2 軸 4 象限
- 半期に 1 回、社員アンケート（匿名）で「光を受けた / 当てた」事象を 3 件ずつヒアリング
- 「リーダー型留まる」象限が増えれば組織の引力が拡張

→ 出典：SHIRO 今井（04）

### ツール 3：**「経営者人生体重 × 圧倒的熱狂 100 人」マトリクス**
- 経営者人生体重の伝達度 × 社員の内的動機接続度
- 目指す象限：**左下＝圧倒的熱狂 100 人**（両立）
- カリスマ依存（後継リスク）/ 数字依存組織（緩やかな自己否定）/ 個人事業所（持続不能）を避ける

→ 出典：フミヤ（10）+ 高木

### ツール 4：**「卒業の階段設計」マトリクス**（70 名以上プラン差別化）
```
影響力（経営参画度）の段階
執行役員（100%）→ 子会社社長（70%）→ 副会長（40%）→ 顧問（20%）→ 引退（0%）
時間軸：0 → 3 → 5 → 7 → 10 年
```
各段階で：権限 / 報酬 / 心理契約 / 次の食立て を事前合意
→ 出典：小杉善信（08）+ 上野山勝也（03）

### ツール 5：**「やめ方のデザインスイート」5 サブメニュー**（70 名以上プラン月 45 万差別化）
1. **顧問・名誉職の制度設計**：肩書きを残しつつ意思決定から外す健全構造
2. **名刺後ライフ設計**：別名刺・別肩書きを引退前に作る伴走
3. **次の夢設計**：夢を作る側だった人が自分のために作れない構造の解除
4. **卒業の階段設計**：執行役員 → 子会社社長 → 副会長 → 顧問 → 引退の段階的離脱パス
5. **権力勾配観察ツール**：本人の権力低下を周囲がどう扱うかを観測する月次サーベイ

→ 出典：小杉善信（08）+ MIMIGURI 安斎（260517）

### ツール 6：**「原体験 × 偏愛 × くだらなさ DB」**（70 名以上プラン留まる施策）
社員別 留まる力ダッシュボード（月次更新）

## 5.1 章の位置付け

v1.0 で「§7 CODE/Coaching/ワンパッケージ反映候補」として 1 章にまとめていた内容を **個人軸独立章**として拡張：

- **CODE への現場活用**：4 型診断補強 + 5 型 / 6 型拡張候補

- **個人軸の現場ツール集 + 即時導入候補**

**重要前提**：個人軸 = **経営者個人**（社員個人 ≠ 個人軸）。CODE / Gravity Coaching は個人軸別商品として温存。

## 5.2 CODE への現場活用（17 本横串）

| **個人偏愛ズレ型** | 「業界別偏愛翻訳辞書」（テレビ「ラみ」起点・小杉 08）で経営者業界の偏愛言語を診断可能化 |

### 5.2.2 CODE 4 型 → 5 型 / 6 型拡張候補

#### 候補 1：**構造蓋型**（07 佐藤）

既存 CODE 4 型に佐藤裕介ケースは綺麗にハマらない：
- 「強み（構造理解能力）は突出している」
- 「想い（人の役に立ちたい）は存在している」
- 「偏愛（ワイルドスタイル / ピュアオリティへの憧れ）も存在している」
- にもかかわらず、**強みが過剰に発達した結果、想い・偏愛の言語化と表出が蓋されている**状態

→ **「構造蓋型（強み過剰発達による想い・偏愛封印）」を 5 型として追加検討**。経営者上位層（連続起業家・複数事業経験者）に特に多い。Gravity Coaching の上位プラン差別化候補。

#### 候補 2：**2 位コンプレックス経営者型**（05 山口揚平）

| 特徴 | 内容 |
|---|---|
| **症状** | 実績は十分ある（B ランク以上）・周囲からは羨ましがられる・本人は「ガチの 1 位がない」と内面で苦しむ |
| **発症経路** | 幼少期の競技経験・学歴経歴の 2 位連発から形成された構造的セルフイメージ |
| **GrowthFix の解決** | 5 次元（自分の KPI）への移行を 6 ヶ月で支援 → 共通物差し（4 次元まで）から離脱 → 2 位コンプレックスから解放 |

**石井自身の競技時代 2 位コンプレックス**（過去の壁打ち履歴で言及）と同じ構造。**石井 = 元 2 位コンプレックス経営者 → 引力経営に到達した実例**として Gravity Coaching LP のストーリーに接続可能。

#### 候補 3：**過剰適応型**（17 本横断）

| 既存 CODE 4 型 | 追加候補 5 型 |
|---|---|
| 個人整合 / 個人想いズレ / 個人強みズレ / 個人偏愛ズレ | **個人過剰適応型**（強み × 想い × 偏愛全て存在するが、役割優先で封印されている）|

候補が **3 つ並んだ**（構造蓋型 / 2 位コンプレックス型 / 過剰適応型）= **CODE 5 型化判断を user 判断ゲートに上げる候補**。

### 5.2.3 CODE「想い軸」賞味期限問題（14 家入・要検討課題）

家入氏「自分の傷つきを癒すために事業を立ち上げてきた。で、癒されちゃったのかもしれない」

| 3 軸要素 | 賞味期限 | 家入氏の状態 |
|---|---|---|
| **強み** | 半永久（能力は摩耗しにくい） | 連続起業家としての構造化力は健在 |
| **想い** | 5-15 年（傷つきが癒されたら消滅） | **居場所を 15 年掲げて癒されきった → 消滅** |
| **偏愛** | 半永久（執着・興味は更新可能） | 「表現せざるを得ない人」「アート」への偏愛は残存 |

**示唆**：CODE 診断の「想い」軸に「想いの賞味期限チェック」追加：
- 「その想いは、もう 50% 以上満たされていますか？」
- 「満たされた後、次の想いを発見していますか？」

→ Gravity Coaching の中核機能を「想い軸の世代交代支援」として再定義可能。

## 5.3 Gravity Coaching への現場活用（17 本横串）

### 5.3.1 Coaching セッション標準スクリプト候補

#### 6 ヶ月セッション構造への組み込み案（高木 4 段階構造輸入）
- **月 1（初回）**：4 段階標準「揺さぶり」フェーズ = 現在地の言語化
- **月 2-3**：「深掘り」フェーズ = 既存ミッションの解凍 + 自己無矛盾告白引き出し
- **月 4**：「提案ジャブ」フェーズ = ソリューション語彙解除 + 中核言語書き換え
- **月 5**：「深掘り深部」フェーズ = エッセンス語彙の再構築
- **月 6（卒業）**：「余韻」フェーズ = 卒業セレモニー（博士号方式 + 第 2 成人式 5 構成要素）

## 10.2 経緯：なぜ 5 軸化したか

### 10.2.1 v1.0 で「§1 通奏低音」だったものが §1 思想層に拡張された経緯

v1.0 では 4 通奏低音を「3 軸の通奏低音」として 1 章に格納していたが、260518 user が論文整理を別エージェントで実施中、**5 軸（集まる / 躍動 / 留まる / 個人軸 / 思想層）で論文が綺麗にマッピングされる**ことを発見。

- 学術基盤（Bandura SE/CE / Edmondson 心安 / Saks-Gruman 17 資源 / Mitchell JE / Rousseau-Hansen-Tomprou / Meyer-Allen 等）は **3 軸の中に格納するには大きすぎる**
- 4 通奏低音（過剰適応 / AI ぽさ / やめ方 / 自我非連続）も **3 軸の通奏低音**として扱うのは矮小化
- **思想層 = 全商品横断の学術基盤 + 引力経営の根源人間観・組織観**として独立章化が必要と判断

→ v2.0 §1 思想層は v1.0 §1 通奏低音 4 本を **学術基盤 + 根源思想 + メタフレーム + 反映候補** の 4 区分に拡張

### 10.2.2 v1.0 で「§7 CODE/Coaching 反映候補」だったものが §5 個人軸に独立化された経緯

v1.0 では CODE / Coaching を「3 軸ワンパッケージへの反映」と並列して 1 章に格納していたが、5 軸定義で **個人軸 = 経営者個人**（社員個人 ≠ 個人軸）として独立。

- CODE / Gravity Coaching は **個人軸別商品**（ワンパッケージとは別商品）
- 17 本動画分析でも **個人軸の最濃動画 10 本**が存在（佐藤 / 山口 / フミヤ / 川邊 / 深井 / 家入 / 田中 / SHIRO / 小杉 / 本田）
- **3 軸の補助扱い ≠ 個人軸別商品の独立軸**として位置付け直し

→ v2.0 §5 個人軸は v1.0 §7 を **CODE 補強 + Coaching 差別化 + dual 運営 + ペルソナ補強 + ツール集** の 5 区分に拡張
