| 指標 | 計算式 | 目標 | データ取得元 |
|---|---|---|---|

| 指標 | 計算式 | 目標 | データ取得元 |
|---|---|---|---|
| **受注率（商談 → 契約）** | 受注数 ÷ 商談数 | 30-50% | パイプラインマスター手動 |

## 3. ボトルネック特定（週次・1 点だけ）

> **運用：** 毎週金曜、上記テーブルから「**最も目標から離れている指標 1 つ**」だけを特定する。それ以外は手をつけない。
> **理由：** 同時に複数を改善しようとすると無効化する。1 点集中で翌週の仮説検証を回す。

| 週 | ボトルネック指標 | 数値 | 改善仮説 | 翌週の打ち手 | 結果 |
|---|---|---|---|---|---|
| W17 | （5/3 に決定） | - | - | - | - |
| W18 | - | - | - | - | - |
| W19 | - | - | - | - | - |

**判断ロジック：**

```
[1] 各指標を「目標 - 実数値」で並べる
[2] 最大乖離の 1 つを特定
[3] 仮説立てる（コピー / 配信先 / タイミング / 手数 / 質）
[4] 翌週は 1 点だけ動かす（他は据え置き）
[5] 翌週末に効果測定
```
