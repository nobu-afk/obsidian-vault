## 🎯 プロダクトの1行定義

> **1on1録音を解析して、幹部の「思考OS」と「熱量4象限マッピング」を可視化するAI診断ツール**

---

## 📐 MVPスコープ（最小実装）

### 入力

| 項目 | 内容 | 形式 |
|---|---|---|
| **1on1録音** | 経営者×幹部1名の60分程度の1on1 | m4a／mp3／wav |
| **組織基礎情報** | 組織規模・事業フェーズ（0→1／10→100／スケール／運用／拡大） | テキスト短文 |
| **経営者視点の幹部評価** | 「この幹部に期待している役割・現状の課題」 | テキスト300字程度 |

### 処理フロー

```
1on1録音
  ↓ [音声→文字起こし：Whisper API or Claude API]
トランスクリプト
  ↓ [Claude API：思考OS診断プロンプト]
幹部の思考OS診断（CODE応用）
  + 熱量4象限マッピング
  ↓ [HTML レポート生成]
診断レポート出力
```

### 出力（HTMLレポート）

1. **幹部の引力タイプ**（CODEの4型応用）
   - ジャイアン型（支配×情）／スネ夫型（計算×義理）／出木杉型（完璧×危うさ）／のび太型（優柔不断×決定力）
   - 1on1での発言パターンから型判定
2. **熱量4象限マッピング**
   - X軸：本人の熱量（高↔低）
   - Y軸：組織への貢献度（高↔低）
   - 「宝」「人材」「罪」「無」の4象限（Q27で扱った人材4象限と接続）
3. **思考OS診断**
   - 意思決定パターン（即断／熟考／合議／依存）
   - 反応パターン（責任回避／担当化／委譲／自分事化）
   - 経営者との認知ギャップ（言葉のズレ／前提のズレ）
4. **次のアクション提案**
   - この幹部に次どう関わるか（経営者向けのヒント）
   - Shift Month2「特定の詰まり部署」診断との接続導線

## 🛠 技術スタック（暫定）

### 既存流用・新規開発の区別

| 要素 | 実装方針 |
|---|---|
| **音声→文字起こし** | 新規（Whisper API または Claude API v4.7の音声対応） |
| **思考OS診断プロンプト** | 既存CODEプロンプト（260415版）を流用・改訂 |
| **HTMLレポート生成** | 既存CODE Executive generate.php 流用（`05_プロダクト/GravityCode/診断_executive_本番/generate.php`） |
| **フォーム／UI** | 新規（シンプルなアップロード＋テキスト入力） |
| **ホスティング** | growthfix.jp配下（Xserverの既存構成） |

### 技術的論点

- **音声サイズ上限**：60分録音で約30-60MB（mp3）→ PHPの `upload_max_filesize` 設定要
- **処理時間**：Whisper文字起こし（1-3分）＋Claude API（1-2分）＝合計3-5分
- **ユーザー待機**：バックグラウンド処理＋メール通知が理想（MVPでは同期処理でもOK）
- **セキュリティ**：録音は一時保存→診断後自動削除（機密情報配慮）

## ⚠️ リスクと対応

| リスク | 対応 |
|---|---|
| **音声データの機密性** | 一時保存→診断後自動削除／オプトイン明示／利用規約明記 |
| **診断精度が低い** | Phase 1でテスト回数増やす／プロンプト改訂／最低限「これは合ってる／合ってない」が出るレベルを担保 |
| **Claude API コスト増** | 音声文字起こし＋診断で1件あたり50-100円想定／100件で5,000-10,000円 → 無料フックとして許容範囲 |
| **Shiftへの誘導率が低い** | レポートにShift体験セッションCTAを目立つ位置に配置／診断結果の"次にやるべきこと"セクションを具体化 |
| **法的リスク（録音）** | 幹部本人の承諾を経営者が取った前提で運用／利用規約に明記 |
