## 1. 転換装置 7 パターン（外的フック → 内的本論への橋渡し）

> **使い分け原則：** チャネルの転換濃度（外的中心 / ハイブリッド / 内的中心）と funnel 上の段階で、複数パターンを組み合わせる。1 つのコンテンツで全部を使う必要はないが、最低 2 パターンは入れる（フック + ブリッジ）。
> **堀田 35 選との対応：** 各パターンに堀田フレーム番号を付記。本 7 パターンは堀田 #15 逆説提示法・#16 メタファー先行・#20 認知 OS 書き換え・#23 ペインポイント解像度・#32 えげつない具体の **下位実装パターン**として位置付け。

### パターン #1：「実は ◯◯ ではなく △△」反転パターン（堀田 #15 #20）

- **構造：** 外的痛み（◯◯）→ 反転接続詞「実は」「これは ◯◯ の問題ではない」→ 内的本質（△△）
- **用例：** 「採用が回らない。── これは採用 How To の問題ではなく、**組織の引力**の問題」
- **用例：** 「離脱が止まらない。── これは制度設計の問題ではなく、**入社初日から始まっている**ミスマッチの問題」
- **用例：** 「会議で意見が出ない。── これは心理的安全性の問題ではなく、**経営者の言葉が幹部 3 人目で死ぬ**構造の問題」
- **適用チャネル：** LP Sub Hero ／ Note 序盤 ／ セミナー転換 ／ WP 章間ブリッジ
- **濃度：** 外的→内的の正面ゲート（最強の転換装置）
- **NG：** 反転先が抽象すぎる（「実は組織変革の話」レベル）→ 具体メカニズムまで言い切る
- **堀田由来の強化軸：** 主語チェック（堀田 #10）── 反転先の主語を「組織の問題」ではなく「事業の成長を止めている構造」にする

### パターン #2：対比構造パターン（Before/After 並置・堀田 #20 認知 OS 書き換え Before→After 変換）

- **構造：** 同一条件 / 異なる結果 → 構造の違いを示唆
- **用例：** 「同じ採用予算 300 万で、応募 5 名の会社と 50 名の会社の差は ──」
- **用例：** 「評価制度を入れた A 社・入れなかった B 社、3 年後に優秀層が残ったのはどちらか」
- **用例：** 「カリスマ社長 vs 引力経営者：5 年後の幹部数の差」
- **適用チャネル：** LP Hero ／ FB ／ Meta 広告 ／ Note 序盤
- **濃度：** 外的のみで成立（内的への橋は本文で）
- **NG：** 対比が陳腐（給料が高いから / 福利厚生が良いから）→ 構造軸を異化
- **堀田由来の強化軸：** 認知 OS 書き換え 5 例の援用（「働いて稼ぐ→仕組みが稼ぐ」「人事の問題を解く→事業のボトルネックを外す」）

### パターン #3：構造可視化パターン（メカニズム提示・堀田 #16 メタファー先行 + #22 一発芸スライド）

- **構造：** 漠然とした感覚 → 名前のあるメカニズム化（「◯◯症候群」「◯◯の壁」「◯◯構造」）
- **用例：** 「経営者の言葉が、**幹部 3 人目で死ぬ**構造」
- **用例：** 「**ナンバー2 不在症候群**」
- **用例：** 「**もじもじエンジニア vs バキバキ営業**の依存構造」
- **用例：** 「**採用 MAX が明治大学レベルで頭打ち**になる経営者の盲点」
- **適用チャネル：** Note ／ WP ／ セミナー ／ 商談
- **濃度：** 50:50 ハイブリッド（メカニズム＝外的可視化、原因解説＝内的）
- **NG：** メカニズム名が長すぎる（5 単語以上）／既知概念に近すぎる（差別化できない）
- **堀田由来の強化軸：** 一発芸スライド原則（#22）── 1 枚で世界観が伝わる構造図に昇格させる（北野唯我「天才を殺す凡人」三角形スライドと同パターン）／お気に入り贔屓の滲ませ（#28）── 4 象限なら 1 つだけアンフェアに特別扱いする

### パターン #4：数値ショックパターン（具体エビデンス直撃・堀田 #19 体験の蒸留 + #32 えげつない具体）

- **構造：** 具体数値 / 固有名詞 / 場面で痛みを再認識させる
- **用例：** 「**明治大学レベル**以上が応募してこない」
- **用例：** 「**年 300-500 万**の採用代行費用、決定 1-2 名」
- **用例：** 「**入社 3 ヶ月**で辞めた人の本当の理由」
- **用例：** 「**幹部 3 人目**で死ぬ言葉」
- **適用チャネル：** LP Hero ／ FB ／ Meta 広告 ／ Note 冒頭 ／ セミナータイトル
- **濃度：** ほぼ外的（内的への橋は別パターンで）
- **NG：** 数値がふんわり（「数百万円」「数年」）／固有名詞リスク（炎上回避と引き換え）→ 具体度ギリギリで運用
- **堀田由来の強化軸：** 「言っちゃいけないやつの言葉」レベル（#32）まで具体度を上げる／美輪明宏メンタル（#26）で批判受容覚悟して尖り維持

### パターン #5：逆説的問いパターン（パラドックス提示・堀田 #15 逆説提示法）

- **構造：** 常識的解決策 vs 不変の現実 → 別の説明軸を提示
- **用例：** 「**なぜ評価制度を入れたのに、組織は変わらないのか？**」
- **用例：** 「**なぜ識学を入れたのに、採用も離脱も変わらなかったのか？**」
- **用例：** 「**なぜ社長が一番頑張っているのに、組織は冷えていくのか？**」
- **適用チャネル：** Note ／ WP ／ セミナー ／ 商談 ／ Note タイトル
- **濃度：** 外的問い → 内的回答の構造（本文 7-8 割が内的になる典型形）
- **NG：** 問いが弱い／自答が陳腐（「それは本気度が足りないから」レベル）→ 引力経営独自の説明軸まで降ろす
- **堀田由来の強化軸：** 逆説カタログ援用（「単価を上げると客が離れる」「優秀であるほどビジネスを回せない」「強みからプロダクトを作るな」）／結論の順序（#7）── 「A→B→C→D」ではなく「A の原因は D です。なぜなら…」の先出し型

### パターン #6：登場人物の解像度パターン（場面描写・堀田 #23 ペインポイント解像度）

- **構造：** 具体場面で経営者の自己同一化を促す
- **用例：** 「**最終面接で目が覚めたように辞退連絡が来る** 経営者」
- **用例：** 「**全社会議で熱が伝わらない** 経営者」
- **用例：** 「**幹部に話を合わせられている** 感覚を持つ経営者」
- **用例：** 「**立派な経営者を演じる** 疲れを抱える 30-50 代経営者」
- **堀田 #23 由来の追加語彙：** 「**事業が踊り場にある**」「**成長痛**」「**COO とうまくいかない**」「**幹部のやる気問題**」（経営者が自分でお金を払いたくなる言い方の具体例）
- **適用チャネル：** LP Hero ／ Note 冒頭 ／ FB ／ 広告コピー
- **濃度：** ほぼ外的（自己同一化トリガー）
- **NG：** 場面が一般的すぎる（「忙しい経営者」レベル）→ 言語マップのスコア A 表現を直接使う／「組織課題」を主語にしない（堀田 #10 主語チェック・経営者は組織課題にお金を出さない）
- **堀田由来の強化軸：** 「経営課題っぽくない入口で売る」（#23）── モテケンの「幹部の絆を強める合宿」型／針の穴ほどの差で刺さる言い方を発見する取材軸

### パターン #7：思想の翻訳パターン（既知概念へのブリッジ・堀田 #16 メタファー先行 + #10 主語チェック + #28 お気に入り贔屓）

- **構造：** 内的概念を既知の概念で言い換え
- **用例：** 「**引力＝集客力**」（核心 5 語彙）
- **用例：** 「**スポーツドリンクの売り方**」（堀田 4/15 メタファー）
- **用例：** 「**組織の引力タイプを 60 分で丸裸にする**」（診断系の funnel 入口）
- **用例：** 「**社長の限界を、事業の限界にしない**」（GrowthFix タグライン本体）
- **適用チャネル：** 全チャネル（特に WP / セミナー / 商談 / コーポレート Hero）
- **濃度：** 内的中心（既知概念で内的世界観を装備）
- **NG：** 翻訳が浅い（「組織変革とは ◯◯」レベル）→ 経営者の常識を一段ずらす表現まで降ろす
- **堀田由来の強化軸：** メタファーカタログ援用（ウニ丼／シェルパ／PL vs BS／ビジネスの配管工／自分で作った牢獄／貫禄おばちゃん）／お気に入り贔屓の滲ませ（#28）── Gravity マップの ❹「グラビティ型成長」だけ思想セリフをキャプションで添える型

## 1.5 堀田由来の上位設計原則 4 件（パターン使用時の必須メタルール）

> 7 パターンを実装する際、以下 4 つの上位原則を **常時テスト**する。パターン単体では満たせない・複数パターンの組み合わせ運用で実現される。本セクションは堀田 35 選から接続装置に直結する 4 件を抽出。

### 原則 A：前振り×意外性サイクル（堀田 #7 ・最重要メタルール）

- **構造：** 共感（前振り）→ 意外性 → 回収（「なるほど、そういうことか」）の 3 段サイクルを **10 サイクル以上散りばめる**
- **前振りの本質：** 相手に共感するのではなく、**こっちが言った言葉に相手が共感する**。根源的欲求（楽になりたい・スケールしたい）に触れる
- **意外性のサイズ：** 大きな意外性 1-2 個 + カクテルパーティー効果型の小さな意外性を多数
- **結論の順序：** 「A→B→C→だから D」ではなく **「A の原因は D です。なぜなら…」**（結論先出し）
- **マジシャン比喩：** 前振り 7 割・畳みかけ 3 割（時間配分）／記憶の 8 割は畳みかけ
- **接続装置への適用：** **本 7 パターンは前振り×意外性サイクルの「素材」**。1 サイクル = #6 場面（前振り）→ #1 反転（意外性）→ #7 翻訳（回収）の 3 パターン組み合わせが最小単位
- **検収：** LP / Note / セミナー / WP に **10 サイクル以上**前振り×意外性が入っているか

### 原則 B：コンバージョン 3 条件（堀田 #21）

- **30 秒以内に以下 3 つが全てピンとこないとコンバートしない：**
  1. **必然性（Why now）：** 今やらなきゃいけない理由（「いつかやろう」を「今やろう」に変える）
  2. **有効性（It works）：** やったらうまくいく根拠（実績・事例・フレームワーク）
  3. **実現可能性（I can do it）：** 自分でもできる感（難しそうに見えると離脱）
- **多くの初心者が陥る罠：** 必然性だけ語って実現可能性を伝えていない
- **接続装置への適用：** 転換装置（特に Hero + Sub Hero + Pricing 補強）を 3 条件で点検。たとえば Recruit LP の Hero「優秀人材が吸い寄せられる会社を、3 ヶ月で実装する」── 必然性 ◯（採用コスト悪循環）/ 有効性 ◯（3 ヶ月の具体納品物）/ 実現可能性 ？（3 ヶ月で本当に実装可能と感じるか）
- **検収：** 全 LP / セミナー / WP に **3 条件全部が 30 秒以内にピン**と来るか音読チェック

### 原則 C：思想 6:ロジック 4 配分（堀田 #25）

- **配分原則：** コンテンツは「思想 6 割 + ロジック 4 割」。ロジック過多になった瞬間に学者モード化し、個性が希薄化し、売れなくなる
- **学者モード化の症状：** AI にブレストを先に投げる／受け狙い／批判恐怖で正確性に逃げる／配分がロジック 8 割：思想 2 割
- **処方箋：** 主軸・Why・偏愛を先に自分で確立 → 後から AI に相談（逆順厳禁）
- **「リーダーの仮面が売れた理由 = 思いが乗りまくっているから」原理**を適用
- **接続装置への適用：** 7 パターンの選択偏重で配分が崩れる例 ── #3 構造可視化＋ #5 逆説的問いに偏ると「論理的だが冷たい」コンテンツになる ／ #6 場面 + #4 数値 + #7 翻訳を厚くすると「思想が滲む」コンテンツになる
- **検収：** 章ごと / セクションごとに「Gravity 思想が滲んでいるか」で判定。滲んでいなければ書き直す

### 原則 D：えげつない具体 3 層構造（堀田 #32 #35）

- **3 層構造：**
  ```
  [トップランディング層]    抽象 OK（思想・興味喚起）
       ↓
  [コア層]                えげつない具体（実装経験から書く）
       ↓
  [CTA 層]                具体ベネフィット（採用うはうは・離脱止まる・売上 1 億行く）
  ```
- **「言っちゃいけないやつの言葉」レベル**まで具体度を上げる ── ✅「面接でこう質問する経営者は採れない」／ ❌「組織には引力がある」（誰でも書ける）
- **マーケ系 vs 人事系決め文句区分（堀田 #35）：**
  - マーケ系商品 → CTA は「**売上 1 億行く**」
  - 人事・組織系商品（GrowthFix 全 6 サービス相当）→ CTA は「**採用うはうは・離脱止まる**」
- **接続装置への適用：** トップ層は #7 翻訳・#3 構造可視化（抽象 OK）／コア層は #4 数値・#6 場面・#3 メカニズム（えげつない具体）／ CTA 層は #2 対比結果・採用接地具体（堀田 #30 #35 直撃）
- **検収：** 全コンテンツで 3 層構造が成立しているか／コア層に抽象論で完結しているセクションがないか／ CTA 層が思想宣言で終わっていないか（採用接地具体への着地必須）

### 4 原則 × 7 パターンの統合チェックリスト

| 検収軸 | 必須パターン |
|---|---|
| 原則 A（前振り×意外性 10 サイクル）| #6 → #1 → #7 の組み合わせを 10 回 |
| 原則 B（コンバージョン 3 条件）| Hero #6 #4 + Sub #1 #5 + Pricing 補強 #2 で 3 条件カバー |
| 原則 C（思想 6:ロジック 4）| #6 #4 #7 を厚く・#3 #5 を抑える |
| 原則 D（えげつない具体 3 層）| トップ #7 → コア #4 #6 #3 → CTA #2 採用接地 |
