# 別エージェントセッションへの貼り付けプロンプト

以下を **別エージェントセッション開始時に最初に貼り付け**てください。

---

```
天プロフレーム R3 セッション開始。

まず以下の引き継ぎ書を Read し、その指示に従って実行せよ。

`04_GrowthFix/02_マーケティング/260517_天プロR3_別エージェント引き継ぎ_v1.0.md`

引き継ぎ書 §3「主要 SSOT パス」と §4「memory」も指示通り順次 Read してから提案を始めること。

最初のタスクは **B（コーポレート系 LP R1 改修）** から開始。対象は profile / achievement / news / knowledge の 4 LP。R1+R2 完全未着手領域なので、Gravity 4 LP（CODE / 引力設計 / Coaching / 無料 Web 診断）で確立した「足す＋消す」運用 + Gravity 用語隔離原則を適用し、構造改修ペア提案 → user 取捨選択 → 都度承認 → 実装 → deploy.sh lp のサイクルで進める。

最初のステップ：
1. 引き継ぎ書を Read
2. R1+R2 で使った主要 SSOT を Read（戦術フレーム集 v1.0 / 理論パート v1.0 / 改善案マトリクス v0.1）
3. memory 6 件を Read（reference 2 + feedback 2 + user 1 + project 1）
4. B 着手 LP を user に選択させる（4 LP のうちどれから？・推奨：profile）
5. 該当 LP を Read → 構造把握 → 5-6 ペア提案を user に提示

注意：
- 自動 deploy 禁止・user 明示承認必須
- 天プロ用語（執着 / 象徴エピソード / 落とし込み 等）を Gravity 公開物に持ち込まない
- 追加だけのペア提案は NG（必ず削除 / 圧縮候補とセット）

完了後、引き継ぎ書 §9 の「最終アウトプット」を本書末尾に追記すること。
```

---

## 補足

このプロンプトを別 thread で貼り付けると、その thread は引き継ぎ書を起点に R3 を開始します。本 thread（メイン）の作業内容は引き継ぎ書と memory にすべて記録済みなので、別 thread は独立に動けます。

別 thread での進行報告は、引き継ぎ書末尾への追記で実施。完了後に本 thread（メイン）に「R3 完了」と一言いただければ、最終統合のための work-log 統合・SSOT 更新を本 thread でサポートします。
