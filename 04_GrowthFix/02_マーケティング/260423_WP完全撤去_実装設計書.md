## 🎨 静的HTML デザイン方針

**ヘッダー／フッター：** 既存 `/gravity/` LP の構造を流用（`styles.css` も共通化）

```html
<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <title>{{ページタイトル}} | GrowthFix</title>
    <meta name="description" content="{{ページ説明}}">
    <link rel="stylesheet" href="/common/styles.css">  <!-- Gravity LP と同じスタイル流用 -->
</head>
<body>
    <!-- 共通ヘッダー（既存と同じデザイン） -->
    <header>...</header>

    <main>
        <!-- ページ固有コンテンツ -->
    </main>

    <!-- 共通フッター（既存と同じ 4層構造） -->
    <footer>...</footer>
</body>
</html>
```

## 🔗 URL リダイレクト戦略（.htaccess）

WP 撤去時に必要な設定（Phase B で実施）：

```apache
# /news/ 個別記事の廃止 → 一覧へリダイレクト
Redirect 301 /news/gravity-release /news/
Redirect 301 /news/site-renewal /news/

# /wp-admin/ / /wp-content/uploads/ は画像・アーカイブ資産のため維持（image URLsが機能）
# ※ uploads フォルダ自体は残す必要あり（既存画像 URL）
```

---

## 📸 画像資産の扱い

`/wp-content/uploads/` 配下の画像は既に多数の HTML から参照されている：
- 新3テーマバナー（`Gemini_Generated_Image_*.png`）
- `/knowledge/` で使用
- トップページで使用

**WP 撤去後も `/wp-content/uploads/` ディレクトリ自体は残す**か、画像を `/assets/images/` に移動して URL を置換するか。

**推奨：** 画像は `/wp-content/uploads/` のまま残す（URL 変更せず）。WP 本体は撤去するが、画像保管庫としてディレクトリは維持。
