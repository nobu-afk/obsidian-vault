# 『数値化の鬼』分析 ── GrowthFixへの応用

> **書籍:** 数値化の鬼 ──「仕事ができる人」に共通する、たった1つの思考法
> **著者:** 安藤広大（株式会社識学 代表）
> **発行:** 2022年、ダイヤモンド社
> **対象:** プレーヤー〜マネジャー
> **核心:** 「行動量を増やす ＞ 確率を上げる」「変数を絞る」「長期から逆算」
> **分析日:** 2026-04-10

---

## 5つのステップの要約

### Step 1：行動量を増やす

- KPIは「分母（行動量）」に置く。「成約率（確率）」ではない
- 「半年で12件成約」→「1日4件のアポ」のように具体化
- 曖昧な評価（「半分以上はやれた」）はNG

### Step 2：確率の罠に気をつける

- 成約率80%/10件 ＜ 成約率50%/50件
- 「成功率を下げたくない」心理が行動量を殺す
- 「平均値」の中身を疑う

### Step 3：変数を見つける

- 結果を変える要因＝変数を実験で発見する
- 上司の成功事例をそのまま模倣するな（仮説に過ぎない）
- 「行動 → 結果 → 理由分析 → 次の試行」のサイクルを回す

### Step 4：真の変数に絞る

- 変数が増えると時間が溶ける
- 改善に繋がらない変数は手放す
- 1つか2つに絞る勇気

### Step 5：長い期間から逆算する

- 短期目標と長期成長は別軸
- 5年・10年後から現在を逆算
- 「目標未達でも行動量が増えた人」を評価する

## GrowthFixの**業務**への応用（日々の活動）

### 1. 行動量KPIの再設定

**現状の問題：** GrowthFixのKPIは「Gravity受注数」「セミナー申込数」など**結果指標**が中心。これは数値化の鬼が言う「確率」の側に寄っている。

**応用：** 行動量を分母に置き直す

| 結果指標（旧） | 行動量KPI（新） |
|---|---|
| Gravity受注数 | 月間Scan提案数 / 商談数 |
| セミナー申込数 | 月間広告クリエイティブ更新回数 / Note記事公開数 |
| Scanモニター数 | 月間声かけ数 / 対面接触数 |
| 4月入金 | 月間アウトバウンド回数 |

**今日からやること：**
- デイリーログに「行動量KPI」セクションを追加
- 「今日やった対話・声かけ・記事執筆」の本数を毎日記録
- 結果が出ない時は「行動量が足りているか？」を最初に問う

### 2. ファネルCVRの罠を回避

**現状：** Scanモニター4名のうち何名がGravityに進むか、を意識している → これは確率指標

**数値化の鬼の警告：** 確率を上げることに集中すると、母数（モニター数）が増えなくなる

**応用：**
- 「Scanモニター10名/月」を行動量KPIにする
- 1名1名の成約確率は気にしない
- 10名やればGravity候補が出る、という前提で動く

### 3. 「真の変数」の特定

**現状：** Note連載・Meta広告・セミナー・対談・LP改修・スキル整備…と動かしている変数が多すぎる

**数値化の鬼の警告：** 変数が多いと時間が溶ける。1-2個に絞れ

**応用（Q2の真の変数）：**
- 候補1：Meta広告クリエイティブの更新頻度
- 候補2：セミナー参加数（4/24セミナーへの集客）
- 候補3：対面接触数（飲み会・対談）

→ 1-2個に絞って、他は手放す勇気が必要。**堀田セッション（4/15）で「真の変数」を確定すべき。**

### 4. 長期からの逆算

**現状：** 4月入金11万、累計358.5万、年間1,100万目標 → これは短期

**長期の視点（2032年オーナー化、2046年）：**
- 2032年：オーナー化（マクロ投資家＋著者＋少数コーチング）
- 2046年：AI時代の組織コンサル領域でカテゴリー定義者
- 2026年（今）：書籍出版とフレームワーク普及の起点

**逆算すると：**
- 1-3年：書籍出版＋セミナー集客＋ファネル稼働
- 3-5年：データ蓄積＋認定パートナーネットワーク
- 5-10年：人＋AI組織設計への拡張

→ **今やるべきは「書籍出版に必要な実績作り」と「フレームワーク普及」。** 短期の月次入金より、書籍化に向けた事例蓄積を優先する判断もあり得る。

## GrowthFixの**サービス**への応用

### 1. Scanレポートに「行動量KPI設計」セクション追加

**現状：** Scanレポートは「伸びしろの構造特定」で終わっている

**応用：** レポートのセクション05「最初の一手」に**行動量KPI**を組み込む

```
最初の一手：
- 何を変えるか（変数の特定）
- どのくらいやるか（行動量KPI）
- いつ振り返るか（評価のタイミング）
```

例：「採用基準の言語化」を一手とする場合
- 変数：採用判断基準
- 行動量KPI：「3名のメンバーに『成功した採用と失敗した採用の違いは何か』を聞く」
- 評価：1週間後

「やってください」から「30分×3回やってください」に変えるだけで実行率が変わる。

### 2. Gravity Shiftに「判断量KPI」を組み込む

**現状の設計：** 3ヶ月で1-2箇所のループ要素から社長を抜く

**数値化の鬼の応用：** 「幹部の判断回数」を行動量KPIに置く

| 月 | 判断量KPI |
|---|---|
| Month 1 | 幹部が「判断した」項目を週5件記録 |
| Month 2 | 幹部だけで判断した件数を週10件 |
| Month 3 | 社長介入なしの判断を週15件 |

**回数で成功基準を可視化する。** これが識学メソッドの中核であり、Shiftの成果可視化に直結する。

### 3. Gravityの月次評価に「行動量レビュー」を導入

**現状：** Gravityは月2回×60分のセッション＋Slack＋成果物納品

**応用：** 月次レビューで「行動量」と「真の変数」を振り返る

```
月次レビュー：
1. 今月の行動量（何をどれだけやったか）
2. 結果に繋がった変数は何か
3. 来月絞る真の変数は何か
4. 長期目標からの逆算は合っているか
```

**経営者にとって毎月「数値化の鬼」をやってもらう機会**を設計する。これがGravityの体験を「ふんわり伴走」から「数値で動く伴走」に変える。

### 4. Orbitに「数値化レビュー」を組み込む

**現状：** 月15万円〜の継続支援（仕組みの維持・最適化）

**応用：** 「行動量レビュー」「真の変数レビュー」を月次で必ずやる

→ Orbitが「ふんわり継続」ではなく「**数値で組織を動かし続ける月次オペレーション**」になる。これはOrbitの差別化要素として強い。

## マーケティング戦略への応用

### 1. Note記事のKPIを「行動量」に置く

**現状：** スキ数・PV数などの結果指標を見ている

**数値化の鬼の応用：** 「記事公開本数」「シェア依頼数」を行動量KPIに

→ スキが少ない時に「記事の質を上げよう」とは考えない。まず「本数を増やす」が第1優先。

### 2. セミナー集客の「真の変数」を特定

**現状：** Meta広告・LP改修・SNS拡散・既存顧客への声かけ…と動かしている変数が多すぎる

**応用：** セミナー4/24までの2週間で**1つの変数だけ**動かす

候補：
- Meta広告クリエイティブの更新頻度
- 既存クライアント・モニターへの直接声かけ数
- セミナーLPのCTA改修

→ **1つ選んで、それだけを動かす。**「これだけはやり切った」と言える状態を作る。

### 3. 書籍出版への逆算

**長期目標：** 書籍『事業の伸ばし方は4パターンしかない』出版

**逆算：**
- 出版社へのアプローチに必要な「実績」は何か（PV数？フォロワー数？導入企業数？）
- それを揃えるための「行動量」は何か
- 月単位でブレイクダウンする

→ これを次の堀田セッションで設計する。

## GrowthFixの「思想」と「数値化の鬼」の関係

| 識学の言葉 | GrowthFixの言葉 |
|---|---|
| 数値化の鬼 | 行動量で勝負する |
| 確率の罠 | 質より母数 |
| 真の変数 | 1つに絞る勇気 |
| 長い期間 | 5年後から逆算 |
| 母数を増やす | まず動け |

**識学は「マネジメント技術」を売っている。GrowthFixは「事業設計の思想」を売っている。** だが両者の根底にある「数値化と行動量」の哲学は同じ。GrowthFixも識学の数値化メソッドを業務とサービスに組み込むことで、「ふんわり伴走」から「数値で動かす伴走」に進化できる。

## 「数値化の鬼」を石井1人事業に応用する時の注意点

### 注意1：1人事業の行動量KPIは「自分との約束」

組織のKPIと違い、1人事業のKPIは自分が自分に課すもの。誰も監視しない。だからこそ「数値化」の力で自分を動かす必要がある。

### 注意2：質を捨てる勇気が要る

GrowthFixは「フレームワークが綺麗」「サービス設計が緻密」など質を高めることに時間を使っている。**質を保ちながら量を増やす**のは至難。質を一部捨てても量を取る判断が必要な時がある。

### 注意3：長期の北極星を見失わない

行動量を増やすと「目の前の動きすぎ」になりがち。**5年後・10年後の北極星（書籍出版・カテゴリー定義者）を月1回確認する**仕組みが要る。週次クローズ（/weekly-close）に組み込むのが現実的。
