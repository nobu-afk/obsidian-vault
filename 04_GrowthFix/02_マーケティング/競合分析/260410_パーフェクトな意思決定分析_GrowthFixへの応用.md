### 核心メッセージ

> **「決められない人は、10年後も決められない。」**
> **「正しい意思決定をしよう」ではなく「決めた後で正しくする」順番が大事。**
> **「水のようにしなやかな意思決定」── 固い氷にも、柔らかい水にもなる。**

## 「3つの箱」のフレームワーク

本書最大の実装可能ツール。意思決定を3種類に振り分ける。

### 1. 即決の箱 ── 情報十分、選択肢明確

**判断基準：** すでに必要な情報が揃っていて、選択肢が明確に存在している
**やること：** その場で即決する
**例：** 紙の予約管理 → スプレッドシート化

### 2. 情報不足の箱 ── 判断材料が足りない

**判断基準：** 情報が少ない、判断材料がない
**やること：** 何の情報が必要かを特定し、チームに収集させる（自分1人で抱えない）
**例：** オンラインサロン立ち上げ判断 → 他社調査をメンバーに依頼

### 3. 期限を設定する箱 ── 結果を見たい

**判断基準：** 試行錯誤が必要、結果を見ないと決められない
**やること：** 「1週間後」「1ヶ月後」など決定期限を必ず定める
**例：** 新商品の継続判断 → 初月の会員数を見て判断

## NG行動

| NG行動 | なぜダメか |
|---|---|
| 決定を他人に預けて責任転嫁する | 決まらない |
| 決定後に不安そうに伝える | 信頼を損なう |
| 決めたのに実行しない | 決めていないのと同じ |
| 失敗を理由に決定者を批判する | 次から誰も決めなくなる |
| 「正しさ」を求めすぎて決めない | 永遠に決まらない |
| 全員の理解を得ようとする | 即決の箱の事項を期限の箱に入れている |

## GrowthFixの**業務**への応用（石井1人事業）

### 1. 「3つの箱」を毎朝の業務開始時に使う

**現状の問題：** 動かしている変数が12個。何から手をつけるか毎日迷う

**応用：** 朝のタスクリストを「3つの箱」に振り分ける

```
即決の箱（午前中に終わらせる）：
- 行動量KPIの記入
- Note記事1本下書き
- 既存LP CSS微調整

情報不足の箱（収集タスクを別途設定）：
- 出版社のリスト作成（誰に紹介してもらうか）
- 対談候補の連絡先リサーチ

期限を設定する箱（4/15堀田で判断）：
- Q2の真の変数の確定
- Meta広告再開の可否
```

**毎朝5分のタスク振り分け**で1日の動きが明確になる。

### 2. 「期限の箱」で先送り癖を撲滅

**現状の問題：** 「いつかやろう」が積み上がっている（書籍化準備、Shift LP制作、対談企画）

**応用：** すべての「いつかやろう」に期限を設定する

| 先送りタスク | 期限 |
|---|---|
| 書籍企画書ドラフト | 4/30 |
| Shift LP叩き台 | 5/15 |
| 対談候補リスト10名 | 4/20 |
| 高野さんへの対談打診 | 4/25 |

**期限なき先送りは存在を許さない。** 期限を設定したらカレンダーに入れる。

### 3. 「決めた後で正しくする」を採用

**現状の問題：** Q2の真の変数を堀田4/15まで決められない＝1週間判断保留

**応用：** 自分で仮決めしてから堀田に持っていく

例：「Q2の真の変数は『対面接触週2件』に仮決めする。堀田4/15で確認・修正する」

→ **保留ではなく仮決め＋修正前提。** これだけで1週間動ける。

### 4. 「意思決定スピード」をKPIにする

**応用：** 毎日「今日決めた数」を行動量KPIに追加する

| カテゴリ | 目標/日 |
|---|---|
| 今日決めた数（即決の箱で処理した数） | 5件以上 |
| 今日「期限の箱」に入れて期限を設定した数 | 1件以上 |
| 今日先送りした数 | 0件目標 |

**識学の「数値化の鬼」を意思決定に応用する。** 「決めた数」を数値化して可視化する。

## GrowthFixの**サービス**への応用

### 1. Scanレポートに「3つの箱」を組み込む

**現状：** Scanレポートのセクション05「最初の一手」に行動量KPIが追加された（260410）

**応用：** さらに進化させて、最初の一手を**3つの箱に振り分ける**

```
セクション05：最初の一手
├── 即決の箱（明日からやる）
│   - 行動：「成功した採用と失敗した採用の共通点を3名に聞く」
│   - 行動量KPI：3名 × 30分
│
├── 情報不足の箱（情報収集タスク）
│   - 行動：「過去1年の退職者の理由を整理する」
│   - 期限：1週間後
│
└── 期限を設定する箱（試行と判断）
    - 行動：「採用基準を仮設定し、次の3名で試す」
    - 期限：1ヶ月後に振り返り
```

経営者がScanレポートを受け取った瞬間に**「明日何をやるか」「いつまでに何を決めるか」**が明確になる。

### 2. Gravity ShiftにNG行動セットを追加

**現状：** Shiftは「社長の判断を幹部にシフトする」サービス

**応用：** 安藤本のNG行動リストをShiftの教材として使う

```
Shiftプログラムで毎月確認するNG行動チェックリスト：
□ 決定を幹部に投げているが、実は責任転嫁している
□ 「任せた」と言いながら、後から覆している
□ 幹部が決めたことを社長が修正しすぎている
□ 「全員の理解を得てから決めよう」と先送りしている
□ 失敗した幹部を批判して、次から決められなくなっている
```

これは**「Shiftで防ぐべき社長のNG行動」**として、サービスの差別化要素になる。

### 3. Gravityに「3つの箱」セッションを月1回組み込む

**現状：** Gravityは月2回×60分のセッション

**応用：** 月1回のセッションで**経営者の「決めるべきこと」を3つの箱に振り分ける**ワークを実施

```
セッションフロー（30分）：
1. 経営者に「今、決められていないこと」を10個書き出してもらう（5分）
2. 一緒に3つの箱に振り分ける（10分）
3. 即決の箱はその場で決める（5分）
4. 情報不足の箱は「誰が何を集めるか」を決める（5分）
5. 期限の箱は決定期限を設定する（5分）
```

**経営者がGravityセッションを受けるたびに「決まる体験」を持ち帰る。** これがGravityの体験価値を一段引き上げる。

### 4. 月次レビューに「決めた数」を追加

**現状：** 260410作成の月次レビューフォーマットに「行動量レビュー」がある

**応用：** 「意思決定量」を行動量KPIに追加

```
月次レビュー：
| カテゴリ | 月次目標 | 実績 |
|---|---|---|
| 即決の箱で処理した意思決定数 | 30件 | |
| 情報不足の箱→決定済みになった数 | 5件 | |
| 期限の箱→決定済みになった数 | 3件 | |
| 先送り（どの箱にも入れずに翌月持ち越し） | 0件 | |
```

**「決めた数」を月次で振り返る。** これは経営者にとって新鮮な指標。

## マーケティング戦略への応用

### 1. セミナー構成への組み込み

『パーフェクトな意思決定』は「決められない経営者」を直接ターゲットにしている。GrowthFixのセミナーでも以下のメッセージを使える：

> 「人を増やしても事業が伸びない理由は、組織の問題ではない。**経営者が決められていないこと**が積み上がっているからだ。」
> 「決められないことは、決めない選択をしているのと同じ。事業を止めているのは、施策の数ではなく、決めていないことの数だ。」

→ セミナータイトル候補に追加：「**決められない社長が、事業の天井を作っている**」

### 2. 書籍構成への組み込み

GrowthFixの書籍『事業の伸ばし方は4パターンしかない』に**「決断力」の章**を追加できる：

```
新章候補：「組織を変える前に、決めていないことを片付けろ」
- 経営者の決めていないことが積み上がっている現実
- 3つの箱フレームワーク（識学からの応用）
- 勝ち筋ループの各要素で「決めるべきこと」を可視化
- 決める習慣が組織を動かす
```

### 3. 「決められない経営者」へのメッセージ

GrowthFixの広告クリエイティブに使える切り口：

| 広告コピー候補 |
|---|
| 「決められない社長は、10年後も決められない」 |
| 「決められないことが、事業の天井を作っている」 |
| 「決めるべきことを3つの箱に振り分ける、たったそれだけで事業は動く」 |

## 識学3部作との統合

| 書籍 | 核心 | GrowthFixでの応用先 |
|---|---|---|
| **リーダーの仮面**（5原則） | マネジメント技術 | 書籍タイトル設計のお手本 |
| **数値化の鬼**（5ステップ） | 行動量で勝負 | デイリー/週次/月次のKPI設計 |
| **パーフェクトな意思決定**（3つの箱） | 決断の習慣化 | Scan/Shift/Gravityへの組み込み |

**3部作の共通点：** 全て「経営者・リーダーが普段やっている曖昧な行動」を**仕組み・数値・分類で構造化**している。これが識学のメソッドの本質。

GrowthFixも同じ構造を採用できる：

```
GrowthFix3部作（仮）：
1. 『事業の伸ばし方は4パターンしかない』── 思想とフレーム
2. 『社長の判断力』── 数値化の鬼の応用（判断量KPI）
3. 『勝ち筋を仕組みにする』── パーフェクトな意思決定の応用（3つの箱で決める）
```

## 戦略的示唆：識学とGrowthFixの差別化はどこか

3冊を読んで明確になった：

| 識学 | GrowthFix |
|---|---|
| マネジメント技術（部下の管理） | 事業設計（勝ち筋から逆算） |
| ルール・位置・利益・結果・成長 | 勝ち筋・逆算・翻訳・言語化・仕組み化 |
| リーダーの仮面 | 経営者の翻訳 |
| 即決・情報不足・期限の3つの箱 | 獲得・提供・成果・拡大の4要素 |
| 部下を動かす | 事業を動かす |
| 主語：人 | 主語：事業 |

**GrowthFixは識学の続編として機能する。** 識学を学んで部下を動かせるようになった経営者の次の問いは「事業を動かすには？」。そこにGrowthFixが入る。
