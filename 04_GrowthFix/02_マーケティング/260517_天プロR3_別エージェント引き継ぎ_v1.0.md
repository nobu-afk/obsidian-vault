- 20 ノード中 13 ノード使用済（73% フレーム量活用）
- 4 LP の構造的改修は完了

→ **R3 は Gravity 4 LP ではなく、未開拓領域に天プロフレームを展開する**。

## 5. 運用ルール（必須遵守・破ったら user が止める）

### a. 「足す＋消す」ペア運用

- LP 改修は **追加だけ NG**（モリモリ化逆戻り）
- 追加候補 × 削除/圧縮候補を **ペアで提示**
- 「単に情報が追加されるだけ」は意味なし

### b. Gravity 用語隔離原則

- **天プロ用語**（執着・象徴エピソード・落とし込み・アート型ビジネス・動機づけシーケンス 等）を **Gravity 公開物に持ち込まない**
- Gravity 用語（**偏愛性・想い・強み・引力**）に統一
- 「天プロ流」「やまけん流」など塾名言及も NG

### c. 自動反映禁止・都度承認制

- ペア提案 → user 取捨選択 → 承認分だけ実装
- 複数案をまとめて自動反映しない
- 本番 deploy は必ず user 明示承認

### d. CODE 関連の数値

- CODE は **40 分** の診断（260516 確定）
- 古い「60 分」「90 分」記述があれば訂正対象としてフラグ

### e. 機械チェック自動実行

- LP 編集後は `audit_mobile_sync.py` + `lint_lp_internal_terms.py` が自動実行（hook）
- mobile.css 未カバークラスゼロ / 社内用語 HIT ゼロを維持
- 違反検出時は user 確認

## 6. デプロイ運用

- 個別 curl 禁止・`deploy.sh` 経由のみ
- LP 改修：`bash 06_開発/scripts/deploy/deploy.sh lp`
- 共通アセット（site-chrome.js）：`bash deploy.sh shared`
- WP V9：`bash deploy.sh wp`

- 自動 verify 連鎖あり（verify_deployment.sh で FAIL 検出時 exit 2）
