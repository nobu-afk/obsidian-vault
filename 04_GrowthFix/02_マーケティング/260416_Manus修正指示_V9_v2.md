## 🔴 優先度A：必須修正（2件）

### A-1. P7 Gravityマップ図表の文字サイズ拡大

**現状の問題：**
- 軸ラベル「仕組みで回す／人で回す」「今のステージを回す／次のステージを作る」が小さすぎて読みづらい
- 各象限の説明文（「CEO関与度 : ○○%」「社長が週末を休んでも事業は止まらない」等）が小さい
- ❹の偏愛扱いの思想セリフが視覚的に弱い

**修正指示：**
1. 軸ラベル（縦軸・横軸）の文字サイズを **現状の1.5倍以上** に拡大
2. 各象限内の「CEO関与度：○○%」を **Bold 強調**
3. ❹グラビティ型成長の象限内に、以下の思想セリフを **目立つフォントで追加**：
   > 「重力が、すべてを引き寄せるように──」
4. ❹象限の背景色（現状ダークネイビー）を維持しつつ、**内部の白文字のコントラストとサイズを強化**
5. 図全体の横幅を現状より広げ、読みやすさ優先

### A-2. Ch04（P10）症状テーブルの例を全面差し替え

**現状の問題：**
現在の3例は「**社長が本来やるべき仕事**」を列挙しており、誤認の証拠になっていない：
- ❌ イレギュラーな顧客対応は必ず社長の判断を仰ぐ
- ❌ 新しい施策のGOサインは常に社長が出す
- ❌ 採用最終面接で社長が「直感」で落とす

これらは経営者が普通にやる仕事で、「❹と思っているが実は❷」の症状として機能しない。

**修正指示：**
症状テーブルを以下4例に差し替えてください（現状3例→4例に拡張）：

| 症状 | 本当の意味 |
|---|---|
| 幹部が判断して持ってきた提案に、社長が最終的に手を入れる癖が抜けない | 判断基準が幹部に「翻訳」されておらず、最後は社長の暗黙知で上書きされている |
| 評価制度は整っているのに、社長が「違和感」で最終評価を調整してしまう | 形式的な仕組みはあっても、「評価の本質」が言語化されず社長の感覚に依存している |
| 社長が1週間休むと、重要な意思決定のスピードが明らかに落ちる | 意思決定の権限が"実質的には"委譲されておらず、社長の処理能力が律速になっている |
| 幹部が「社長ならどう判断しますか？」と頻繁に聞いてくる | 本来の勝ち筋が組織の言葉になっておらず、幹部が自分で判断できる土台がない |

**表の直後のリード文**も現状「これらはすべて、判断基準が組織に翻訳されておらず...」を以下に差し替え：

> これらは、一見すると「社長が関与している」だけに見える。しかし本質は違う。**仕組みも組織図も整っているのに、判断の"核"が翻訳されていないから、結局社長の"人"の力で回している**。これが❷人力拡大の正体であり、❹グラビティ型成長との決定的な違いだ。

## 🟡 優先度B：推奨修正（3件）

### B-1. Ch03（P7-8）❹グラビティ型成長の扱いを強化

**現状の問題：**
P8 の「❹グラビティ型成長とは」セクションが独立ページの最後に小さく置かれている。❹の偏愛扱い（この章のクライマックス）として弱い。

**修正指示：**
1. 「❹グラビティ型成長とは」セクションを、Gravityマップ図表（P7）**の直後・同ページ内**に配置する
2. セクション自体を **ハイライトボックス（ダークネイビー背景）** に変更し視覚的に強調
3. セクション冒頭に以下のリード文を追加：
   > ❹「グラビティ型成長」は、他の3象限と本質的に異なる。目指して到達するものではなく、**経営者が本来の自分に戻った時、自然と育ってくる状態**だ。
4. 既存4項目（判断の翻訳／社長不在でも自走／自然と引き寄せられる／引力が生まれた状態）は維持
5. セクション末尾に強い締めの一文を追加：
   > このグラビティ型成長を"作ろう"とする経営者ほど、❷人力拡大から抜け出せない。次章で、その誤認の構造を見ていく。

### B-2. Ch07（P15）とCh08（P16）の役割分担を明確化

**現状の問題：**
- Ch07「経営者を取り戻す3プロセス」（演じるのをやめる／本来の判断を言語化／翻訳して渡す）
- Ch08「翻訳の4ステップ」（止まる工程特定／判断言語化／問いに分解／幹部に渡す）

この2章は同じ骨格（特定→言語化→翻訳）を2回繰り返しており、Ch07の思想クライマックスとしての力が弱まっている。

**修正指示：**

**Ch07 を「思想の章」として書き直す：**
1. 「経営者を取り戻す3プロセス」の箱は**削除**
2. 代わりに以下の構成で思想に徹する：
   - 「組織を変える」ゲームが終わらない理由（強化・現状維持）
   - **経営者を取り戻すとは、何を取り戻すことか**（現状のセクションを拡張・思想的に深く）
   - **50社で見た、経営者が本来に戻った瞬間**（新規・Case前に感情的描写。例：「ある経営者は、組織改革の書籍を20冊読んだ後、すべて棚にしまって、自分の原点に戻ることから始めた」等のストーリー系ナラティブ）
   - **本来の自分に戻った時、組織は動き出す**（クライマックス・タイトルの畳み込み）
3. 「具体的な方法は次章で」と明示し、Ch08への橋渡しを強化

**Ch08 は「方法論の章」に純化：**
1. 現状の「翻訳の4ステップ」はそのまま維持
2. ただし章冒頭のリード文を：
   > 前章で、経営者を取り戻すことの意味をお話した。ここからは、具体的な技術である。
3. Ch07の「3プロセス」と Ch08の「4ステップ」の重複感を解消

### B-3. Ch09（P17-18）Case C の分量強化

**現状の問題：**
Case A（SaaS・週15h→週2h）と Case B（受託・10%→80%）は充実しているが、Case C（専門コンサル・差し戻し90%→20%）は3行で浅い。

**修正指示：**
Case C を以下のレベルまで厚くしてください（Case A・Bと同じ密度）：

```
CASE C
レビュー差し戻し90%が20%に

専門コンサルティングを提供するC社（従業員25名）の社長は、
コンサルタントが作成したレポートのレビューで、差し戻し率が90%に達していた。
「この質、うちの看板では出せない」と、毎回赤入れで修正していた。
コンサルタントたちは疲弊し、「何をやっても差し戻される」と自信を失っていた。

社長の「うちの看板に合う」という直感を翻訳した結果、
5つの明確なチェックポイントが浮かび上がった：
①論理の飛躍がないか ②顧客の言葉で語られているか
③数字の根拠が示されているか ④対案が具体的か
⑤実行者が特定されているか──これを「論理の飛躍チェックポイント5箇条」として言語化し、
レポート作成ガイドラインに組み込んだ。

結果：差し戻し率が20%に激減。コンサルタントは自信を取り戻し、
社長は「本質的な価値向上のための議論ができるようになった」と本来の喜びを取り戻した。
```

### C-2. Ch05（P15）「引力が発生する4つの段階」の視覚化

**現状の問題：**
4つの段階がテキスト箱の縦並びで、構造がフラット。引力が「積み上がっていく」感覚が出ていない。

**修正指示：**
4段階を**上向きの階段状ビジュアル** or **矢印で繋いだフロー図**に変更してください：
```
④仕組みとして機能する → 引力発生
     ↑
③翻訳可能になる
     ↑
②判断が一貫する
     ↑
①経営者が本来の自分
```
各段階の説明文は現状のものを維持。視覚的な「積み上がり」を出すことで、引力が自然に育つ思想を表現。
