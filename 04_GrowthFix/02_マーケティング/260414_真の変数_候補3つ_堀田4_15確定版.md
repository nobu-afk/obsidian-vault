## 2. 真の変数 候補3つ（4/14版）

### 候補A：**体験セッション実施数 / 月** 【本命】

| 項目 | 内容 |
|-----|------|
| 測る指標 | 60分体験セッションの実施件数 |
| なぜ真の変数か | **契約直前の実効KPI**。石井の能力の輪の最内側（翻訳・診断・コーチング）が発揮される場 |
| 石井のコントロール | 間接的（来てもらう必要がある） |
| 他指標への連動 | 直接Gravity／AI Shift契約に連動。Scan販売・Orbit契約にも波及 |
| 月次目標 | **4件**（Gravity転換率25%でGravity月1件＝210万） |
| 測定頻度 | 週次 |

**なぜこれが本命か：**
- 動かせば他が動く：体験セッション増やすために広告・WP・メールが全部機能する
- 動かなければ他も無意味：体験セッションが0なら広告費もWPも意味がない
- **能力の輪の内側が直接発揮される唯一の場**

### 候補B：**WPダウンロード数 / 月** 【先行指標】

| 項目 | 内容 |
|-----|------|
| 測る指標 | WP「事業の勝ち筋ループの作り方」のダウンロード数 |
| なぜ先行指標か | ファネル最上流。DLが増えれば2-4週間後に体験セッションが増える |
| 石井のコントロール | 直接（広告出稿・WP内容・LP改善） |
| 他指標への連動 | 2-4週間遅れで体験セッションに連動 |
| 月次目標 | **100-200件**（体験セッション4件を逆算） |
| 測定頻度 | 日次（累計） |

**候補Aとの関係：**
- 候補Aが契約直前の「実効KPI」
- 候補Bが2-4週間前に察知できる「先行指標」
- **候補Bが先に動き、候補Aが後から動く**（因果順序）

### 候補C：**Xエンゲージメント数 / 週** 【補助指標・既存資産活用】

**4/14時点のSNS資産：X 4,050（主力）／Note 430（育成中）**

| 項目 | 内容 |
|-----|------|
| 測る指標 | Xのいいね・RT・リプライ・DM合計（X中心に計測） |
| なぜ補助指標か | **X 4,050フォロワーが最大の既存資産**。認知構築の動力源 |
| 石井のコントロール | 直接（毎日の発信） |
| 他指標への連動 | Xから直接WP DLへ誘導可能。体験セッションへの転換実績は未測定 |
| 月次目標 | **X週次エンゲージメント +200／Note +100** |
| 測定頻度 | 週次 |

**Note（430）との役割分担：**
- X：広く撒く＋WP配布の入口
- Note：WP本文の一部を記事化→深く読ませる入口

**候補A/Bとの関係：**
- 補完関係。認知層を広げる役割
- ただしエンゲージメント→体験セッションの転換率が未知
- **本命にはしない**（転換データが揃うまで補助指標扱い）

## 6. やめる判断（数値化の鬼 Step5）

**真の変数を絞る＝他をやめる、だが何をやめるか？**

### やめる
- **セミナー直接集客**（リストが100人溜まるまで保留）
- **LP細部改修**（既に4/9で大幅改修済み、触らない）
- **新規フレーム整備の細部**（既存3フレームで十分、拡張は発信経由で検証）

### やめない（継続）
- Note記事執筆（書籍化準備＋補助指標）
- 既存3クライアントのコーチング（安定収入＋紹介起点）
- マクロ経済分析（投資戦略・イラン戦争発信）

### 新規着手（ファネル実装）
- WP執筆「事業の勝ち筋ループの作り方」
- WP LP制作
- Meta広告（WP訴求に転換）
- メール配信ツール設定＋育成メール
