## P0 接続装置（4 切り口）

> **背景：** 260430 確定の最大投資領域。GrowthFix が売れない真因は「内的コンセプト（引力ドリブン組織変革）」と「外的コンセプト（経営者のコンプレックス解消）」の間の **接続装置未整備**。タグライン = 外的コンセプト層なので、タグラインが完成しても接続装置が空白なら浮いたまま。
> **参照：** `memory/project_internal_external_concept_260430.md` ／ `04_GrowthFix/02_マーケティング/260430_5層モデル実装設計図_5サービスLP_診断UI_コーポレート.md`

### 切り口 1：外的コンセプト言語マップ v0.1 → v1.0（5/6 水・昼〜夕）

- **目的：** 経営者のコンプレックス・痛みの完全リスト化＋よだれ表現候補化。LP/WP/Note/セミナーの全コピー素材源にする
- **既存：** `04_GrowthFix/02_マーケティング/260430_外的コンセプト言語マップ.md`（v0.1）
- **成果物：**
  - 痛みリスト 30 件以上（採用 / 離脱 / 優秀人材問題 / マネジメント疲弊 / 後継者 / 拡張壁 等カテゴリ別）

  - 各表現に「えげつない具体度」スコア（A/B/C）と「使えるチャネル」タグ（LP/WP/Note/FB/セミナー/商談）
- **完了基準：** v1.0 として SSOT 化＋ MEMORY ポインタ更新

### 切り口 3：ターゲットペルソナ解像度向上（5/11 月）

## 運用ルール

1. **1 日 1 軸主義：** 1 軸完了するまで次の軸に手を出さない（並行禁止）
2. **完了 = SSOT 化：** 各軸完了で対応する 09_会社OS / memory / SSOT 用語と定義 に連鎖反映
3. **進捗ボード単一更新点：** 本ファイルの「進捗トラッキング」表を SSOT として更新
4. **daily 連動：** 各日の `04_GrowthFix/04_デイリーログ/YYMMDD_daily.md` 「今日やること」に当日の軸を記載
5. **週次レビュー：** 毎週金曜の `/weekly-close` で進捗・遅延・優先順位再評価
6. **5/15 セミナー・5/8 モニターは外乱要因：** これらの日は軸進行を停止して当該イベントに集中
