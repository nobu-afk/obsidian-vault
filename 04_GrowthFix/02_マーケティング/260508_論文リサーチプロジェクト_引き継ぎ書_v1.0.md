## 0. プロジェクト全体像

### 主軸構造（不可侵）

- **主軸（60%）：** 50 社診断 + 16 年現場経験 + AI ネイティブ実装（**現場 60% 側は SSOT_Gravity_コア主張.md で完成済**）
- **補強材（40%）：** 世界中の学術論文・先行研究で裏付け（**本プロジェクトで構築**）

### 比率原則

レポート / LP / 対話の権威性訴求は **石井 50 社診断 + 16 年現場経験が常に主軸**。論文は補強材として 40% 以下。**学者モード化禁止**。

### 配置戦略

| 媒体 | コア主張数 | 詳細度 | 期日 |
|---|---|---|---|
| CODE / Scan レポート theory-background-box | 5 項目固定文言 | 5 行 / レポート | 既に 260508 夜実装済 |
| CODE / Scan LP「理論背景」セクション | 各 LP 2-3 件深掘り | 短文 × 2-3 | Q3 後期 |
| **WhitePaper V10**（無料 DL・経営者向け）| 全 10 件 + 論文引用 | 1 章 / 1 主張 | Q4 中期 |
| **書籍 2027 Q1 KDP**（11 章構成）| 全 10 件 + 章末参考文献 + 引用注釈 | 1 章 / 1 主張 | 2027 Q1 |
| **認定コーチ研修教材**（2027〜）| 全 10 件 + 対話実例 + 論文要約 | 教材ブロック / 1 主張 | 2027〜 |

## 2. 想定リスク + 対策

| リスク | 対策 |
|---|---|
| **学者モード化**（論文に寄って石井の偏愛・現場感が薄まる） | 比率原則「現場 60% : 論文 40%」を SSOT 化（Phase 1 完了）／ 各媒体で「論文 → 石井独自解釈」の構造を明示 |
| **AI による論文捏造**（実在しない論文の引用） | 三段検証プロトコル（DOI 実在確認 → 原文読み合わせ）必須 |
| **過剰適合**（自分の主張に合う論文だけ集める） | Phase 3 で **反論・懐疑論文を 5-10 本含める**（確証バイアス回避） |
| **更新負荷**（毎月新しい論文が出る） | 年次見直しに留める（2027 / 2028 / 2029 でアップデート版 WhitePaper・書籍改訂版）|
| **法務リスク**（引用範囲・著作権） | APA / MLA 形式で適切引用 + 必要なら出版社に許諾依頼 |

### Semantic Scholar から OpenAlex への切替（260509 学び）

Phase 3 着手時に Semantic Scholar API を試したが、API キー無しでは **rate limit 厳しすぎ**（429 Too Many Requests を 12 クエリすべてで発生）。代替として **OpenAlex API**（https://api.openalex.org・mailto パラメータで polite pool・無料・1 秒間隔で 12 クエリ完走）に切り替え成功。

**今後の運用：** OpenAlex を主軸 + 必要時のみ Semantic Scholar API キー取得。Elicit は手動利用（API なし）に限定。
