## Day 2：事例1＋「あなたはどのパターンか？」

### 件名
**【事例】❷人力拡大から抜けられない、7人の会社の話**

### 本文

```
{お名前}様

WPは読んでいただけましたか？

今日は、実際のモニター企業から抽出した事例をお送りします。
業種・規模は匿名化していますが、実話です。

───────────────────────────

【事例：サービス業・従業員7名のA社】

社長は「現場のオペレーションはリーダー2名に任せている。
だからうちは自走している」と語っていました。

しかし深掘りすると、以下の「隠れた社長依存」が明らかになりました：

・イレギュラーな顧客対応は必ず社長の判断を仰ぐ
・新しい施策のGOサインは社長しか出せない
・採用の最終面接で社長が「なんか違う」とひっくり返す

これは自走（❹）ではなく、単に「社長の作業を優秀な社員が
代替しているだけ（❷）」の状態です。

───────────────────────────

【組織の"引力"が社員に伝わっていない状態】

A社の社長は、優秀でした。
事業センスも、判断も正しかった。

問題は、その判断の"引力"が組織に伝わらず、
結局すべての判断を社長が引き受けている、という構造でした。

「組織を変えるな。引力を作れ。」

これがGravityシリーズの根本メッセージです。
制度を整えるのではなく、社長の引力が組織に伝わる仕掛けを作る。

───────────────────────────

【診断後、A社がやったこと】

採用基準を言語化し、幹部に判断を委譲。
3ヶ月で採用フローが自走し始めました。

具体的には、社長の「この人は任せられるか？」という感覚を
4つの問いに分解しました。

・過去3社で同じ役職を経験しているか
・離職理由に「期待と実態のズレ」が含まれるか
・初回面接で「自分で決めた意思決定」のエピソードが3つ以上出たか
・我々の事業モデルに対する質問が5つ以上あったか

この4項目中3つ以上Yesなら採用判断の土俵に乗せる。
それだけで、幹部が自走できるようになりました。

───────────────────────────

あなたの会社は、A社と似ていませんか？

もし「うちも同じ」と感じたら、WPの第9章
「❷→❸に進むための最初の一手」を再読してみてください。

次回Day 4は、飲食ブランド（10名）の事例をお送りします。

GrowthFix 石井伸幸
```

## Day 4：事例2＋「最も多いのは❷人力拡大」

### 件名
**【事例】6-7事業を1人で抱える、10名の飲食ブランド社長**

### 本文

```
{お名前}様

前回のA社（サービス業7名）の事例はいかがでしたか？

今日は、もう一つ別の業種の事例をお送りします。

───────────────────────────

【事例：飲食ブランド・従業員10名のB社】

B社の社長は、海外ブランドのライセンス販売、FC運営、
DXコンサルなど6-7本の事業ラインを10人で回していました。

「百貨店の外商がモデル」という高級路線を志向し、
新ブランドの発掘から交渉、販売戦略、百貨店への営業、
在庫管理まで全てを自ら手がけていました。

しかし、このまま続けるほど、新しいブランド開拓の時間が
構造的に生まれない状態に陥っていました。

───────────────────────────

【診断結果：85-90%の社長関与】

社長は「問題なく回っている」と感じていましたが、
実際は以下の全てに社長が直接関与していました：

・ライセンス交渉
・百貨店開拓
・高級路線の品質維持
・6-7事業の戦略判断

目指すべき関与度は40%。
残りの60%は「番頭」役に任せることで、
社長はプロデューサーとしての新規IP開拓に集中できます。

───────────────────────────

【B社がやったこと】

「判断の番頭制」を導入し、各事業に日常判断ができる
責任者を配置。3ヶ月で社長の業務時間を40%削減し、
新規事業展開のスピードが上がりました。

社長への相談は「番頭が判断に迷った時のみ」という
ルールを設定しただけです。

重要なのは、「ルール」ではなく、
社長の判断の"引力"が番頭に正しく伝わる"設計図"を
作ったこと。

───────────────────────────

【組織を変えるな。引力を作れ。】

WPの第10章で、3社の事例を紹介しています。
共通しているのは、「社長の判断を組織に渡すこと」が
すべての変化の起点になっていることです。

組織を変える必要はない。
変えるのは、社長の判断基準の渡し方だけ。

───────────────────────────
```
