# チャネル戦略整理（260318 堀田さんセッション起点）

> **起点：** 堀田さんコーチング（2026-03-18）で得た示唆を整理。
> 広告運用開始後、流入はあるがCVゼロ → 根本原因は「温度感設計不足」と「チャネル×プロダクトの最適化不足」。

---

## 勝ち筋の2本柱

### ルートA：対面起点 → CODE → Coaching

| ステップ | 内容 |
|---|---|
| 接点 | 飲み会・イベント・コミュニティ |
| フック | Gravity CODE（無料提供）で「隠れた才能がわかる」 |
| 転換 | CODEレポート納品 → コーチングへ接続 |
| 受注 | Gravity Coaching（6ヶ月） |

**根拠：** 今までの受注は全て飲み会経由。対面で会うと一気に信頼残高20点。CODE無料提供で確実にフックできる。

### ルートB：Web起点 → Webinar/WS → SCAN → Gravity → Orbit

| ステップ | 内容 |
|---|---|
| 集客 | Meta広告 → Webinar/ワークショップLP |
| 中間 | Webinar/WS参加（意外性で信頼残高を積む） |
| 転換 | WS後の個別相談（15分壁打ち） |
| 受注 | Gravity Scan → Gravity → Gravity Orbit |

**根拠：** 診断LP経由ではエキサイトメント不足で離脱。WSで束にして温度感を作る方が営業負荷も下がる。

## 全チャネル候補

### 1. 飲み会・イベント（現在の主力）

- **対象プロダクト：** CODE → Coaching
- **スケール性：** 低（身一つ）
- **CPA：** 最低（飲み代のみ）
- **課題：** 2-3ヶ月に1回しか誘われない → 主催者ネットワークの周辺に入り込む必要あり
- **アクション：** AI系、経営系、人事系のコミュニティに顔を出す頻度を上げる。飲み会→次の飲み会へのブリッジを意識的に作る

### 2. Webinar/ワークショップ（新規構築中）

- **対象プロダクト：** SCAN → Gravity → Orbit
- **スケール性：** 高（広告で集客可能）
- **CPA：** 中〜高（広告費 + 運営工数）
- **設計ポイント：**
  - 少人数限定（10名等）で希少性
  - 開催日程を確定して募集（「優先案内」ではなく「この日に参加する」）
  - 意外性中心のタイトル設計
  - WS後に個別相談 or CODE無料提供で接続
- **堀田さん助言：** ウェビナー形式なら少人数でもこけた感が出ない。サクラを入れてもOK

### 3. 対談・コラボ配信（堀田さん自身の成功パターン）

- **対象プロダクト：** CODE → Coaching / SCAN認知
- **スケール性：** 中（相手のオーディエンスに乗る）
- **CPA：** 極低（工数のみ）
- **メカニズム：**
  - 堀田さん自身が平本章男さんとの対談→平本系列のお客さんが来た、と語っている（51:49付近）
  - 石井の「AI×組織開発」の切り口は、HR系・経営系インフルエンサーにとっても対談する意味がある
  - 対談そのものが**権威付け**になる
- **アクション：** 飲み会で出会った人を「対談しませんか」に持っていく。飲み会の一回性を資産化する
- **プラットフォーム：** YouTube / Podcast / Note対談記事

### 4. 既存クライアントからのリファラル

- **対象プロダクト：** SCAN（組織課題の横展開）/ CODE → Coaching
- **スケール性：** 低（クライアント数に依存）
- **CPA：** 最低
- **メカニズム：** 紹介元の信頼残高が移転（紹介先に20-30点が乗る）
- **アクション：** CODE無料提供を「紹介特典」として設計。「うちの組織も見てほしい」→ SCAN案件化

### 5. ゲスト登壇（他社イベント・研修に入り込む）

- **対象プロダクト：** SCAN → Gravity → Orbit
- **スケール性：** 中
- **CPA：** 低（工数のみ）
- **メカニズム：** 堀田さんの「プレゼンを聞いたら1人申し込んでくる」パターン（50:54付近）
- **アクション：** 90分ワークショップをそのまま「お試し版」として他社イベントに持ち込む。人事系カンファレンス、経営者勉強会にゲスト枠で登壇

### 6. Note連載（信頼残高の非同期蓄積）

- **対象プロダクト：** SCAN / Gravity（認知・信頼構築）
- **スケール性：** 中（SEO + SNS拡散）
- **CPA：** 低（執筆工数のみ）
- **メカニズム：** 「本に書いてなさそうなこと」＝意外性を記事で実証。堀田さんの「経営者は既知の話ではなく新しい視点を求める」に直接応える
- **位置付け：** 単体では弱い。WebinarやWSへの導線として組み合わせる

### 7. Meta広告（WSへの送客手段）

- **対象プロダクト：** WS/Webinar集客
- **スケール性：** 高
- **CPA：** 中〜高
- **現状の学び：** 診断LP直結ではCVしない。WSに直接送客する方が有望
- **注意：** 広告クリエイティブの意外性はAIに任せず自分で作る（堀田さん助言）

## 堀田さんの原則（チャネル戦略に共通する判断基準）

1. **意外性** — すべてのチャネルで「フリー→意外性→解消」の順序を意識
2. **心理的動線** — クリック/参加後の感情変化を設計。エキサイトメントが上がる仕掛け
3. **権威付け** — 対談・導入実績・推薦で「この人は何者か」を明示
4. **点数積み上げ** — 0→1→2と地道に。高単価購買は50点超えが必要
5. **ブランディング** — 軽いページではなく、深さと価値観が伝わる設計
