## §0 目的・適用範囲・本ファイルの位置づけ

### 目的

GrowthFix の事業指標（広告 / 流入 / コンテンツ / 営業 / コスト）を横断管理し、**フェーズ転換判断（Phase 1 採用集中 → Phase 2 多軸展開）を数値で裏付ける**データ駆動経営基盤を確立する。

### 現状の課題

| 課題 | 詳細 |
|---|---|
| データ散在 | Meta 広告・反応率 MD・パイプライン MD・デイリーログが各所に分散 |
| 横断集計なし | ファネル全体（認知〜受注〜LTV）を 1 枚で見られる仕組みが未整備 |
| 月次レポート未整備 | 「今月の経営判断に使える数字」を自動または半自動で出す仕組みがない |
| KPI 目標値未確定 | 反応率ダッシュボードに目標値あるが **年次・四半期目標は TASK-11 と連動待ち** |

### 適用範囲

- **対象期間：** 2026 年 Q2〜Q3（Phase 1 集中期）
- **対象ファイル：** 本 MD が 6 データソース × 運用ルーチン × 月次テンプレの SSOT
- **スコープ外：** 個別 LP の A/B テスト詳細・競合分析・財務会計（別軸）

## §2 横断ダッシュボード要件

> ✅ **確定（260507）：実装方法 = (D) A+C ハイブリッド** — 自作 Python + Markdown 月次レポート + daily skill 拡張で日次自動更新。以下 3 候補の比較は判断参考として残す。

### 2-1 候補 A：自作 Python + Markdown 月次レポート（推奨候補）

**概要：** 既存の `meta_ads_daily.py` / `token_usage.py` を参考に、月次集計スクリプト `monthly_report.py` を新規作成。出力先は `04_GrowthFix/04_デイリーログ/` 配下の月次 MD。

**メリット：**
- 既存スクリプト資産を流用・拡張できる
- Vault 内完結（外部ツール不要・コスト追加なし）
- daily skill への組み込みが容易

**デメリット：**
- GA・UTAGE の API 連携は別途構築が必要
- 可視化はテキストテーブル止まり（グラフ非対応）

**実装工数概算：** 既存スクリプト拡張 2-4 時間 + GA API 連携 4-8 時間

---

### 2-2 候補 B：既存ツール統合（Looker Studio / Metabase 等）

**概要：** Google Looker Studio（無料）に Meta / GA / UTAGE データを接続し、ブラウザで可視化。

**メリット：**
- グラフ・ダッシュボード UI がリッチ
- GA との公式コネクタあり（接続容易）

**デメリット：**
- Vault との連携なし（MD への書き戻し不可）
- 外部ツール依存（ロックイン・運用コスト）
- UTAGE・Meta 以外のデータ統合に手間

**判断軸：** 「数字をブラウザで見たい」なら B。「Vault 内で完結させたい」なら A。

---

### 2-3 候補 C：daily skill 拡張で日次自動更新

**概要：** `daily` skill の STEP を拡張し、日次で全指標を自動収集・前日比差分を報告。

**メリット：**
- 毎朝自動でデータが揃う
- 既存の daily ワークフローに乗れる

**デメリット：**
- 手動入力指標（Note・FB）はスキップされる
- daily skill が肥大化するリスク

**✅ 確定（260507）：A（月次レポート）+ C（日次デルタ）ハイブリッドを採用。更新頻度 = Meta 日次 / Note 週次 / SNS 月次（19-2 確定）。**

## §3 月次レポートテンプレ要件

> ✅ **確定（260507）：優先取り込みデータソース = (B) 広告 + 営業層**（Meta + パイプライン + 反応率ダッシュボード）。GA・SNS は後続フェーズ。KPI 項目の数値目標は TASK-11 との統合後に確定（§4 参照）。

### 3-1 月次レポート候補項目（石井さんが赤入れして確定）

```
## [YYYYMM] 月次事業レポート（テンプレ候補 v0.1）

### A. 認知・流入
- Meta 広告：消化額 / CTR / CPC / クリック数
- GA：オーガニック訪問数（LP 別）
- Note：月間ビュー合計・新規フォロワー数
- FB / LinkedIn：インプレッション合計

### B. 興味化・リード獲得
- WP DL 数（月計）
- オプトイン CVR（WP DL 数 ÷ オプトイン LP 訪問）
- セミナー申込数（該当月開催分）
- DM 返信率（アウトリーチ対象別）

### C. 営業
- 新規商談数（30 分壁打ち実施）
- 提案数 / 受注数 / 受注額
- ファネル転換率（認知→興味→商談→受注）
- 失注数・失注理由分類

### D. コスト
- Meta 広告費（月計）
- Claude トークン費用（月計・Retail 換算）
- CAC（顧客獲得単価）= 広告費 ÷ 新規受注数
- ROI = 新規受注額 ÷ 広告費

### E. 累計ポジション
- 累計受注件数 / 受注額（Coaching / Shift R / 別）
- アクティブクライアント数
- Phase 1 → 2 移行トリガー進捗（🔴 TBD：TASK-11 で定義）
```
