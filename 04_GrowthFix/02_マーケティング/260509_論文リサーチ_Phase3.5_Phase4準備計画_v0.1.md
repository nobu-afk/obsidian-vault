## 2. Phase 3.5 ワークフロー設計

### 2.1 入力準備（5 月後半・1 週間）

**A. 全 296 論文の abstract 自動取得スクリプト作成**
- 入力：Phase 3 v0.2 の 328 論文 DOI リスト
- 処理：OpenAlex API or Crossref API で abstract 取得
- 出力：CSV / MD 形式で `02_セッション記録/論文リサーチ/Phase3.5_abstracts/` に格納
- 推定工数：スクリプト 2 時間 + 実行 1 時間

**B. 論文要約自動生成スクリプト**
- 入力：abstract 296 件
- 処理：Claude API（Sonnet）で 100 字要約 + 主軸領域タグ付け
- 出力：論文ごとに「100 字要約 + 主軸領域 + Gravity 主張番号 + 賛成/反論」付きの構造化データ
- 推定工数：スクリプト 3 時間 + API 実行 30 分（Sonnet バッチ）
- コスト目安：Sonnet 1 論文 ≈ $0.003 × 296 = **約 $1**

### 2.2 Layer 3 三段検証（6 月・2 週間）

**C. 石井さん目利きシート設計**

各論文 1 件あたり **1-3 分**で判断できるシート：

| 項目 | 値（5 段階）| 備考 |
|---|---|---|
| 現場感整合度 | 1（全く違う）〜 5（完全に整合） | 50 社診断の現場感に照らして判定 |
| Gravity 主張接点強度 | 1（薄い）〜 5（コア主張の核心を補強） | コア主張番号 #1-#12 のうちどれに紐付くか |
| 反論論文として機能するか | Y / N / 該当なし | 賛成枠 or 反論枠 or 除外 |
| 媒体配置適合度 | 入口 / 信用 / 理論 | 学術濃度 3 層モデル（260509 SSOT）|
| 採用判定 | ⭕ / △ / ❌ | △ は Phase 4 で再検討 |

## 5. 石井さん目利きの判断基準（Layer 3 三段検証）

### 5 軸チェックリスト

各論文の abstract + 100 字要約を見て、1-3 分で判断する：

#### 軸 1：現場感整合度（5 段階）
- **5：** 50 社診断で何度も見た現象を完全に説明している
- **4：** 50 社のうち多くで観察された現象と整合
- **3：** 一部のクライアントで観察した現象と整合
- **2：** 理論的には整合するが現場での実感は薄い
- **1：** 現場の実感と全く違う / 反する

#### 軸 2：Gravity 主張接点強度（5 段階）
- **5：** コア主張 #X の核心を直接補強する
- **4：** コア主張 #X の周辺・サブ要素を補強する
- **3：** 接点はあるが弱い・複数主張に薄く絡む
- **2：** 接点が無理矢理感ある
- **1：** 接点が見えない（除外）

#### 軸 3：反論論文として機能するか
- **Y（反論枠）：** Gravity 主張への有効な反論を含む（10 本枠候補）
- **N（賛成枠）：** Gravity 主張を補強する（30 本枠候補）
- **該当なし：** どちらにも該当しない（除外）

#### 軸 4：媒体配置適合度
- **入口層**（学術濃度 0-5%）：原則 LP に出さない（除外候補）
- **信用層**（20-40%）：WhitePaper / 商談資料 / Note 連載で使う
- **理論層**（50-70%）：書籍 / 認定コーチ研修教材で使う

#### 軸 5：採用判定（最終）
- **⭕**：Phase 4 確定枠候補
- **△**：Phase 4 で再検討
- **❌**：除外

### Pass 条件（Phase 3.5 通過）

- 軸 1 ≥ 3 AND 軸 2 ≥ 3 AND 軸 5 = ⭕ or △

### 想定通過率

- 296 本 → 60-100 本通過（**約 25-35%**）
- 反論論文 80 本 → 25-35 本通過想定（最終 10 本に絞る）
- 賛成論文 248 本 → 35-65 本通過想定（最終 30 本に絞る）

## 6. リスク管理

### リスク 1：Layer 3 工数オーバー
**症状：** 1 本判定に 5 分かかると 296 本 = 25 時間（週末 3 日分）
**対策：**
- 1 本 1-3 分制約をハーネス化（タイマー設置・週 30-40 本ペース厳守）
- 100 字要約 + abstract のみで判断（原文 PDF は確定 40 本のみ後日精読）
- バッチ単位（50-70 本）で 90 分セッション化

### リスク 2：学者モード化リスク
**症状：** 296 本見ているうちに学術用語に毒され「現場感整合度」がブレる
**対策：**
- セッション開始前に **`SSOT_Gravity_コア主張.md` 第 1 章「現場 60% : 学術 40%」原則** を音読
- 1 セッションに 1 回「50 社診断の具体エピソード」を思い出すブレイク
- 確証バイアス防止：反論論文を意図的に 1 セッション目に読む

### リスク 3：比率原則（60:40）からの逸脱
**症状：** 学術 40% を超えて 50%、60% に膨張
**対策：**
- Phase 4 完成時に「全媒体での学術引用密度監査」を実施
- WhitePaper V10 / 書籍 / LP「理論背景」セクション 全体で 40% 上限ハード制約

### リスク 4：反論論文の偏り
**症状：** 反論 10 本がコア主張 1-3 つに偏る（残主張は反論ゼロ）
**対策：**
- Phase 4 厳選時に **コア主張 1-12 で各 1 本最低** の制約をハード化
- 不足コア主張があれば追加クエリ実行（v0.3）

### リスク 5：捏造リスク（OpenAlex データ品質）
**症状：** OpenAlex で取得した DOI / 著者名 / 引用数が誤っている
**対策：**
- Phase 4 確定 40 本のみ **Semantic Scholar で再検証**（DOI + 著者所属）
- 原文 PDF を 1 段落以上読み合わせ（Layer 3 真の最終確認）
