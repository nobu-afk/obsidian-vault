### 1. minimal LP パターンの汎用化（Orbit → Shift への展開）

5/1 で Orbit を minimal LP 化した運用（「サービスは存在するが LP は最小化・商談時のみ案内」）が、本日 Gravity Shift にも適用された。**ユーザー直感（「顧客はピン痛で検索する」「フルパッケージは商談時に提案するのが筋」）から判断が降りてきた**点が重要。

### 2. Note 記事ネタ：3 段階戦略変更を 1 セッションで本番デプロイ

朝（命名 B 案）→ 昼（18 ファイル実装）→ 夕（minimal LP 化で再修正）→ 夜（Succession 骨子）の 4 段階を 1 セッションで完遂。**戦略変更 → SSOT → 18 ファイル本番デプロイの一気通貫実装**は AI ネイティブ経営者の標準パターンとして Note 記事化候補。

### 3. lint_consistency.sh のハーネス装置強化

[5.5] minimal LP 運用警告（Hub/コーポレート/Footer に Shift カード/リンク残存を機械検出）を新設。今後 minimal LP 違反が発生した場合は自動検出可能。**「絶滅危惧種の運用ルール」を機械的に守るハーネス装置**が拡張された。

### 4. Shift R/A/Succession 3 冊体制対称構造の完成

引力経営 3 軸（採用 × 躍動 × 承継）が各 3 冊体制で完全対称になった。Succession の本格化は 2027〜だが、構想段階で骨子を先に作っておくことで 2027 案件化時に SSOT 反映保留解除するだけで済む。**「先回り構造化」の運用例**。

### 5. perl lookahead 正規表現の有用性

「同じ行内に Gravity Recruit テキスト + gravity-shift URL」のミスマッチを `(?=[^<]*?Gravity Recruit)` で安全に修正した。sed では難しい「文脈依存の置換」を実装できるため、AI コーディングの安全策として標準テクニック化したい。

### 6. サブエージェント分離 = ハーネスエンジニアリングの拡張

「議論=Opus / コード=Sonnet」の自動切替はネイティブ無いが、`~/.claude/agents/*.md` で **モデル指定 + system prompt + tools 限定**を 1 枚にパッケージ化できる。これは AI.md のハーネス装置概念（コア / ハーネス / 実行エージェント 3 層）の **実行エージェント層を物理ファイル化したもの**と整理可能。LP 編集後の mobile audit 自動実行ルール（feedback メモリ）を system prompt に転記することで「ハーネスがメモリ依存ではなくエージェント定義に内蔵される」状態を実現。今後の standardpattern として位置付け候補。
