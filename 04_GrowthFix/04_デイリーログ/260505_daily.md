## 気づき・判断メモ

### 1. 「思想 → 命名 → 物理整合」の連鎖コスト構造

今日の Activate→Cultivate 変更は **思想判断 60 分 / 物理整合 480+ 分**（ファイル変更・URL リネーム・301・FTP）。命名変更は思想判断より物理整合の方がコストが大きい。AI でこの整合作業を機械化できないと、命名変更が後回しになって「思想と表記がズレた状態」が長期化する。

### 2. 「外的 → 内的」連鎖更新の必要性

外的 Cultivate に変えたら内部 Shift A も Shift C に揃える必要がある。R/A → R/C 動詞ペアは対称性維持・mnemonic 一致でクリーン。「内部だから後回し」と放置すると「Shift A（外的：Cultivate）」のような annotation だらけになる。

### 3. シャープ化は「思想転換」を含む

Orbit v3.0 のシャープ化は単なる削ぎ落としではなく、260424 幹部複製軸（守り）→ Gravity 型成長加速軸（攻め）の **思想転換** を含む。新タグライン「集まり続け、躍動し続ける会社を、月次で進化させる」と Cultivate（耕す）思想連続性で「土台を耕す → 土壌で引力を月次強化」のストーリーが完成。

### 4. _archive + INDEX が Claude 参照ノイズの物理遮断装置

完全削除ではなく `_archive/` 物理移動 + `_INDEX.md` で「なぜ捨てたか」を辿れる仕組み。MEMORY.md 軽量化哲学（260502）と同パターンを vault 全体に適用。今後の戦略変更時のデフォルトプロセスとして確立。

### 5. KISS 原則の機能：Standard/Pro 単一化判断

「Pro の差別化軸が新タグラインと整合する明確な軸を持たない」なら単一に削ぐ。経営者の判断負荷削減（YES/NO のみ）。Apple 的シンプル価格 + カスタム対応は商談時オプションで担保。
