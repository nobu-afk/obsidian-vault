> **目的：** 全 memory + 主要 SSOT に散在しているペンディングタスクを 1 枚で俯瞰する単一の真実

> **位置付け：** 本ファイル＝統合タスクの SSOT・各 memory は個別詳細の参照源（個別 memory が正・本ファイルは index）
> **初版作成：** 2026-05-12（260512）

## 📐 更新ルール

1. **新規追加：** 新規 memory 追加 / 既存 memory に「TODO / 残課題 / 予定」が増えた時に本ファイルにも反映
2. **完了処理：** タスク完了時は **削除せず `[x] 完了` マーカー + 完了日**を付与（履歴保持）
3. **期日変更：** 元の期日に取消線 + 新期日記載
4. **週次集計：** 毎週土曜 weekly_close で「完了率・遅延・新規」を集計
5. **重大変更：** 例：5/15 セミナーペンディング → memory 永続化 → 本ファイル反映
